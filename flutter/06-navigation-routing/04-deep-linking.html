Part 1 — Deep Link Types
Custom Schemes vs Verified App Links
Two fundamentally different deep link mechanisms — custom URI schemes (fragile, no verification) and verified App Links/Universal Links (HTTPS-based, OS-verified, no app chooser dialog).

1. Deep Link Types Comparison

TypeExampleVerificationFallbackSecurity
Custom URI scheme myapp://product/123 None — any app can claim Error if app not installed Low — hijackable
Android App Links https://myapp.com/product/123 Digital Asset Links JSON Opens in browser High — domain-verified
iOS Universal Links https://myapp.com/product/123 apple-app-site-association Opens in browser High — domain-verified
Staff-Level Insight
At Google scale, custom URI schemes are deprecated in favour of App Links. The key advantage: if the app is not installed, HTTPS links open in the browser (where you can prompt the user to install the app or show the content directly). Custom schemes fail silently or show an error. For a production app, always implement App Links/Universal Links with a web fallback.

2. Android App Links Setup

Step 1: AndroidManifest.xml Intent Filter

<!-- android/app/src/main/AndroidManifest.xml -->
<activity
    android:name=".MainActivity"
    android:exported="true"
    android:launchMode="singleTask">

  <!-- Standard launch intent -->
  <intent-filter>
    <action android:name="android.intent.action.MAIN"/>
    <category android:name="android.intent.category.LAUNCHER"/>
  </intent-filter>

  <!-- App Links intent filter -->
  <intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <!-- HTTPS scheme required for App Links -->
    <data
        android:scheme="https"
        android:host="myapp.com"
        android:pathPrefix="/product"/>
    <data
        android:scheme="https"
        android:host="myapp.com"
        android:pathPrefix="/user"/>
    <!-- Catch-all for all paths on this domain -->
    <data
        android:scheme="https"
        android:host="myapp.com"/>
  </intent-filter>

  <!-- Custom scheme (legacy support) -->
  <intent-filter>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="myapp"/>
  </intent-filter>
</activity>

Tip

3. iOS Universal Links Setup

Step 1: Xcode Entitlements (Associated Domains)

<!-- ios/Runner/Runner.entitlements -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>com.apple.developer.associated-domains</key>
  <array>
    <!-- applinks: prefix enables Universal Links -->
    <string>applinks:myapp.com</string>
    <!-- webcredentials: for password AutoFill -->
    <string>webcredentials:myapp.com</string>
  </array>
</dict>
</plist>

Step 2: Apple App Site Association (AASA)

// Hosted at: https://myapp.com/.well-known/apple-app-site-association
// OR: https://myapp.com/apple-app-site-association (legacy path)
{
  "applinks": {
    "apps": [],  // must be empty array
    "details": [
      {
        // TeamID.BundleID format
        "appID": "TEAMID1234.com.example.myapp",
        "paths": [
          "/product/*",    // matches /product/anything
          "/user/*",
          "/order/*",
          "NOT /api/*",    // exclude API paths
          "/*"             // catch-all
        ],
        // iOS 13+ format with components:
        "components": [
          { "/": "/product/*", "comment": "Product pages" },
          { "/": "/user/*" },
          { "/": "/api/*", "exclude": true }
        ]
      }
    ]
  }
}
Part 2 — Flutter Integration
Handling Deep Links in Flutter
GoRouter automatic handling, manual initial link, foreground links, and the cold vs warm start distinction.

4. GoRouter Deep Link Handling

GoRouter handles deep links automatically when the app is configured correctly. The OS passes the incoming URL to Flutter, which GoRouter interprets as the initial route.

// GoRouter automatically handles the initial deep link URL.
// No additional setup needed beyond AndroidManifest/AASA.

final _router = GoRouter(
  initialLocation: '/home',  // used only if no deep link
  routes: [
    GoRoute(
      path: '/product/:id',
      builder: (context, state) =>
        ProductScreen(id: state.pathParameters['id']!),
    ),
    GoRoute(
      path: '/home',
      builder: (context, state) => const HomeScreen(),
    ),
  ],
);

// When user clicks https://myapp.com/product/123:
// GoRouter sets initial location to '/product/123'
// ProductScreen is shown directly — no home screen in back stack

5. Cold Start vs Warm Start

ScenarioApp StateDeep Link DeliveryHandling
Cold start App not running (process dead) Via getInitialLink() / GoRouter initialLocation Set initial route in GoRouter
Warm start (background) App running, in background Via link stream / onLink stream Navigate imperatively via GoRouter
Already in foreground App visible Via link stream Same as warm start

Manual Handling with app_links Package

import 'package:app_links/app_links.dart';

class DeepLinkService {
  final AppLinks _appLinks = AppLinks();
  StreamSubscription? _subscription;
  final GoRouter _router;

  DeepLinkService(this._router);

  Future<void> initialize() async {
    // Cold start: get the link that launched the app
    final initialLink = await _appLinks.getInitialLink();
    if (initialLink != null) {
      _handleLink(initialLink);
    }

    // Warm start: listen for links while app is running
    _subscription = _appLinks.uriLinkStream.listen(
      _handleLink,
      onError: (error) {
        debugPrint('Deep link error: $error');
      },
    );
  }

  void _handleLink(Uri uri) {
    // Convert the incoming URI to a GoRouter path
    // e.g., https://myapp.com/product/123 → /product/123
    final path = uri.path;
    final query = uri.query.isEmpty ? '' : '?${uri.query}';
    _router.go('$path$query');
  }

  void dispose() {
    _subscription?.cancel();
  }
}

// Initialize in main or app startup:
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  runApp(const App());
}

class App extends ConsumerStatefulWidget {
  @override
  ConsumerState<App> createState() => _AppState();
}

class _AppState extends ConsumerState<App> {
  late final DeepLinkService _deepLinkService;

  @override
  void initState() {
    super.initState();
    _deepLinkService = DeepLinkService(_router);
    _deepLinkService.initialize();
  }

  @override
  void dispose() {
    _deepLinkService.dispose();
    super.dispose();
  }
}

6. Auth-Aware Deep Links

A common production requirement: deep link to a protected page, redirect to login if unauthenticated, then redirect to the original destination after login.

final _router = GoRouter(
  redirect: (context, state) {
    final isAuth = context.read<AuthBloc>().state is Authenticated;
    final isLoginPage = state.matchedLocation == '/login';

    if (!isAuth && !isLoginPage) {
      // Preserve the intended destination
      final intended = Uri.encodeComponent(state.uri.toString());
      return '/login?from=$intended';
    }
    return null;
  },
  refreshListenable: GoRouterRefreshStream(authBloc.stream),
  routes: [
    GoRoute(
      path: '/login',
      builder: (context, state) {
        final from = state.uri.queryParameters['from'];
        return LoginScreen(intendedDestination: from);
      },
    ),
    GoRoute(
      path: '/product/:id',
      builder: (context, state) =>
        ProductScreen(id: state.pathParameters['id']!),
    ),
  ],
);

// LoginScreen after successful login:
class LoginScreen extends StatelessWidget {
  final String? intendedDestination;

  Future<void> _onLoginSuccess(BuildContext context) async {
    if (intendedDestination != null) {
      context.go(Uri.decodeComponent(intendedDestination!));
    } else {
      context.go('/home');
    }
  }
}

8. Migrating from Firebase Dynamic Links

Firebase Dynamic Links was deprecated in August 2023 and shut down in August 2025. Migration path:

  1. Set up App Links / Universal Links on your own domain (as above)
  2. Create a redirection service — your backend receives short link clicks and redirects to the appropriate HTTPS deep link
  3. Handle the redirect chain — GoRouter follows the final URL after redirects, so your app opens at the correct destination
  4. Replace Dynamic Links in push notifications — use FCM data payloads with explicit path parameters instead
// Example: FCM notification with deep link data
// FCM payload (no Dynamic Link):
{
  "data": {
    "type": "product",
    "productId": "123",
    "deepLink": "/product/123"
  },
  "notification": {
    "title": "Flash Sale!",
    "body": "Flutter Developer Kit is 50% off"
  }
}

// Handle in Flutter:
FirebaseMessaging.onMessageOpenedApp.listen((message) {
  final deepLink = message.data['deepLink'];
  if (deepLink != null) {
    _router.go(deepLink);
  }
});
