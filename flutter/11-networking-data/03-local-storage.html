SharedPreferences

SharedPreferences is the simplest local storage option. On iOS it wraps NSUserDefaults; on Android it wraps android.content.SharedPreferences. It is appropriate only for small, primitive configuration values — feature flags, user preferences, last-seen timestamps.

import 'package:shared_preferences/shared_preferences.dart';

// Reading
final prefs = await SharedPreferences.getInstance();
final isDarkMode = prefs.getBool('dark_mode') ?? false;
final userId = prefs.getString('user_id');
final retryCount = prefs.getInt('retry_count') ?? 0;

// Writing
await prefs.setBool('dark_mode', true);
await prefs.setString('user_id', 'abc-123');

// Removing
await prefs.remove('user_id');
await prefs.clear(); // remove all keys
Do not use SharedPreferences for
  • Large objects or lists (it serializes the entire file on every write on Android)
  • Sensitive data like tokens or passwords (not encrypted)
  • Structured data with relationships
  • Data that needs to be queried or sorted

Testing SharedPreferences

// In test setUp — install an in-memory mock
SharedPreferences.setMockInitialValues({
  'dark_mode': true,
  'user_id': 'test-123',
});

final prefs = await SharedPreferences.getInstance();
expect(prefs.getBool('dark_mode'), isTrue);

flutter_secure_storage

flutter_secure_storage stores strings in the platform's secure enclave:

  • iOS: Keychain Services — encrypted by the Secure Enclave, locked when device is locked
  • Android: Android Keystore — keys stored in hardware-backed keystore on API 23+
  • macOS: Keychain
  • Windows: DPAPI (Data Protection API)
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

const storage = FlutterSecureStorage(
  aOptions: AndroidOptions(encryptedSharedPreferences: true),
  iOptions: IOSOptions(accessibility: KeychainAccessibility.first_unlock),
);

// Write a token
await storage.write(key: 'auth_token', value: 'Bearer eyJ...');

// Read
final token = await storage.read(key: 'auth_token');

// Delete
await storage.delete(key: 'auth_token');

// Delete all
await storage.deleteAll();
Tip

Use flutter_secure_storage exclusively for credentials and secrets. All other structured data should go through Hive, Drift, or Isar. Never store JWTs in SharedPreferences.

Hive — NoSQL for Flutter

Hive is a lightweight, pure-Dart NoSQL database stored as binary files on disk. It is extremely fast for reads because it keeps frequently accessed boxes in memory and uses a custom binary encoding rather than JSON or SQLite.

Setup and Opening a Box

import 'package:hive_flutter/hive_flutter.dart';

void main() async {
  await Hive.initFlutter(); // sets the documents directory as Hive root

  // Register type adapters before opening boxes
  Hive.registerAdapter(UserAdapter());
  Hive.registerAdapter(OrderAdapter());

  await Hive.openBox<User>('users');
  await Hive.openBox<String>('settings');

  runApp(const MyApp());
}

TypeAdapters for Custom Objects

Hive cannot serialize arbitrary Dart objects — you must generate a TypeAdapter. Use the hive_generator package:

import 'package:hive/hive.dart';

part 'user.g.dart'; // generated

@HiveType(typeId: 0) // typeId must be unique across all adapters
class User extends HiveObject {
  @HiveField(0)
  late String id;

  @HiveField(1)
  late String name;

  @HiveField(2)
  late String email;
}

# Generate the adapter
dart run build_runner build

CRUD Operations

final box = Hive.box<User>('users');

// Write — key can be String or int
await box.put('user-1', User()
  ..id = 'user-1'
  ..name = 'Alice'
  ..email = 'alice@example.com');

// Read
final user = box.get('user-1');

// Update (User extends HiveObject)
user?.name = 'Alice Smith';
await user?.save();

// Delete
await box.delete('user-1');

// All values
final allUsers = box.values.toList();

// Listen to changes
box.watch(key: 'user-1').listen((event) {
  print('User changed: ${event.value}');
});

Lazy Boxes for Large Data

A regular box loads all values into memory when opened. For large datasets, use a lazy box — values are read from disk on demand:

final lazyBox = await Hive.openLazyBox<Message>('messages');

// Reading from a lazy box is async
final message = await lazyBox.get('msg-1');

Hive Encrypted Box

// Generate a 32-byte encryption key and store it securely
final secureStorage = const FlutterSecureStorage();
var encryptionKey = await secureStorage.read(key: 'hive_key');

if (encryptionKey == null) {
  final key = Hive.generateSecureKey();
  await secureStorage.write(
    key: 'hive_key',
    value: base64UrlEncode(key),
  );
  encryptionKey = base64UrlEncode(key);
}

final keyBytes = base64Url.decode(encryptionKey);

// Open the box with encryption
final encryptedBox = await Hive.openBox<String>(
  'secure_data',
  encryptionCipher: HiveAesCipher(keyBytes),
);

Drift — Type-Safe SQLite ORM

Drift (formerly Moor) is a type-safe reactive SQLite wrapper. You define your schema in Dart, and Drift generates query classes, statement builders, and data classes. It is the right choice when you need relational data, complex queries, or reactive streams from a local database.

Defining Tables

import 'package:drift/drift.dart';

// Tables are plain Dart classes
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text().withLength(min: 1, max: 100)();
  TextColumn get email => text().unique()();
  DateTimeColumn get createdAt => dateTime().withDefault(currentDateAndTime)();
}

class Orders extends Table {
  IntColumn get id => integer().autoIncrement()();
  IntColumn get userId => integer().references(Users, #id)();
  RealColumn get total => real()();
  TextColumn get status => text().withDefault(const Constant('pending'))();
}

Database Class

import 'package:drift/drift.dart';
import 'package:drift/native.dart';
import 'package:path_provider/path_provider.dart';
import 'package:path/path.dart' as p;

part 'app_database.g.dart'; // generated

@DriftDatabase(tables: [Users, Orders])
class AppDatabase extends _$AppDatabase {
  AppDatabase() : super(_openConnection());

  @override
  int get schemaVersion => 2;

  @override
  MigrationStrategy get migration => MigrationStrategy(
    onCreate: (m) async {
      await m.createAll();
    },
    onUpgrade: (m, from, to) async {
      await m.runMigrationSteps(
        from: from,
        to: to,
        steps: migrationSteps(
          from1To2: (m, schema) async {
            // Add a new column in migration from v1 to v2
            await m.addColumn(schema.orders, schema.orders.status);
          },
        ),
      );
    },
    beforeOpen: (details) async {
      // Enable foreign key enforcement
      await customStatement('PRAGMA foreign_keys = ON');
    },
  );
}

LazyDatabase _openConnection() {
  return LazyDatabase(() async {
    final dir = await getApplicationDocumentsDirectory();
    final file = File(p.join(dir.path, 'app.sqlite'));
    return NativeDatabase.createInBackground(file);
  });
}

Generated Queries and CRUD

// Insert
Future<int> insertUser(UsersCompanion user) => into(users).insert(user);

// Usage
final id = await db.insertUser(UsersCompanion.insert(
  name: 'Alice',
  email: 'alice@example.com',
));

// Select all
Future<List<User>> getAllUsers() => select(users).get();

// Select with filter
Future<User?> getUserByEmail(String email) {
  return (select(users)..where((u) => u.email.equals(email))).getSingleOrNull();
}

// Update
Future<bool> updateUser(User user) => update(users).replace(user);

// Delete
Future<int> deleteUser(int id) {
  return (delete(users)..where((u) => u.id.equals(id))).go();
}

Reactive Queries with watch()

This is Drift's killer feature. watch() returns a Stream that emits a new value whenever the underlying table changes — perfect for driving reactive UI with StreamBuilder or Riverpod StreamProvider.

// Returns a Stream that emits whenever the users table changes
Stream<List<User>> watchAllUsers() => select(users).watch();

// Watch a single row
Stream<User?> watchUser(int id) {
  return (select(users)..where((u) => u.id.equals(id))).watchSingleOrNull();
}

// In the widget
StreamBuilder<List<User>>(
  stream: db.watchAllUsers(),
  builder: (context, snapshot) {
    if (!snapshot.hasData) return const CircularProgressIndicator();
    final users = snapshot.data!;
    return ListView.builder(
      itemCount: users.length,
      itemBuilder: (context, i) => ListTile(title: Text(users[i].name)),
    );
  },
)

Joins

// Join users and orders
Future<List<TypedResult>> getUsersWithOrders() {
  return (select(users)
    ..join([
      leftOuterJoin(orders, orders.userId.equalsExp(users.id)),
    ])
  ).get();
}

Drift Isolate — Background Database Access

For heavy database operations that should not block the UI, use DriftIsolate:

// Spawn the database on a background isolate
Future<AppDatabase> openDatabaseInBackground() async {
  final receivePort = ReceivePort();

  await Isolate.spawn(_startBackground, receivePort.sendPort);

  final sendPort = await receivePort.first as SendPort;
  final connection = await DriftIsolate.fromConnectPort(sendPort).connect();
  return AppDatabase.connect(connection);
}

void _startBackground(SendPort port) {
  final isolate = DriftIsolate.inCurrent(_openConnection);
  port.send(isolate.connectPort);
}

Isar — High-Performance NoSQL

Isar is a newer NoSQL database for Flutter that aims to combine Hive's speed with more powerful querying. Key differentiators:

  • No manual TypeAdapter writing — code generation handles everything
  • Built-in Isar Inspector (browser-based DB viewer during development)
  • Supports links and backlinks (like foreign keys for NoSQL)
  • Reactive queries via watchLazy() and watch()
  • Full-text search support
import 'package:isar/isar.dart';

part 'user.g.dart';

@collection
class User {
  Id id = Isar.autoIncrement;

  @Index(type: IndexType.value)
  late String email;

  late String name;

  // Link to another collection
  final orders = IsarLinks<Order>();
}

// Open the database
final isar = await Isar.open([UserSchema, OrderSchema]);

// Write
await isar.writeTxn(() async {
  await isar.users.put(User()
    ..name = 'Alice'
    ..email = 'alice@example.com');
});

// Read
final user = await isar.users.filter().emailEqualTo('alice@example.com').findFirst();

// Reactive query
isar.users.watchLazy().listen((_) {
  // Table changed — refetch
});
Stability note

Isar is not yet stable on all platforms (web support is limited, and there have been breaking API changes between major versions). Evaluate maturity carefully before using in a production app. Drift is the safer choice for teams that need long-term stability and SQLite's full power.

Encrypted SQLite with Drift

For apps that need full database encryption (e.g. healthcare, finance), use sqlcipher_flutter_libs with Drift:

# pubspec.yaml
dependencies:
  drift: ^2.14.0
  sqlcipher_flutter_libs: ^0.5.0  # Android + iOS
  sqlite3_flutter_libs: ^0.5.0    # non-encrypted fallback

import 'package:sqlcipher_flutter_libs/sqlcipher_flutter_libs.dart';
import 'package:drift/native.dart';

LazyDatabase _openEncryptedConnection(String passphrase) {
  return LazyDatabase(() async {
    final dir = await getApplicationDocumentsDirectory();
    final file = File(p.join(dir.path, 'encrypted.sqlite'));

    // Apply SQLCipher encryption key
    return NativeDatabase.createInBackground(
      file,
      setup: (rawDb) {
        rawDb.execute("PRAGMA key = '$passphrase'");
      },
    );
  });
}

Migrations with Drift

Drift's migration system uses schema versioning. The schemaVersion integer in your database class is the current version. When the app detects the on-disk version is lower, it runs the onUpgrade strategy.

@override
MigrationStrategy get migration => MigrationStrategy(
  onCreate: (m) => m.createAll(),
  onUpgrade: (m, from, to) async {
    if (from < 2) {
      // v1 -> v2: add profile_picture column to users
      await m.addColumn(users, users.profilePicture);
    }
    if (from < 3) {
      // v2 -> v3: create tags table
      await m.createTable(tags);
    }
  },
  beforeOpen: (details) async {
    if (details.wasCreated) {
      // Seed initial data
      await into(settings).insert(SettingsCompanion.insert(theme: 'light'));
    }
  },
);
Staff-level insight

Always generate and commit a drift_schemas/ directory using dart run drift_dev schema dump lib/database.dart drift_schemas/. This records the exact schema at each version, allowing Drift to verify your migration logic is correct with verifyAll() in tests. Without this, schema migration bugs only surface in production upgrades.

Testing Local Storage

Testing Drift with In-Memory Database

import 'package:drift/native.dart';
import 'package:flutter_test/flutter_test.dart';

AppDatabase createInMemoryDatabase() {
  // NativeDatabase.memory() — fast, no file I/O, isolated per test
  return AppDatabase.connect(
    DatabaseConnection(NativeDatabase.memory()),
  );
}

void main() {
  late AppDatabase db;

  setUp(() {
    db = createInMemoryDatabase();
  });

  tearDown(() async {
    await db.close();
  });

  test('inserts and retrieves a user', () async {
    final id = await db.insertUser(UsersCompanion.insert(
      name: 'Alice',
      email: 'alice@test.com',
    ));

    final user = await db.getUserByEmail('alice@test.com');
    expect(user?.id, equals(id));
    expect(user?.name, equals('Alice'));
  });

  test('watchAllUsers emits when user is added', () async {
    final stream = db.watchAllUsers();

    // First emission — empty list
    expect(stream, emits(isEmpty));

    await db.insertUser(UsersCompanion.insert(name: 'Bob', email: 'bob@test.com'));

    // Second emission — one user
    expect(stream, emits(hasLength(1)));
  });
}

Testing Hive with Temp Directory

import 'dart:io';
import 'package:hive/hive.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:path/path.dart' as p;

void main() {
  late Directory tempDir;

  setUp(() async {
    tempDir = await Directory.systemTemp.createTemp('hive_test_');
    Hive.init(tempDir.path);
    Hive.registerAdapter(UserAdapter());
    await Hive.openBox<User>('users');
  });

  tearDown(() async {
    await Hive.close();
    await tempDir.delete(recursive: true);
  });

  test('stores and retrieves a user', () async {
    final box = Hive.box<User>('users');
    await box.put('u1', User()..id = 'u1'..name = 'Alice'..email = 'a@b.com');

    final retrieved = box.get('u1');
    expect(retrieved?.name, equals('Alice'));
  });
}
