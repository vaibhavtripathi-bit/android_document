Networking & Data — Section Index

HTTP · Serialization · Local Storage · Offline-First Architecture

Production Flutter apps require more than a simple http.get() call. This section covers the full data layer stack: configuring a robust HTTP client with retries and auth, generating type-safe serialization code, choosing and using the right local storage solution, and designing an offline-first architecture that works when the network doesn't. These topics span from L4 basics to L6 system design and are asked in nearly every senior Flutter interview.

The offline-first section in particular reflects the architectural complexity that separates mid-level from staff-level engineers — managing sync queues, conflict resolution, and reactive local-first data flow requires genuine design judgment, not just API knowledge.

Section 01

HTTP & Dio

Dio setup, interceptor chain, retry logic, token refresh on 401, request cancellation, multipart uploads, timeout configuration, and testing HTTP calls with mock adapters.
Section 02

Serialization & Code Generation

json_serializable and Freezed with build_runner. Generated file structure, custom converters for DateTime and enums, sealed union types for API response states, and Dart 3 pattern matching.
Section 03

Local Storage

Comparison of SharedPreferences, flutter_secure_storage, Hive, Isar, and Drift. When to use each, Drift reactive queries, schema migrations, and DAO patterns for a clean data layer.
Section 04

Offline-First Architecture

Local DB as source of truth. Repository pattern, cache-then-network, optimistic updates with rollback, durable sync queue with exponential backoff, conflict resolution strategies, and delta sync with cursors.
Section 05

Interview Q&A

18+ Q&A pairs covering L4–L6 questions: Dio interceptors, Hive vs Drift, token refresh, json_serializable internals, secure storage backends, offline-first system design, conflict resolution, Freezed sealed classes, and certificate pinning.
Study Order
If you are comfortable with HTTP basics, start with section 03 (Local Storage) and 04 (Offline-First) — these are where L6 interview questions focus. Sections 01 and 02 are essential foundations but are less likely to be the primary interview topic at Staff level. End with 05 (Q&A) to test recall across the full section.
