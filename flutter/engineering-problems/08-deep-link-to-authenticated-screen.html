08 — Deep Link → Auth Gate → Intended Destination

Problem: Deep link to /orders/123/detail — handle auth, cold start, and warm start

Problem Statement

The app receives a deep link: https://app.example.com/orders/123/detail. The user may be in any of these states:

User stateApp stateExpected behaviour
Logged inCold start (app not running)Launch → navigate directly to order 123 detail
Logged inWarm start (app backgrounded)Bring to foreground → push order 123 detail
Logged outCold startLaunch → show login → after login, navigate to order 123 detail
Logged outWarm startBring to foreground → show login → after login, navigate to order 123 detail
Staff Signal
Interviewers expect you to distinguish cold start (platform passes the link as the initial route) from warm start (platform calls onNewIntent / openURL while the engine is alive). The pending route pattern for post-login redirect is well-known; the cold-start wiring through GoRouter's initialLocation is less commonly known.

Auth State (Riverpod)

// lib/src/auth/auth_state.dart

@freezed
class AuthState with _$AuthState {
  const factory AuthState({
    required bool isAuthenticated,
    String? pendingRoute,  // deep link destination saved before login
    User? user,
  }) = _AuthState;
}

// lib/src/auth/auth_notifier.dart

@riverpod
class AuthNotifier extends _$AuthNotifier {
  @override
  AuthState build() {
    // Restore auth state from secure storage on startup.
    final token = ref.read(secureStorageProvider).read('auth_token');
    return AuthState(isAuthenticated: token != null);
  }

  void setPendingRoute(String route) {
    state = state.copyWith(pendingRoute: route);
  }

  void clearPendingRoute() {
    state = state.copyWith(pendingRoute: null);
  }

  Future<void> login(String email, String password) async {
    final user = await ref.read(authRepositoryProvider).login(email, password);
    await ref.read(secureStorageProvider).write('auth_token', user.token);
    // Setting isAuthenticated triggers GoRouterRefreshStream,
    // which causes GoRouter to re-evaluate its redirect function.
    state = state.copyWith(isAuthenticated: true, user: user);
  }

  Future<void> logout() async {
    await ref.read(secureStorageProvider).delete('auth_token');
    state = const AuthState(isAuthenticated: false);
  }
}

GoRouter Configuration

// lib/src/router/app_router.dart

@riverpod
GoRouter appRouter(AppRouterRef ref) {
  final authState = ref.watch(authNotifierProvider);

  return GoRouter(
    // Cold start: if the app was launched via a deep link, the platform
    // sets PlatformDispatcher.defaultRouteName to the link path.
    // GoRouter reads this automatically as the initial location.
    initialLocation: '/',

    // refreshListenable causes GoRouter to re-run redirect() whenever
    // auth state changes (e.g., after successful login).
    refreshListenable: GoRouterRefreshStream(
      ref.watch(authNotifierProvider.stream),
    ),

    redirect: (BuildContext context, GoRouterState state) {
      final auth = ref.read(authNotifierProvider);
      final isLoggedIn = auth.isAuthenticated;
      final isAuthRoute = state.matchedLocation == '/login';

      if (!isLoggedIn && !isAuthRoute) {
        // Store the intended destination before redirecting to login.
        ref
            .read(authNotifierProvider.notifier)
            .setPendingRoute(state.uri.toString());
        return '/login';
      }

      if (isLoggedIn && isAuthRoute) {
        // After login, consume the pending route (or fall back to home).
        final pending = auth.pendingRoute;
        ref.read(authNotifierProvider.notifier).clearPendingRoute();
        return pending ?? '/home';
      }

      // No redirect needed.
      return null;
    },

    routes: [
      GoRoute(
        path: '/',
        redirect: (_, __) => '/home',
      ),
      GoRoute(
        path: '/home',
        builder: (context, state) => const HomeScreen(),
      ),
      GoRoute(
        path: '/login',
        builder: (context, state) => const LoginScreen(),
      ),
      GoRoute(
        path: '/orders/:orderId/detail',
        builder: (context, state) {
          final orderId = state.pathParameters['orderId']!;
          return OrderDetailScreen(orderId: orderId);
        },
      ),
    ],
  );
}

GoRouterRefreshStream Helper

// Bridges a Dart Stream to a Listenable so GoRouter can react to
// async state changes (Riverpod streams, BLoC streams, etc.)
class GoRouterRefreshStream extends ChangeNotifier {
  late final StreamSubscription<dynamic> _subscription;

  GoRouterRefreshStream(Stream<dynamic> stream) {
    notifyListeners(); // trigger initial evaluation
    _subscription = stream.listen((_) => notifyListeners());
  }

  @override
  void dispose() {
    _subscription.cancel();
    super.dispose();
  }
}

Cold Start Wiring

On cold start, the platform encodes the deep link as the initial route before the Flutter engine starts. GoRouter reads PlatformDispatcher.instance.defaultRouteName automatically — it becomes the router's initial location. No extra code is needed if you are using GoRouter 6+.

The redirect function fires immediately for the initial location. If the user is logged out, it stores /orders/123/detail as the pending route and redirects to /login. After login, the pending route is consumed and the user lands on the order detail screen.

Warm Start Wiring

When the app is already running and a deep link arrives:

  • Android: the platform calls onNewIntent on the Activity, which Flutter forwards to the app_links package's link stream.
  • iOS: the platform calls application(_:open:options:) or application(_:continue:restorationHandler:), again forwarded by app_links.
// lib/src/router/deep_link_handler.dart

class DeepLinkHandler {
  final GoRouter router;

  DeepLinkHandler(this.router);

  StreamSubscription? _sub;

  void init() {
    final appLinks = AppLinks();

    // Handle link that launched the app (cold start fallback — GoRouter
    // handles this automatically, but you may want it here for logging).
    appLinks.getInitialLink().then((uri) {
      if (uri != null) {
        // GoRouter already handles this via initialLocation.
        // Log or track it here if needed.
        debugPrint('Cold start link: $uri');
      }
    });

    // Handle links arriving while the app is already running (warm start).
    _sub = appLinks.uriLinkStream.listen((uri) {
      // router.go() replaces the current stack and triggers redirect().
      router.go(uri.path +
          (uri.query.isNotEmpty ? '?${uri.query}' : ''));
    });
  }

  void dispose() => _sub?.cancel();
}
// Wire the handler in main.dart or via a Riverpod provider.

@riverpod
DeepLinkHandler deepLinkHandler(DeepLinkHandlerRef ref) {
  final router = ref.watch(appRouterProvider);
  final handler = DeepLinkHandler(router);
  handler.init();
  ref.onDispose(handler.dispose);
  return handler;
}

// In your app widget, force the provider to initialise early.
class App extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    ref.watch(deepLinkHandlerProvider); // initialise on first build
    final router = ref.watch(appRouterProvider);
    return MaterialApp.router(routerConfig: router);
  }
}

Login Screen: Consuming the Pending Route

The login screen does not need to know about routing. It simply calls authNotifier.login(). After the state update, GoRouterRefreshStream notifies GoRouter, which re-runs redirect(), reads the pending route, clears it, and navigates automatically. 

class LoginScreen extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    return Scaffold(
      body: Center(
        child: ElevatedButton(
          onPressed: () async {
            // After this completes, GoRouter redirect() fires automatically.
            await ref.read(authNotifierProvider.notifier).login(
              'user@example.com',
              'password123',
            );
            // No manual navigation needed here — the redirect handles it.
          },
          child: const Text('Log in'),
        ),
      ),
    );
  }
}

Edge Cases

Edge caseHandling
Deep link to a route that requires a specific resource permission (e.g., someone else's order)OrderDetailScreen fetches the order and shows a 403 error screen if the user doesn't own it — routing layer does not validate business-level permissions
Two deep links arriving in rapid succession (warm start)router.go() replaces the navigation stack — the second call wins, which is correct
Deep link with query params (/orders/123/detail?highlight=item456)GoRoute receives query params via state.uri.queryParameters; pass them to the screen as constructor arguments
App killed while pending route is storedPending route is in-memory only — lost on kill. For persistence, write it to SharedPreferences and read it back on cold start before auth check
User cancels loginPending route remains stored; if user navigates to home manually, clear it to avoid surprise redirect on next login

Testing

void main() {
  group('GoRouter redirect', () {
    test('redirects logged-out user to /login and stores pending route', () {
      final container = ProviderContainer(overrides: [
        authNotifierProvider.overrideWith(() => FakeAuthNotifier(loggedIn: false)),
      ]);

      final router = container.read(appRouterProvider);
      // Simulate navigating to the protected route.
      router.go('/orders/123/detail');

      // After redirect, location should be /login.
      expect(router.location, '/login');

      // Pending route should be stored.
      final auth = container.read(authNotifierProvider);
      expect(auth.pendingRoute, '/orders/123/detail');
    });

    test('redirects to pending route after login', () async {
      final container = ProviderContainer(overrides: [
        authNotifierProvider.overrideWith(
          () => FakeAuthNotifier(loggedIn: false, pendingRoute: '/orders/123/detail'),
        ),
      ]);

      final notifier = container.read(authNotifierProvider.notifier);
      await notifier.login('user@example.com', 'pass');

      final router = container.read(appRouterProvider);
      // After login, router should have consumed the pending route.
      expect(router.location, '/orders/123/detail');
      expect(container.read(authNotifierProvider).pendingRoute, isNull);
    });

    test('logged-in user reaches destination directly', () {
      final container = ProviderContainer(overrides: [
        authNotifierProvider.overrideWith(() => FakeAuthNotifier(loggedIn: true)),
      ]);

      final router = container.read(appRouterProvider);
      router.go('/orders/456/detail');

      expect(router.location, '/orders/456/detail');
    });
  });
}

// integration_test/deep_link_test.dart

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  testWidgets('cold start deep link lands on order detail after login',
      (tester) async {
    // Override auth to start logged out.
    await tester.pumpWidget(
      ProviderScope(
        overrides: [
          authNotifierProvider.overrideWith(
            () => FakeAuthNotifier(loggedIn: false),
          ),
        ],
        child: const App(),
      ),
    );

    // Simulate deep link navigation (warm start equivalent in tests).
    final router = tester
        .element(find.byType(App))
        .read(appRouterProvider);
    router.go('/orders/789/detail');

    await tester.pumpAndSettle();

    // Should land on login screen.
    expect(find.byType(LoginScreen), findsOneWidget);

    // Tap login button.
    await tester.tap(find.text('Log in'));
    await tester.pumpAndSettle();

    // Should now be on order detail.
    expect(find.byType(OrderDetailScreen), findsOneWidget);
    expect(find.text('Order #789'), findsOneWidget);
  });
}
Package Note
Use the app_links package (not the deprecated uni_links) for warm-start link handling. It correctly handles Android 12+ restrictions on ACTION_VIEW intents and iOS 13+ universal link callbacks. GoRouter's built-in deep link support handles cold start automatically if FlutterDeepLinkingEnabled is set in Info.plist.
