06 — Custom RenderObject: Radial Layout

Problem: Arrange children in a circle — Widget/Element/RenderObject from scratch

Problem Statement

Build a RadialLayout widget that positions N children evenly spaced around a circle of configurable radius, centered in the widget's bounding box. The implementation must:

  1. Work correctly for any number of children and any child size.
  2. Support hit testing — tapping a child fires its gesture recognizers.
  3. Support accessibility — each child must be reachable via semantics traversal.
  4. Allow custom starting angle and clockwise/counter-clockwise ordering.
Staff Signal
Most Flutter engineers never write a custom RenderObject. This problem tests whether you understand the three-layer architecture (Widget → Element → RenderObject), the difference between ContainerRenderObjectMixin and single-child render objects, and why ParentData is needed. Be ready to explain each layer's responsibility without conflating them.

The Three-Layer Architecture

LayerClassResponsibilityMutable?
WidgetRadialLayoutImmutable configuration. Describes what to build.No — rebuilt freely
ElementAuto-generated by MultiChildRenderObjectWidgetManages lifecycle, reconciles widget tree with render tree.Yes — long-lived
RenderObjectRenderRadialLayoutLayout, paint, hit test. The actual engine work.Yes — mutated by element
Key Insight
You never directly instantiate the Element — MultiChildRenderObjectWidget provides a MultiChildRenderObjectElement that handles child slot management. Your job is to implement the Widget and RenderObject layers only.

ParentData

ParentData stores per-child data that the parent render object needs about each child. It lives on the child's RenderObject but is owned and written by the parent. For radial layout, each child needs to know its computed offset — where to paint itself relative to the parent's origin. 

import 'package:flutter/rendering.dart';

// ParentData attached to each child RenderBox by the parent.
class RadialLayoutParentData extends ContainerBoxParentData<RenderBox> {
  // ContainerBoxParentData already provides:
  //   Offset offset        — where to paint this child
  //   RenderBox? nextSibling  — linked list of siblings
  //   RenderBox? previousSibling

  // We can add extra per-child data here if needed, e.g.:
  double angle = 0.0; // stored for debugging / accessibility label
}

Widget Layer

import 'dart:math';
import 'package:flutter/rendering.dart';
import 'package:flutter/widgets.dart';

class RadialLayout extends MultiChildRenderObjectWidget {
  final double radius;
  final double startAngle;  // radians, default -π/2 (top of circle)
  final bool clockwise;

  const RadialLayout({
    super.key,
    required super.children,
    required this.radius,
    this.startAngle = -pi / 2,
    this.clockwise = true,
  });

  @override
  RenderRadialLayout createRenderObject(BuildContext context) {
    return RenderRadialLayout(
      radius: radius,
      startAngle: startAngle,
      clockwise: clockwise,
    );
  }

  // Called when the widget rebuilds with new configuration.
  // Updates the existing RenderObject in place — no rebuild needed.
  @override
  void updateRenderObject(
    BuildContext context,
    RenderRadialLayout renderObject,
  ) {
    renderObject
      ..radius = radius
      ..startAngle = startAngle
      ..clockwise = clockwise;
  }
}

RenderObject Layer

class RenderRadialLayout extends RenderBox
    with
        ContainerRenderObjectMixin<RenderBox, RadialLayoutParentData>,
        RenderBoxContainerDefaultsMixin<RenderBox, RadialLayoutParentData> {

  RenderRadialLayout({
    required double radius,
    required double startAngle,
    required bool clockwise,
  })  : _radius = radius,
        _startAngle = startAngle,
        _clockwise = clockwise;

  // Each setter calls markNeedsLayout() so Flutter reflows when config changes.
  double _radius;
  double get radius => _radius;
  set radius(double value) {
    if (_radius == value) return;
    _radius = value;
    markNeedsLayout();
  }

  double _startAngle;
  double get startAngle => _startAngle;
  set startAngle(double value) {
    if (_startAngle == value) return;
    _startAngle = value;
    markNeedsLayout();
  }

  bool _clockwise;
  bool get clockwise => _clockwise;
  set clockwise(bool value) {
    if (_clockwise == value) return;
    _clockwise = value;
    markNeedsLayout();
  }

  // Tell Flutter what ParentData type to attach to each child.
  @override
  void setupParentData(RenderBox child) {
    if (child.parentData is! RadialLayoutParentData) {
      child.parentData = RadialLayoutParentData();
    }
  }

  // Compute intrinsic sizes — needed for widgets like IntrinsicWidth/Height.
  @override
  double computeMinIntrinsicWidth(double height) => _radius * 2;

  @override
  double computeMaxIntrinsicWidth(double height) => _radius * 2;

  @override
  double computeMinIntrinsicHeight(double width) => _radius * 2;

  @override
  double computeMaxIntrinsicHeight(double width) => _radius * 2;

  @override
  void performLayout() {
    // Radial layout takes as much space as constraints allow.
    size = constraints.biggest;
    final center = Offset(size.width / 2, size.height / 2);
    final n = childCount;

    if (n == 0) return;

    // Children are laid out with loose constraints — they size to their content.
    final childConstraints = BoxConstraints.loose(constraints.biggest);

    var child = firstChild;
    int i = 0;

    while (child != null) {
      child.layout(childConstraints, parentUsesSize: true);

      // Distribute children evenly around the circle.
      final fraction = i / n;
      final angle = _startAngle +
          (_clockwise ? 1 : -1) * 2 * pi * fraction;

      final parentData = child.parentData as RadialLayoutParentData;
      parentData.angle = angle;

      // Position the child so its center lands on the circle.
      parentData.offset = Offset(
        center.dx + _radius * cos(angle) - child.size.width / 2,
        center.dy + _radius * sin(angle) - child.size.height / 2,
      );

      child = parentData.nextSibling;
      i++;
    }
  }

  // Paint all children at their computed offsets.
  // defaultPaint iterates the child linked list and calls paintChild for each.
  @override
  void paint(PaintingContext context, Offset offset) {
    defaultPaint(context, offset);
  }

  // Hit test children in reverse paint order (last painted = topmost = first hit).
  // defaultHitTestChildren walks the linked list in reverse.
  @override
  bool hitTestChildren(BoxHitTestResult result, {required Offset position}) {
    return defaultHitTestChildren(result, position: position);
  }

  // Accessibility: expose all children to the semantics tree.
  // Without this, children are invisible to screen readers.
  @override
  void visitChildrenForSemantics(RenderObjectVisitor visitor) {
    var child = firstChild;
    while (child != null) {
      visitor(child);
      child = (child.parentData as RadialLayoutParentData).nextSibling;
    }
  }
}

Usage Example

class ActionWheel extends StatelessWidget {
  final List<ActionItem> items;

  @override
  Widget build(BuildContext context) {
    return SizedBox(
      width: 300,
      height: 300,
      child: Stack(children: [
        // Center hub
        const Center(child: CircleAvatar(radius: 32, child: Icon(Icons.person))),

        // Radial action buttons
        RadialLayout(
          radius: 110,
          startAngle: -pi / 2, // start at top
          clockwise: true,
          children: items.map((item) => _ActionButton(item: item)).toList(),
        ),
      ]),
    );
  }
}

class _ActionButton extends StatelessWidget {
  final ActionItem item;

  @override
  Widget build(BuildContext context) {
    return Semantics(
      label: item.label,       // accessibility label for screen reader
      button: true,
      child: GestureDetector(
        onTap: item.onTap,
        child: Container(
          width: 48,
          height: 48,
          decoration: BoxDecoration(
            color: item.color,
            shape: BoxShape.circle,
            boxShadow: [BoxShadow(color: Colors.black26, blurRadius: 4)],
          ),
          child: Icon(item.icon, color: Colors.white),
        ),
      ),
    );
  }
}

Testing the RenderObject

void main() {
  testWidgets('RadialLayout positions children evenly', (tester) async {
    const radius = 100.0;

    await tester.pumpWidget(
      Center(
        child: SizedBox(
          width: 300,
          height: 300,
          child: RadialLayout(
            radius: radius,
            children: List.generate(
              4,
              (i) => SizedBox(key: ValueKey(i), width: 20, height: 20),
            ),
          ),
        ),
      ),
    );

    // For 4 children starting at top (-π/2), positions should be:
    // child 0: top    (150, 50)   — center.x + 100*cos(-π/2) = 150, center.y + 100*sin(-π/2) = 50
    // child 1: right  (250, 150)
    // child 2: bottom (150, 250)
    // child 3: left   (50, 150)

    final child0 = tester.getTopLeft(find.byKey(const ValueKey(0)));
    expect(child0.dx, closeTo(140, 2)); // 150 - 20/2 = 140 (child center at 150)
    expect(child0.dy, closeTo(40, 2));  // 50 - 20/2 = 40

    final child2 = tester.getTopLeft(find.byKey(const ValueKey(2)));
    expect(child2.dy, closeTo(240, 2)); // bottom
  });

  testWidgets('hit test reaches child tap handlers', (tester) async {
    bool tapped = false;
    await tester.pumpWidget(
      Center(
        child: SizedBox(
          width: 300,
          height: 300,
          child: RadialLayout(
            radius: 100,
            children: [
              GestureDetector(
                onTap: () => tapped = true,
                child: const SizedBox(width: 40, height: 40),
              ),
            ],
          ),
        ),
      ),
    );

    // Child should be at top of circle — tap there.
    await tester.tapAt(const Offset(150, 50));
    expect(tapped, isTrue);
  });
}

Common Mistakes to Avoid

MistakeSymptomFix
Forgetting setupParentDataClassCastException at runtime when casting to RadialLayoutParentDataAlways override setupParentData and check type before assigning
Not calling markNeedsLayout() in settersLayout does not update when radius changesEvery mutable property setter must call markNeedsLayout() or markNeedsPaint()
Calling child.layout() without parentUsesSize: trueParent uses stale child size in offset calculationPass parentUsesSize: true whenever you read child.size after layout
Overriding paint() without calling defaultPaintChildren not paintedUse defaultPaint(context, offset) from the mixin, or iterate manually
Missing visitChildrenForSemanticsChildren invisible to TalkBack/VoiceOverOverride and visit all children
Staff Signal
Know the difference between markNeedsLayout() (triggers performLayout next frame), markNeedsPaint() (triggers paint only — cheaper), and markNeedsSemanticsUpdate(). Using markNeedsLayout() when only paint changed is a correctness-preserving but unnecessary performance penalty.

Extension: Animating the Radius

// Because RenderRadialLayout exposes a setter with markNeedsLayout,
// you can drive it directly from an AnimationController without
// rebuilding any widgets.

class AnimatedRadialLayout extends StatefulWidget {
  final List<Widget> children;
  final double maxRadius;

  @override
  State<AnimatedRadialLayout> createState() => _AnimatedRadialLayoutState();
}

class _AnimatedRadialLayoutState extends State<AnimatedRadialLayout>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _radius;
  // Hold a reference to the render object for direct mutation.
  final _key = GlobalKey();

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 600),
      vsync: this,
    );
    _radius = Tween(begin: 0.0, end: widget.maxRadius).animate(
      CurvedAnimation(parent: _controller, curve: Curves.elasticOut),
    );
    _radius.addListener(() {
      final ro = _key.currentContext?.findRenderObject() as RenderRadialLayout?;
      ro?.radius = _radius.value; // direct mutation — no setState needed
    });
    _controller.forward();
  }

  @override
  Widget build(BuildContext context) {
    return RadialLayout(
      key: _key,
      radius: widget.maxRadius,
      children: widget.children,
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }
}
