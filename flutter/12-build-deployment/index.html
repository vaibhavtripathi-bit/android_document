Key Concepts Summary

Build Modes in One Line Each

  • Debug: JIT + hot reload + debug overlays + VM service — slow startup, large binary, full dev tools.
  • Profile: AOT + profiling hooks — measures real performance without debug overhead.
  • Release: AOT + tree-shaking + obfuscation — smallest binary, fastest startup, no debug hooks.

Flavor vs --dart-define

  • Flavors (Android productFlavors / iOS schemes): configure native side — app ID, app name, icons, Firebase config files, native resource variants.
  • --dart-define: configures the Dart side — API endpoints, feature flags, environment name. Available at compile time via String.fromEnvironment().
  • Both are typically combined: flavor selects the right native config, --dart-define selects the right Dart config.

Code Signing Essentials

  • Android: Keystore → upload key (you hold) → Google Play signs release APK (Play App Signing). Don't lose the upload key.
  • iOS: Signing certificate (proves developer identity) + Provisioning Profile (lists allowed devices/entitlements). Fastlane match syncs both via Git repo or cloud storage.

CI/CD Non-Negotiables

  • Never commit signing secrets to source control — use CI environment variables or secret managers.
  • Always run tests in CI before building a release artifact.
  • Use App Bundle (not APK) for Play Store — enables Dynamic Delivery and per-ABI optimization.
  • Increment build number (pubspec version+build) automatically in CI to ensure unique build identifiers per run.
