Series 1: Flutter Architecture

From the Dart VM to Skia rendering — the internal mechanics every Staff-level Flutter engineer must master for Google L6 interviews.

This series deconstructs Flutter's layered architecture at the depth expected for a Google L6 / Staff-level interview. We trace how Dart code becomes pixels on screen through the Framework, Engine, and Embedder layers. Each document focuses on internals that distinguish a Staff engineer from a Senior one.

Documents in This Series

01 — Architecture Overview

Flutter's layered architecture: Framework (Dart), Engine (C++), Embedder. How these layers communicate and their responsibilities.

02 — Flutter Engine

Engine internals: Skia graphics library, Dart runtime, platform channels, text rendering, and the shell that ties it together.

03 — Dart VM & Runtime

Dart VM architecture, JIT vs AOT compilation, snapshot types, isolate model, garbage collection, and memory management.

04 — Skia & Impeller

Graphics rendering: Skia 2D engine, GPU acceleration, shader compilation, and Impeller — Flutter's next-gen renderer.

05 — Embedder & Platform

Platform embedders for iOS, Android, Web, Desktop. Surface management, input handling, lifecycle, and platform runners.

06 — Compilation Modes

Debug, profile, release modes. JIT hot reload, AOT compilation, tree shaking, code size optimization.

07 — Frame Rendering Pipeline

Complete frame lifecycle: vsync, build, layout, paint, composite. GPU thread, raster thread, platform thread coordination.

08 — Interview Q&A

55+ L6-level interview questions covering Flutter architecture, Engine, Dart VM, Skia, embedders, and compilation.
