Part 1 — Core Concept
Explicit Animation Model
You own the AnimationController. You drive timing, sequencing, and physics. Full control — full responsibility.

1. Why Explicit Animations?

Explicit animations give you direct control over an AnimationController. You decide when the animation starts, stops, reverses, repeats, and what value it drives. This is necessary for:

  • Looping animations (spinners, pulsing indicators)
  • Gesture-driven animations where controller.value tracks a drag offset
  • Staggered or sequenced multi-step animations with precise timing
  • Physics simulations (spring, friction, gravity)
  • Animations coordinated across multiple widgets via a shared controller

The tradeoff: you must dispose the controller. A forgotten controller.dispose() leaks a Ticker, which continues scheduling frames indefinitely and drains battery.

2. AnimatedBuilder

AnimatedBuilder is the idiomatic way to rebuild a subtree in response to an animation without lifting the AnimationController into the parent's setState. It listens to a Listenable (typically an Animation) and rebuilds only its builder subtree on each tick.

2.1 Core Pattern

class _SpinnerState extends State<Spinner>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,
      duration: const Duration(seconds: 2),
    )..repeat(); // loop forever
  }

  @override
  void dispose() {
    _controller.dispose(); // critical — stops the Ticker
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedBuilder(
      animation: _controller,
      // child is built once and passed into builder every frame
      child: const Icon(Icons.sync, size: 48),
      builder: (context, child) {
        return Transform.rotate(
          angle: _controller.value * 2 * 3.14159,
          child: child,
        );
      },
    );
  }
}

2.2 Why AnimatedBuilder over setState

If you called _controller.addListener(() => setState(() {})) in the parent, the entire parent widget tree rebuilds every frame — potentially rebuilding expensive sibling widgets, header bars, or list items. AnimatedBuilder scopes the rebuild to just the builder subtree. The child optimization further limits work: the child expression is evaluated once, not on every frame.

// AVOID: entire parent rebuilds every frame
class _BadParentState extends State<BadParent>
    with SingleTickerProviderStateMixin {
  late AnimationController _ctrl;

  @override
  void initState() {
    super.initState();
    _ctrl = AnimationController(vsync: this, duration: const Duration(seconds: 1))
      ..addListener(() => setState(() {})); // rebuilds ALL children
  }

  @override
  Widget build(BuildContext context) {
    return Column(children: [
      const ExpensiveSiblingWidget(), // rebuilt every frame — BAD
      Transform.scale(scale: _ctrl.value, child: const AnimatedTarget()),
    ]);
  }
}

// PREFER: only Transform.scale's subtree rebuilds
class _GoodParentState extends State<GoodParent>
    with SingleTickerProviderStateMixin {
  late AnimationController _ctrl;

  @override
  void initState() {
    super.initState();
    _ctrl = AnimationController(vsync: this, duration: const Duration(seconds: 1))
      ..forward();
  }

  @override
  Widget build(BuildContext context) {
    return Column(children: [
      const ExpensiveSiblingWidget(), // never rebuilt by animation
      AnimatedBuilder(
        animation: _ctrl,
        builder: (context, child) =>
            Transform.scale(scale: _ctrl.value, child: child),
        child: const AnimatedTarget(), // built once
      ),
    ]);
  }
}

3. AnimatedWidget

AnimatedWidget is an alternative to AnimatedBuilder for creating reusable animated components. It is a StatelessWidget subclass that takes a Listenable and rebuilds whenever it notifies. The controller is still owned externally.

// Reusable rotating widget — controller comes from parent
class RotatingWidget extends AnimatedWidget {
  final Widget child;

  const RotatingWidget({
    super.key,
    required Animation<double> animation,
    required this.child,
  }) : super(listenable: animation);

  // Convenience getter — casts the listenable back to Animation<double>
  Animation<double> get _animation => listenable as Animation<double>;

  @override
  Widget build(BuildContext context) {
    return Transform.rotate(
      angle: _animation.value * 2 * 3.14159,
      child: child,
    );
  }
}

// Usage — parent owns the controller and the tween
class _ParentState extends State<Parent> with SingleTickerProviderStateMixin {
  late final AnimationController _ctrl;
  late final Animation<double> _animation;

  @override
  void initState() {
    super.initState();
    _ctrl = AnimationController(vsync: this, duration: const Duration(seconds: 1))
      ..repeat();
    // Apply a curve to the raw controller value
    _animation = CurvedAnimation(parent: _ctrl, curve: Curves.easeInOut);
  }

  @override
  Widget build(BuildContext context) {
    return RotatingWidget(animation: _animation, child: const FlutterLogo());
  }
}
AnimatedBuilder vs AnimatedWidget
Use AnimatedWidget when building a reusable animated component that will be used in many places. Use AnimatedBuilder for one-off in-place animations. The functional difference is minimal — both scope rebuilds to the animated subtree. AnimatedBuilder is more common in production code due to its inline nature.

4. Staggered Animations with Interval

Staggered animations run multiple property animations in sequence or with overlap, all driven by a single AnimationController. The key tool is Interval — a Curve that maps a sub-range of the controller's 0.0–1.0 range to 0.0–1.0, and outputs 0.0 outside that range.

4.1 Interval Mechanics

// Interval(begin, end, curve: ...)
// At controller.value = 0.0 → Interval output = 0.0
// At controller.value = 0.5 (midpoint of 0.2-0.8) → output = 0.5
// At controller.value = 1.0 → Interval output = 1.0

final Animation<double> _fadeIn = CurvedAnimation(
  parent: _controller,
  curve: const Interval(0.0, 0.4, curve: Curves.easeIn),
);

final Animation<double> _slideUp = CurvedAnimation(
  parent: _controller,
  curve: const Interval(0.2, 0.7, curve: Curves.easeOut),
);

final Animation<double> _scale = CurvedAnimation(
  parent: _controller,
  curve: const Interval(0.5, 1.0, curve: Curves.elasticOut),
);

4.2 Full Staggered Animation Example

class _StaggeredCardState extends State<StaggeredCard>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;

  // Phase 1 (0.0 – 0.3): card fades and slides in
  late final Animation<double> _opacity;
  late final Animation<Offset> _slideOffset;

  // Phase 2 (0.3 – 0.65): title text appears
  late final Animation<double> _titleOpacity;

  // Phase 3 (0.55 – 1.0): action button scales in with bounce
  late final Animation<double> _buttonScale;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 900),
    );

    _opacity = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.0, 0.3, curve: Curves.easeIn),
      ),
    );

    _slideOffset = Tween<Offset>(
      begin: const Offset(0, 0.15),
      end: Offset.zero,
    ).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.0, 0.4, curve: Curves.easeOut),
      ),
    );

    _titleOpacity = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.3, 0.65, curve: Curves.easeIn),
      ),
    );

    _buttonScale = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.55, 1.0, curve: Curves.elasticOut),
      ),
    );

    _controller.forward();
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedBuilder(
      animation: _controller,
      builder: (context, _) {
        return FadeTransition(
          opacity: _opacity,
          child: SlideTransition(
            position: _slideOffset,
            child: Card(
              child: Padding(
                padding: const EdgeInsets.all(16),
                child: Column(
                  crossAxisAlignment: CrossAxisAlignment.start,
                  children: [
                    FadeTransition(
                      opacity: _titleOpacity,
                      child: const Text('Card Title',
                          style: TextStyle(fontSize: 20, fontWeight: FontWeight.bold)),
                    ),
                    const SizedBox(height: 16),
                    ScaleTransition(
                      scale: _buttonScale,
                      child: ElevatedButton(
                        onPressed: () {},
                        child: const Text('Action'),
                      ),
                    ),
                  ],
                ),
              ),
            ),
          ),
        );
      },
    );
  }
}

5. TweenSequence<T>

TweenSequence chains multiple tweens end-to-end along the controller's timeline, with configurable relative weights. It is an alternative to Interval for sequential (non-overlapping) animations.

// Animate color through a sequence: red → yellow → green
final Animation<Color?> _colorSequence = TweenSequence<Color?>([
  TweenSequenceItem(
    tween: ColorTween(begin: Colors.red, end: Colors.yellow),
    weight: 40, // takes 40% of the timeline
  ),
  TweenSequenceItem(
    tween: ColorTween(begin: Colors.yellow, end: Colors.green),
    weight: 60, // takes 60% of the timeline
  ),
]).animate(_controller);

// Chained scale: grow → shrink → grow
final Animation<double> _scalePulse = TweenSequence<double>([
  TweenSequenceItem(
    tween: Tween(begin: 1.0, end: 1.5)
        .chain(CurveTween(curve: Curves.easeOut)),
    weight: 50,
  ),
  TweenSequenceItem(
    tween: Tween(begin: 1.5, end: 1.0)
        .chain(CurveTween(curve: Curves.easeIn)),
    weight: 50,
  ),
]).animate(_controller);

5.1 .chain() for Curves on Individual Segments

The .chain(CurveTween(curve: Curves.easeIn)) call composes a curve onto a single tween segment. This is equivalent to wrapping the tween's output through a curve transformation, and it applies within that segment's weight range only.

6. AnimationController.drive() and Tween.animate()

These two APIs are mirror images of each other. Both produce an Animation<T> that maps the controller's 0.0–1.0 value through a tween.

final AnimationController _ctrl = AnimationController(
  vsync: this, duration: const Duration(milliseconds: 500));

// Method 1: tween.animate(controller) — traditional
final Animation<double> _anim1 =
    Tween<double>(begin: 0, end: 300).animate(_ctrl);

// Method 2: controller.drive(tween) — chain-friendly
final Animation<double> _anim2 =
    _ctrl.drive(Tween<double>(begin: 0, end: 300));

// controller.drive shines in chains:
final Animation<double> _anim3 = _ctrl
    .drive(CurveTween(curve: Curves.easeInOut))   // apply curve first
    .drive(Tween<double>(begin: 50, end: 250));   // then map to range
L6 Insight
drive() reads left-to-right — the controller drives the curve, which drives the tween. animate() reads right-to-left: the tween is animated by the controller. Both are semantically equivalent. Prefer drive() in code review feedback because it composes in reading order, reducing cognitive load for complex chains.

7. Physics Simulations

Physics simulations replace the simple linear/curved interpolation of controller.forward() with physically-modeled dynamics. You pass a Simulation to controller.animateWith(simulation), and the controller queries the simulation for position and velocity each frame until simulation.isDone(time) returns true.

7.1 SpringSimulation

import 'package:flutter/physics.dart';

// Spring parameters
const SpringDescription spring = SpringDescription(
  mass: 1.0,
  stiffness: 180.0,  // higher = faster oscillation
  damping: 20.0,     // higher = less oscillation (more damped)
);

// Critical damping: damping = 2 * sqrt(stiffness * mass) = ~26.8
// Under-damped (damping < critical): oscillates before settling
// Over-damped (damping > critical): slow, no oscillation
// Critically-damped: fastest settling without oscillation

void _launchSpring() {
  final simulation = SpringSimulation(
    spring,
    _controller.value, // start position (0.0–1.0)
    1.0,               // end position
    0.0,               // initial velocity
  );
  _controller.animateWith(simulation);
}

// Gesture-driven spring: match finger, then spring to snap point
void _onPanEnd(DragEndDetails details) {
  final velocity = details.velocity.pixelsPerSecond.dy;
  final normalizedVelocity = velocity / MediaQuery.of(context).size.height;

  final simulation = SpringSimulation(
    const SpringDescription(mass: 1, stiffness: 200, damping: 25),
    _controller.value,
    _controller.value > 0.5 ? 1.0 : 0.0, // snap to nearest
    normalizedVelocity,                    // preserve fling velocity
  );
  _controller.animateWith(simulation);
}

7.2 FrictionSimulation

// Models deceleration — like a ball rolling on a surface
// friction: 0.0 = no friction, higher = stops faster
void _fling(double velocity) {
  final simulation = FrictionSimulation(
    0.015,            // friction coefficient (drag)
    _controller.value, // initial position
    velocity,          // initial velocity (normalized, px/s / screen_height)
  );
  _controller.animateWith(simulation);
}

// Typical use: after a fling gesture on a custom draggable widget
void _onPanEnd(DragEndDetails details) {
  _controller.animateWith(
    FrictionSimulation(
      0.02,
      _controller.value,
      details.velocity.pixelsPerSecond.dx / 1000.0,
    ),
  );
}

7.3 GravitySimulation

// Models constant acceleration (gravity)
// acceleration: positive = falls down, negative = launches up
void _dropBall() {
  final simulation = GravitySimulation(
    9.8,               // acceleration (units/s²)
    _controller.value, // start position
    1.0,               // end position (floor)
    0.0,               // initial velocity
  );
  _controller.animateWith(simulation);
}

7.4 Custom Simulation

// Implement Simulation for fully custom physics
class BouncingSimulation extends Simulation {
  final double _start;
  final double _velocity;

  BouncingSimulation(this._start, this._velocity);

  @override
  double x(double time) {
    // Simple bounce: parabolic arc reflected at 0 and 1
    double pos = _start + _velocity * time - 4.9 * time * time;
    return pos.clamp(0.0, 1.0);
  }

  @override
  double dx(double time) {
    // Derivative of position = velocity
    return _velocity - 9.8 * time;
  }

  @override
  bool isDone(double time) => time > 2.0 || x(time) <= 0.0;
}

8. Curves: Built-in and Custom

A Curve maps an input in [0,1] to an output, typically also in [0,1] but not required (elasticOut exceeds 1.0). Curves are applied via CurvedAnimation.

8.1 Key Built-in Curves

CurveCharacterUse Case
Curves.easeInSlow start, fast endElements exiting the screen
Curves.easeOutFast start, slow endElements entering the screen
Curves.easeInOutSlow start and endGeneral transitions
Curves.elasticOutOvershoots then settlesPlayful entrance animations
Curves.bounceOutBounces at endDropping/landing objects
Curves.fastOutSlowInMaterial standardMaterial Design transitions
Curves.linearConstant velocityLooping progress indicators
Curves.decelerateStarts fast, deceleratesMomentum-style panning

8.2 Custom Cubic Bezier Curve

// Cubic(x1, y1, x2, y2) — cubic bezier control points
// CSS equivalent: cubic-bezier(x1, y1, x2, y2)

const Curve myCustomCurve = Cubic(0.17, 0.67, 0.83, 0.67);
// Creates an S-shaped curve

// For Material You motion spec:
const Curve emphasizedCurve = Cubic(0.2, 0.0, 0.0, 1.0);
// Fast initial movement that settles smoothly

9. Sequencing with AnimationStatusListener

For true sequential animations (A finishes → B starts), use addStatusListener to detect when one animation completes and trigger the next:

class _SequentialAnimState extends State<SequentialAnim>
    with TickerProviderStateMixin {

  late final AnimationController _phase1;
  late final AnimationController _phase2;
  late final AnimationController _phase3;

  @override
  void initState() {
    super.initState();

    _phase1 = AnimationController(vsync: this,
        duration: const Duration(milliseconds: 400));
    _phase2 = AnimationController(vsync: this,
        duration: const Duration(milliseconds: 300));
    _phase3 = AnimationController(vsync: this,
        duration: const Duration(milliseconds: 500));

    // Chain: when phase 1 completes, start phase 2
    _phase1.addStatusListener((status) {
      if (status == AnimationStatus.completed) _phase2.forward();
    });

    // When phase 2 completes, start phase 3
    _phase2.addStatusListener((status) {
      if (status == AnimationStatus.completed) _phase3.forward();
    });

    // Start the sequence
    _phase1.forward();
  }

  @override
  void dispose() {
    _phase1.dispose();
    _phase2.dispose();
    _phase3.dispose();
    super.dispose();
  }
}
Caution: Multiple Controllers
Each AnimationController owns a Ticker. Use TickerProviderStateMixin (not SingleTickerProviderStateMixin) when you need multiple controllers. Forgetting this causes a runtime assertion error.

10. Spring-Based Pan Gesture Physics

A common pattern for draggable sheets, dismissible cards, and bottom panels: track finger with the controller's value, then spring to a snap point on release.

class _DraggablePanelState extends State<DraggablePanel>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;
  double _dragStart = 0;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(vsync: this, lowerBound: 0, upperBound: 1);
  }

  void _onPanStart(DragStartDetails details) {
    _controller.stop(); // stop any ongoing spring
    _dragStart = details.globalPosition.dy;
  }

  void _onPanUpdate(DragUpdateDetails details) {
    final screenH = MediaQuery.of(context).size.height;
    final delta = (details.globalPosition.dy - _dragStart) / screenH;
    _controller.value = (_controller.value + delta).clamp(0.0, 1.0);
    _dragStart = details.globalPosition.dy;
  }

  void _onPanEnd(DragEndDetails details) {
    final velocity = details.velocity.pixelsPerSecond.dy /
        MediaQuery.of(context).size.height;

    // Spring to nearest snap point, preserving fling velocity
    final target = _controller.value > 0.5 ? 1.0 : 0.0;
    _controller.animateWith(
      SpringSimulation(
        const SpringDescription(mass: 1, stiffness: 200, damping: 28),
        _controller.value,
        target,
        velocity,
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return GestureDetector(
      onPanStart: _onPanStart,
      onPanUpdate: _onPanUpdate,
      onPanEnd: _onPanEnd,
      child: AnimatedBuilder(
        animation: _controller,
        builder: (context, child) {
          final h = MediaQuery.of(context).size.height;
          return Transform.translate(
            offset: Offset(0, _controller.value * h * 0.7),
            child: child,
          );
        },
        child: const BottomPanel(),
      ),
    );
  }
}

11. Complete Staggered List Entrance Example

class StaggeredList extends StatefulWidget {
  final List<String> items;
  const StaggeredList({required this.items});

  @override
  State<StaggeredList> createState() => _StaggeredListState();
}

class _StaggeredListState extends State<StaggeredList>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,
      duration: Duration(milliseconds: 200 + widget.items.length * 80),
    )..forward();
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return ListView.builder(
      itemCount: widget.items.length,
      itemBuilder: (context, index) {
        // Each item occupies a window of the total animation timeline
        final itemStart = index / widget.items.length;
        final itemEnd = (index + 1) / widget.items.length;

        final opacity = Tween<double>(begin: 0.0, end: 1.0).animate(
          CurvedAnimation(
            parent: _controller,
            curve: Interval(itemStart, itemEnd, curve: Curves.easeOut),
          ),
        );

        final slide = Tween<Offset>(
          begin: const Offset(0.3, 0),
          end: Offset.zero,
        ).animate(
          CurvedAnimation(
            parent: _controller,
            curve: Interval(itemStart, itemEnd, curve: Curves.easeOut),
          ),
        );

        return FadeTransition(
          opacity: opacity,
          child: SlideTransition(
            position: slide,
            child: ListTile(title: Text(widget.items[index])),
          ),
        );
      },
    );
  }
}
L6 Design Consideration
When interviewing at L6, be able to discuss the trade-off between multiple controllers (simpler per-phase logic, higher overhead) vs. single controller with Intervals (more efficient, requires careful interval math). For most UI animations, a single controller with Intervals is preferred. Multiple controllers are justified when animations run on different durations that cannot be normalized (e.g., a loading animation that runs for an indeterminate period, then a completion animation of fixed duration).
