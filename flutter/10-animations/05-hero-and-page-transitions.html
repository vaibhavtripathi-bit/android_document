Part 1 — Hero Internals
How Hero Flight Works
Hero widgets lift matched elements to an Overlay, animate them across the screen, then land them in the destination route — all coordinated by HeroController.

1. Hero Widget Fundamentals

The Hero widget creates a shared element transition between two routes. A widget tagged with the same tag on both the source and destination routes appears to "fly" from its position on the source to its position on the destination during the route transition.

// Source route — a grid of product cards
Hero(
  tag: 'product-image-${product.id}', // must be unique across all visible heroes
  child: Image.network(product.imageUrl, width: 100, height: 100),
)

// Destination route — product detail page
Hero(
  tag: 'product-image-${product.id}', // same tag = matched pair
  child: Image.network(product.imageUrl, width: double.infinity, height: 300),
)

The tag is an Object, not necessarily a String. You can use any object that correctly implements equality — often the model object itself or a typed record.

2. HeroController: The NavigatorObserver

HeroController is a NavigatorObserver registered with the Navigator. It intercepts every push and pop, scans the widget tree for matched Hero tags, and orchestrates the flight animation. MaterialApp automatically installs a HeroController — you only need to add one manually when using a bare Navigator or a nested navigator.

// Bare Navigator with HeroController — needed for nested navigators
Navigator(
  observers: [HeroController()],
  onGenerateRoute: (settings) => MaterialPageRoute(
    builder: (_) => const MyScreen(),
  ),
)

// MaterialApp — HeroController is automatic
MaterialApp(
  // HeroController already registered internally
  home: const HomeScreen(),
)

2.1 What HeroController Does on Push

  1. Intercepts didPush(Route newRoute, Route? previousRoute)
  2. Scans the from route's subtree for all Hero widgets, recording their global positions via RenderBox.localToGlobal
  3. Scans the to route's subtree for matching tags
  4. For each matched pair, inserts a "shuttle" widget into the Navigator's Overlay — this is the widget that visually flies across the screen
  5. Hides the source and destination Hero originals (replaced with a transparent placeholder) while the shuttle is in flight
  6. Animates the shuttle from the source Rect to the destination Rect using a RectTween
  7. On completion, removes the shuttle and reveals the destination Hero in place

3. _HeroState and createRectTween

Each Hero widget's state holds a reference to its RenderBox to compute its global position at the moment of the navigation event. The createRectTween property on both the Hero widget and the HeroController determines how the intermediate rectangles are computed during the flight.

3.1 MaterialRectArcTween vs RectTween

// Default in MaterialApp: MaterialRectArcTween
// The hero follows a curved arc path rather than a straight diagonal line.
// This looks more natural because the center point of the hero
// follows a circular arc, mimicking how objects move in physical space.

// Linear RectTween — straight line from source to destination
Hero(
  tag: 'my-hero',
  createRectTween: (begin, end) => RectTween(begin: begin, end: end),
  child: const MyWidget(),
)

// Arc tween — curved path (default Material behavior)
Hero(
  tag: 'my-hero',
  createRectTween: (begin, end) =>
      MaterialRectArcTween(begin: begin, end: end),
  child: const MyWidget(),
)

// Custom tween — e.g., always slide from left
Hero(
  tag: 'my-hero',
  createRectTween: (begin, end) {
    // begin: source rect in global coordinates
    // end: destination rect in global coordinates
    // Return any Tween<Rect?> to control the flight path
    return RectTween(begin: begin, end: end);
  },
  child: const MyWidget(),
)
L6 Depth
MaterialRectArcTween decomposes the source and destination rects into center + size, interpolates the center along a circular arc (the arc is computed so that both endpoints are on the circle), and lerps the size linearly. This produces the characteristic Material Design "shared axis" motion where the hero seems to glide naturally. When both source and destination have the same x or y coordinate, the arc collapses to a straight line — understanding this edge case shows deep knowledge.

4. flightShuttleBuilder

By default, the flying shuttle widget is a copy of the destination Hero's child, scaled to fit the animating rect. flightShuttleBuilder lets you replace the shuttle with any widget — useful for cross-fading between source and destination appearances.

Hero(
  tag: 'avatar-${user.id}',
  flightShuttleBuilder: (
    BuildContext flightContext,
    Animation<double> animation,    // 0.0 at source, 1.0 at destination
    HeroFlightDirection flightDirection, // push or pop
    BuildContext fromHeroContext,
    BuildContext toHeroContext,
  ) {
    // Cross-fade between source and destination during flight
    return AnimatedBuilder(
      animation: animation,
      builder: (context, _) {
        return Stack(
          fit: StackFit.expand,
          children: [
            // Source appearance fades out
            Opacity(
              opacity: 1.0 - animation.value,
              child: fromHeroContext.widget, // the source Hero's child
            ),
            // Destination appearance fades in
            Opacity(
              opacity: animation.value,
              child: toHeroContext.widget,
            ),
          ],
        );
      },
    );
  },
  child: CircleAvatar(backgroundImage: NetworkImage(user.avatarUrl)),
)

5. placeholderBuilder

While the hero is in flight, the original widget in the source route is replaced by a placeholder (transparent by default). placeholderBuilder lets you show something in that space during the transition:

Hero(
  tag: 'product-${product.id}',
  placeholderBuilder: (context, heroSize, child) {
    // heroSize: the size of the hero at the moment flight started
    // Show a shimmer/skeleton while the original is "in flight"
    return SizedBox.fromSize(
      size: heroSize,
      child: Container(
        decoration: BoxDecoration(
          color: Colors.grey[200],
          borderRadius: BorderRadius.circular(8),
        ),
      ),
    );
  },
  child: ProductImage(product: product),
)

6. HeroMode: Disabling Heroes Conditionally

// Wrap a subtree to disable Hero animations within it
// Useful when you want to suppress heroes during certain transitions
// (e.g., tab changes within the same route)
HeroMode(
  enabled: !_isTabbing, // disable when switching tabs
  child: Hero(
    tag: 'my-element',
    child: const MyWidget(),
  ),
)

7. Custom PageRoute: buildTransitions

Every route's visual entry/exit animation is defined by overriding buildTransitions in a PageRoute subclass. The framework calls this method with the animation (0→1 on push, 1→0 on pop) and passes the built child page widget.

7.1 Slide Transition Route

class SlideRightRoute extends PageRouteBuilder {
  final Widget page;

  SlideRightRoute({required this.page})
      : super(
          pageBuilder: (context, animation, secondaryAnimation) => page,
          transitionsBuilder: (context, animation, secondaryAnimation, child) {
            // animation: this route's entrance/exit (0→1 push, 1→0 pop)
            // secondaryAnimation: secondary route's entrance (used for push-off effect)
            const begin = Offset(1.0, 0.0); // enter from right
            const end = Offset.zero;

            final slideTween = Tween(begin: begin, end: end)
                .chain(CurveTween(curve: Curves.easeInOut));
            final offsetAnimation = animation.drive(slideTween);

            // Current route slides off to the left on push
            final secondarySlide = Tween(
              begin: Offset.zero,
              end: const Offset(-0.3, 0.0),
            ).animate(CurvedAnimation(
                parent: secondaryAnimation, curve: Curves.easeInOut));

            return SlideTransition(
              position: offsetAnimation,
              child: SlideTransition(
                position: secondarySlide,
                child: child,
              ),
            );
          },
          transitionDuration: const Duration(milliseconds: 350),
        );
}

7.2 Scale + Fade Route

class ScaleFadeRoute<T> extends PageRoute<T> {
  final WidgetBuilder builder;
  final Duration _duration;

  ScaleFadeRoute({required this.builder, Duration duration = const Duration(milliseconds: 300)})
      : _duration = duration;

  @override
  Color? get barrierColor => null;

  @override
  String? get barrierLabel => null;

  @override
  bool get maintainState => true;

  @override
  Duration get transitionDuration => _duration;

  @override
  Widget buildPage(BuildContext context, Animation<double> animation,
      Animation<double> secondaryAnimation) {
    return builder(context);
  }

  @override
  Widget buildTransitions(BuildContext context, Animation<double> animation,
      Animation<double> secondaryAnimation, Widget child) {
    final curved = CurvedAnimation(parent: animation, curve: Curves.easeOut);
    return FadeTransition(
      opacity: curved,
      child: ScaleTransition(
        scale: Tween<double>(begin: 0.92, end: 1.0).animate(curved),
        child: child,
      ),
    );
  }
}

// Usage
Navigator.of(context).push(ScaleFadeRoute(builder: (_) => const DetailScreen()));

8. PageTransitionsTheme

PageTransitionsTheme sets the default route transition per platform. This is the correct way to globally customize transitions without wrapping every push call.

MaterialApp(
  theme: ThemeData(
    pageTransitionsTheme: const PageTransitionsTheme(
      builders: {
        TargetPlatform.android: ZoomPageTransitionsBuilder(), // Material 3 zoom
        TargetPlatform.iOS: CupertinoPageTransitionsBuilder(), // iOS swipe-back
        TargetPlatform.macOS: CupertinoPageTransitionsBuilder(),
      },
    ),
  ),
  home: const HomeScreen(),
)

// Built-in PageTransitionsBuilder options:
// FadeUpwardsPageTransitionsBuilder — Material 2 default (Android pre-M3)
// ZoomPageTransitionsBuilder — Material 3 default (zoom in from center)
// CupertinoPageTransitionsBuilder — iOS slide from right
// OpenUpwardsPageTransitionsBuilder — slides up from bottom

9. Material Motion: SharedAxisTransition and FadeThroughTransition

The animations package (pub.dev) provides Material Motion spec-compliant transitions. These are more polished than hand-rolled transitions and match the official spec.

// pubspec.yaml
// dependencies:
//   animations: ^2.0.0

import 'package:animations/animations.dart';

// SharedAxisTransition: coordinated movement along X, Y, or Z axis
SharedAxisTransition(
  animation: animation,
  secondaryAnimation: secondaryAnimation,
  transitionType: SharedAxisTransitionType.horizontal, // or vertical, scaled
  child: child,
)

// FadeThroughTransition: fade out old, fade in new — for unrelated destinations
FadeThroughTransition(
  animation: animation,
  secondaryAnimation: secondaryAnimation,
  child: child,
)

// FadeScaleTransition: for modal dialogs appearing from nothing
FadeScaleTransition(animation: animation, child: child)

// OpenContainer: for container-transform (card expands to full screen)
OpenContainer(
  closedBuilder: (context, openContainer) =>
      Card(child: ListTile(title: const Text('Tap me'), onTap: openContainer)),
  openBuilder: (context, closeContainer) =>
      DetailScreen(onClose: closeContainer),
  transitionDuration: const Duration(milliseconds: 400),
  transitionType: ContainerTransitionType.fade, // or fadeThrough
)

10. Implementing Shared Element Without Hero

Sometimes you need a Hero-like effect but cannot use the Hero widget (e.g., animating between widgets in the same route, or across tab boundaries). The manual approach uses Overlay and coordinate conversion.

// Step 1: Get the source widget's global rect
RenderBox? box = sourceKey.currentContext?.findRenderObject() as RenderBox?;
if (box == null) return;
final sourceOffset = box.localToGlobal(Offset.zero);
final sourceSize = box.size;
final sourceRect = sourceOffset & sourceSize;

// Step 2: Insert a clone into the Overlay at the source position
late OverlayEntry _entry;
_entry = OverlayEntry(
  builder: (context) => AnimatedBuilder(
    animation: _controller,
    builder: (context, child) {
      // Lerp rect from source to destination
      final currentRect = RectTween(
        begin: sourceRect,
        end: destinationRect,
      ).evaluate(_controller)!;

      return Positioned(
        left: currentRect.left,
        top: currentRect.top,
        width: currentRect.width,
        height: currentRect.height,
        child: Material(
          color: Colors.transparent,
          child: FlyingWidget(progress: _controller.value),
        ),
      );
    },
  ),
);

Overlay.of(context).insert(_entry);

// Step 3: Run animation
await _controller.forward();

// Step 4: Remove overlay entry and show destination widget
_entry.remove();
setState(() => _showDestination = true);

11. Hero with Different Widget Types

// Source: small circular avatar
Hero(
  tag: 'user-avatar',
  child: CircleAvatar(
    radius: 24,
    backgroundImage: NetworkImage(user.avatarUrl),
  ),
)

// Destination: large rectangular header image
// The flutter framework stretches/fits the source widget
// to fill the destination size during flight
Hero(
  tag: 'user-avatar',
  // flightShuttleBuilder handles the shape mismatch gracefully
  flightShuttleBuilder: (_, animation, direction, from, to) {
    final Image image = Image.network(
      user.avatarUrl,
      fit: BoxFit.cover,
    );
    return AnimatedBuilder(
      animation: animation,
      builder: (context, _) => ClipRRect(
        borderRadius: BorderRadius.lerp(
          const BorderRadius.all(Radius.circular(24)),
          BorderRadius.zero,
          animation.value,
        )!,
        child: image,
      ),
    );
  },
  child: Image.network(user.avatarUrl, fit: BoxFit.cover, height: 300),
)

12. Common Hero Pitfalls

ProblemCauseFix
Hero doesn't animateTags don't match, or HeroController missing on nested NavigatorVerify tag equality; add HeroController() to nested Navigator observers
Hero clips to wrong shape during flightSource/destination have different clip constraintsUse flightShuttleBuilder to interpolate the clip shape
Multiple heroes with same tag crashTag not unique (e.g., using a non-unique string in a list)Include a unique id: tag: 'product-${product.id}'
Hero overshoots its destinationMaterialRectArcTween arc exceeds bounds on small distancesUse RectTween (linear) for short-distance flights
Hero not visible during flight in nested navigatorOverlay is scoped to the navigator — shuttle appears below the parentUse a root-level navigator or restructure routes
Hero pops back wrong (jumps)Destination widget size changed after push completedEnsure destination layout is stable before pop; avoid layout-affecting async loads
L6 Design Discussion
At L6, you should be able to explain why Hero uses an Overlay rather than animating within the route's own widget tree: the hero must visually transcend both the source and destination route layers, which are stacked in the Navigator's overlay. Only by inserting the shuttle into the Navigator's Overlay at the top of the stack can it appear above both routes simultaneously during the transition. This is a constraint imposed by the rendering architecture, not an arbitrary choice.

13. Custom Transition: iOS-style Modal Sheet

class ModalBottomSheetRoute<T> extends PageRoute<T> {
  final WidgetBuilder builder;

  ModalBottomSheetRoute({required this.builder});

  @override
  Color get barrierColor => Colors.black54;

  @override
  String get barrierLabel => 'Modal barrier';

  @override
  bool get barrierDismissible => true;

  @override
  bool get maintainState => true;

  @override
  Duration get transitionDuration => const Duration(milliseconds: 350);

  @override
  Widget buildPage(BuildContext context, Animation<double> animation,
      Animation<double> secondaryAnimation) {
    return builder(context);
  }

  @override
  Widget buildTransitions(BuildContext context, Animation<double> animation,
      Animation<double> secondaryAnimation, Widget child) {
    return SlideTransition(
      position: Tween<Offset>(
        begin: const Offset(0, 1), // start from bottom
        end: Offset.zero,
      ).animate(CurvedAnimation(
        parent: animation,
        curve: Curves.fastOutSlowIn,
        reverseCurve: Curves.easeIn,
      )),
      child: child,
    );
  }
}
Summary
Hero uses an Overlay-based flight mechanism coordinated by HeroController (a NavigatorObserver). The flight path follows a MaterialRectArcTween by default. Customize the shuttle appearance with flightShuttleBuilder, the flight path with createRectTween, and the source placeholder with placeholderBuilder. For global transition customization use PageTransitionsTheme; for per-push control subclass PageRoute and override buildTransitions.
