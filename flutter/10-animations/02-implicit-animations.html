Part 1 — Core Concept
The Implicit Animation Philosophy
Animate by describing the desired end state, not the animation mechanics. Flutter interpolates automatically.

1. What Are Implicit Animations?

Implicit animations are the simplest animation model Flutter provides. The key insight: you change a value, Flutter animates to it. There is no AnimationController you manage, no Tween you configure manually, no addListener you set up — all of that machinery is hidden inside the widget.

The mental model is declarative: instead of saying "start from A, animate to B over 300ms", you say "I want the value to be B" and Flutter transitions from whatever it currently is to B using the provided duration and optional curve.

This makes implicit animations ideal for:

  • Toggling UI state (expanded/collapsed, selected/unselected)
  • Responding to data changes where animation is incidental polish
  • Simple transitions that don't require precise timing control or sequencing
  • Reducing boilerplate for common interpolatable properties
Key mental model
Implicit animations are to explicit animations what setState is to StreamController. Use the simpler tool unless you need precise control.

2. ImplicitlyAnimatedWidget Base Class Internals

All Flutter built-in implicit animation widgets extend ImplicitlyAnimatedWidget, which itself is a StatefulWidget. Its companion State class is ImplicitlyAnimatedWidgetState<T>. Understanding how this base class works demystifies every widget in the family.

2.1 The AnimationController It Hides

Inside ImplicitlyAnimatedWidgetState, a private AnimationController _controller is created in initState() using the widget's duration and reverseDuration. This controller is ticked by a Ticker obtained from TickerProviderStateMixin — the state mixin used automatically. When the widget is disposed, the controller is disposed too, preventing ticker leaks.

// Conceptual internals of ImplicitlyAnimatedWidgetState
class ImplicitlyAnimatedWidgetState<T extends ImplicitlyAnimatedWidget>
    extends State<T> with SingleTickerProviderStateMixin {

  late AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: widget.duration,
      vsync: this,
    );
    _constructTweens();
    // If there are tweens to animate (not the initial build), forward
  }

  @override
  void didUpdateWidget(T oldWidget) {
    super.didUpdateWidget(oldWidget);
    // If duration changed, update controller
    if (widget.duration != oldWidget.duration) {
      _controller.duration = widget.duration;
    }
    // Key: reconstruct tweens from old value to new value
    if (_constructTweens()) {
      // There are changes — animate forward
      _controller
        ..value = 0.0
        ..forward();
    }
  }

  bool _constructTweens() {
    // Each subclass calls forEachTween() to update its tweens
    // Returns true if any tween has a non-trivial change
    bool shouldStartAnimation = false;
    forEachTween((Tween<dynamic>? tween, dynamic targetValue,
        TweenConstructor<dynamic> constructor) {
      if (targetValue != null) {
        tween ??= constructor(targetValue);
        if (_shouldAnimateTween(tween, targetValue)) {
          shouldStartAnimation = true;
        }
        // Set begin to current value (lerped position), end to target
        tween.end = targetValue;
      }
      return tween;
    });
    return shouldStartAnimation;
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }
}

2.2 The forEachTween Pattern

Each concrete subclass (like _AnimatedContainerState) overrides forEachTween to declare which properties it interpolates and how. This is the extension point the base class provides:

// Inside _AnimatedContainerState
@override
void forEachTween(TweenVisitor<dynamic> visitor) {
  _color = visitor(
    _color,                          // current tween (nullable on first call)
    widget.color,                    // target value
    (dynamic value) => ColorTween(begin: value as Color?),  // constructor
  ) as ColorTween?;

  _width = visitor(
    _width,
    widget.width,
    (dynamic value) => Tween<double>(begin: value as double?),
  ) as Tween<double>?;
  // ... same pattern for all animatable properties
}

2.3 The Lerp Cycle

Each frame, the animation controller's value (0.0 to 1.0, modified by the curve) is used to call tween.evaluate(animation), which returns the interpolated value. This is combined with the widget's other properties and used to build the actual rendering widget. The key is: begin is always the current rendered value at the moment a new animation starts, not the original starting value. This allows smooth interruption — if you change the target mid-animation, the new animation starts from wherever the widget currently is.

// How the animation value flows to the rendered widget
// (conceptual — actual implementation in subclass build())
@override
Widget build(BuildContext context) {
  return Container(
    color: _color?.evaluate(animation),       // animation = CurvedAnimation
    width: _width?.evaluate(animation),
    height: _height?.evaluate(animation),
    // ... other properties
    child: widget.child,
  );
}
L6 Depth
When asked about implicit animation internals, explain the forEachTween visitor pattern: it lets the base class orchestrate tween lifecycle (creation, begin-update, evaluation) while subclasses declare what to animate. This is the Strategy pattern applied to animation. Also note that begin is updated to the current lerped value on each didUpdateWidget call — enabling smooth mid-animation direction changes without jumps.

3. AnimatedContainer

AnimatedContainer is the workhorse of implicit animations. It animates virtually every property of a Container: width, height, color, decoration, padding, margin, alignment, constraints, and transform.

3.1 Basic Usage

class _ExpandableCardState extends State<ExpandableCard> {
  bool _expanded = false;

  @override
  Widget build(BuildContext context) {
    return GestureDetector(
      onTap: () => setState(() => _expanded = !_expanded),
      child: AnimatedContainer(
        duration: const Duration(milliseconds: 300),
        curve: Curves.easeInOut,
        width: _expanded ? 300.0 : 150.0,
        height: _expanded ? 200.0 : 80.0,
        decoration: BoxDecoration(
          color: _expanded ? Colors.blue : Colors.grey[200],
          borderRadius: BorderRadius.circular(_expanded ? 16.0 : 8.0),
          boxShadow: _expanded
              ? [BoxShadow(color: Colors.blue.withOpacity(0.3), blurRadius: 12)]
              : [],
        ),
        padding: EdgeInsets.all(_expanded ? 16.0 : 8.0),
        child: const Text('Tap me'),
      ),
    );
  }
}

3.2 What AnimatedContainer Cannot Do

AnimatedContainer only interpolates properties that have a defined lerp function. Some BoxDecoration properties — particularly complex gradient combinations and certain border types — may not interpolate smoothly. When lerp returns null for incompatible types (e.g., switching from LinearGradient to RadialGradient), the animation snaps rather than interpolates.

Common gotcha
Switching between a solid color (color: Colors.blue) and a gradient (decoration: BoxDecoration(gradient: ...)) will throw an assertion: you cannot set both color and decoration. If animating decoration, move the color into the BoxDecoration.color property.

4. TweenAnimationBuilder<T>

TweenAnimationBuilder is the escape hatch within implicit animations: it lets you create a custom implicit animation for any type T that has a Tween, without creating a full custom ImplicitlyAnimatedWidget subclass.

4.1 Signature and Behavior

TweenAnimationBuilder<double>(
  tween: Tween<double>(begin: 0.0, end: 1.0),
  duration: const Duration(milliseconds: 500),
  curve: Curves.easeOut,
  builder: (BuildContext context, double value, Widget? child) {
    // Called every frame with the current interpolated value
    return Opacity(
      opacity: value,
      child: child,  // child is the optional pre-built subtree
    );
  },
  child: const ExpensiveSubtree(), // built once, reused across frames
);

4.2 The child Optimization

The optional child parameter is critical for performance. The builder is called on every animation frame. Any widget subtree that does not depend on the animated value should be passed as child rather than constructed inside the builder. Flutter builds this child once and passes the same widget instance to every builder invocation, allowing the framework to avoid unnecessary rebuilds of the subtree.

// Inefficient: ExpensiveList rebuilt every frame
TweenAnimationBuilder<double>(
  tween: Tween(begin: 0.0, end: 1.0),
  duration: const Duration(milliseconds: 600),
  builder: (context, value, _) {
    return Transform.scale(
      scale: value,
      child: const ExpensiveList(), // BAD: rebuilt every frame
    );
  },
);

// Efficient: ExpensiveList built once
TweenAnimationBuilder<double>(
  tween: Tween(begin: 0.0, end: 1.0),
  duration: const Duration(milliseconds: 600),
  builder: (context, value, child) {
    return Transform.scale(scale: value, child: child);
  },
  child: const ExpensiveList(), // GOOD: built once, reused
);

4.3 Animating Non-Standard Types

// Animate between two colors using ColorTween
TweenAnimationBuilder<Color?>(
  tween: ColorTween(begin: Colors.red, end: Colors.blue),
  duration: const Duration(seconds: 1),
  builder: (context, color, child) {
    return Container(color: color, child: child);
  },
  child: const SomeContent(),
);

// Animate a custom value object with a custom Tween
class PriceDisplayTween extends Tween<double> {
  PriceDisplayTween({required double begin, required double end})
      : super(begin: begin, end: end);
}

// Usage — animates the price counter value
TweenAnimationBuilder<double>(
  tween: PriceDisplayTween(begin: 0, end: widget.price),
  duration: const Duration(milliseconds: 800),
  curve: Curves.easeOut,
  builder: (context, value, _) {
    return Text('\$${value.toStringAsFixed(2)}',
        style: const TextStyle(fontSize: 24));
  },
);

4.4 Reacting to Target Changes

When the tween's end value changes across widget rebuilds, TweenAnimationBuilder automatically animates from the current value to the new target. This mirrors the ImplicitlyAnimatedWidget.didUpdateWidget behavior. The begin of the new tween is effectively overridden to be the current animated value, ensuring smooth transitions.

// Drives a progress bar — animates whenever progress changes
class ProgressIndicatorWidget extends StatelessWidget {
  final double progress; // 0.0 to 1.0

  const ProgressIndicatorWidget({required this.progress});

  @override
  Widget build(BuildContext context) {
    return TweenAnimationBuilder<double>(
      tween: Tween<double>(end: progress), // begin omitted = starts at 0
      duration: const Duration(milliseconds: 400),
      curve: Curves.easeInOut,
      builder: (context, value, _) {
        return LinearProgressIndicator(value: value);
      },
    );
  }
}

5. AnimatedSwitcher

AnimatedSwitcher animates the transition when its child widget changes. It maintains old and new children simultaneously, running an exit animation on the old and an entrance animation on the new.

5.1 Key Requirement: Widget Keys

The most important thing to understand about AnimatedSwitcher: it determines that the child has "changed" by comparing the new child's runtimeType and key. If you replace a Text("A") with a Text("B"), Flutter sees the same runtime type with no key and considers them the same widget — no animation occurs. You must use a ValueKey to force AnimatedSwitcher to treat them as different:

AnimatedSwitcher(
  duration: const Duration(milliseconds: 300),
  child: Text(
    _count.toString(),
    key: ValueKey<int>(_count), // CRITICAL: different key = new child
    style: const TextStyle(fontSize: 48),
  ),
);

5.2 Default Transition and transitionBuilder

// Default: FadeTransition
AnimatedSwitcher(
  duration: const Duration(milliseconds: 400),
  transitionBuilder: (Widget child, Animation<double> animation) {
    // Called for BOTH incoming and outgoing children
    // 'animation' goes 0→1 for incoming, 1→0 for outgoing
    return FadeTransition(opacity: animation, child: child);
  },
  child: _current,
);

// Custom: slide from bottom, fade out upward
AnimatedSwitcher(
  duration: const Duration(milliseconds: 350),
  transitionBuilder: (child, animation) {
    final offsetAnimation = Tween<Offset>(
      begin: const Offset(0, 0.3),
      end: Offset.zero,
    ).animate(CurvedAnimation(parent: animation, curve: Curves.easeOut));

    return FadeTransition(
      opacity: animation,
      child: SlideTransition(position: offsetAnimation, child: child),
    );
  },
  layoutBuilder: (currentChild, previousChildren) {
    // Control how current and previous children stack
    return Stack(
      alignment: Alignment.center,
      children: [
        ...previousChildren,
        if (currentChild != null) currentChild,
      ],
    );
  },
  child: _current,
);

5.3 layoutBuilder

The layoutBuilder parameter controls how the outgoing and incoming children are positioned relative to each other during the transition. The default stacks all children centered. A common customization is to size the switcher to match the current child rather than growing to the maximum of all children:

layoutBuilder: (Widget? currentChild, List<Widget> previousChildren) {
  return Stack(
    alignment: Alignment.topCenter,
    children: <Widget>[
      ...previousChildren,
      if (currentChild != null) currentChild,
    ],
  );
},

6. AnimatedOpacity, AnimatedPositioned, AnimatedAlign, AnimatedPadding

These are the purpose-built single-property implicit animation widgets. They are more efficient than AnimatedContainer when you only need to animate one property, since they compile to simpler render object operations.

// AnimatedOpacity: fade in/out
AnimatedOpacity(
  opacity: _visible ? 1.0 : 0.0,
  duration: const Duration(milliseconds: 300),
  curve: Curves.easeIn,
  child: const MyWidget(),
);

// AnimatedPositioned: requires Stack parent
Stack(
  children: [
    AnimatedPositioned(
      duration: const Duration(milliseconds: 400),
      left: _expanded ? 0 : 100,
      top: _expanded ? 0 : 50,
      width: _expanded ? 300 : 100,
      height: _expanded ? 200 : 50,
      child: const Card(),
    ),
  ],
);

// AnimatedAlign: shift alignment within parent
AnimatedAlign(
  alignment: _aligned ? Alignment.topRight : Alignment.bottomLeft,
  duration: const Duration(milliseconds: 500),
  curve: Curves.elasticOut,
  child: const FloatingActionButton(onPressed: null, child: Icon(Icons.add)),
);

// AnimatedPadding: expand/contract content area
AnimatedPadding(
  padding: EdgeInsets.all(_focused ? 24.0 : 8.0),
  duration: const Duration(milliseconds: 200),
  child: const TextField(),
);

7. AnimatedList

AnimatedList provides animated insertion and removal of list items. Unlike regular ListView.builder, it maintains an internal list state and requires you to notify it of insertions and removals so it can run entry/exit animations.

7.1 GlobalKey Pattern

class _AnimatedListDemoState extends State<AnimatedListDemo> {
  final GlobalKey<AnimatedListState> _listKey = GlobalKey<AnimatedListState>();
  final List<String> _items = ['Apple', 'Banana', 'Cherry'];

  void _addItem() {
    final int index = _items.length;
    _items.add('Item ${index + 1}');
    // Notify AnimatedList to animate insertion at index
    _listKey.currentState!.insertItem(index,
        duration: const Duration(milliseconds: 300));
  }

  void _removeItem(int index) {
    final String removedItem = _items.removeAt(index);
    // AnimatedList keeps a "ghost" entry during exit animation
    _listKey.currentState!.removeItem(
      index,
      (context, animation) => _buildItem(removedItem, animation),
      duration: const Duration(milliseconds: 250),
    );
  }

  Widget _buildItem(String item, Animation<double> animation) {
    return SizeTransition(
      sizeFactor: animation,
      child: FadeTransition(
        opacity: animation,
        child: ListTile(
          title: Text(item),
          trailing: IconButton(
            icon: const Icon(Icons.delete),
            onPressed: () => _removeItem(_items.indexOf(item)),
          ),
        ),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedList(
      key: _listKey,
      initialItemCount: _items.length,
      itemBuilder: (context, index, animation) {
        return _buildItem(_items[index], animation);
      },
    );
  }
}
Common mistake
Modifying the underlying list without calling insertItem/removeItem on the AnimatedList state causes index mismatch errors. Always update the list and the AnimatedList state atomically — update the list first for insertions, update the list after for removals (so the animation builder can still access the removed item's data).

8. AnimatedSize

AnimatedSize wraps a child and animates layout size changes when the child's intrinsic size changes. This is useful for text that grows, expandable content areas, or any widget that changes its own layout size.

AnimatedSize(
  duration: const Duration(milliseconds: 300),
  curve: Curves.easeInOut,
  // alignment controls how the content shifts as size changes
  alignment: Alignment.topCenter,
  child: _expanded
      ? const ExpandedContent()
      : const CollapsedSummary(),
);

// With AnimatedSwitcher for combined size + content animation
AnimatedSize(
  duration: const Duration(milliseconds: 300),
  child: AnimatedSwitcher(
    duration: const Duration(milliseconds: 200),
    child: _showDetails
        ? const DetailView(key: ValueKey('detail'))
        : const SummaryView(key: ValueKey('summary')),
  ),
);

AnimatedSize requires a TickerProvider which it sources from its ancestor's TickerProviderStateMixin. Since Flutter 2.2, it no longer requires explicit vsync in newer APIs — it finds the ticker via context. Check the Flutter version your project targets.

9. Performance Considerations

9.1 Implicit Animations Still Need a Controller

Despite the "no controller needed" ergonomics, implicit animations create and manage an AnimationController internally. This means:

  • Each ImplicitlyAnimatedWidget instance owns a Ticker registered with the engine
  • Tickers run on every vsync even when the animation is complete (they check if forward() was called)
  • Having many simultaneous implicit animations is not "free" — each has overhead
  • Animation completes and the controller stops — no ongoing overhead after completion

9.2 Avoid Animating Expensive Subtrees

// PROBLEMATIC: the entire widget rebuilds every frame
AnimatedContainer(
  duration: const Duration(milliseconds: 300),
  color: _color,
  child: const ComplexExpensiveWidget(), // still rebuilt each frame
);

// BETTER: Use RepaintBoundary to isolate repaints
AnimatedContainer(
  duration: const Duration(milliseconds: 300),
  color: _color,
  child: RepaintBoundary(
    child: const ComplexExpensiveWidget(),
  ),
);

9.3 When to Use Implicit vs Explicit

ScenarioUse ImplicitUse Explicit
Simple toggle (show/hide, expand/collapse)YesOverkill
Repeating/looping animationNoYes — controller.repeat()
Sequenced multi-step animationNoYes — Interval-based
Physics-based animationNoYes — animateWith(simulation)
Driven by gesture positionNoYes — controller.value = offset
Simple state-change decorationYesNo
Coordinated multi-widget animationAwkwardYes — shared controller
L6 Interview Signal
Knowing when NOT to use implicit animations is as important as knowing how to use them. At the staff level, you should recognize that loading spinners, progress animations, and parallax scrolling effects require explicit controllers. Proposing an implicit animation for a gesture-driven interaction signals a gap in Flutter animation knowledge.

10. Creating a Custom ImplicitlyAnimatedWidget

When you need an implicit animation for a non-standard property that no existing widget covers, subclass ImplicitlyAnimatedWidget directly:

// Animates a radial progress indicator's progress value
class AnimatedRadialProgress extends ImplicitlyAnimatedWidget {
  final double progress; // 0.0 to 1.0
  final Color color;
  final double strokeWidth;

  const AnimatedRadialProgress({
    super.key,
    required this.progress,
    required this.color,
    this.strokeWidth = 8.0,
    super.duration = const Duration(milliseconds: 400),
    super.curve = Curves.easeOut,
  });

  @override
  ImplicitlyAnimatedWidgetState<AnimatedRadialProgress> createState() =>
      _AnimatedRadialProgressState();
}

class _AnimatedRadialProgressState
    extends ImplicitlyAnimatedWidgetState<AnimatedRadialProgress> {
  // Declare tweens for each animated property
  Tween<double>? _progress;
  ColorTween? _color;

  @override
  void forEachTween(TweenVisitor<dynamic> visitor) {
    // The visitor pattern: base class calls this to update tweens
    _progress = visitor(
      _progress,
      widget.progress,
      (value) => Tween<double>(begin: value as double),
    ) as Tween<double>?;

    _color = visitor(
      _color,
      widget.color,
      (value) => ColorTween(begin: value as Color),
    ) as ColorTween?;
  }

  @override
  Widget build(BuildContext context) {
    // animation is the CurvedAnimation provided by the base class
    return CustomPaint(
      painter: _RadialPainter(
        progress: _progress?.evaluate(animation) ?? widget.progress,
        color: _color?.evaluate(animation) ?? widget.color,
        strokeWidth: widget.strokeWidth,
      ),
      size: const Size(80, 80),
    );
  }
}

class _RadialPainter extends CustomPainter {
  final double progress;
  final Color color;
  final double strokeWidth;

  _RadialPainter({
    required this.progress,
    required this.color,
    required this.strokeWidth,
  });

  @override
  void paint(Canvas canvas, Size size) {
    final center = Offset(size.width / 2, size.height / 2);
    final radius = (size.width - strokeWidth) / 2;

    // Background track
    canvas.drawCircle(
      center, radius,
      Paint()
        ..color = color.withOpacity(0.2)
        ..style = PaintingStyle.stroke
        ..strokeWidth = strokeWidth,
    );

    // Progress arc
    canvas.drawArc(
      Rect.fromCircle(center: center, radius: radius),
      -3.14159 / 2,            // start at top
      2 * 3.14159 * progress,  // sweep angle
      false,
      Paint()
        ..color = color
        ..style = PaintingStyle.stroke
        ..strokeWidth = strokeWidth
        ..strokeCap = StrokeCap.round,
    );
  }

  @override
  bool shouldRepaint(_RadialPainter oldDelegate) {
    return oldDelegate.progress != progress ||
           oldDelegate.color != color ||
           oldDelegate.strokeWidth != strokeWidth;
  }
}

Quick Summary
Implicit animations are declarative: change a value, Flutter handles the rest. Internally they use a hidden AnimationController and the forEachTween visitor pattern to interpolate from current-to-target. Use TweenAnimationBuilder for custom types without subclassing. Use AnimatedSwitcher for child transitions (but remember the ValueKey requirement). Prefer explicit animations for repeating, sequenced, gesture-driven, or physics-based needs.
