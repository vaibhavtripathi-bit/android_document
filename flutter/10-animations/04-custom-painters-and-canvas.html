Part 1 — Core Concept
Direct GPU Drawing with Canvas
CustomPainter drops below the widget layer and draws directly to the Skia/Impeller canvas — the same API the framework uses internally.

1. The CustomPainter Contract

CustomPainter is an abstract class with two required methods. You place it inside a CustomPaint widget which allocates a canvas region and calls your painter.

class MyPainter extends CustomPainter {
  // Called every time the painter needs to redraw
  // Canvas size is provided by the widget's layout constraints
  @override
  void paint(Canvas canvas, Size size) {
    // All drawing happens here
  }

  // Called before paint() to determine if a repaint is needed
  // Return true only if the visual output would differ from oldDelegate
  @override
  bool shouldRepaint(MyPainter oldDelegate) {
    return false; // no properties changed — skip repaint
  }
}

// Usage in widget tree:
CustomPaint(
  size: const Size(200, 200),   // explicit size, or let parent constrain
  painter: MyPainter(),          // paints behind child
  foregroundPainter: MyPainter(), // paints in front of child
  child: const SomeWidget(),
)

2. shouldRepaint: The Critical Optimization

shouldRepaint is called by the framework when the parent rebuilds and passes a new painter instance. If it returns false, the raster cache for this paint operation is reused — no redraw occurs. This is one of the most impactful performance levers in custom drawing.

class CircleProgressPainter extends CustomPainter {
  final double progress;   // 0.0 to 1.0
  final Color color;
  final double strokeWidth;

  const CircleProgressPainter({
    required this.progress,
    required this.color,
    required this.strokeWidth,
  });

  @override
  void paint(Canvas canvas, Size size) {
    // ... drawing code
  }

  @override
  bool shouldRepaint(CircleProgressPainter oldDelegate) {
    // Only repaint if the visual output would actually change
    return oldDelegate.progress != progress ||
           oldDelegate.color != color ||
           oldDelegate.strokeWidth != strokeWidth;
  }
}

// ANTI-PATTERN: always returning true causes a repaint every build
// even when the parent rebuilds for unrelated reasons
@override
bool shouldRepaint(covariant CustomPainter oldDelegate) => true; // BAD
L6 Interview Signal
The distinction between widget rebuild and canvas repaint is a key Flutter performance concept. A widget rebuild triggers a new painter instance, but shouldRepaint gates whether the raster thread actually re-executes the drawing commands. At L6, you must explain this two-phase check and its role in the render pipeline.

3. Canvas Drawing Commands

3.1 Basic Shapes

@override
void paint(Canvas canvas, Size size) {
  final paint = Paint()
    ..color = Colors.blue
    ..style = PaintingStyle.fill;   // fill vs stroke

  // Rectangle
  canvas.drawRect(
    Rect.fromLTWH(10, 10, 100, 60),
    paint,
  );

  // Rounded rectangle
  canvas.drawRRect(
    RRect.fromRectAndRadius(
      const Rect.fromLTWH(10, 90, 100, 60),
      const Radius.circular(12),
    ),
    paint..color = Colors.green,
  );

  // Circle
  canvas.drawCircle(
    Offset(size.width / 2, size.height / 2),
    40,  // radius
    paint..color = Colors.red,
  );

  // Arc (drawArc uses a bounding Rect)
  canvas.drawArc(
    Rect.fromCenter(
        center: Offset(size.width / 2, size.height / 2),
        width: 100, height: 100),
    -3.14159 / 2,      // startAngle (radians, 0 = 3 o'clock, -π/2 = 12 o'clock)
    3.14159 * 1.5,     // sweepAngle
    false,             // useCenter: true draws pie slice, false draws arc only
    paint
      ..style = PaintingStyle.stroke
      ..strokeWidth = 6,
  );

  // Line
  canvas.drawLine(
    const Offset(0, 0),
    Offset(size.width, size.height),
    paint
      ..color = Colors.black
      ..strokeWidth = 2
      ..style = PaintingStyle.stroke,
  );

  // Oval
  canvas.drawOval(
    Rect.fromCenter(
        center: Offset(size.width / 2, size.height * 0.8),
        width: 80, height: 40),
    paint..color = Colors.purple..style = PaintingStyle.fill,
  );
}

final paint = Paint()

  ..strokeWidth = 4.0
  ..strokeCap = StrokeCap.round        // butt, round, square

3.3 Gradients as Shaders

@override
void paint(Canvas canvas, Size size) {
  // Linear gradient
  final linearGradient = LinearGradient(
    colors: [Colors.blue, Colors.purple],
    begin: Alignment.topLeft,
    end: Alignment.bottomRight,
  ).createShader(Rect.fromLTWH(0, 0, size.width, size.height));

  canvas.drawRect(
    Rect.fromLTWH(0, 0, size.width, size.height),
    Paint()..shader = linearGradient,
  );

  // Radial gradient
  final radialGradient = RadialGradient(
    colors: [Colors.yellow, Colors.orange, Colors.red],
    stops: const [0.0, 0.5, 1.0],
    center: Alignment.center,
    radius: 0.5,
  ).createShader(Rect.fromCircle(
      center: Offset(size.width / 2, size.height / 2),
      radius: size.width / 2));

  canvas.drawCircle(
    Offset(size.width / 2, size.height / 2),
    size.width / 2,
    Paint()..shader = radialGradient,
  );
}

4. Path: Complex Shape Drawing

The Path class is the workhorse for complex shapes. It records a sequence of drawing commands (moveTo, lineTo, cubicTo, arcTo) that are executed when you call canvas.drawPath(path, paint).

4.1 Basic Path Operations

@override
void paint(Canvas canvas, Size size) {
  final path = Path();

  // Move pen without drawing
  path.moveTo(0, size.height);

  // Straight line
  path.lineTo(size.width / 2, 0);
  path.lineTo(size.width, size.height);

  // Close the shape (line back to moveTo point)
  path.close();

  // Bezier curves
  final curvePath = Path()
    ..moveTo(0, size.height / 2)
    ..cubicTo(
      size.width * 0.25, 0,           // control point 1
      size.width * 0.75, size.height, // control point 2
      size.width, size.height / 2,    // end point
    );

  // Quadratic bezier (one control point)
  final quadPath = Path()
    ..moveTo(0, size.height)
    ..quadraticBezierTo(
      size.width / 2, 0,    // control point
      size.width, size.height, // end point
    );

  // Arc within a bounding rect
  final arcPath = Path()
    ..moveTo(50, 50)
    ..arcToPoint(
      const Offset(150, 50),
      radius: const Radius.circular(60),
      clockwise: false,
    );

  // Add predefined shapes
  path.addRect(const Rect.fromLTWH(10, 10, 80, 80));
  path.addOval(const Rect.fromLTWH(100, 10, 80, 80));
  path.addRRect(RRect.fromRectAndRadius(
      const Rect.fromLTWH(200, 10, 80, 80), const Radius.circular(16)));

  canvas.drawPath(path, Paint()..color = Colors.blue);
}

4.2 Path.combine for Boolean Operations

// Combine two paths with set operations
final circle = Path()
  ..addOval(Rect.fromCircle(center: const Offset(100, 100), radius: 60));

final rect = Path()
  ..addRect(const Rect.fromLTWH(70, 70, 120, 120));

// Intersection: only the overlapping area
final intersection = Path.combine(PathOperation.intersect, circle, rect);

// Difference: circle minus rect
final difference = Path.combine(PathOperation.difference, circle, rect);

// Union: merged shape
final union = Path.combine(PathOperation.union, circle, rect);

// Exclusive or: non-overlapping areas
final xor = Path.combine(PathOperation.xor, circle, rect);

canvas.drawPath(intersection, Paint()..color = Colors.red);

5. Clipping

@override
void paint(Canvas canvas, Size size) {
  // Clip to a circle before drawing
  canvas.save(); // save the unclipped state
  canvas.clipPath(
    Path()..addOval(Rect.fromCircle(
        center: Offset(size.width / 2, size.height / 2),
        radius: size.width / 2)),
  );
  // All subsequent drawing is clipped to the circle
  canvas.drawRect(
    Rect.fromLTWH(0, 0, size.width, size.height),
    Paint()..color = Colors.blue,
  );
  canvas.restore(); // restore unclipped state

  // Clip to rounded rect
  canvas.save();
  canvas.clipRRect(RRect.fromRectAndRadius(
    Rect.fromLTWH(0, 0, size.width, size.height),
    const Radius.circular(20),
  ));
  // draw inside the rounded rect
  canvas.restore();
}

6. Image Drawing

// Drawing a dart:ui Image onto the canvas
// (Images loaded via ImageProvider, then resolved to dart:ui.Image)
@override
void paint(Canvas canvas, Size size) {
  if (_image == null) return;

  // Draw image at full size
  canvas.drawImage(_image!, Offset.zero, Paint());

  // Draw scaled/positioned image (src rect → dst rect)
  canvas.drawImageRect(
    _image!,
    Rect.fromLTWH(0, 0,          // source rect (within the image)
        _image!.width.toDouble(), _image!.height.toDouble()),
    Rect.fromLTWH(10, 10, 100, 100), // destination rect on canvas
    Paint(),
  );

  // Draw image with nine-patch-like slicing
  canvas.drawImageNine(
    _image!,
    const Rect.fromLTWH(30, 30, 10, 10), // center slice
    Rect.fromLTWH(0, 0, size.width, size.height),
    Paint(),
  );
}

7. Shadows with drawShadow

@override
void paint(Canvas canvas, Size size) {
  final path = Path()
    ..addRRect(RRect.fromRectAndRadius(
        Rect.fromLTWH(20, 20, size.width - 40, size.height - 40),
        const Radius.circular(16)));

  // Draw shadow first (behind the shape)
  canvas.drawShadow(
    path,
    Colors.black.withOpacity(0.3), // shadow color
    8.0,                            // elevation (shadow spread)
    true,                           // transparentOccluder: true for semi-transparent shapes
  );

  // Then draw the shape on top
  canvas.drawPath(path, Paint()..color = Colors.white);
}

8. TextPainter for Custom Text

@override
void paint(Canvas canvas, Size size) {
  final textPainter = TextPainter(
    text: const TextSpan(
      text: '42%',
      style: TextStyle(
        color: Colors.black,
        fontSize: 24,
        fontWeight: FontWeight.bold,
      ),
    ),
    textDirection: TextDirection.ltr,
  );

  // Must call layout before paint
  textPainter.layout(
    minWidth: 0,
    maxWidth: size.width,
  );

  // Center the text
  final offset = Offset(
    (size.width - textPainter.width) / 2,
    (size.height - textPainter.height) / 2,
  );

  textPainter.paint(canvas, offset);
}

// IMPORTANT: TextPainter should be created/cached outside paint()
// Recreating it every frame is expensive. Cache as a field and
// only re-layout when text content or constraints change.

9. Animated CustomPainter

The key to animating a CustomPainter efficiently is the repaint parameter of CustomPainter's constructor — it takes a Listenable. When this listenable notifies, the canvas repaints without triggering a widget rebuild. This bypasses the widget tree entirely and goes straight to the raster thread.

9.1 Animated Arc (Progress Ring)

class _AnimatedRingState extends State<AnimatedRing>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;
  late final Animation<double> _progressAnim;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this, duration: const Duration(seconds: 2));
    _progressAnim = CurvedAnimation(
      parent: _controller, curve: Curves.easeInOut);
    _controller.repeat(reverse: true);
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return CustomPaint(
      size: const Size(120, 120),
      // Pass the animation as repaint listenable
      // canvas.repaint() is called each tick WITHOUT rebuilding
      painter: _RingPainter(animation: _progressAnim),
    );
  }
}

class _RingPainter extends CustomPainter {
  final Animation<double> animation;

  // Pass animation as the repaint listenable
  _RingPainter({required this.animation}) : super(repaint: animation);

  @override
  void paint(Canvas canvas, Size size) {
    final center = Offset(size.width / 2, size.height / 2);
    final radius = size.width / 2 - 8;

    // Background track
    canvas.drawCircle(
      center, radius,
      Paint()
        ..color = Colors.grey.shade200
        ..style = PaintingStyle.stroke
        ..strokeWidth = 8,
    );

    // Animated progress arc
    canvas.drawArc(
      Rect.fromCircle(center: center, radius: radius),
      -3.14159 / 2,                       // start at top
      2 * 3.14159 * animation.value,      // sweep to current progress
      false,
      Paint()
        ..color = Colors.blue
        ..style = PaintingStyle.stroke
        ..strokeWidth = 8
        ..strokeCap = StrokeCap.round,
    );
  }

  @override
  bool shouldRepaint(_RingPainter oldDelegate) {
    // Animation handles repaints via repaint listenable
    // shouldRepaint only needed if non-animation properties change
    return oldDelegate.animation != animation;
  }
}

9.2 Animated Wave Painter

class _WavePainter extends CustomPainter {
  final Animation<double> animation;
  final Color color;

  _WavePainter({required this.animation, required this.color})
      : super(repaint: animation);

  @override
  void paint(Canvas canvas, Size size) {
    final t = animation.value; // 0.0 to 1.0
    final path = Path();
    const waveHeight = 20.0;
    const waveFrequency = 2.0;

    path.moveTo(0, size.height / 2);

    for (double x = 0; x <= size.width; x++) {
      final y = size.height / 2 +
          waveHeight *
              _sin((x / size.width * waveFrequency * 2 * 3.14159) +
                  (t * 2 * 3.14159));
      path.lineTo(x, y);
    }

    path.lineTo(size.width, size.height);
    path.lineTo(0, size.height);
    path.close();

    canvas.drawPath(path, Paint()..color = color.withOpacity(0.6));
  }

  double _sin(double x) => (x - x * x * x / 6 + x * x * x * x * x / 120) % 6.28;

  @override
  bool shouldRepaint(_WavePainter old) => old.color != color;
}

10. Canvas Transforms

@override
void paint(Canvas canvas, Size size) {
  final center = Offset(size.width / 2, size.height / 2);

  // save() preserves current transform/clip state
  canvas.save();

  // Translate origin to center
  canvas.translate(center.dx, center.dy);

  // Rotate around the translated origin
  canvas.rotate(3.14159 / 4); // 45 degrees

  // Scale
  canvas.scale(1.5, 1.5);

  // Now draw — transforms are applied in reverse stack order
  canvas.drawRect(
    const Rect.fromLTWH(-40, -40, 80, 80),
    Paint()..color = Colors.blue,
  );

  // restore() pops the transform stack
  canvas.restore();

  // saveLayer for compositing with opacity/blend modes
  // (more expensive than save — creates an intermediate bitmap)
  canvas.saveLayer(
    Rect.fromLTWH(0, 0, size.width, size.height),
    Paint()..color = Colors.white.withOpacity(0.5),
  );
  canvas.drawCircle(center, 50, Paint()..color = Colors.red);
  canvas.restore();
}
Performance: saveLayer is expensive
canvas.saveLayer() allocates an intermediate offscreen bitmap, composites into it, then blends the result back. Use it only when you need true opacity blending or complex blend modes on a group of drawing operations. For simple transforms and clips, use save()/restore() which are essentially free.

11. Performance: Reusing Paint and Path Objects

Creating Paint and Path objects inside paint() allocates heap memory on every frame — at 60fps, this generates significant garbage collection pressure. The solution is to cache these objects as fields and update their properties rather than recreating them.

// WRONG: allocates new objects every frame
class BadPainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()   // allocated every frame
      ..color = Colors.blue
      ..strokeWidth = 4;
    final path = Path();    // allocated every frame
    path.moveTo(0, 0);
    path.lineTo(size.width, size.height);
    canvas.drawPath(path, paint);
  }
}

// CORRECT: reuse cached objects, only update changed properties
class GoodPainter extends CustomPainter {
  final Color color;

  GoodPainter({required this.color});

  // Cache Paint as a field — reused across frames
  final Paint _paint = Paint()..strokeWidth = 4;
  final Path _path = Path();

  @override
  void paint(Canvas canvas, Size size) {
    _paint.color = color; // update only what changes

    _path.reset(); // clear path data, keep allocated buffer
    _path.moveTo(0, 0);
    _path.lineTo(size.width, size.height);

    canvas.drawPath(_path, _paint);
  }

  @override
  bool shouldRepaint(GoodPainter old) => old.color != color;
}

12. ClipPath for Custom Widget Shapes

// Use ClipPath to clip any widget tree to a custom shape
class DiagonalClip extends StatelessWidget {
  final Widget child;
  const DiagonalClip({required this.child});

  @override
  Widget build(BuildContext context) {
    return ClipPath(
      clipper: _DiagonalClipper(),
      child: child,
    );
  }
}

class _DiagonalClipper extends CustomClipper<Path> {
  @override
  Path getClip(Size size) {
    return Path()
      ..lineTo(0, size.height * 0.85)
      ..lineTo(size.width, size.height)
      ..lineTo(size.width, 0)
      ..close();
  }

  @override
  bool shouldReclip(_DiagonalClipper oldClipper) => false;
}

// Wave bottom clip
class _WaveClipper extends CustomClipper<Path> {
  final double waveHeight;
  const _WaveClipper({this.waveHeight = 30});

  @override
  Path getClip(Size size) {
    final path = Path();
    path.lineTo(0, size.height - waveHeight);

    final firstControlPoint = Offset(size.width / 4, size.height);
    final firstEndPoint = Offset(size.width / 2, size.height - waveHeight);
    path.quadraticBezierTo(
        firstControlPoint.dx, firstControlPoint.dy,
        firstEndPoint.dx, firstEndPoint.dy);

    final secondControlPoint = Offset(size.width * 3 / 4, size.height - waveHeight * 2);
    final secondEndPoint = Offset(size.width, size.height - waveHeight);
    path.quadraticBezierTo(
        secondControlPoint.dx, secondControlPoint.dy,
        secondEndPoint.dx, secondEndPoint.dy);

    path.lineTo(size.width, 0);
    path.close();
    return path;
  }

  @override
  bool shouldReclip(_WaveClipper old) => old.waveHeight != waveHeight;
}

13. canvas.save() / restore() Pattern Summary

canvas.save()Very low — push matrix stackTransform/clip isolation
canvas.restore()Very low — pop matrix stackPaired with save()
canvas.saveLayer()High — allocates offscreen bitmapGroup opacity, blend modes
canvas.translate()Very lowMove coordinate origin
canvas.rotate()Very lowRotation around current origin
canvas.scale()Very lowScale drawing

Quick Reference
The repaint listenable parameter is the key to efficient animated CustomPainters — it bypasses widget rebuilds entirely and goes directly to the raster thread. Cache Paint and Path objects as fields. Return false from shouldRepaint whenever possible. Avoid saveLayer in tight animation loops.
