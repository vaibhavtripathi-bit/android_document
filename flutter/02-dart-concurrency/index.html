Key Concepts Summary

The Mental Model That Matters

Dart is single-threaded. "Async" in Dart does not mean "runs on another thread" — it means "cooperatively suspends at await points, letting other work run." Only isolates provide true parallelism. The event loop has two queues: microtasks (high priority, drain fully) and events (one per loop tick). Every await point is a potential interleaving point.

Common Interview Traps

  • Future.value().then() schedules a microtask — not an event queue item. Critical for ordering puzzles.
  • compute() requires a top-level function — not a closure. Use Isolate.run() if you need closure semantics.
  • Dart has no data races, but has logical race conditions — await points are interleaving points.
  • Isolates are for CPU-bound work, not IO-bound — async/await is sufficient for IO.
  • Always check mounted after every await in StatefulWidget.
  • Always cancel subscriptions in dispose() — uncancelled subscriptions are memory leaks.

Production Patterns to Know

  • Long-running worker isolate pattern (avoid respawn overhead for frequent tasks)
  • IsolateNameServer for cross-isolate port discovery
  • runZonedGuarded + FlutterError.onError for comprehensive error observability
  • CompositeSubscription for clean subscription lifecycle management
  • package:synchronized Lock for async mutual exclusion
  • Timer-based debounce for search-as-you-type
