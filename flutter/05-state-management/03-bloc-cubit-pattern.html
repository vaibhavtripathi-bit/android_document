Part 1 — Foundations
BLoC: Business Logic Component
Originated at Google I/O 2018 by Paolo Soares & Cong Hui. The pattern separates UI from business logic using Dart Streams as the communication channel.

1. Origin and Motivation

BLoC was introduced by Google engineers at I/O 2018 to address a fundamental problem: Flutter's reactive UI model needed a way to keep business logic out of widgets while remaining testable and reusable across platforms (Flutter, AngularDart).

The core insight is that every interaction in an app can be modeled as a stream of events that produces a stream of states. The BLoC sits in between, pure Dart code with no Flutter dependency, transforming events into states via business logic.

  • Separation of concerns — widgets only dispatch events and render state
  • Testability — BLoC is pure Dart, testable without a widget tree
  • Reusability — same BLoC can be used by multiple widgets or platforms
  • Predictability — given the same sequence of events, BLoC always produces the same states
Key Concept
The BLoC pattern enforces a strict contract: Events in, States out. Nothing else crosses the boundary between UI and business logic.

2. Stream-Based Architecture

The Mental Model

UI Widget
  │  add(event)           [EventSink]
  ▼
┌──────────────────────────────────┐
│             BLoC                 │
│  on<Event>(handler,             │
│    transformer: ...)             │
│                                  │
│  mapEventToState (pre-v8)        │
│  or on<E> handlers (v8+)         │
└──────────────────────────────────┘
  │  Stream<State>        [Stream]
  ▼
BlocBuilder rebuilds UI

Internally the bloc package uses a StreamController for the event sink and a BehaviorSubject-like mechanism (actually _stateController, a broadcast StreamController) for states. The current state is always available synchronously via state.

BlocBase Internal State Machine

abstract class BlocBase<State> {
  // Internal state controller (broadcast stream)
  late final _stateController = StreamController<State>.broadcast();

  State _state;

  State get state => _state;

  Stream<State> get stream => _stateController.stream;

  // Called by Cubit.emit() or Bloc's event handlers
  void emit(State state) {
    if (_stateController.isClosed) return;
    if (state == _state) return; // equality check — no redundant emits
    onChange(Change(currentState: _state, nextState: state));
    _state = state;
    _stateController.add(_state);
  }

  void onChange(Change<State> change) {} // override for logging

  void addError(Object error, [StackTrace? stackTrace]) {
    onError(error, stackTrace ?? StackTrace.current);
  }

  void onError(Object error, StackTrace stackTrace) {} // override

  Future<void> close() async {
    await _stateController.close();
  }
}
Tip
The equality check in emit() means BLoC skips re-emitting if the new state == the current state. For complex state objects, you must implement proper equality (via Equatable or == override) or BLoC will never emit repeated-value updates.

3. Cubit — The Simplified BLoC

Cubit was introduced in bloc v6.1 as a lighter alternative. Instead of defining an event hierarchy, you call methods directly that call emit().

// State
class CounterState extends Equatable {
  final int count;
  const CounterState(this.count);

  @override
  List<Object> get props => [count];
}

// Cubit
class CounterCubit extends Cubit<CounterState> {
  CounterCubit() : super(const CounterState(0));

  void increment() => emit(CounterState(state.count + 1));
  void decrement() => emit(CounterState(state.count - 1));
  void reset()     => emit(const CounterState(0));
}

Cubit uses the same BlocBase infrastructure — the only difference is that it exposes public methods instead of an event sink. This makes Cubit simpler to use but gives up EventTransformer capabilities.

Cubit Lifecycle

class MyCubit extends Cubit<MyState> {
  MyCubit() : super(MyState.initial());

  @override
  void onChange(Change<MyState> change) {
    super.onChange(change);
    // Called before state update — good for logging
    print('${change.currentState} -> ${change.nextState}');
  }

  @override
  void onError(Object error, StackTrace stackTrace) {
    super.onError(error, stackTrace);
    // Called when addError() is invoked
  }
}

4. BLoC — Full Event-Driven Pattern

Defining Events

Events should be immutable sealed classes (or use Equatable). Use a sealed class (Dart 3) or a base class with subclasses.

// Using sealed classes (Dart 3+)
sealed class CartEvent extends Equatable {
  const CartEvent();
}

final class AddItemToCart extends CartEvent {
  final String productId;
  final int quantity;
  const AddItemToCart({required this.productId, required this.quantity});

  @override
  List<Object> get props => [productId, quantity];
}

final class RemoveItemFromCart extends CartEvent {
  final String productId;
  const RemoveItemFromCart(this.productId);

  @override
  List<Object> get props => [productId];
}

final class ClearCart extends CartEvent {
  const ClearCart();

  @override
  List<Object> get props => [];
}

Defining States

sealed class CartState extends Equatable {
  const CartState();
}

final class CartInitial extends CartState {
  const CartInitial();
  @override List<Object> get props => [];
}

final class CartLoading extends CartState {
  const CartLoading();
  @override List<Object> get props => [];
}

final class CartLoaded extends CartState {
  final List<CartItem> items;
  final double total;
  const CartLoaded({required this.items, required this.total});
  @override List<Object> get props => [items, total];
}

final class CartError extends CartState {
  final String message;
  const CartError(this.message);
  @override List<Object> get props => [message];
}

Implementing the BLoC

class CartBloc extends Bloc<CartEvent, CartState> {
  final CartRepository _repository;

  CartBloc(this._repository) : super(const CartInitial()) {
    on<AddItemToCart>(_onAddItem);
    on<RemoveItemFromCart>(_onRemoveItem);
    on<ClearCart>(_onClearCart);
  }

  Future<void> _onAddItem(
    AddItemToCart event,
    Emitter<CartState> emit,
  ) async {
    emit(const CartLoading());
    try {
      final cart = await _repository.addItem(event.productId, event.quantity);
      emit(CartLoaded(items: cart.items, total: cart.total));
    } catch (e, st) {
      addError(e, st);
      emit(CartError(e.toString()));
    }
  }

  Future<void> _onRemoveItem(
    RemoveItemFromCart event,
    Emitter<CartState> emit,
  ) async {
    // Optimistic update — use current state items
    if (state is CartLoaded) {
      final current = state as CartLoaded;
      final updated = current.items
        .where((i) => i.productId != event.productId)
        .toList();
      emit(CartLoaded(
        items: updated,
        total: updated.fold(0, (s, i) => s + i.lineTotal),
      ));
    }
    await _repository.removeItem(event.productId);
  }

  Future<void> _onClearCart(
    ClearCart event,
    Emitter<CartState> emit,
  ) async {
    emit(const CartLoading());
    await _repository.clearCart();
    emit(const CartLoaded(items: [], total: 0));
  }

  @override
  void onTransition(Transition<CartEvent, CartState> transition) {
    super.onTransition(transition);
    // event, currentState, nextState all accessible
  }

  @override
  void onError(Object error, StackTrace stackTrace) {
    // Report to Crashlytics or logging
    super.onError(error, stackTrace);
  }
}
Part 2 — Advanced
EventTransformers & Concurrency
Control how concurrent events of the same type are handled — a critical production concern.

5. EventTransformer Deep Dive

By default, BLoC processes events sequentially — one event handler must complete before the next starts. The bloc_concurrency package provides four transformers for controlling concurrent event processing.

TransformerBehaviorUse Case
sequential() Default. Queue all events, process one at a time. Cart add/remove operations that must not overlap
concurrent() Process all events simultaneously, no waiting. Independent parallel fetches
droppable() Ignore new events while one is processing. Button tap that triggers a network call (debounce-like)
restartable() Cancel current handler and start fresh with new event. Search-as-you-type (cancel previous request) 
import 'package:bloc_concurrency/bloc_concurrency.dart';

class SearchBloc extends Bloc<SearchEvent, SearchState> {
  final SearchRepository _repo;

  SearchBloc(this._repo) : super(SearchInitial()) {
    // Restartable: each new search cancels the previous one
    on<SearchQueryChanged>(
      _onQueryChanged,
      transformer: restartable(),
    );

    // Droppable: ignore taps while loading
    on<SearchSubmitted>(
      _onSubmitted,
      transformer: droppable(),
    );
  }

  Future<void> _onQueryChanged(
    SearchQueryChanged event,
    Emitter<SearchState> emit,
  ) async {
    if (event.query.length < 3) {
      emit(SearchInitial());
      return;
    }

    emit(SearchLoading());

    // With restartable(), if a new SearchQueryChanged arrives,
    // this await is cancelled (the Emitter becomes "closed")
    await Future.delayed(const Duration(milliseconds: 300)); // debounce

    // Check emitter is still valid after await
    final results = await _repo.search(event.query);

    // emit() checks if the emitter is closed before emitting
    emit(SearchLoaded(results));
  }
}

How EventTransformer Works Internally

// The EventTransformer type signature:
typedef EventTransformer<Event> =
  Stream<Event> Function(Stream<Event> events, EventMapper<Event> mapper);

// restartable() implementation concept:
EventTransformer<Event> restartable<Event>() {
  return (events, mapper) => events.switchMap(mapper);
}

// droppable() implementation concept:
EventTransformer<Event> droppable<Event>() {
  return (events, mapper) => events.exhaustMap(mapper);
}

// concurrent() — no transformation:
EventTransformer<Event> concurrent<Event>() {
  return (events, mapper) => events.flatMap(mapper);
}

// sequential() — default queue behavior:
EventTransformer<Event> sequential<Event>() {
  return (events, mapper) => events.asyncExpand(mapper);
}
Staff-Level Insight
Understanding that restartable() maps to switchMap and droppable() maps to exhaustMap is critical. In a search screen, using sequential() for search queries means old results arrive after newer ones — a classic race condition. restartable() solves this by cancelling the in-flight HTTP request when the user types again.

6. BlocBuilder, BlocListener, BlocConsumer

BlocBuilder — Rebuilding UI

BlocBuilder<CartBloc, CartState>(
  // Optional: only rebuild when this function returns true
  buildWhen: (previous, current) {
    // Avoid rebuilding for intermediate loading states
    return current is! CartLoading;
  },
  builder: (context, state) {
    return switch (state) {
      CartInitial() => const EmptyCartPlaceholder(),
      CartLoading()  => const CircularProgressIndicator(),
      CartLoaded(items: final items, total: final total) =>
        CartItemList(items: items, total: total),
      CartError(message: final msg) => ErrorDisplay(message: msg),
    };
  },
)

BlocBuilder internally subscribes to the BLoC's stream. When a new state arrives, it calls buildWhen (defaults to always rebuild). If buildWhen returns true, it calls setState on its internal State object, triggering a rebuild.

BlocListener — Side Effects Only

BlocListener<CartBloc, CartState>(
  // Only trigger listener for specific state transitions
  listenWhen: (previous, current) =>
    current is CartError && previous is! CartError,
  listener: (context, state) {
    if (state is CartError) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text(state.message)),
      );
    }
  },
  child: const CartScreen(),
)
Warning
Never put navigation, SnackBars, or dialogs inside BlocBuilder. These are side effects that should live in BlocListener. BlocBuilder can be called multiple times (rebuild), so side effects there will fire repeatedly.

BlocConsumer — Both UI and Side Effects

BlocConsumer<AuthBloc, AuthState>(
  listenWhen: (prev, curr) => curr is AuthAuthenticated || curr is AuthError,
  listener: (context, state) {
    if (state is AuthAuthenticated) {
      context.go('/home'); // GoRouter navigation
    } else if (state is AuthError) {
      showErrorDialog(context, state.message);
    }
  },
  buildWhen: (prev, curr) => curr is AuthLoading || curr is AuthInitial,
  builder: (context, state) {
    return LoginForm(isLoading: state is AuthLoading);
  },
)

MultiBlocProvider & MultiBlocListener

MultiBlocProvider(
  providers: [
    BlocProvider(create: (ctx) => CartBloc(ctx.read<CartRepository>())),

7. BlocProvider and Dependency Injection

// Providing a BLoC
BlocProvider(
  // create is called lazily (first time BLoC is accessed)
  create: (context) => CartBloc(context.read<CartRepository>()),
  // lazy: false — create immediately
  lazy: false,
  child: const CartScreen(),
)

// Accessing in descendants
context.read<CartBloc>();   // one-time read, no subscription
context.watch<CartBloc>();  // subscribe (causes rebuild)
context.select<CartBloc, CartState>((bloc) => bloc.state); // granular

// Providing existing BLoC (e.g., passing down to sub-route)
BlocProvider.value(
  value: context.read<CartBloc>(),
  child: const CartDetailScreen(),
)
Important
BlocProvider automatically calls bloc.close() when the widget is removed from the tree. Use BlocProvider.value when you provide an already-created BLoC — this does NOT auto-close it, since you don't own the lifecycle.

8. BloC Observers (Global Monitoring)

class AppBlocObserver extends BlocObserver {
  @override
  void onCreate(BlocBase bloc) {
    super.onCreate(bloc);
    log('BLoC created: ${bloc.runtimeType}');
  }

  @override
  void onEvent(Bloc bloc, Object? event) {
    super.onEvent(bloc, event);
    log('Event: ${bloc.runtimeType} - $event');
  }

  @override
  void onChange(BlocBase bloc, Change change) {
    super.onChange(bloc, change);
    log('Change: ${bloc.runtimeType}\n'
        '  From: ${change.currentState}\n'
        '  To:   ${change.nextState}');
  }

  @override
  void onTransition(Bloc bloc, Transition transition) {
    super.onTransition(bloc, transition);
    // Transition includes the triggering event
    log('Transition: ${transition.event}');
  }

  @override
  void onError(BlocBase bloc, Object error, StackTrace stackTrace) {
    log('Error: ${bloc.runtimeType} - $error');
    FirebaseCrashlytics.instance.recordError(error, stackTrace);
    super.onError(bloc, error, stackTrace);
  }

  @override
  void onClose(BlocBase bloc) {
    super.onClose(bloc);
    log('BLoC closed: ${bloc.runtimeType}');
  }
}

// Register in main()
void main() {
  Bloc.observer = AppBlocObserver();
  runApp(const App());
}

9. BLoC-to-BLoC Communication

BLoCs should not directly reference each other. There are two recommended patterns:

Pattern 1: Stream Subscription in Constructor

class NotificationBloc extends Bloc<NotificationEvent, NotificationState> {
  final AuthBloc _authBloc;
  late final StreamSubscription<AuthState> _authSubscription;

  NotificationBloc(this._authBloc) : super(NotificationInitial()) {
    on<AuthStateChanged>(_onAuthStateChanged);

    // Listen to AuthBloc and add internal events
    _authSubscription = _authBloc.stream.listen((authState) {
      if (authState is AuthAuthenticated) {
        add(AuthStateChanged(userId: authState.userId));
      } else if (authState is AuthUnauthenticated) {
        add(const AuthStateChanged(userId: null));
      }
    });
  }

  Future<void> _onAuthStateChanged(
    AuthStateChanged event,
    Emitter<NotificationState> emit,
  ) async {
    if (event.userId != null) {
      final notifs = await _repository.getNotifications(event.userId!);
      emit(NotificationLoaded(notifs));
    } else {
      emit(NotificationInitial());
    }
  }

  @override
  Future<void> close() {
    _authSubscription.cancel();
    return super.close();
  }
}

Pattern 2: Emitter.forEach for Stream Binding

class ProfileBloc extends Bloc<ProfileEvent, ProfileState> {
  final AuthBloc _authBloc;

  ProfileBloc(this._authBloc) : super(ProfileInitial()) {
    on<ProfileStarted>(_onStarted);
  }

  Future<void> _onStarted(
    ProfileStarted event,
    Emitter<ProfileState> emit,
  ) async {
    // emit.forEach auto-manages the subscription lifecycle
    // and will cancel it when the event handler is done or
    // a new event cancels this one (e.g., with restartable())
    await emit.forEach<AuthState>(
      _authBloc.stream,
      onData: (authState) => switch (authState) {
        AuthAuthenticated(:final userId) =>
          ProfileLoaded(userId: userId),
        _ => ProfileInitial(),
      },
    );
  }
}
Staff-Level Insight
emit.forEach is the preferred modern approach over manual StreamSubscription management. It automatically handles cancellation when the emitter closes (due to BLoC close or restartable() cancellation), preventing memory leaks that are common with manual subscriptions.

10. Testing with bloc_test

Basic blocTest Structure

import 'package:bloc_test/bloc_test.dart';
import 'package:mocktail/mocktail.dart';
import 'package:test/test.dart';

class MockCartRepository extends Mock implements CartRepository {}

void main() {
  late MockCartRepository repository;
  late CartBloc bloc;

  setUp(() {
    repository = MockCartRepository();
    bloc = CartBloc(repository);
  });

  tearDown(() => bloc.close());

  group('CartBloc', () {
    blocTest<CartBloc, CartState>(
      'emits [CartLoading, CartLoaded] when AddItemToCart succeeds',
      // setup — configure mocks
      setUp: () {
        when(() => repository.addItem('prod-1', 1))
          .thenAnswer((_) async => Cart(
            items: [CartItem(productId: 'prod-1', quantity: 1, price: 9.99)],
            total: 9.99,
          ));
      },
      build: () => CartBloc(repository),
      // seed — initial state before act
      seed: () => const CartInitial(),
      // act — add events
      act: (bloc) => bloc.add(const AddItemToCart(productId: 'prod-1', quantity: 1)),
      // expect — ordered list of emitted states
      expect: () => [
        const CartLoading(),
        CartLoaded(
          items: [CartItem(productId: 'prod-1', quantity: 1, price: 9.99)],
          total: 9.99,
        ),
      ],
      // verify — check side effects
      verify: (_) {
        verify(() => repository.addItem('prod-1', 1)).called(1);
      },
    );

    blocTest<CartBloc, CartState>(
      'emits [CartLoading, CartError] when repository throws',
      setUp: () {
        when(() => repository.addItem(any(), any()))
          .thenThrow(Exception('Network error'));
      },
      build: () => CartBloc(repository),
      act: (bloc) => bloc.add(const AddItemToCart(productId: 'x', quantity: 1)),
      expect: () => [
        const CartLoading(),
        const CartError('Exception: Network error'),
      ],
      errors: () => [isA<Exception>()], // expects onError to be called
    );
  });
}

11. Error Handling in BLoC

class DataBloc extends Bloc<DataEvent, DataState> {
  DataBloc(this._repo) : super(DataInitial()) {
    on<FetchData>(_onFetch);
  }

  Future<void> _onFetch(FetchData event, Emitter<DataState> emit) async {
    emit(DataLoading());
    try {
      final data = await _repo.fetch();
      emit(DataLoaded(data));
    } on NetworkException catch (e, st) {
      // Specific error handling
      addError(e, st); // notifies observer
      emit(DataError(type: ErrorType.network, message: e.message));
    } on AuthException catch (e, st) {
      addError(e, st);
      emit(DataError(type: ErrorType.auth, message: 'Session expired'));
    } catch (e, st) {
      addError(e, st);
      emit(DataError(type: ErrorType.unknown, message: 'Unexpected error'));
    }
  }

  @override
  void onError(Object error, StackTrace stackTrace) {
    // Centralized error reporting
    ErrorReporter.report(error, stackTrace, context: runtimeType.toString());
    super.onError(error, stackTrace);
  }
}

12. BLoC vs Cubit: Decision Guide

ConsiderationCubitBLoC
Complexity Lower — just call methods Higher — event classes, sealed hierarchies
Event traceability Limited — method calls in stack trace Full — event logged in observer/transition
Concurrency control Not supported EventTransformer (restartable, droppable)
Event de-duplication Not supported droppable() or custom transformer
Testing granularity Test outcomes of method calls Test event-to-state mapping precisely
Boilerplate Minimal Significant (offset by code gen or Freezed)
Best for Simple UI state, forms, toggles Complex flows, search, auth, multi-step
Staff-Level Insight — When to Reach for BLoC

At Google scale, BLoC's event traceability is its killer feature. Every user action maps to an event in logs, making it straightforward to replay user sessions for debugging and A/B testing. The added boilerplate is justified by the observability. For simple feature flags and UI toggles, Cubit is perfectly appropriate — use the right tool for the complexity level.

A pattern used in large apps: Cubit for local UI state (form validation, tab selection), BLoC for domain state (auth, shopping cart, real-time feeds).

13. HydratedBloc — Persistent State

HydratedBloc from the hydrated_bloc package automatically persists and restores BLoC state using SharedPreferences or a custom storage backend.

class ThemeBloc extends HydratedBloc<ThemeEvent, ThemeState> {
  ThemeBloc() : super(ThemeState.light()) {
    on<ToggleTheme>((event, emit) {
      emit(state.isDark ? ThemeState.light() : ThemeState.dark());
    });
  }

  // Serialize state to JSON for persistence
  @override
  ThemeState? fromJson(Map<String, dynamic> json) {
    return ThemeState.fromJson(json);
  }

  @override
  Map<String, dynamic>? toJson(ThemeState state) {
    return state.toJson();
  }
}

// Initialize in main
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  HydratedBloc.storage = await HydratedStorage.build(
    storageDirectory: await getApplicationDocumentsDirectory(),
  );
  runApp(const App());
}
