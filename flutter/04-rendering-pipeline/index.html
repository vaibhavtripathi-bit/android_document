Section 04 — Rendering Pipeline

Complete frame pipeline from widget rebuild to GPU presentation — 7 files covering every phase, optimization technique, and interview scenario

Section Overview
This section covers Flutter's complete frame rendering pipeline end-to-end: how a setState() call becomes pixels on screen, how layout constraints flow through the render tree, how the layer tree enables GPU compositing, how vsync drives animations, and how to diagnose and fix jank. These are among the most frequently tested topics at Google L6/Staff level for Flutter roles.
Study Path
Read files 01 through 06 in order on first pass — each builds on the previous. Then use file 07 for active recall. For targeted review before an interview, prioritize 01 (pipeline phases), 02 (BoxConstraints), 04 (layers), and 06 (jank diagnosis).

Files in This Section

File 01
Frame Pipeline Overview Core

The six phases of Flutter's rendering pipeline — Animate, Build, Layout, Compositing Bits, Paint, Composite — with the class names responsible for each phase (SchedulerBinding, BuildOwner, PipelineOwner, RenderView). Covers the UI thread vs raster thread split and how frames are pipeline-staged.

File 02
Layout Deep Dive Core

BoxConstraints semantics (tight, loose, unbounded), the RenderObject layout protocol, single-pass constraint/size propagation, RelayoutBoundary for O(N) layout, and the performance cost of intrinsic sizing. Includes the LayoutBuilder and CustomMultiChildLayout APIs.

File 03
Paint & Compositing Core

How RenderObject.paint() records into PaintingContext / Canvas, the difference between paint() and paintChild(), when needsCompositing is true, and how the layer tree is assembled during the paint phase. Covers PictureLayer, OffsetLayer, and RepaintBoundary.

File 04
Layers & Scene Deep

The full layer type hierarchy (PictureLayer, TransformLayer, OpacityLayer, ClipRectLayer, PhysicalModelLayer, TextureLayer, PlatformViewLayer), how layers enable GPU compositor caching, and how buildScene() serializes the layer tree for the raster thread. The mechanism behind 60fps scrolling.

File 05
Vsync & Scheduler Core

How the OS delivers vsync signals via window.onBeginFrame, how SchedulerBinding orchestrates frame callbacks, how Ticker and AnimationController integrate with the vsync signal, and the four scheduler phases (idle, transient, persistent, post-frame). Covers high-refresh-rate display support.

File 06
Jank Diagnosis & Performance Profiling Practical

The four types of jank (UI thread, raster thread, shader compilation, GC pause), how to use Flutter DevTools Performance tab and flame chart, Repaint Rainbow, shader compilation jank and Impeller's fix, GC pause mitigation, and a systematic 9-step diagnosis workflow. Includes profile mode vs debug mode distinction.

File 07
Interview Q&A Deep

20+ Q&A blocks covering pipeline phases, BoxConstraints, RepaintBoundary, Opacity cost, shader compilation jank, UI vs raster thread, RenderObject layout protocol, paint vs paintChild, vsync and AnimationController, layer tree performance. Includes deep staff-level questions: full setState() walkthrough, cross-platform jank diagnosis, O(N) layout analysis, 60fps chart design, and ListView compositing strategy.
