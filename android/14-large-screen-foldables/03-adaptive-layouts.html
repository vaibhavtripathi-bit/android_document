2. NavigationSuiteScaffold — Adaptive Navigation

NavigationSuiteScaffold (Material3 Adaptive) automatically switches between BottomNavigationBar, NavigationRail, and NavigationDrawer based on WindowSizeClass — one composable, all screen sizes.

// Dependency: androidx.compose.material3.adaptive:adaptive-navigation-suite
@Composable
fun AppNavigation() {
    val navController = rememberNavController()
    val currentDestination = navController.currentBackStackEntryAsState().value?.destination

    NavigationSuiteScaffold(
        navigationSuiteItems = {
            AppDestination.entries.forEach { destination ->
                item(
                    icon = { Icon(destination.icon, contentDescription = null) },
                    label = { Text(stringResource(destination.label)) },
                    selected = currentDestination?.hierarchy?.any {
                        it.route == destination.route
                    } == true,
                    onClick = {
                        navController.navigate(destination.route) {
                            popUpTo(navController.graph.findStartDestination().id) {
                                saveState = true
                            }
                            launchSingleTop = true
                            restoreState = true
                        }
                    }
                )
            }
        }
    ) {
        NavHost(navController = navController, startDestination = AppDestination.HOME.route) {
            composable(AppDestination.HOME.route) { HomeScreen() }
            composable(AppDestination.SEARCH.route) { SearchScreen() }
            composable(AppDestination.PROFILE.route) { ProfileScreen() }
        }
    }
}
L6 Insight — What NavigationSuiteScaffold does internally
It reads WindowAdaptiveInfo (which wraps WindowSizeClass + FoldingFeature) and switches: Compact width → BottomNavigationBar. Medium width OR Compact width + Medium height → NavigationRail. Expanded width → NavigationDrawer. You can override with layoutType parameter if the automatic choice doesn't match your design.

3. ListDetailPaneScaffold — Canonical List-Detail

// Dependency: androidx.compose.material3.adaptive:adaptive-layout
@Composable
fun EmailApp() {
    val navigator = rememberListDetailPaneScaffoldNavigator<EmailId>()

    BackHandler(navigator.canNavigateBack()) {
        navigator.navigateBack()
    }

    ListDetailPaneScaffold(
        directive = navigator.scaffoldDirective,
        value = navigator.scaffoldValue,
        listPane = {
            AnimatedPane {
                EmailListScreen(
                    onEmailClick = { emailId ->
                        navigator.navigateTo(ListDetailPaneScaffoldRole.Detail, emailId)
                    }
                )
            }
        },
        detailPane = {
            AnimatedPane {
                val emailId = navigator.currentDestination?.content
                if (emailId != null) {
                    EmailDetailScreen(emailId = emailId)
                } else {
                    // Placeholder when no email is selected
                    Box(Modifier.fillMaxSize(), contentAlignment = Alignment.Center) {
                        Text("Select an email")
                    }
                }
            }
        }
    )
}

On phone (Compact): only the list pane is shown. Tapping an item navigates to detail (full-screen). On tablet/foldable (Expanded): both panes visible side-by-side. BackHandler handles the back stack correctly in both cases.

4. Supporting Pane Layout

// Three-pane layout: list + detail + extra pane (e.g., metadata/filters)
@Composable
fun ThreePaneApp() {
    val navigator = rememberSupportingPaneScaffoldNavigator()

    SupportingPaneScaffold(
        directive = navigator.scaffoldDirective,
        value = navigator.scaffoldValue,
        mainPane = { MainContent() },
        supportingPane = {
            AnimatedPane { SupportingPanel() }
        },
        extraPane = {
            AnimatedPane { ExtraDetailsPanel() }
        }
    )
}

5. Adaptive Layout for View-based Apps

If you're not on Compose yet, use SlidingPaneLayout — a ViewGroup that shows one or two panes based on available width:

<!-- layout/activity_main.xml -->
<androidx.slidingpanelayout.widget.SlidingPaneLayout
    android:id="@+id/sliding_pane_layout"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <!-- List pane — minimum 320dp, can grow -->
    <fragment
        android:name=".ListFragment"
        android:layout_width="320dp"
        android:layout_height="match_parent"
        android:layout_weight="1" />

    <!-- Detail pane — minimum 480dp -->
    <fragment
        android:name=".DetailFragment"
        android:layout_width="480dp"
        android:layout_height="match_parent"
        android:layout_weight="3" />

</androidx.slidingpanelayout.widget.SlidingPaneLayout>
// Handle back navigation with SlidingPaneLayout
class MainActivity : AppCompatActivity() {
    private lateinit var slidingPaneLayout: SlidingPaneLayout

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        slidingPaneLayout = binding.slidingPaneLayout

        // Handle back button — go back to list if detail is showing
        onBackPressedDispatcher.addCallback(
            this,
            SlidingPaneLayout.SimpleSlidingPaneLayout.BackPressedCallback(slidingPaneLayout)
        )
    }
}

6. Adaptive Grid — LazyVerticalGrid

// Responsive grid that adapts column count to window width
@Composable
fun ProductGrid(products: List<Product>) {
    LazyVerticalGrid(
        // Automatically calculates column count: window_width / 180dp
        columns = GridCells.Adaptive(minSize = 180.dp),
        contentPadding = PaddingValues(16.dp),
        horizontalArrangement = Arrangement.spacedBy(12.dp),
        verticalArrangement = Arrangement.spacedBy(12.dp)
    ) {
        items(products) { product ->
            ProductCard(product)
        }
    }
}
// Phone (360dp wide)   → 1 column  (360/180 = 2, but 2×180 = 360 with no padding, so ~2)
// Tablet (800dp wide)  → 4 columns
// Foldable (1200dp)    → 6 columns

7. Window Insets on Large Screens

Tablets and foldables have different inset configurations — the status bar, navigation bar, and system gesture zones change. Always use WindowInsets APIs:

// Compose — consume insets correctly
Scaffold(
    modifier = Modifier.fillMaxSize(),
    contentWindowInsets = ScaffoldDefaults.contentWindowInsets
) { innerPadding ->
    // innerPadding includes status bar + nav bar + display cutouts
    LazyColumn(
        contentPadding = innerPadding
    ) { /* items */ }
}

// Extend content edge-to-edge (required for tablets / foldables)
// In Activity.onCreate:
WindowCompat.setDecorFitsSystemWindows(window, false)
Keyboard on Large Screens
On tablets, the software keyboard shows differently — it may float or dock to one side in split-screen. Always use WindowInsetsAnimationCompat to animate your layout when the IME appears/disappears. Hardcoded padding for keyboard will break on tablets.

Interview Cheat Sheet — Adaptive Layouts

  • Three canonical layouts? — List-Detail (Gmail), Supporting Pane (Maps+search), Feed (Photos). Design for all three in L6 interviews.
  • NavigationSuiteScaffold? — Single composable that automatically switches Bottom Nav → Rail → Drawer based on window size. From material3-adaptive-navigation-suite.
  • ListDetailPaneScaffold? — Compose API for list-detail pattern. Single pane on phone (navigates), two panes on tablet (side by side). Handles back stack automatically.
  • View equivalent of ListDetail? — SlidingPaneLayout. Define min widths for each pane; it stacks or slides based on available width.
  • Responsive grid? — GridCells.Adaptive(minSize = Xdp) in LazyVerticalGrid. Automatically calculates columns. Never hardcode column count.
  • Edge-to-edge? — Call WindowCompat.setDecorFitsSystemWindows(window, false). Consume insets via Scaffold contentWindowInsets or Modifier.windowInsetsPadding().
  • Keyboard insets on tablet? — Use WindowInsetsAnimationCompat. Never use a fixed bottom padding for IME.
