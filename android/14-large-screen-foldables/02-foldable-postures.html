3. Jetpack WindowManager — FoldingFeature

Use androidx.window:window to observe hinge state. FoldingFeature describes the physical hinge.

// Observe hinge state in a lifecycle-aware coroutine
lifecycleScope.launch {
    lifecycle.repeatOnLifecycle(Lifecycle.State.STARTED) {
        WindowInfoTracker.getOrCreate(this@MainActivity)
            .windowLayoutInfo(this@MainActivity)
            .collect { layoutInfo ->
                val foldingFeature = layoutInfo.displayFeatures
                    .filterIsInstance<FoldingFeature>()
                    .firstOrNull()

                updateLayoutForPosture(foldingFeature)
            }
    }
}

private fun updateLayoutForPosture(fold: FoldingFeature?) {
    if (fold == null) {
        // No fold — regular phone
        showSinglePane()
        return
    }

    when {
        fold.state == FoldingFeature.State.FLAT &&
        fold.orientation == FoldingFeature.Orientation.VERTICAL -> {
            // Book-fold fully open → show two-pane layout
            showTwoPane()
        }

        fold.state == FoldingFeature.State.HALF_OPENED &&
        fold.orientation == FoldingFeature.Orientation.HORIZONTAL -> {
            // Clamshell / tabletop mode → show content top, controls bottom
            showTabletopLayout()
        }

        fold.state == FoldingFeature.State.HALF_OPENED &&
        fold.orientation == FoldingFeature.Orientation.VERTICAL -> {
            // Book-fold half open → treat like two separate portrait screens
            showBookLayout()
        }
    }
}

4. FoldingFeature Properties

PropertyValuesMeaning
stateFLAT, HALF_OPENEDWhether the device is fully open or at an angle
orientationHORIZONTAL, VERTICALDirection the hinge runs (not the fold direction)
occlusionTypeNONE, FULLFULL = hinge physically covers part of the screen (some older Fold devices). NONE = hinge is visible but no pixels are occluded.
isSeparatingBooleanTrue when the hinge logically divides the screen into two distinct areas. Always true for HALF_OPENED, may be true for FLAT too on some devices.
boundsRectPhysical pixel bounds of the hinge/fold area. Use this to avoid placing interactive content behind the hinge.

5. Avoiding Content Behind the Hinge

When occlusionType == FULL, content placed over foldingFeature.bounds is physically hidden. Even with NONE, placing interactive content (buttons, text input) straddling the hinge creates a bad UX — the hinge distorts touch targets.

// Compose: use GuidedPane to split at the hinge boundary
@Composable
fun TwoPaneLayout(foldingFeature: FoldingFeature?) {
    if (foldingFeature?.isSeparating == true) {
        // Use Jetpack's TwoPaneLayout to automatically split at hinge
        TwoPane(
            first = { ListContent() },
            second = { DetailContent() },
            strategy = FoldAwareHorizontalTwoPaneStrategy(
                splitFraction = 0.5f,
                gapWidth = with(LocalDensity.current) {
                    foldingFeature.bounds.width().toDp()
                }
            )
        )
    } else {
        // Single pane
        ListContent()
    }
}
L6 Insight — isSeparating is the key flag
Don't check state == FLAT to decide whether to show two panes. Use isSeparating. On some devices a fully-open (FLAT) fold still has a visible crease that should be respected as a separator. isSeparating = true means the OEM considers the hinge a logical boundary — split your content at it.

6. Tabletop Mode — Camera & Video Apps

When a clamshell is half-open horizontally (HALF_OPENED + HORIZONTAL), the device sits on a surface like a laptop. This is the "tabletop mode". Ideal layout:

  • Top half: viewfinder / video content (user looks at this).
  • Bottom half: controls, settings, playback bar (user taps here).
// Compose — tabletop layout
@Composable
fun CameraTabletopLayout(fold: FoldingFeature) {
    val hingeBoundsDp = with(LocalDensity.current) {
        fold.bounds.top.toDp()
    }
    Column {
        // Viewfinder fills top half (up to hinge)
        Box(modifier = Modifier
            .fillMaxWidth()
            .height(hingeBoundsDp)
        ) {
            CameraPreview()
        }
        // Controls fill bottom half (below hinge)
        Box(modifier = Modifier
            .fillMaxWidth()
            .weight(1f)
        ) {
            CameraControls()
        }
    }
}

7. Compose — collectFoldingFeatureAsState

// Simpler Compose helper from the Window library
@Composable
fun rememberFoldingFeature(activity: Activity): FoldingFeature? {
    val windowInfoTracker = WindowInfoTracker.getOrCreate(activity)
    val windowLayoutInfo by windowInfoTracker
        .windowLayoutInfo(activity)
        .collectAsStateWithLifecycle(initialValue = WindowLayoutInfo(emptyList()))

    return windowLayoutInfo.displayFeatures
        .filterIsInstance<FoldingFeature>()
        .firstOrNull()
}

// Usage
@Composable
fun MyScreen(activity: Activity) {
    val fold = rememberFoldingFeature(activity)
    when {
        fold?.isSeparating == true &&
        fold.orientation == FoldingFeature.Orientation.VERTICAL ->
            TwoPaneLayout()
        fold?.state == FoldingFeature.State.HALF_OPENED ->
            TabletopLayout()
        else -> SinglePaneLayout()
    }
}

Interview Cheat Sheet — Foldable Postures

  • Two fold states? — FLAT (fully open) and HALF_OPENED (at an angle). No "closed" state — that's detected via WindowSizeClass going to Compact (outer screen).
  • Hinge orientation? — VERTICAL = book-fold (hinge runs top-to-bottom). HORIZONTAL = clamshell (hinge runs left-to-right).
  • isSeparating vs FLAT? — Use isSeparating to decide two-pane layout. Don't rely on FLAT alone — some FLAT devices still have a separating crease.
  • occlusionType FULL? — Hinge physically covers pixels. Never place content there. Use foldingFeature.bounds to know exact pixel area.
  • Tabletop mode? — HALF_OPENED + HORIZONTAL hinge. Content on top, controls on bottom. Split at foldingFeature.bounds.top.
  • API to use? — Jetpack WindowManager: WindowInfoTracker.windowLayoutInfo(activity). Collect as a Flow. Never use display metrics or screen size.
  • Testing? — TestWindowInfoTracker.overrideDecorator() to inject fake FoldingFeature in instrumented tests.
