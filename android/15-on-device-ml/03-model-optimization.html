Model Optimization

Quantization, pruning, distillation, TFLite conversion — making models small and fast enough for mobile

1. Why Optimize?

A typical ResNet-50 model is ~100MB and takes ~500ms per inference on a mid-range CPU. That's unusable on mobile. Optimization brings this down to ~10MB and ~20ms — the difference between a feature and a frustration.

FP32 → FP16 (half precision)2×1.5–2× on GPUNegligible (<0.5%)

2. Quantization — The Most Important Technique

Quantization reduces the precision of weights and activations. The model uses fewer bits per number — smaller file, faster arithmetic on hardware that has integer SIMD units.

Post-Training Dynamic Range Quantization (easiest)

# Python — TFLite Converter
import tensorflow as tf

converter = tf.lite.TFLiteConverter.from_saved_model("my_model")

# Dynamic quantization: weights are INT8, activations remain FP32 at runtime
converter.optimizations = [tf.lite.Optimize.DEFAULT]

tflite_model = converter.convert()
with open("model_dynamic_quant.tflite", "wb") as f:
    f.write(tflite_model)

# Result: ~4× smaller. CPU inference 2× faster. No calibration data needed.

Post-Training Full Integer Quantization (best for DSP/NPU)

# Full INT8: weights AND activations are INT8
# Requires a representative dataset for calibration

def representative_dataset():
    for image_batch in calibration_dataset.take(100):
        yield [image_batch]  # ~100 samples is sufficient

converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.representative_dataset = representative_dataset
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
converter.inference_input_type = tf.int8   # or tf.uint8
converter.inference_output_type = tf.int8

# Result: ~4× smaller. Runs on NNAPI/DSP/Hexagon at full speed.
# Required for Hexagon delegate and most NPUs.
L6 Insight — Why calibration data matters
Full INT8 quantization maps FP32 value ranges to [-128, 127]. To do this accurately, the converter needs to observe the actual range of activations at each layer — that's what the representative dataset provides. Using only 100 diverse samples from your production distribution is sufficient. Using random noise or unrepresentative data causes accuracy degradation because the quantization scale factors are miscalibrated.

Quantization-Aware Training (best accuracy)

# Apply QAT during training — model learns to be robust to quantization
import tensorflow_model_optimization as tfmot

# Wrap model with fake quantization nodes during training
model = tf.keras.applications.MobileNetV2(...)
quant_aware_model = tfmot.quantization.keras.quantize_model(model)

quant_aware_model.compile(optimizer='adam', loss='categorical_crossentropy')
quant_aware_model.fit(train_dataset, epochs=5)  # fine-tune with QAT

# Convert — weights and activations are already calibrated
converter = tf.lite.TFLiteConverter.from_keras_model(quant_aware_model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()

# Result: best accuracy at INT8 — losses <0.5% vs FP32 baseline

3. Pruning

Pruning removes weights that contribute little to model output (zeroing them out). A sparse model with 80% zeros can be stored more compactly and computed faster on hardware with sparse matrix support.

import tensorflow_model_optimization as tfmot

# Apply pruning schedule during training
pruning_schedule = tfmot.sparsity.keras.PolynomialDecay(
    initial_sparsity=0.0,
    final_sparsity=0.80,   # 80% of weights will be zero
    begin_step=0,
    end_step=1000
)

pruned_model = tfmot.sparsity.keras.prune_low_magnitude(
    model,
    pruning_schedule=pruning_schedule
)

# Train with pruning callbacks
callbacks = [tfmot.sparsity.keras.UpdatePruningStep()]
pruned_model.fit(train_dataset, callbacks=callbacks, epochs=10)

# Strip pruning wrappers before conversion
final_model = tfmot.sparsity.keras.strip_pruning(pruned_model)
# Convert to TFLite with quantization applied on top

4. Knowledge Distillation

Train a small student model to mimic the outputs of a large teacher model. The student learns not just the correct labels but the teacher's soft probability distribution — which encodes more information than one-hot labels.

# Distillation training loop (conceptual)
teacher = load_large_model()   # e.g., EfficientNet-L2, 500MB
student = build_small_model()  # e.g., MobileNetV3-Small, 5MB

for batch in train_dataset:
    teacher_outputs = teacher(batch, training=False)  # soft labels
    with tf.GradientTape() as tape:
        student_outputs = student(batch, training=True)
        # Combine: soft distillation loss + hard label loss
        distill_loss = kl_divergence(teacher_outputs / T, student_outputs / T)
        hard_loss = cross_entropy(true_labels, student_outputs)
        total_loss = alpha * distill_loss + (1 - alpha) * hard_loss
    gradients = tape.gradient(total_loss, student.trainable_variables)
    optimizer.apply_gradients(zip(gradients, student.trainable_variables))
L6 Insight — Temperature in distillation
Temperature T (typically 3–10) softens the teacher's probability distribution before computing KL divergence. Without temperature, the teacher outputs near-one-hot predictions (high confidence), and the student learns nothing from the soft labels. Higher T = softer distribution = more information transfer from teacher to student. Google used this to build MobileNet — small models that punch above their size.

5. TFLite Model Maker — Easiest Path to Custom Models

# TFLite Model Maker — fine-tunes pre-trained models on your data
# No ML expertise required
pip install tflite-model-maker

from tflite_model_maker import image_classifier
from tflite_model_maker import ImageClassifierDataLoader

# Load your labelled image dataset
data = ImageClassifierDataLoader.from_folder("my_dataset/")
train_data, test_data = data.split(0.9)

# Fine-tune MobileNetV2 on your data (transfer learning)
model = image_classifier.create(
    train_data,
    model_spec=image_classifier.EfficientNetLite0Spec(),
    epochs=20
)

# Evaluate
loss, accuracy = model.evaluate(test_data)
print(f"Accuracy: {accuracy:.2%}")

# Export: generates .tflite file + labels.txt + metadata
model.export(export_dir="output/")
# Output: model.tflite (~5MB, INT8 quantized)

6. Firebase ML — Remote Model Delivery

Firebase ML allows you to host TFLite models in the cloud and download them to devices — without an app update. This is how you ship model improvements without a Play Store release cycle.

// Download model from Firebase ML
val remoteModel = CustomRemoteModel.Builder(
    FirebaseModelSource.Builder("my_image_classifier").build()
).build()

val downloadConditions = CustomModelDownloadConditions.Builder()
    .requireWifi()
    .build()

FirebaseModelDownloader.getInstance()
    .getModel(remoteModel, DownloadType.LOCAL_MODEL_UPDATE_IN_BACKGROUND, downloadConditions)
    .addOnSuccessListener { model ->
        // model.file = File pointing to the downloaded .tflite
        val interpreter = Interpreter(model.file!!)
        // Run inference with downloaded model
    }
Always ship a bundled fallback
Never rely solely on Firebase ML for a critical feature. If the device has no internet connection and the model hasn't been downloaded yet, the feature will fail. Bundle a smaller/older model in your APK as a fallback, and use the Firebase-downloaded model as an upgrade.

Interview Cheat Sheet — Model Optimization

  • Three quantization types? — Dynamic range (weights INT8, activations FP32, no calibration data), Full INT8 (both INT8, needs representative dataset), QAT (quantization-aware training, best accuracy).
  • Why representative dataset for full INT8? — Calibrates the scale factors that map FP32 ranges to [-128,127] at each layer. Without it, scale factors are wrong → accuracy drops.
  • FP16 vs INT8? — FP16: 2× smaller, best on GPU. INT8: 4× smaller, best on CPU/DSP/NPU. INT8 required for Hexagon delegate.
  • Knowledge distillation? — Small student model trained to mimic large teacher's soft output distribution (not just hard labels). Temperature T softens the distribution. Produces models that outperform their size class.
  • Pruning? — Zero out low-magnitude weights. 80% sparsity = 80% of weights are zero. Saves storage. Speeds up inference on sparse-aware hardware.
  • TFLite Model Maker? — High-level Python API for fine-tuning pre-trained models on custom data via transfer learning. Outputs INT8 quantized .tflite automatically.
  • Firebase ML model delivery? — Host models in cloud, download without app update. Always bundle a fallback model in APK for offline/first-launch.
