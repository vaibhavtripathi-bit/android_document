TFLite Fundamentals

Interpreter, model loading, tensors, delegates, threading — the core runtime every Android ML engineer must know

1. What is TFLite?

TensorFlow Lite is Google's ML inference runtime designed for mobile and embedded devices. It takes a trained TensorFlow or Keras model, converts it to the .tflite flatbuffer format, and runs it on-device with minimal memory and compute overhead.

2. The TFLite Interpreter — Core API

// Add dependency
// implementation("org.tensorflow:tensorflow-lite:2.14.0")
// implementation("org.tensorflow:tensorflow-lite-gpu:2.14.0")  // GPU delegate

class ImageClassifier(private val context: Context) {

    private val interpreter: Interpreter

    init {
        // Load model from assets
        val model = loadModelFile("mobilenet_v1.tflite")

        val options = Interpreter.Options().apply {
            numThreads = 4          // CPU thread count
            useNNAPI = false        // we'll use GPU delegate instead
        }

        interpreter = Interpreter(model, options)
    }

    private fun loadModelFile(fileName: String): MappedByteBuffer {
        val assetFileDescriptor = context.assets.openFd(fileName)
        val inputStream = FileInputStream(assetFileDescriptor.fileDescriptor)
        val fileChannel = inputStream.channel
        return fileChannel.map(
            FileChannel.MapMode.READ_ONLY,
            assetFileDescriptor.startOffset,
            assetFileDescriptor.declaredLength
        )
    }

    fun classify(bitmap: Bitmap): List<ClassificationResult> {
        // 1. Pre-process: resize to model input size, normalize
        val inputBuffer = preprocessBitmap(bitmap)

        // 2. Prepare output buffer
        // Model output: [1, 1001] — 1001 class probabilities
        val outputBuffer = Array(1) { FloatArray(1001) }

        // 3. Run inference
        interpreter.run(inputBuffer, outputBuffer)

        // 4. Post-process: sort by probability, return top-5
        return outputBuffer[0]
            .mapIndexed { index, prob -> ClassificationResult(labels[index], prob) }
            .sortedByDescending { it.probability }
            .take(5)
    }

    private fun preprocessBitmap(bitmap: Bitmap): ByteBuffer {
        val scaledBitmap = Bitmap.createScaledBitmap(bitmap, 224, 224, true)
        val byteBuffer = ByteBuffer.allocateDirect(1 * 224 * 224 * 3 * 4) // FLOAT32
        byteBuffer.order(ByteOrder.nativeOrder())

        val pixels = IntArray(224 * 224)
        scaledBitmap.getPixels(pixels, 0, 224, 0, 0, 224, 224)

        pixels.forEach { pixel ->
            // Normalize to [-1, 1] for MobileNet
            byteBuffer.putFloat(((pixel shr 16 and 0xFF) - 127.5f) / 127.5f)  // R
            byteBuffer.putFloat(((pixel shr 8  and 0xFF) - 127.5f) / 127.5f)  // G
            byteBuffer.putFloat(((pixel        and 0xFF) - 127.5f) / 127.5f)  // B
        }
        return byteBuffer
    }

    fun close() = interpreter.close()
}

3. Understanding Tensors

TFLite models operate on tensors — multi-dimensional arrays with a specific shape and data type.

// Inspect model input/output tensors at runtime
val inputTensor = interpreter.getInputTensor(0)
println("Input shape: ${inputTensor.shape().toList()}")  // [1, 224, 224, 3]
println("Input dtype: ${inputTensor.dataType()}")         // FLOAT32

val outputTensor = interpreter.getOutputTensor(0)
println("Output shape: ${outputTensor.shape().toList()}") // [1, 1001]
println("Output dtype: ${outputTensor.dataType()}")       // FLOAT32

4. Delegates — Accelerating Inference

By default, TFLite runs on the CPU. Delegates offload computation to specialized hardware:

// GPU Delegate — fastest for image models on most Android devices
val gpuDelegate = GpuDelegate(
    GpuDelegate.Options().apply {
        setPrecisionLossAllowed(true)   // allow FP16 on GPU (2× faster, minor accuracy loss)
        setQuantizedModelsAllowed(true) // run INT8 models on GPU
    }
)

val options = Interpreter.Options()
    .addDelegate(gpuDelegate)

val interpreter = Interpreter(model, options)
// IMPORTANT: interpreter and gpuDelegate must be used on the same thread
// GPU delegate is NOT thread-safe
// NNAPI Delegate — uses Android Neural Networks API
// Routes to NPU/DSP if available, falls back to CPU
val nnapiDelegate = NnApiDelegate(
    NnApiDelegate.Options().apply {
        allowFp16 = true
        executionPreference = NnApiDelegate.Options.EXECUTION_PREFERENCE_SUSTAINED_SPEED
    }
)
val options = Interpreter.Options().addDelegate(nnapiDelegate)
L6 Insight — Delegate Selection Strategy
GPU delegate: Best for image/vision models (CNNs). Parallelizes convolution operations efficiently. Not all ops are supported — unsupported ops fall back to CPU automatically.

NNAPI delegate: Best for models with heavy matrix multiplication (transformers, dense layers). Routes to NPU on devices that have one (Pixel 6+ Tensor chip, Samsung Exynos with NPU). Falls back gracefully.

Hexagon delegate: Qualcomm Snapdragon DSP. Best for INT8 quantized models. Dramatically faster and more power-efficient than CPU for supported ops. Only available on Snapdragon devices.

5. Threading Model

Critical Threading Rule
Interpreter is not thread-safe. You cannot share a single Interpreter across threads. Options:
  • Create one Interpreter per thread (recommended for concurrent inference).
  • Use a single background thread (via Dispatchers.IO or a dedicated HandlerThread) and serialize all inference calls.
  • GPU delegate additionally requires all calls on the same thread it was created on — use a single dedicated thread.
// Safe pattern: single inference thread with coroutines
class SafeClassifier(context: Context) {
    private val inferenceDispatcher = Executors.newSingleThreadExecutor().asCoroutineDispatcher()
    private val interpreter = buildInterpreter(context)

    suspend fun classify(bitmap: Bitmap): List<ClassificationResult> =
        withContext(inferenceDispatcher) {
            // Safe — always runs on the same single thread
            val input = preprocessBitmap(bitmap)
            val output = Array(1) { FloatArray(1001) }
            interpreter.run(input, output)
            postprocess(output)
        }

    fun close() {
        inferenceDispatcher.close()
        interpreter.close()
    }
}

6. Model Signatures — Modern API

Models converted with signatures (TF SavedModel with @tf.function) expose named inputs/outputs — more readable than index-based tensor access:

// Named input/output via signature runner (TFLite 2.6+)
val runner = interpreter.getSignatureRunner("serving_default")

// Map inputs by name
val inputs = mapOf("image" to inputBuffer)
val outputs = mapOf("probabilities" to outputBuffer)

runner.run(inputs, outputs)

7. Memory Management

// Always close resources — Interpreter holds native memory
class MyViewModel : ViewModel() {
    private val classifier = ImageClassifier(application)

    override fun onCleared() {
        classifier.close()  // releases native Interpreter + delegate
        super.onCleared()
    }
}

// Use TFLite's ByteBuffer pooling for repeated inference
// Avoid allocating new ByteBuffer on every frame (camera inference)
private val inputBuffer = ByteBuffer.allocateDirect(INPUT_SIZE).apply {
    order(ByteOrder.nativeOrder())
}

fun runInference(bitmap: Bitmap) {
    inputBuffer.rewind()      // reset position — reuse the buffer
    fillBuffer(inputBuffer, bitmap)
    interpreter.run(inputBuffer, outputBuffer)
}

Interview Cheat Sheet — TFLite Fundamentals

  • What is a .tflite file? — FlatBuffer-serialized model graph. Smaller and faster to load than SavedModel. Contains ops, weights (possibly quantized), and metadata.
  • Interpreter.run() inputs? — ByteBuffer (most efficient) or multi-dimensional arrays. ByteBuffer avoids data copy; arrays incur JNI copy overhead.
  • Thread safety? — Interpreter is NOT thread-safe. One interpreter per thread, or serialize inference on a single thread. GPU delegate additionally requires a single fixed thread.
  • Delegates purpose? — Offload ops from CPU to GPU (GpuDelegate), NPU/DSP (NnApiDelegate), or Snapdragon DSP (HexagonDelegate). Unsupported ops fall back to CPU automatically.
  • Input pre-processing? — Resize to model input size. Normalize pixel values (model-dependent: MobileNet uses [-1,1], EfficientNet uses [0,1]). Pack into ByteBuffer in NHWC format.
  • Memory best practice? — Allocate ByteBuffer once, call rewind() before each inference. Always close() Interpreter when done. Hold in ViewModel, close in onCleared().
  • Signatures? — Named input/output access via getSignatureRunner("name"). Cleaner than index-based run(). Requires model exported with tf.function signature.
