ML Kit

Google's pre-built ML APIs — text recognition, barcode, face, objects, pose, translation — on-device and cloud

1. What is ML Kit?

ML Kit is Google's mobile ML SDK that packages pre-trained, production-quality models into simple APIs. You don't need to manage TFLite models, tensors, or pre-processing — ML Kit abstracts all of it. Available for Android and iOS.

2. Bundled vs Downloadable Models

ML Kit models ship in two forms:

  • Bundled: model included in your APK/AAR. Larger app size, but works immediately with no downloads. Use for critical features (barcode scanning, core OCR).
  • Downloadable: model is not in the APK. Downloaded via Google Play Services on first use (~10–50MB per model). Smaller app size. Must handle the case where download hasn't completed yet.

3. Text Recognition — Full Example

// Text recognition from camera image (works with CameraX ImageAnalysis)
class TextRecognitionAnalyzer(
    private val onTextFound: (String) -> Unit
) : ImageAnalysis.Analyzer {

    private val recognizer = TextRecognition.getClient(
        TextRecognizerOptions.DEFAULT_OPTIONS  // Latin script
        // For Devanagari (Hindi): use DevanagariTextRecognizerOptions.Builder().build()
    )

    @androidx.camera.core.ExperimentalGetImage
    override fun analyze(imageProxy: ImageProxy) {
        val mediaImage = imageProxy.image ?: run {
            imageProxy.close()
            return
        }

        val inputImage = InputImage.fromMediaImage(
            mediaImage,
            imageProxy.imageInfo.rotationDegrees
        )

        recognizer.process(inputImage)
            .addOnSuccessListener { visionText ->
                // visionText.text = full text string
                // visionText.textBlocks = list of TextBlock (paragraph)
                //   TextBlock.lines = list of Line
                //     Line.elements = list of Element (word)
                //       Element.boundingBox = Rect on screen

                val extractedText = visionText.text
                if (extractedText.isNotBlank()) {
                    onTextFound(extractedText)
                }
            }
            .addOnFailureListener { e ->
                Log.e("TextRecognition", "Failed", e)
            }
            .addOnCompleteListener {
                imageProxy.close()  // MUST close — or camera freezes
            }
    }
}
Always close ImageProxy
In addOnCompleteListener, always call imageProxy.close(). If you close it in onSuccess only, a failure will leave the proxy open and freeze the camera pipeline — no more frames will be delivered.

4. Barcode Scanning — Real-Time

val scanner = BarcodeScanning.getClient(
    BarcodeScannerOptions.Builder()
        .setBarcodeFormats(
            Barcode.FORMAT_QR_CODE,
            Barcode.FORMAT_EAN_13,
            Barcode.FORMAT_CODE_128
        )
        .build()
)

// In ImageAnalysis.Analyzer:
scanner.process(inputImage)
    .addOnSuccessListener { barcodes ->
        barcodes.forEach { barcode ->
            when (barcode.valueType) {
                Barcode.TYPE_URL -> openUrl(barcode.url?.url)
                Barcode.TYPE_PRODUCT -> lookupProduct(barcode.rawValue)
                Barcode.TYPE_CONTACT_INFO -> saveContact(barcode.contactInfo)
                Barcode.TYPE_WIFI -> connectToWifi(barcode.wifi)
            }
        }
    }

5. Object Detection & Tracking

val options = ObjectDetectorOptions.Builder()
    .setDetectorMode(ObjectDetectorOptions.STREAM_MODE)  // for real-time camera
    // vs SINGLE_IMAGE_MODE for static images
    .enableMultipleObjects()          // detect up to 5 objects
    .enableClassification()           // get object labels
    .build()

val detector = ObjectDetection.getClient(options)

// Result processing
detector.process(inputImage)
    .addOnSuccessListener { detectedObjects ->
        detectedObjects.forEach { obj ->
            val boundingBox = obj.boundingBox      // Rect on original image
            val trackingId = obj.trackingId        // stable ID across frames (STREAM_MODE)
            val label = obj.labels.firstOrNull()
            val confidence = label?.confidence ?: 0f
            val category = label?.text ?: "Unknown"

            drawBoundingBox(boundingBox, "$category ${(confidence * 100).toInt()}%")
        }
    }
L6 Insight — STREAM_MODE tracking ID
In STREAM_MODE, ML Kit assigns a stable trackingId to each detected object across consecutive frames. This allows you to track "object #3" as it moves across the screen — critical for AR experiences or picking a specific object from a crowded scene. The ID is only stable within a session; it resets when the object leaves the frame.

6. Pose Estimation

val poseDetector = PoseDetection.getClient(
    PoseDetectorOptions.Builder()
        .setDetectorMode(PoseDetectorOptions.STREAM_MODE)
        .build()
)

poseDetector.process(inputImage)
    .addOnSuccessListener { pose ->
        // 33 landmarks: nose, shoulders, elbows, wrists, hips, knees, ankles, etc.
        val leftShoulder = pose.getPoseLandmark(PoseLandmark.LEFT_SHOULDER)
        val rightShoulder = pose.getPoseLandmark(PoseLandmark.RIGHT_SHOULDER)
        val leftElbow = pose.getPoseLandmark(PoseLandmark.LEFT_ELBOW)

        leftShoulder?.let {
            val x = it.position.x   // pixel coords on input image
            val y = it.position.y
            val z = it.position3D.z // depth (relative — not absolute meters)
            val confidence = it.inFrameLikelihood  // 0.0 – 1.0
        }

        // Calculate angle for exercise rep counting

    }

7. On-Device Translation

// Download translation model first (Hindi → English)
val conditions = DownloadConditions.Builder()
    .requireWifi()   // only download on WiFi
    .build()

val translator = Translation.getClient(
    TranslatorOptions.Builder()
        .setSourceLanguage(TranslateLanguage.HINDI)
        .setTargetLanguage(TranslateLanguage.ENGLISH)
        .build()
)

// Download model (~30MB per language pair)
translator.downloadModelIfNeeded(conditions)
    .addOnSuccessListener {
        // Model ready — translate offline
        translator.translate("नमस्ते दुनिया")
            .addOnSuccessListener { translatedText ->
                // "Hello World"
                showTranslation(translatedText)
            }
    }
    .addOnFailureListener { e ->
        // No WiFi, or download failed
        showError("Translation model not available offline")
    }

// Clean up when done
translator.close()

Interview Cheat Sheet — ML Kit

  • ML Kit vs raw TFLite? — ML Kit wraps TFLite models in easy APIs — no tensor management, no pre-processing code, no model file to manage. Use ML Kit for common vision/NLP tasks. Use raw TFLite for custom models.
  • Bundled vs downloadable? — Bundled: in APK, immediate, larger size. Downloadable: fetched via Play Services, smaller APK, must handle download-in-progress state.
  • Always close ImageProxy? — In addOnCompleteListener, not just onSuccess. Failure to close freezes the camera pipeline.
  • STREAM_MODE vs SINGLE_IMAGE_MODE? — STREAM_MODE: optimized for video frames, provides tracking IDs, runs faster. SINGLE_IMAGE_MODE: higher accuracy, no tracking, for static photos.
  • trackingId in object detection? — Stable ID for an object across frames in STREAM_MODE. Resets when object leaves frame. Used for AR tracking and selection.
  • Pose landmarks count? — 33 landmarks covering full body. Each has x/y pixel position, z depth (relative), and inFrameLikelihood confidence.
  • Translation model size? — ~30MB per language pair. Download on WiFi with DownloadConditions. Always call translator.close() when done.
