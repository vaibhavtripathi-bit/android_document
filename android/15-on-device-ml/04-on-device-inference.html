3. GPU Delegate — Setup and Caveats

// GPU delegate must be initialized and used on the SAME thread
// Use a single-thread executor for this
val gpuExecutor = Executors.newSingleThreadExecutor()

var gpuDelegate: GpuDelegate? = null
var interpreter: Interpreter? = null

fun initOnGpuThread() {
    gpuExecutor.execute {
        gpuDelegate = GpuDelegate(
            GpuDelegate.Options().apply {
                setPrecisionLossAllowed(true)  // FP16 on GPU — ~2× faster
                inferencePreference = GpuDelegate.Options.INFERENCE_PREFERENCE_FAST_SINGLE_ANSWER
            }
        )
        val options = Interpreter.Options().addDelegate(gpuDelegate!!)
        interpreter = Interpreter(loadModel(), options)
    }
}

fun runInference(input: ByteBuffer, output: FloatArray) {
    gpuExecutor.execute {
        interpreter?.run(input, output)
    }
}

fun cleanup() {
    gpuExecutor.execute {
        interpreter?.close()
        gpuDelegate?.close()
    }
    gpuExecutor.shutdown()
}
GPU Delegate Thread Rule
The GPU delegate binds to an OpenGL context on the thread it's created. All subsequent interpreter.run() calls must happen on the same thread. Calling from a different thread silently produces wrong results or crashes. Always use a dedicated single-thread executor for GPU inference.

4. NNAPI Delegate — Best for Pixel 6+ / Exynos NPU

val nnapiOptions = NnApiDelegate.Options().apply {
    // Execution preference
    executionPreference = NnApiDelegate.Options.EXECUTION_PREFERENCE_SUSTAINED_SPEED
    // vs EXECUTION_PREFERENCE_LOW_POWER (slower but less battery drain)
    // vs EXECUTION_PREFERENCE_FAST_SINGLE_ANSWER (burst, then sleep)

    allowFp16 = true              // allow FP16 precision on NNAPI
    useNnapiCpu = false           // don't use NNAPI's CPU path (use TFLite CPU instead)
    acceleratorName = null        // null = let NNAPI pick best accelerator
    // Or specify: "google-edgetpu", "qti-default", "samsung-npu"
}

val nnapiDelegate = NnApiDelegate(nnapiOptions)
val options = Interpreter.Options().addDelegate(nnapiDelegate)

// Check if NNAPI is actually available and accelerated
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.P) {
    val manager = context.getSystemService(NeuralNetworksManager::class.java)
    val devices = manager.devices
    val hasNpu = devices.any { it.type == Device.TYPE_ACCELERATOR }
    Log.d("ML", "NPU available: $hasNpu, devices: ${devices.map { it.name }}")
}

5. Delegate Fallback Strategy

// Robust delegate selection with fallback chain
fun buildInterpreter(context: Context, model: MappedByteBuffer): Interpreter {
    // Try GPU first (best for image models on most devices)
    return try {
        val gpuDelegate = GpuDelegate()
        Interpreter(model, Interpreter.Options().addDelegate(gpuDelegate))
            .also { Log.d("ML", "Using GPU delegate") }
    } catch (e: Exception) {
        Log.w("ML", "GPU delegate failed: ${e.message}, trying NNAPI")
        try {
            val nnapiDelegate = NnApiDelegate()
            Interpreter(model, Interpreter.Options().addDelegate(nnapiDelegate))
                .also { Log.d("ML", "Using NNAPI delegate") }
        } catch (e2: Exception) {
            Log.w("ML", "NNAPI failed: ${e2.message}, falling back to CPU")
            Interpreter(model, Interpreter.Options().apply { numThreads = 4 })
                .also { Log.d("ML", "Using CPU with 4 threads") }
        }
    }
}

L6 Insight — Thermal Throttling
Running inference at 30fps on the CPU heats the device. After 2–3 minutes, the thermal governor throttles CPU clocks to 50–60% — your 30ms inference becomes 60ms and frame rate halves. Solutions:
  • Use GPU or NNAPI delegate — more power-efficient per inference.
  • Reduce inference rate: run at 10fps instead of 30fps for non-real-time tasks.
  • Use EXECUTION_PREFERENCE_LOW_POWER in NNAPI for background tasks.
  • Monitor with PowerManager.isDeviceIdleMode() and reduce rate when hot.

7. CameraX + TFLite Integration

// ImageAnalysis use case — processes camera frames for inference
val imageAnalysis = ImageAnalysis.Builder()
    .setTargetResolution(Size(224, 224))     // match model input size
    .setBackpressureStrategy(ImageAnalysis.STRATEGY_KEEP_ONLY_LATEST)
    // ↑ CRITICAL: drop frames if inference is slower than camera fps
    // Without this, frames queue up → increasing latency → OOM
    .setOutputImageFormat(ImageAnalysis.OUTPUT_IMAGE_FORMAT_RGBA_8888)
    .build()

imageAnalysis.setAnalyzer(inferenceExecutor) { imageProxy ->
    // imageProxy is delivered on inferenceExecutor's thread
    val bitmap = imageProxy.toBitmap()

    // Run inference synchronously on this thread
    val results = classifier.classify(bitmap)

    // Post results to main thread for UI update
    mainHandler.post { updateUI(results) }

    imageProxy.close()  // release frame back to camera pipeline
}

// Bind use case
cameraProvider.bindToLifecycle(
    lifecycleOwner,
    CameraSelector.DEFAULT_BACK_CAMERA,
    imageAnalysis

// TFLite Benchmark Tool — run from adb
// adb shell am start -S -n org.tensorflow.lite.benchmark/.BenchmarkModelActivity \

//              --use_gpu=true \
//              --num_runs=50"

// In-code benchmarking
fun benchmarkModel(interpreter: Interpreter, input: ByteBuffer, output: FloatArray): Long {
    val warmupRuns = 5
    val benchmarkRuns = 50

    // Warmup — JIT and GPU shader compilation
    repeat(warmupRuns) { interpreter.run(input, output) }

    val start = SystemClock.elapsedRealtimeNanos()
    repeat(benchmarkRuns) { interpreter.run(input, output) }
    val elapsed = SystemClock.elapsedRealtimeNanos() - start

    return elapsed / benchmarkRuns / 1_000_000L  // ms per inference
}

Interview Cheat Sheet — Inference Pipeline & Delegates

  • GPU delegate thread rule? — Create and use on the same single thread. Binds to an OpenGL context. Use a single-thread Executor dedicated to GPU inference.
  • NNAPI delegate best on? — Pixel 6+ (Google Tensor NPU), Exynos with NPU, Snapdragon with CDSP. Specify acceleratorName or let NNAPI pick. Falls back to CPU gracefully.
  • Hexagon delegate requirement? — Qualcomm Snapdragon only, model must be fully INT8 quantized. Fastest for INT8 models on Snapdragon (2–4× faster than CPU INT8).
  • CameraX backpressure strategy? — Always use STRATEGY_KEEP_ONLY_LATEST for inference. Without it, frames queue up, latency grows, and OOM is possible.
  • Thermal throttling solution? — GPU/NNAPI delegate (more efficient). Reduce inference fps. Use LOW_POWER execution preference for background tasks.
  • Benchmarking rule? — Always do warmup runs (5+) before measuring. GPU needs shader compilation on first runs. JIT affects CPU timing. Use SystemClock.elapsedRealtimeNanos() not System.currentTimeMillis().
  • Fallback chain? — Try GPU → NNAPI → CPU. Wrap each in try/catch. Log which path is taken for observability.
