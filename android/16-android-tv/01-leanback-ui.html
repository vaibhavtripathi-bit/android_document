Leanback UI Library

BrowseSupportFragment, card presenters, ObjectAdapter, PlaybackSupportFragment — the building blocks of every Android TV app

1. Leanback Library Overview

Leanback is Google's UI library for Android TV. It provides fragments and widgets purpose-built for the 10-foot experience — large cards, horizontal row browsing, focus-driven interaction, and D-pad-first navigation.

Android TV App Structure (Leanback) ───────────────────────────────────────────────────────── MainActivity └── BrowseSupportFragment ← main content browser ├── HeadersSupportFragment ← left rail (categories) └── RowsSupportFragment ← right content area ├── ListRow (Trending Now) │ └── [Card] [Card] [Card] ... ├── ListRow (Continue Watching) │ └── [Card] [Card] ... └── PageRow → SearchFragment / SettingsFragment DetailsSupportFragment ← selected item details ├── DetailsOverviewRow ← poster + title + actions └── ListRow (Related Content) PlaybackSupportFragment ← fullscreen video player └── PlaybackControlsRow ← transport controls overlay

2. BrowseSupportFragment — The Home Screen

class MainFragment : BrowseSupportFragment() {

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // Branding
        title = "Hotstar"
        headersState = HEADERS_ENABLED   // show left rail
        isHeadersTransitionOnBackEnabled = true

        // Background color shown behind cards
        brandColor = ContextCompat.getColor(requireContext(), R.color.hotstar_blue)

        // Search icon in top-right — triggers SearchFragment
        setOnSearchClickedListener {
            startActivity(Intent(requireContext(), SearchActivity::class.java))
        }

        // Load content
        loadRows()
    }

    private fun loadRows() {
        val rowsAdapter = ArrayObjectAdapter(ListRowPresenter())

        // Row 1: Continue Watching
        val continueHeader = HeaderItem(0, "Continue Watching")
        val continueAdapter = ArrayObjectAdapter(VideoCardPresenter())
        continueAdapter.addAll(0, viewModel.getContinueWatching())
        rowsAdapter.add(ListRow(continueHeader, continueAdapter))

        // Row 2: Trending
        val trendingHeader = HeaderItem(1, "Trending Now")
        val trendingAdapter = ArrayObjectAdapter(VideoCardPresenter())
        trendingAdapter.addAll(0, viewModel.getTrending())
        rowsAdapter.add(ListRow(trendingHeader, trendingAdapter))

        // Row 3: Sports (PageRow — loads a separate fragment)
        val sportsHeader = HeaderItem(2, "Sports")
        rowsAdapter.add(PageRow(sportsHeader))

        adapter = rowsAdapter
    }

    // PageRow navigation — swap right panel to a custom fragment
    override fun onCreateMainFragment(): Fragment? {
        val header = selectedHeader ?: return null
        return when (header.id) {
            2L -> SportsFragment()  // loads live scores, match cards
            else -> null
        }
    }

    // Item click handler
    override fun onItemViewClicked(
        itemViewHolder: Presenter.ViewHolder?,
        item: Any?,
        rowViewHolder: RowPresenter.ViewHolder?,
        row: Row?
    ) {
        if (item is VideoItem) {
            val intent = Intent(requireContext(), DetailActivity::class.java)
            intent.putExtra("video_id", item.id)

            // Leanback shared element transition — card image slides to detail
            val bundle = ActivityOptionsCompat.makeSceneTransitionAnimation(
                requireActivity(),
                (itemViewHolder?.view as ImageCardView).mainImageView,
                DetailActivity.SHARED_ELEMENT_NAME
            ).toBundle()
            startActivity(intent, bundle)
        }
    }
}

3. Card Presenter — How Items Are Displayed

A Presenter is Leanback's equivalent of RecyclerView.ViewHolder + Adapter. It inflates views and binds data for each card in a row.

class VideoCardPresenter : Presenter() {

    override fun onCreateViewHolder(parent: ViewGroup): ViewHolder {
        // ImageCardView is the standard Leanback card — image + title + content
        val cardView = ImageCardView(parent.context).apply {
            isFocusable = true
            isFocusableInTouchMode = true
            setMainImageDimensions(320, 180)  // 16:9 thumbnail
        }
        return ViewHolder(cardView)
    }

    override fun onBindViewHolder(viewHolder: ViewHolder, item: Any?) {
        val video = item as VideoItem
        val cardView = viewHolder.view as ImageCardView

        cardView.titleText = video.title
        cardView.contentText = video.duration  // "45 min" or "S1 E3"

        // Load thumbnail with Glide
        Glide.with(cardView.context)
            .load(video.thumbnailUrl)
            .centerCrop()
            .into(cardView.mainImageView)
    }

    override fun onUnbindViewHolder(viewHolder: ViewHolder) {
        val cardView = viewHolder.view as ImageCardView
        cardView.mainImage = null  // clear to avoid image persistence
        Glide.with(cardView.context).clear(cardView.mainImageView)
    }
}
L6 Insight — Custom Card Layouts
ImageCardView covers most cases, but for metadata-rich cards (live match scores, progress bar for continue-watching, LIVE badge), extend BaseCardView with a custom layout. Override onFocusChanged() in the card view to animate scale/elevation on focus — this is the "pop" effect every streaming TV app uses. Scale to 1.1× on focus with a 150ms interpolator.

4. DiffCallback — Efficient Adapter Updates

// Use DiffCallback to update rows without full rebind (like DiffUtil for RecyclerView)
class VideoDiffCallback : DiffCallback() {

    override fun areItemsTheSame(oldItem: VideoItem, newItem: VideoItem): Boolean {
        return oldItem.id == newItem.id
    }

    override fun areContentsTheSame(oldItem: VideoItem, newItem: VideoItem): Boolean {
        return oldItem == newItem  // data class equals
    }
}

// Update an existing row adapter efficiently
val diffResult = DiffUtil.calculateDiff(
    ArrayObjectAdapterDiffCallback(currentAdapter, newItems, VideoDiffCallback())
)
currentAdapter.setItems(newItems, VideoDiffCallback())

5. DetailsSupportFragment — Content Detail Page

class VideoDetailFragment : DetailsSupportFragment() {

    private val detailsPresenter = FullWidthDetailsOverviewRowPresenter(DetailsDescriptionPresenter())
    private val rowsAdapter = ArrayObjectAdapter(detailsPresenter)

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val video = viewModel.selectedVideo

        // Main details row — large backdrop + actions
        val detailsRow = DetailsOverviewRow(video).apply {
            imageDrawable = ContextCompat.getDrawable(requireContext(), R.drawable.placeholder)
            // Load actual backdrop
            Glide.with(requireContext())
                .asBitmap()
                .load(video.backdropUrl)
                .into(object : CustomTarget() {
                    override fun onResourceReady(resource: Bitmap, t: Transition?) {
                        imageDrawable = BitmapDrawable(resources, resource)
                    }
                    override fun onLoadCleared(placeholder: Drawable?) {}
                })
        }

        // Action buttons — Play, Add to Watchlist, Download
        val actionsAdapter = SparseArrayObjectAdapter()
        actionsAdapter.set(0, Action(ACTION_PLAY, "Play"))
        actionsAdapter.set(1, Action(ACTION_WATCHLIST, "Add to List"))
        actionsAdapter.set(2, Action(ACTION_DOWNLOAD, "Download"))
        detailsRow.actionsAdapter = actionsAdapter

        rowsAdapter.add(detailsRow)

        // Related content row
        val relatedAdapter = ArrayObjectAdapter(VideoCardPresenter())
        relatedAdapter.addAll(0, viewModel.getRelatedVideos())
        rowsAdapter.add(ListRow(HeaderItem("More Like This"), relatedAdapter))

        adapter = rowsAdapter

        // Listen for action button clicks
        detailsPresenter.setOnActionClickedListener { action ->
            when (action.id) {
                ACTION_PLAY -> startPlayback(video)
                ACTION_WATCHLIST -> viewModel.toggleWatchlist(video)
                ACTION_DOWNLOAD -> viewModel.downloadVideo(video)
            }
        }
    }

    private fun startPlayback(video: VideoItem) {
        val intent = Intent(requireContext(), PlaybackActivity::class.java)
        intent.putExtra("video_id", video.id)
        startActivity(intent)
    }

    companion object {
        const val ACTION_PLAY = 1L
        const val ACTION_WATCHLIST = 2L
        const val ACTION_DOWNLOAD = 3L
    }
}

7. PlaybackSupportFragment — Video Player UI

class PlaybackFragment : VideoSupportFragment() {

    private lateinit var mTransportControlGlue: PlayerAdapter
    private lateinit var player: ExoPlayer

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        player = ExoPlayer.Builder(requireContext()).build()

        // LeanbackPlayerAdapter bridges ExoPlayer to Leanback transport controls
        val playerAdapter = LeanbackPlayerAdapter(requireContext(), player, UPDATE_DELAY_MS)

        // Glue connects the adapter to Leanback's transport control UI
        mTransportControlGlue = PlaybackTransportControlGlue(requireContext(), playerAdapter).apply {
            host = VideoSupportFragmentGlueHost(this@PlaybackFragment)
            title = videoTitle
            subtitle = videoSubtitle
            isSeekEnabled = true
            // Auto-hide controls after 3 seconds of inactivity
        }

        // Load and play
        val mediaItem = MediaItem.fromUri(videoUrl)
        player.setMediaItem(mediaItem)
        player.prepare()
        player.play()
    }

    override fun onPause() {
        super.onPause()
        mTransportControlGlue.pause()
    }

    override fun onDestroy() {
        super.onDestroy()
        player.release()
    }

    companion object {
        private const val UPDATE_DELAY_MS = 16  // ~60fps control update
    }
}
L6 Insight — LeanbackPlayerAdapter vs Custom Controls
LeanbackPlayerAdapter gives you free D-pad transport controls (play/pause, seek bar, ±10s skip). But Hotstar and Netflix use fully custom playback UIs — they extend VideoSupportFragment and build their own control overlay with custom animations, episode lists, subtitle track selectors, and audio quality indicators. The custom approach requires manually handling every KEYCODE_DPAD_* event and focus state on your overlay views. Use LeanbackPlayerAdapter for prototyping; go custom for production-quality playback UX.

8. GuidedStepSupportFragment — Wizard / Onboarding Flows

// Used for: login flow, parental controls PIN entry, subscription selection
class LoginGuidedFragment : GuidedStepSupportFragment() {

    override fun onCreateGuidance(savedInstanceState: Bundle?): GuidanceStylist.Guidance {
        return GuidanceStylist.Guidance(
            title = "Sign In",
            description = "Enter your email and password",
            breadcrumb = "Account Setup",
            iconDrawable = ContextCompat.getDrawable(requireContext(), R.drawable.ic_account)
        )
    }

    override fun onCreateActions(actions: MutableList, savedInstanceState: Bundle?) {
        // Email input field
        actions.add(
            GuidedAction.Builder(requireContext())
                .id(ACTION_EMAIL)
                .title("Email")
                .description("user@example.com")
                .editDescription(true)       // tappable — opens on-screen keyboard
                .descriptionEditable(true)
                .descriptionInputType(InputType.TYPE_TEXT_VARIATION_EMAIL_ADDRESS)
                .build()
        )

        // Password field
        actions.add(
            GuidedAction.Builder(requireContext())
                .id(ACTION_PASSWORD)
                .title("Password")
                .editDescription(true)
                .descriptionEditable(true)
                .descriptionInputType(InputType.TYPE_TEXT_VARIATION_PASSWORD or InputType.TYPE_CLASS_TEXT)
                .build()
        )
    }

    override fun onCreateButtonActions(actions: MutableList, savedInstanceState: Bundle?) {
        actions.add(
            GuidedAction.Builder(requireContext())
                .id(ACTION_SIGN_IN)
                .label("Sign In")
                .build()
        )
    }

    override fun onGuidedActionClicked(action: GuidedAction) {
        when (action.id) {
            ACTION_SIGN_IN -> {
                val email = findActionById(ACTION_EMAIL)?.description?.toString() ?: ""
                val password = findActionById(ACTION_PASSWORD)?.description?.toString() ?: ""
                viewModel.login(email, password)
            }
        }
    }

    companion object {
        const val ACTION_EMAIL = 1L
        const val ACTION_PASSWORD = 2L
        const val ACTION_SIGN_IN = 3L
    }
}

9. SearchSupportFragment — Voice + Text Search

class TvSearchFragment : SearchSupportFragment(), SearchSupportFragment.SearchResultProvider {

    private val rowsAdapter = ArrayObjectAdapter(ListRowPresenter())

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setSearchResultProvider(this)

        // Item click in search results
        setOnItemViewClickedListener { _, item, _, _ ->
            if (item is VideoItem) navigateToDetail(item)
        }
    }

    // Called every time the user types a character
    override fun getResultsAdapter(): ObjectAdapter = rowsAdapter

    override fun onQueryTextChange(newQuery: String): Boolean {
        search(newQuery)
        return true
    }

    override fun onQueryTextSubmit(query: String): Boolean {
        search(query)
        return true
    }

    private fun search(query: String) {
        if (query.length < 2) return
        viewModel.search(query).observe(viewLifecycleOwner) { results ->
            rowsAdapter.clear()
            val resultAdapter = ArrayObjectAdapter(VideoCardPresenter())
            resultAdapter.addAll(0, results)
            rowsAdapter.add(ListRow(HeaderItem("Results for \"$query\""), resultAdapter))
        }
    }
}

Interview Cheat Sheet — Leanback UI

  • BrowseSupportFragment structure? — HeadersSupportFragment (left rail, categories) + RowsSupportFragment (right, horizontal card rows). Each row is a ListRow = HeaderItem + ObjectAdapter + Presenter.
  • Presenter role? — Leanback's ViewHolder factory + binder. onCreateViewHolder inflates the card view; onBindViewHolder loads data. One presenter per card type. Always clear images in onUnbindViewHolder.
  • PageRow vs ListRow? — ListRow shows cards in horizontal scroll. PageRow replaces the right panel with a custom Fragment — used for sections like Sports/Live that need a unique layout.
  • DiffCallback? — setItems(list, DiffCallback) updates ArrayObjectAdapter efficiently — only rebinds changed items. Equivalent of DiffUtil for Leanback.
  • LeanbackPlayerAdapter? — Bridges ExoPlayer to Leanback transport controls (seek bar, play/pause). Use PlaybackTransportControlGlue + VideoSupportFragmentGlueHost. For custom playback UI, extend VideoSupportFragment directly.
  • GuidedStepSupportFragment? — Wizard-style UI for login, PIN entry, settings. Left panel shows guidance; right panel shows actions. Used for parental controls, subscription flows.
  • SearchSupportFragment? — Integrates voice + text search. Implement SearchResultProvider. onQueryTextChange called on every keystroke — debounce search calls. Register in manifest with SEARCH intent for system-level voice search.
