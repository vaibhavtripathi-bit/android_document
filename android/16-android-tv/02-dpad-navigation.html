D-pad Navigation & Focus

Focus traversal, key event handling, focus highlight, input edge cases — the biggest source of TV app bugs

1. How Focus Works on TV

On Android TV there is no touch input. The user navigates exclusively using a D-pad remote (up/down/left/right/center/back). Android's view system handles focus traversal automatically — but only if every interactive view is correctly configured. A broken focus chain means a user literally cannot reach a button.

D-pad Remote Input Flow ─────────────────────────────────────────────────────── User presses D-pad RIGHT │ ▼ KeyEvent(KEYCODE_DPAD_RIGHT, ACTION_DOWN) │ ▼ Activity.dispatchKeyEvent() │ ▼ (not consumed?) Current focused View.onKeyDown() │ ▼ (not consumed?) ViewGroup.focusSearch(FOCUS_RIGHT) │ Uses nextFocusRight xml attr, OR │ Android's geometric focus search algorithm ▼ Next View receives focus → requestFocus() │ ▼ View.onFocusChanged(focused=true) → draw focus ring

2. Making Views Focusable

<!-- XML — all interactive views on TV must be focusable -->
<ImageCardView
    android:focusable="true"
    android:focusableInTouchMode="true"   <!-- required for some older TVs -->
    android:nextFocusUp="@id/header_row"
    android:nextFocusDown="@id/next_row"
    android:nextFocusLeft="@id/prev_card"
    android:nextFocusRight="@id/next_card" />

<!-- For custom cards in code -->
// In code — set on custom card views
cardView.isFocusable = true
cardView.isFocusableInTouchMode = true

// Explicitly set next focus targets (use when automatic search picks wrong view)
cardView.nextFocusUpId = R.id.header_bar
cardView.nextFocusDownId = R.id.action_buttons
Common Bug — Focus Trap
If a modal dialog or overlay appears and no view inside it is focusable, the D-pad becomes unresponsive — the user is trapped. Always ensure dialogs and overlays have at least one focusable view and call view.requestFocus() when the overlay appears.

3. Focus Highlight — Visual Feedback

Without touch, users must always know which element is selected. Leanback provides FocusHighlight — a scale + shadow animation that "pops" the focused card forward.

// Leanback ListRowPresenter — built-in focus highlight
val rowPresenter = ListRowPresenter(FocusHighlight.ZOOM_FACTOR_MEDIUM)
// ZOOM_FACTOR_NONE, ZOOM_FACTOR_SMALL, ZOOM_FACTOR_MEDIUM, ZOOM_FACTOR_LARGE, ZOOM_FACTOR_XSMALL

// For custom card views — implement focus animation manually
class FocusableCardView @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null
) : FrameLayout(context, attrs) {

    init {
        isFocusable = true
        isFocusableInTouchMode = true
    }

    override fun onFocusChanged(gainFocus: Boolean, direction: Int, previouslyFocusedRect: Rect?) {
        super.onFocusChanged(gainFocus, direction, previouslyFocusedRect)
        if (gainFocus) {
            animate()
                .scaleX(1.1f)
                .scaleY(1.1f)
                .translationZ(8f)  // elevation — pops card above neighbours
                .setDuration(150)
                .setInterpolator(DecelerateInterpolator())
                .start()
        } else {
            animate()
                .scaleX(1.0f)
                .scaleY(1.0f)
                .translationZ(0f)
                .setDuration(150)
                .setInterpolator(DecelerateInterpolator())
                .start()
        }
    }
}

4. Key Event Handling

KEYCODE_DPAD_UPD-pad upMove focus up
KEYCODE_DPAD_DOWND-pad downMove focus down
KEYCODE_DPAD_LEFTD-pad leftMove focus left / seek back
KEYCODE_DPAD_RIGHTD-pad rightMove focus right / seek forward
KEYCODE_DPAD_CENTER / KEYCODE_ENTEROK / SelectClick / confirm
KEYCODE_BACKBack buttonNavigate back
KEYCODE_MEDIA_PLAY_PAUSEPlay/Pause buttonToggle playback

KEYCODE_MEDIA_STOPStop buttonStop playback, exit player
// Intercept key events in Activity or Fragment
override fun onKeyDown(keyCode: Int, event: KeyEvent): Boolean {
    return when (keyCode) {
        KeyEvent.KEYCODE_MEDIA_PLAY_PAUSE -> {
            if (player.isPlaying) player.pause() else player.play()
            true  // consumed — do NOT pass to super
        }
        KeyEvent.KEYCODE_DPAD_LEFT -> {
            if (isPlayerVisible && seekBarFocused) {
                player.seekTo(player.currentPosition - 10_000L)
                true  // custom seek — consume event
            } else {
                false  // let default focus traversal handle it
            }
        }
        KeyEvent.KEYCODE_BACK -> {
            if (isPlayerControlsVisible) {
                hideControls()
                true  // consume back — only hide controls, don't exit
            } else {
                false  // let default back handling exit the activity
            }
        }
        else -> super.onKeyDown(keyCode, event)
    }
}
L6 Insight — Long Press D-pad
D-pad keys repeat when held. KeyEvent.getRepeatCount() returns how many times the key has repeated (0 = first press, 1 = first repeat, ...). Use this for accelerating seek: first press seeks 10s, after 5 repeats seek 30s per event — the same technique Netflix uses for fast scrubbing with a remote.

Always handle both ACTION_DOWN and ACTION_UP if you need press-and-hold behavior. ACTION_DOWN fires repeatedly while held; ACTION_UP fires once on release.

5. Focus Restoration After Navigation

// Problem: user scrolls to row 3, card 5 — then opens detail and returns.
// Default behavior: focus resets to first card in first row. Bad UX.

// Solution: save and restore focus position
class MainFragment : BrowseSupportFragment() {

    // Save the focused item's adapter position
    private var savedFocusedRowIndex = 0
    private var savedFocusedCardIndex = 0

    override fun onPause() {
        super.onPause()
        // selectedPosition comes from BrowseSupportFragment
        savedFocusedRowIndex = selectedPosition
        // selectedItem is the focused card in the current row
    }

    override fun onResume() {
        super.onResume()
        // Restore row focus
        setSelectedPosition(savedFocusedRowIndex, false)
        // Restore card within row — need row adapter reference
    }
}

// In RowsSupportFragment — restore card position
fun restoreCardFocus(rowIndex: Int, cardIndex: Int) {
    val rowViewHolder = findRowViewHolderForAdapterPosition(rowIndex)
    rowViewHolder?.let {
        (it as ListRowPresenter.ViewHolder)
            .gridView
            .setSelectedPosition(cardIndex)
    }
}

6. Handling Focus in Custom Overlays

// Playback controls overlay — appears on D-pad press, auto-hides after timeout
class PlaybackControlsOverlay(context: Context) : FrameLayout(context) {

    private val hideHandler = Handler(Looper.getMainLooper())
    private val hideRunnable = Runnable { hide() }

    fun show() {
        visibility = VISIBLE
        // Give focus to the first control (play/pause button)
        playPauseButton.requestFocus()
        // Reset auto-hide timer
        hideHandler.removeCallbacks(hideRunnable)
        hideHandler.postDelayed(hideRunnable, 3000L)
    }

    fun hide() {
        visibility = GONE
        // Return focus to the video surface so key events still work
        videoView.requestFocus()
    }

    // Any D-pad press resets the hide timer
    override fun dispatchKeyEvent(event: KeyEvent): Boolean {
        if (event.action == KeyEvent.ACTION_DOWN) {
            hideHandler.removeCallbacks(hideRunnable)
            hideHandler.postDelayed(hideRunnable, 3000L)
        }
        return super.dispatchKeyEvent(event)
    }
}

7. Focus Debugging

// Enable focus debug logging — shows which view gains/loses focus
// adb shell setprop log.tag.ViewRootImpl DEBUG
// adb logcat -s ViewRootImpl

// In code — attach a global focus change listener
(requireActivity().window.decorView as ViewGroup)
    .viewTreeObserver
    .addOnGlobalFocusChangeListener { oldFocus, newFocus ->
        Log.d("FocusDebug",
            "Focus: ${oldFocus?.javaClass?.simpleName} → ${newFocus?.javaClass?.simpleName}"
        )
    }

// Dump the current focus chain
fun dumpFocusChain(view: View, depth: Int = 0) {
    val prefix = "  ".repeat(depth)
    val focusStr = if (view.hasFocus()) " ◄ FOCUSED" else ""
    Log.d("FocusChain", "$prefix${view.javaClass.simpleName}${focusStr}")
    if (view is ViewGroup) {
        for (i in 0 until view.childCount) {
            dumpFocusChain(view.getChildAt(i), depth + 1)
        }
    }
}
Testing Without a TV
Use the Android TV emulator (AVD Manager → TV category). To simulate D-pad events from keyboard: enable "D-pad emulation" in emulator extended controls. Alternatively, connect a USB keyboard — arrow keys map to D-pad, Enter maps to KEYCODE_DPAD_CENTER, Escape maps to KEYCODE_BACK. Always test the full navigation flow with a real remote before shipping — USB keyboard and D-pad have subtle timing differences.

8. RecyclerView Focus on TV (non-Leanback)

// If building a custom TV UI with RecyclerView instead of Leanback:

class TvLinearLayoutManager(context: Context) : LinearLayoutManager(context) {

    override fun onRequestChildFocus(
        parent: RecyclerView,
        state: RecyclerView.State,
        child: View,
        focused: View?
    ): Boolean {
        // Scroll the focused child into view — RecyclerView doesn't do this by default on TV
        val position = getPosition(child)
        scrollToPositionWithOffset(position, parent.width / 3)  // keep focused item at 1/3 from left
        return true
    }
}

// Use it:
recyclerView.layoutManager = TvLinearLayoutManager(context).apply {
    orientation = LinearLayoutManager.HORIZONTAL
}

// Ensure each item view is focusable
recyclerView.adapter = object : RecyclerView.Adapter<CardViewHolder>() {
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): CardViewHolder {
        val view = LayoutInflater.from(parent.context).inflate(R.layout.card_item, parent, false)
        view.isFocusable = true
        view.isFocusableInTouchMode = true
        return CardViewHolder(view)
    }
}

Interview Cheat Sheet — D-pad Navigation

  • Focus traversal algorithm? — Android uses geometric search: finds the closest focusable view in the requested direction by overlap and distance. Override with nextFocusUp/Down/Left/Right XML attrs or nextFocusUpId in code when automatic search picks wrong view.
  • Focus trap bug? — Modal/overlay with no focusable view inside locks the user out. Always requestFocus() on overlay appear, and restore focus on dismiss.
  • Focus highlight? — Use scale (1.0→1.1) + translationZ (0→8dp) on onFocusChanged(). Leanback ListRowPresenter does this automatically via FocusHighlight.ZOOM_FACTOR_MEDIUM.
  • Key event priority? — Activity.dispatchKeyEvent → focused View.onKeyDown → ViewGroup focus search. Return true from onKeyDown to consume; false or call super to propagate.
  • D-pad repeat on hold? — KeyEvent.getRepeatCount() increases while held. Use for accelerating seek: base seek 10s, fast seek 30s after N repeats.
  • Auto-hide controls? — Use Handler.postDelayed(hideRunnable, 3000). Any key event resets the timer. On hide, return focus to video surface so media keys still work.
  • Focus save/restore? — Save BrowseSupportFragment.selectedPosition on pause. Restore with setSelectedPosition() on resume. Prevents jarring reset to first item when returning from detail screen.
