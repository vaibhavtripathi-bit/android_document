3. Home Screen Channels — Content Discovery

The Leanback / Google TV launcher shows "channels" from your app directly on the home screen. Users see your content without opening your app. This is critical for engagement — Hotstar's IPL channel on the home screen drives direct playback without a single tap into the app.

// Publish a channel to the TV home screen launcher
class ChannelPublisher(private val context: Context) {

    fun publishChannel(channelName: String): Long {
        // Check permission first
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            val channel = Channel.Builder()
                .setType(TvContractCompat.Channels.TYPE_PREVIEW)
                .setDisplayName(channelName)           // "Hotstar Sports"
                .setAppLinkIntentUri(deepLinkUri)       // opens app when channel header tapped
                .build()

            val channelUri = context.contentResolver.insert(
                TvContractCompat.Channels.CONTENT_URI,
                channel.toContentValues()
            )
            return ContentUris.parseId(channelUri!!)
        }
        return -1L
    }

    fun publishPrograms(channelId: Long, videos: List) {
        // Clear old programs first
        context.contentResolver.delete(
            TvContractCompat.buildPreviewProgramsUriForChannel(channelId),
            null, null
        )

        videos.forEachIndexed { index, video ->
            val program = PreviewProgram.Builder()
                .setChannelId(channelId)
                .setType(TvContractCompat.Programs.Genres.MOVIES)
                .setTitle(video.title)
                .setDescription(video.synopsis)
                .setPosterArtUri(Uri.parse(video.thumbnailUrl))
                .setPosterArtAspectRatio(TvContractCompat.Programs.ASPECT_RATIO_16_9)
                .setDurationMillis(video.durationMs)
                .setWeight(videos.size - index)  // higher weight = shown first
                // Deep link — tapping the card opens playback directly
                .setIntentUri(Uri.parse("hotstar://play/${video.id}"))
                .build()

            context.contentResolver.insert(
                TvContractCompat.PreviewPrograms.CONTENT_URI,
                program.toContentValues()
            )
        }
    }
}

// Update channels periodically via WorkManager
class ChannelUpdateWorker(ctx: Context, params: WorkerParameters) : CoroutineWorker(ctx, params) {
    override suspend fun doWork(): Result {
        val videos = contentRepository.getFeaturedContent()
        ChannelPublisher(applicationContext).publishPrograms(savedChannelId, videos)
        return Result.success()
    }
}
L6 Insight — Channel Visibility
Channels are hidden by default until the user explicitly browses to your app's channel in the launcher settings and enables it. On first launch, trigger a one-time prompt: use ChannelHelper.requestChannelBrowsable(context, channelId) — this opens the launcher's "Add Channel" dialog. Once enabled, your channel persists on the home screen. Only call this once; repeated prompts are bad UX and ignored after the first.

4. Deep Links — Tapping Cards Without Opening the App

// Handle deep link from home screen card tap
// Or from voice search: "Play Mirzapur on Hotstar"
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        handleIntent(intent)
    }

    override fun onNewIntent(intent: Intent) {
        super.onNewIntent(intent)
        handleIntent(intent)
    }

    private fun handleIntent(intent: Intent) {
        when {
            // Deep link from home screen channel card
            intent.data != null -> {
                val videoId = intent.data?.lastPathSegment
                videoId?.let { navigateToPlayback(it) }
            }
            // Voice search result: SEARCH action with query
            Intent.ACTION_SEARCH == intent.action -> {
                val query = intent.getStringExtra(SearchManager.QUERY) ?: ""
                navigateToSearch(query)
            }
        }
    }
}

5. Voice Search Integration

// SearchFragment handles voice input from Google Assistant / TV remote mic
class TvSearchFragment : SearchSupportFragment(), SearchSupportFragment.SearchResultProvider {

    private val resultsAdapter = ArrayObjectAdapter(ListRowPresenter())

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setSearchResultProvider(this)
        setBadgeDrawable(ContextCompat.getDrawable(requireContext(), R.drawable.ic_mic))

        setOnItemViewClickedListener { _, item, _, _ ->
            if (item is VideoItem) {
                startActivity(PlaybackActivity.newIntent(requireContext(), item.id))
            }
        }
    }

    override fun getResultsAdapter(): ObjectAdapter = resultsAdapter

    override fun onQueryTextChange(newQuery: String): Boolean {
        performSearch(newQuery)
        return true
    }

    override fun onQueryTextSubmit(query: String): Boolean {
        performSearch(query)
        return true
    }

    private fun performSearch(query: String) {
        viewModel.search(query).observe(viewLifecycleOwner) { results ->
            resultsAdapter.clear()
            if (results.isNotEmpty()) {
                val cardAdapter = ArrayObjectAdapter(VideoCardPresenter())
                cardAdapter.addAll(0, results)
                resultsAdapter.add(ListRow(HeaderItem("Results"), cardAdapter))
            }
        }
    }
}

6. Performance Constraints on TV Hardware

ConstraintTypical TV specMitigation
RAM1–2 GB (low-end TVs)Limit image cache size; use Glide with small memory cache; avoid multiple large bitmaps in memory
CPUCortex-A53 quad-core, ~1.5 GHzMinimize work on main thread; use coroutines for all I/O; avoid complex custom views
GPUMali-400 or similar (low-end)Avoid overdraw; keep layer count low; don't use hardware acceleration for heavy animations if they drop to <60fps
Storage8–16 GB total (often 4 GB user)Limit download size; clear cache aggressively; check available space before download
NetworkWiFi (no mobile data)Default to higher bitrate ABR targets; use STRATEGY_KEEP_ONLY_LATEST for live streams
4K output3840×2160 via HDMIDecode at native 4K if Widevine L1 + HDCP 2.2 present; SurfaceView only (TextureView can't do 4K securely)
// Limit Glide's memory cache on TV (less RAM available)
Glide.get(context).apply {
    setMemoryCategory(MemoryCategory.LOW)
}

// Or configure in GlideModule:
@GlideModule
class TvGlideModule : AppGlideModule() {
    override fun applyOptions(context: Context, builder: GlideBuilder) {
        val memoryCacheSizeBytes = 1024 * 1024 * 50L  // 50 MB — conservative for TV
        builder.setMemoryCache(LruResourceCache(memoryCacheSizeBytes))
    }
}

7. Ambient Mode (Daydream / Screensaver)

When the TV is idle, Android TV enters ambient mode — a screensaver. Streaming apps show a slideshow of beautiful content artwork. This keeps the app "present" even when not actively watched.

// Declare ambient mode activity in manifest
// <activity android:name=".AmbientActivity">
//   <intent-filter>
//     <action android:name="android.intent.action.MAIN" />
//     <category android:name="android.intent.category.HOME" />
//     <category android:name="tv.danmaku.ijk.media.player.AMBIENT" />
//   </intent-filter>
// </activity>

// Leanback BackgroundManager — set backdrop images that animate during idle
class BackgroundHelper(private val activity: Activity) {

    private val backgroundManager = BackgroundManager.getInstance(activity).apply {
        attach(activity.window)
    }

    private val displayMetrics = DisplayMetrics().also {
        activity.windowManager.defaultDisplay.getMetrics(it)
    }

    fun updateBackground(imageUrl: String) {
        Glide.with(activity)
            .asBitmap()
            .load(imageUrl)
            .override(displayMetrics.widthPixels, displayMetrics.heightPixels)
            .centerCrop()
            .into(object : CustomTarget() {
                override fun onResourceReady(resource: Bitmap, t: Transition?) {
                    backgroundManager.setBitmap(resource)
                }
                override fun onLoadCleared(placeholder: Drawable?) {}
            })
    }

    fun clearBackground() {
        backgroundManager.clearDrawable()
    }
}

// Update background when focused item changes in BrowseSupportFragment
setOnItemViewSelectedListener { _, item, _, _ ->
    if (item is VideoItem) {
        backgroundHelper.updateBackground(item.backdropUrl)
    }
}

8. Widevine L1 on TV — 4K DRM

// TV devices with a TEE (Trusted Execution Environment) get Widevine L1
// This is required for 4K / HD streaming from Netflix, Hotstar Premium

fun checkWidevineSecurity(): Int {
    return try {
        val mediaDrm = MediaDrm(UUID.fromString("edef8ba9-79d6-4ace-a3c8-27dcd51d21ed"))
        val securityLevel = mediaDrm.getPropertyString("securityLevel")
        // "L1" = hardware TEE, can play 4K
        // "L3" = software only, capped at 480p/720p by content owner rules
        mediaDrm.close()
        if (securityLevel == "L1") MediaDrm.SECURITY_LEVEL_HW_SECURE_ALL else 3
    } catch (e: Exception) {
        3  // assume L3 if check fails
    }
}

L6 Insight — TV vs Phone DRM Differences
On phones, Widevine L1 is common (most flagship phones since 2016). On TVs, it depends entirely on the SoC and OEM implementation. Many budget TVs running Android TV (common in India — Vu, Thomson, iFFALCON) ship with L3 only. This means Hotstar Premium 4K content cannot play on those TVs at 4K — you must detect the security level at runtime and cap the maximum resolution accordingly. Never assume L1 on TV; always check and downgrade gracefully to SD/HD with a user-facing message.

9. TV App Architecture Pattern — Recommended

Recommended Architecture for a Streaming TV App ────────────────────────────────────────────────────────── UI Layer (Fragments) BrowseFragment → DetailFragment → PlaybackFragment │ │ │ └─────────────────┴────────────────┘ │ ViewModel (per screen) │ Repository (single source of truth) ├── RemoteDataSource (Retrofit + OkHttp) ├── LocalDataSource (Room) │ ├── watchlist table │ ├── watch_progress table │ └── downloaded_content table └── PlayerManager (singleton) ├── ExoPlayer instance ├── MediaSession └── DownloadManager Singletons (Application scope): - PlayerManager → one ExoPlayer, reused across screens - DownloadManager → background download service - DrmSessionManager → cached Widevine sessions
L6 Insight — Single ExoPlayer Instance
On TV, never create a new ExoPlayer in PlaybackFragment's onCreate(). Create it once in a PlayerManager singleton (Application scope or ViewModel scoped to the activity). Reuse the same instance across Browse → Detail → Playback transitions. This avoids the 300–500ms initialization cost and codec allocation on every navigation event. Release the player only when the app goes to background or the user explicitly stops playback.

10. ExoPlayer + MediaSession for TV Background Audio

// TV users switch apps while audio plays (music, podcast).
// MediaSessionService keeps playback alive when your app loses foreground.
// (See Section 13 — MediaSession for full implementation)

// On TV, MediaSession also enables:
// - Google Assistant voice control: "Hey Google, pause" → KEYCODE_MEDIA_PAUSE → MediaSession
// - Now Playing card on home screen (Android TV 9+)
// - Integration with HDMI-CEC: TV remote controls your app's player

class TvPlaybackService : MediaSessionService() {
    private lateinit var player: ExoPlayer
    private lateinit var mediaSession: MediaSession

    override fun onCreate() {
        super.onCreate()
        player = ExoPlayer.Builder(this).build()
        mediaSession = MediaSession.Builder(this, player).build()
    }

    override fun onGetSession(controllerInfo: MediaSession.ControllerInfo) = mediaSession

    override fun onDestroy() {
        mediaSession.release()
        player.release()
        super.onDestroy()
    }
}

Interview Cheat Sheet — TV Architecture

  • TV manifest requirements? — uses-feature leanback required=true, touchscreen required=false, android:banner (320×180 banner image), LEANBACK_LAUNCHER category on main activity.
  • Home screen channels? — Publish via TvContractCompat.Channels + PreviewProgram. Update with WorkManager. Request browsable with ChannelHelper.requestChannelBrowsable() once on first launch. Cards deep-link directly to playback.
  • Widevine L1 on TV? — Not guaranteed. Many budget India TVs (Vu, Thomson) are L3 only. Check MediaDrm.getPropertyString("securityLevel") and cap resolution accordingly. L1 required for 4K; L3 caps at 480p/720p per studio rules.
  • 4K playback requirements? — Widevine L1 (hardware TEE) + HDCP 2.2 on the HDMI output + SurfaceView (not TextureView). ExoPlayer automatically selects the highest quality track allowed by DRM security level.
  • Single ExoPlayer pattern? — One ExoPlayer instance in Application/Activity scope, reused across fragments. Avoids 300–500ms codec init cost on every screen transition.
  • Voice search integration? — Implement SearchSupportFragment + declare android.intent.action.SEARCH intent filter + searchable.xml metadata. Google Assistant routes "play X on Hotstar" queries to your SearchActivity.
  • Low RAM handling? — Limit Glide memory cache (50 MB), use MemoryCategory.LOW. Avoid loading full-resolution images for background rows — load only the focused card's backdrop. Release player when app backgrounded.
  • Background audio on TV? — Use MediaSessionService (same as phone). TV Android also gets Google Assistant voice control + HDMI-CEC remote passthrough via MediaSession.
