DRM & Widevine

Content protection end-to-end — Widevine L1 vs L3, MediaDrm, license flow, CENC, and what Netflix/Hotstar actually check

1. Why DRM Exists

Without DRM, any user could download an HLS/DASH segment file and keep it forever. Studios (Sony Pictures, Warner, Disney) will not license premium content unless the streaming platform can guarantee:

  1. Content is encrypted — raw H.264 bytes are never exposed in memory.
  2. Decryption keys are never accessible to software — they live in a trusted hardware environment.
  3. Output is protected — HDCP enforced on external displays.

Widevine (acquired by Google in 2010) is the DRM system used on Android. It is one of three systems that implement the Common Encryption (CENC) standard — the others are PlayReady (Microsoft) and FairPlay (Apple).

2. Widevine Security Levels

Widevine L1 — Hardware-backed
  • All crypto, decoding, and rendering happens inside a Trusted Execution Environment (TEE).
  • The decryption key never leaves the TEE — the Android CPU and your app code cannot read it.
  • Required for: HD (720p+) and 4K content on Netflix, Amazon Prime, Disney+, Hotstar.
  • Requires dedicated TEE hardware — most flagship and mid-range devices (Snapdragon 660+, Exynos 9810+, Dimensity 800+).
  • Check: MediaDrm.getPropertyString("securityLevel") == "L1"
  • Netflix HD/4K only streams to L1 devices.
Widevine L3 — Software-only
  • Decryption happens in software (in a sandboxed process, but still accessible to the CPU).
  • Keys are in RAM — theoretically extractable by a sophisticated attacker.
  • Content limited to: SD quality (480p) on most premium services.
  • Common on: older phones, emulators, rooted devices (TEE integrity check fails).
  • Check: MediaDrm.getPropertyString("securityLevel") == "L3"
L2 Exists but Is Rare
Widevine L2 = crypto in TEE but rendering in software. It was an intermediate level used by some older devices. In practice, all modern devices are either L1 or L3. You will not encounter L2 in production.

3. Common Encryption (CENC) — How Content Is Encrypted

Content is encrypted using AES-CTR (or AES-CBC for older HLS). Only the encrypted bytes are stored — the container format (MP4 box headers, NAL unit headers for H.264) remains in the clear so the player can parse the stream without decrypting it.

fMP4 segment structure with CENC encryption: moof box (unencrypted — ExoPlayer reads this) └── saiz / saio boxes — per-sample encryption info (IV, key ID) mdat box └── For each NAL unit in H.264: ┌─ NAL header (unencrypted — parser reads this) └─ NAL payload (AES-CTR encrypted with content key) The content key (CEK) is ~128-bit AES key. The CEK itself is wrapped by the DRM system's key — never stored in clear.

The manifest (MPD/M3U8) contains a PSSH box (Protection System Specific Header). The PSSH is a base64-encoded blob that tells the player:

  • Which DRM system to use (identified by UUID — Widevine UUID = EDEF8BA9-79D6-4ACE-A3C8-27DCD51D21ED).
  • The Key ID — which content key is needed (not the key itself).
  • Other DRM-specific data the license server needs.

5. Implementing DRM in ExoPlayer / Media3

// Full DRM setup with Media3
val drmConfig = MediaItem.DrmConfiguration.Builder(C.WIDEVINE_UUID)
    .setLicenseUri("https://license.hotstar.com/widevine")
    .setLicenseRequestHeaders(
        mapOf(
            "Authorization" to "Bearer ${userSession.token}",
            "X-Content-Id" to contentId
        )
    )
    // Force L3 for testing (will fail for premium content on L1-required streams)
    // .forceDefaultLicenseUri(true)
    .setMultiSession(false)   // true for streams with multiple key IDs (e.g., ad + content)
    .build()

val mediaItem = MediaItem.Builder()
    .setUri(streamUrl)
    .setDrmConfiguration(drmConfig)
    .build()

player.setMediaItem(mediaItem)
player.prepare()

Custom License Request/Response Handling

Sometimes you need to wrap the license request (add custom headers, transform the body). Use a custom DrmSessionManager:

val httpDataSourceFactory = DefaultHttpDataSource.Factory()
    .setDefaultRequestProperties(mapOf("User-Agent" to "HotstarAndroid/5.0"))

val drmSessionManager = DefaultDrmSessionManager.Builder()
    .setUuidAndExoMediaDrmProvider(C.WIDEVINE_UUID, FrameworkMediaDrm.DEFAULT_PROVIDER)
    .setMultiSession(false)
    .build(
        MediaDrmCallback { request, _ ->
            // Intercept: add auth before sending to license server
            val authRequest = request.toBuilder()
                .setRequestProperties(mapOf("Authorization" to "Bearer $token"))
                .build()

            // Send request, get response
            val response = httpDataSourceFactory.createDataSource()
                .use { dataSource ->
                    DataSourceUtil.readToByteArray(dataSource, authRequest.toDataSpec())
                }
            MediaDrmCallback.KeyResponse(response)
        }
    )

val mediaSource = DashMediaSource.Factory(
    DefaultDashChunkSource.Factory(httpDataSourceFactory),
    httpDataSourceFactory
)
    .setDrmSessionManagerProvider { drmSessionManager }
    .createMediaSource(mediaItem)

6. License Policies — What the License Server Controls

A Widevine license is not just a key — it is a key + playback policy. The license server sets:

PolicyMeaningExample
License durationHow long the license is valid24 hours for rental
Playback durationHow long the user can play once started48 hours from first play
Rental durationOffline window for downloaded content30 days to start watching
Output protectionHDCP required on external displaysHDCP 2.2 for 4K
Allowed track typesSD / HD / UHD based on device security levelHD only for L1 devices
Security level requiredMinimum Widevine level to decryptL1 for 4K Netflix
L6 Insight — License Expiry During Playback
A common production bug: the user starts a 2-hour movie, the license expires after 1 hour (license_duration_seconds = 3600 set incorrectly). ExoPlayer throws ERROR_CODE_DRM_LICENSE_EXPIRED. The fix is either: set license duration to cover full content duration + buffer, or implement license renewal via DefaultDrmSessionManager.setMode(MODE_PLAYBACK, offlineLicenseKeySetId) with a refresh callback.

7. Detecting Device DRM Capability

fun getWidevineSecurityLevel(context: Context): String {
    return try {
        val mediaDrm = MediaDrm(C.WIDEVINE_UUID)
        val level = mediaDrm.getPropertyString("securityLevel")
        mediaDrm.release()
        level  // "L1", "L2", or "L3"
    } catch (e: UnsupportedSchemeException) {
        "NOT_SUPPORTED"
    }
}

fun isWidevineL1(context: Context): Boolean =
    getWidevineSecurityLevel(context) == "L1"

// Use this to gate HD content
fun getMaxAllowedResolution(context: Context): Int =
    if (isWidevineL1(context)) 1080 else 480
Rooted Device / Emulator
On rooted devices, the TEE integrity check fails. Widevine downgrades to L3 even on hardware that would normally be L1. Emulators are always L3. This is expected — check Build.FINGERPRINT.contains("generic") to detect emulators in tests.

8. Output Protection — HDCP

When content requires HDCP (High-bandwidth Digital Content Protection), ExoPlayer checks if the connected display supports it:

// ExoPlayer checks HDCP level automatically via Display.getHdrCapabilities()
// and via the DRM policy from the license.
// For manual check:
val displayManager = context.getSystemService(DisplayManager::class.java)
val display = displayManager.getDisplay(Display.DEFAULT_DISPLAY)

// On Android 14+
val hdcpLevel = display.getHdrCapabilities()
// Older: check via MediaDrm property
val mediaDrm = MediaDrm(C.WIDEVINE_UUID)
val hdcpString = mediaDrm.getPropertyString("hdcpLevel")  // e.g. "HDCP_V2_2"

If a user connects a non-HDCP display while watching premium content, ExoPlayer will:

  1. Receive a PlaybackException with ERROR_CODE_DRM_CONTENT_ERROR.
  2. You should show: "This content cannot be played on external displays without HDCP support."

9. Multi-Key Content (Multi-Session DRM)

Some streams use different keys for different parts (e.g., a different key for the ad vs the main content, or different keys per resolution). This is called multi-session DRM:

val drmConfig = MediaItem.DrmConfiguration.Builder(C.WIDEVINE_UUID)
    .setLicenseUri(licenseUrl)
    .setMultiSession(true)  // enable multiple concurrent key sessions
    .build()
L6 Insight — Why Multi-Session?
Netflix uses different content keys per resolution track. This allows the license server to grant a key for 720p but refuse the 4K key for non-L1 devices — in the same stream. Without multi-session, you'd need separate stream URLs per quality tier.

Interview Cheat Sheet — DRM & Widevine

  • Widevine L1 vs L3? — L1: all crypto/decoding in TEE (hardware-backed). L3: software only. L1 required for HD+ content on Netflix, Hotstar, Prime. L3 → SD only.
  • What is a PSSH box? — Protection System Specific Header in the manifest. Contains the Key ID (not the key itself) and tells the CDM which DRM system and license server to contact.
  • License flow steps? — 1) Parse PSSH. 2) CDM generates challenge. 3) App POSTs challenge to license server with auth. 4) License server returns encrypted license. 5) CDM unwraps, stores key in TEE. 6) MediaCodec decrypts content.
  • What is CENC? — Common Encryption standard. AES-CTR encryption of media payload, leaving container headers clear. Used by Widevine, PlayReady, FairPlay.
  • License expiry bug? — Set license_duration_seconds ≥ content duration + buffer. Handle ERROR_CODE_DRM_LICENSE_EXPIRED with license renewal.
  • Rooted device? — TEE attestation fails, downgrades to L3. Premium HD content will be refused by license server policy.
  • HDCP? — Required for external display output with premium content. Check via MediaDrm.getPropertyString("hdcpLevel"). Failure → ERROR_CODE_DRM_CONTENT_ERROR.
  • Multi-session? — Multiple key IDs in one stream (per-resolution keys, ad vs content). Enable via setMultiSession(true) in DrmConfiguration.
  • Widevine UUID? — EDEF8BA9-79D6-4ACE-A3C8-27DCD51D21ED. Use C.WIDEVINE_UUID in ExoPlayer.
