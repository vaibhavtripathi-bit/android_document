Series 12: Browser Internals

Rendering pipeline, V8 engine, event loop, DOM APIs, networking, storage, and Web Workers — the platform knowledge that separates Staff engineers from framework users.

This series dives deep into the browser platform that React runs on. At Google L6, you're expected to understand not just React's abstractions but the underlying platform mechanics — how the rendering pipeline works, how V8 optimizes JavaScript, how the event loop schedules work, and how browser APIs enable advanced patterns. This knowledge lets you make informed performance decisions and debug issues that framework-level thinking can't explain.

Documents in This Series

01 — Rendering Engine

HTML parsing, DOM/CSSOM construction, render tree, layout, paint, composite. Critical rendering path, reflow vs repaint, GPU acceleration, will-change, contain, content-visibility.

02 — JavaScript Engine (V8)

V8 pipeline: parsing, AST, Ignition bytecode, Sparkplug, Maglev, TurboFan. Hidden classes, inline caches, deoptimization, garbage collection (Orinoco), JIT compilation.

03 — Event Loop

Call stack, task queue, microtask queue. setTimeout, rAF, rIC timing. Promise scheduling, queueMicrotask. Event loop and rendering. Long tasks, INP, Web Workers.

04 — DOM APIs

DOM manipulation performance, batch reads/writes, MutationObserver, IntersectionObserver, ResizeObserver, Custom Elements, Shadow DOM, event model, passive listeners.

05 — Networking

Fetch API, HTTP/2 multiplexing, HTTP/3 QUIC, Service Workers, Cache API, WebSocket, SSE, resource hints, Priority Hints, Background Sync, Beacon API.

06 — Storage APIs

localStorage, sessionStorage, IndexedDB, Cache API, cookies, Origin Private File System, storage quotas, cross-tab communication, structured clone algorithm.

07 — Web Workers

Dedicated, shared, service workers. postMessage, transferable objects, SharedArrayBuffer, Comlink, OffscreenCanvas, worklets, thread pool patterns.

08 — Interview Q&A

55+ L6-level questions on browser rendering, V8 internals, event loop, DOM APIs, networking, storage, and web workers with detailed answers.
