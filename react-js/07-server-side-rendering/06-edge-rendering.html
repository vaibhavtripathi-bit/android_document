Edge Computing Model

Edge computing moves server-side logic from centralized data centers to distributed points of presence (PoPs) close to end users. For React SSR, this means rendering HTML at the edge location nearest to the user, reducing network latency from hundreds of milliseconds to single-digit milliseconds.

Traditional Server Rendering: User (Tokyo) ──── 200ms RTT ────▶ Server (US-East) ──── Render ──── Response 50ms Total: ~450ms (200ms request + 50ms render + 200ms response) Edge Rendering: User (Tokyo) ──── 5ms RTT ────▶ Edge (Tokyo PoP) ──── Render ──── Response 50ms Total: ~60ms (5ms request + 50ms render + 5ms response)

V8 Isolates

Edge runtimes use V8 isolates instead of containers or VMs. A V8 isolate is a lightweight instance of the V8 JavaScript engine with its own heap and execution context. Multiple isolates share the same process, enabling near-instant cold starts.

Container-based (Traditional Serverless): ┌────────────────────────────────────┐ │ OS + Runtime + Dependencies │ │ ┌──────────────────────────────┐ │ │ │ Node.js Process │ │ │ │ ┌────────────────────────┐ │ │ │ │ │ Your Application │ │ │ │ │ └────────────────────────┘ │ │ │ └──────────────────────────────┘ │ └────────────────────────────────────┘ Cold start: 100ms - 5s Memory: 128MB - 1GB per instance V8 Isolate-based (Edge): ┌────────────────────────────────────┐ │ Single Process (shared V8 engine) │ │ ┌──────┐ ┌──────┐ ┌──────┐ │ │ │ Iso1 │ │ Iso2 │ │ Iso3 │ ... │ │ │ App1 │ │ App2 │ │ App3 │ │ │ └──────┘ └──────┘ └──────┘ │ └────────────────────────────────────┘ Cold start: <5ms Memory: ~128KB per isolate
V8 Isolate Security Model

Each isolate has its own memory heap — there's no shared state between isolates. This provides security isolation similar to containers but with microsecond-level startup. The tradeoff: isolates run in a restricted environment without access to the filesystem, native modules, or long-running processes.

Cloudflare Workers

// Cloudflare Workers with React SSR
import { renderToReadableStream } from 'react-dom/server';
import App from './App';

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url);

    // Access Cloudflare-specific features
    const cf = request.cf;
    const country = cf?.country ?? 'US';
    const city = cf?.city ?? 'Unknown';

    // Render React to a readable stream
    const stream = await renderToReadableStream(
      <App url={url.pathname} geo={{ country, city }} />,
      {
        bootstrapScripts: ['/static/bundle.js'],
        signal: AbortSignal.timeout(10000),
      }
    );

    return new Response(stream, {
      headers: {
        'Content-Type': 'text/html',
        'Cache-Control': 'public, max-age=60, s-maxage=300',
      },
    });
  },
};

// Using Cloudflare KV for edge state
async function getConfig(env: Env) {
  const config = await env.CONFIG_KV.get('site-config', 'json');
  return config;
}

// Using Cloudflare D1 (edge SQL database)
async function getProducts(env: Env) {
  const { results } = await env.DB.prepare(
    'SELECT * FROM products WHERE active = 1 LIMIT 50'
  ).all();
  return results;
}

Streaming from Edge

Edge runtimes support streaming SSR via the Web Streams API. This is particularly powerful because the edge location is close to the user, meaning streamed chunks arrive with minimal network latency.

// Streaming SSR from edge with Suspense
export const runtime = 'edge';

export default async function Page() {
  return (
    <html>
      <body>
        <Header />

        <Suspense fallback={<ProductSkeleton />}>
          <ProductDetails />
        </Suspense>

        <Suspense fallback={<ReviewsSkeleton />}>
          <Reviews />
        </Suspense>

        <Footer />
      </body>
    </html>
  );
}

// Edge streaming timeline:
// t=0ms:  Edge receives request (5ms from user)
// t=10ms: Shell streams (Header + skeletons + Footer)
// t=50ms: ProductDetails resolves (edge fetches from origin)
// t=120ms: Reviews resolves (separate API call)
//
// User sees content at t=15ms (shell) instead of t=125ms (buffered)

Geolocation-Based Rendering

Edge runtimes provide geolocation data from the CDN layer, enabling personalized rendering without client-side geolocation APIs or separate geo-IP services.

// Geolocation-based rendering at the edge
export const runtime = 'edge';

export default async function StorePage({ request }) {
  const geo = {
    country: request.headers.get('x-vercel-ip-country') ?? 'US',
    region: request.headers.get('x-vercel-ip-country-region'),
    city: request.headers.get('x-vercel-ip-city'),
  };

  // Fetch region-specific data
  const [products, currency, shipping] = await Promise.all([
    fetchProducts({ region: geo.country }),
    getCurrency(geo.country),
    getShippingOptions(geo.country, geo.region),
  ]);

  return (
    <StoreLayout currency={currency} geo={geo}>
      <ProductGrid products={products} />
      <ShippingEstimate options={shipping} />
    </StoreLayout>
  );
}

// Middleware for geo-based routing
// middleware.ts
export function middleware(request: NextRequest) {
  const country = request.geo?.country ?? 'US';
  const url = request.nextUrl.clone();

  // Redirect to country-specific store
  if (url.pathname === '/store' && country !== 'US') {
    url.pathname = `/store/${country.toLowerCase()}`;
    return NextResponse.rewrite(url);
  }

  // A/B testing based on region
  const variant = request.geo?.region?.startsWith('CA')
    ? 'west-coast'
    : 'default';
  const response = NextResponse.next();
  response.headers.set('x-variant', variant);
  return response;
}

Edge Caching Strategies

// Edge caching with Cache API
export const runtime = 'edge';

export async function GET(request: Request) {
  const cache = caches.default;
  const cacheKey = new Request(request.url, request);

  // Check cache first
  let response = await cache.match(cacheKey);
  if (response) {
    return response;
  }

  // Render and cache
  const stream = await renderToReadableStream(<App />);
  response = new Response(stream, {
    headers: {
      'Content-Type': 'text/html',
      'Cache-Control': 'public, s-maxage=300, stale-while-revalidate=600',
    },
  });

  // Store in edge cache (non-blocking)
  const cacheResponse = response.clone();
  await cache.put(cacheKey, cacheResponse);

  return response;
}

// Vary-based caching for personalization
function getCacheHeaders(request: Request) {
  const isAuthenticated = request.headers.get('cookie')?.includes('session');

  if (isAuthenticated) {
    return {
      'Cache-Control': 'private, no-cache',
      'Vary': 'Cookie',
    };
  }

  return {
    'Cache-Control': 'public, s-maxage=3600, stale-while-revalidate=86400',
    'CDN-Cache-Control': 'public, max-age=3600',
    'Vary': 'Accept-Encoding',
  };
}

Edge Middleware Patterns

// Pattern 1: Authentication at the edge
export async function middleware(request: NextRequest) {
  const token = request.cookies.get('session')?.value;

  if (!token && isProtectedRoute(request.nextUrl.pathname)) {
    return NextResponse.redirect(new URL('/login', request.url));
  }

  // Verify JWT at the edge (no server round-trip)
  if (token) {
    try {
      const payload = await verifyJWT(token);
      const headers = new Headers(request.headers);
      headers.set('x-user-id', payload.sub);
      return NextResponse.next({ headers });
    } catch {
      return NextResponse.redirect(new URL('/login', request.url));
    }
  }
}

// Pattern 2: Feature flags at the edge
export async function middleware(request: NextRequest) {
  const flags = await getFeatureFlags(request);
  const response = NextResponse.next();

  // Set flags as cookies for client-side access
  response.cookies.set('feature-flags', JSON.stringify(flags), {
    httpOnly: false,
    sameSite: 'lax',
  });

  // Rewrite to variant pages
  if (flags.newCheckout) {
    const url = request.nextUrl.clone();
    if (url.pathname === '/checkout') {
      url.pathname = '/checkout-v2';
      return NextResponse.rewrite(url);
    }
  }

  return response;
}

// Pattern 3: Rate limiting at the edge
const rateLimiter = new Map<string, { count: number; reset: number }>();

export async function middleware(request: NextRequest) {
  const ip = request.ip ?? 'unknown';
  const now = Date.now();
  const window = 60_000; // 1 minute
  const limit = 100;

  const entry = rateLimiter.get(ip);
  if (entry && now < entry.reset) {
    if (entry.count >= limit) {
      return new NextResponse('Too Many Requests', { status: 429 });
    }
    entry.count++;
  } else {
    rateLimiter.set(ip, { count: 1, reset: now + window });
  }

  return NextResponse.next();
}
Architect Insight

Edge rendering is not universally better than serverless or traditional servers. Use edge when: (1) latency is critical and your users are globally distributed, (2) the rendering logic is simple enough for edge constraints, and (3) your data sources are accessible from edge (HTTP APIs, edge databases). Keep complex processing, heavy computation, and stateful operations on traditional servers or serverless functions.
