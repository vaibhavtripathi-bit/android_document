Series 7: Server-Side Rendering

From renderToString to React Server Components and streaming architecture — mastering the server rendering pipeline, meta-frameworks, edge computing, and caching strategies at Staff-level depth.

Server-side rendering has evolved from a simple renderToString call into a sophisticated streaming architecture with React Server Components, selective hydration, and edge deployment. This series covers the full spectrum — from SSR fundamentals through Next.js internals, meta-framework comparison, edge rendering, and production caching strategies. Each document targets the depth expected in Google L6 / Staff-level interviews.

Documents in This Series

01 — SSR Fundamentals

renderToString vs renderToPipeableStream vs renderToReadableStream. SSR vs SSG vs ISR. Hydration process. Node.js rendering pipeline. HTML streaming. Universal/isomorphic JavaScript.

02 — React Server Components

RSC architecture, server vs client component boundary, 'use client'/'use server' directives, zero-bundle-size components, Flight protocol wire format, serialization constraints.

03 — Streaming Architecture

renderToPipeableStream internals, Suspense as streaming units, progressive rendering, out-of-order streaming, selective hydration, backpressure, Node.js streams vs Web Streams.

04 — Next.js Architecture

App Router internals, file-system routing, layouts/templates, Server Actions, caching layers (Request Memoization, Data Cache, Full Route Cache, Router Cache), partial prerendering.

05 — Meta-Frameworks

Remix (nested routes, loaders, actions), Gatsby (GraphQL data layer), Astro (islands architecture). Framework comparison, rendering strategies, micro-frontend integration.

06 — Edge Rendering

Vercel Edge Runtime, Cloudflare Workers, Deno Deploy. V8 isolates, streaming from edge, geolocation-based rendering, edge caching, runtime limitations, middleware patterns.

07 — Caching Strategies

HTTP caching (Cache-Control, ETag, SWR), CDN caching, application-level caching, ISR, on-demand revalidation, tag-based invalidation, distributed caching, cache stampede prevention.

08 — Interview Q&A

55+ L6/Staff-level questions covering SSR fundamentals, React Server Components, streaming, Next.js internals, meta-frameworks, edge rendering, and caching strategies.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6
SSR vs SSG vs ISR Choosing the right rendering strategy per route is an architectural decision with massive performance and cost implications at scale
React Server Components RSC fundamentally changes the React mental model — understanding the server/client boundary is critical for modern React architecture
Streaming & Selective Hydration Streaming SSR with Suspense boundaries enables progressive loading patterns that dramatically improve perceived performance
Framework Architecture Staff engineers must evaluate and choose meta-frameworks based on deep understanding of their internals, not just API surface
Edge Computing Edge rendering reduces latency globally but introduces runtime constraints — knowing the tradeoffs is essential for global-scale apps
Caching Architecture Multi-layer caching (CDN, application, framework) is the primary lever for SSR performance — misconfiguration causes stale data or cache stampedes

Prerequisites

  • Strong understanding of React fundamentals (Series 1) and rendering/reconciliation (Series 2)
  • Familiarity with Node.js HTTP servers and streams
  • Basic knowledge of HTTP caching headers and CDN concepts
  • Experience with at least one React meta-framework (Next.js, Remix, etc.)
