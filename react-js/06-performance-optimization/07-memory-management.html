2. Garbage Collection

V8 (Chrome/Node.js) uses a generational garbage collector with two main spaces:

V8 Generational GC: Young Generation (Scavenger вҖ” fast, frequent): в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ в”Ӯ Semi-space A (active) в”Ӯ Semi-space B в”Ӯ в”Ӯ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ӯ (empty) в”Ӯ в”Ӯ в”Ӯ в—Ӣ в”Ӯ в”Ӯ в—Ҹ в”Ӯ в”Ӯ в—Ӣ в”Ӯ в”Ӯ в”Ӯ в”Ӯ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в”Ӯ в”Ӯ в”Ӯ в—Ӣ = alive в—Ҹ = dead в”Ӯ в”Ӯ в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ After scavenge: alive objects copied to B, A is wiped. Objects surviving 2 scavenges are promoted to Old Generation. Old Generation (Mark-Sweep-Compact вҖ” slower, less frequent): в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ в”Ӯ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ в”Ӯ в”Ӯ в”Ӯ в—Ӣ в”Ӯ в”Ӯ в—Ҹ в”Ӯ в”Ӯ в—Ӣ в”Ӯ в”Ӯ в—Ҹ в”Ӯ в”Ӯ в—Ӣ в”Ӯ в”Ӯ в”Ӯ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в””в”Җв”Җв”Җв”ҳ в”Ӯ в”Ӯ в”Ӯ в”Ӯ 1. Mark: traverse from roots, mark liveв”Ӯ в”Ӯ 2. Sweep: free unmarked objects в”Ӯ в”Ӯ 3. Compact: defragment (optional) в”Ӯ в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ GC Roots: global object, stack variables, active closures, event listeners, timers, DOM references
Key Insight

An object is "alive" if it's reachable from a GC root through any chain of references. A memory leak occurs when objects that are no longer needed remain reachable вҖ” the GC can't collect them because something still holds a reference. In React, the most common leak sources are: uncleaned event listeners, uncleared timers, closures capturing large objects, and detached DOM nodes.

3. Memory Leaks in React

// The 5 most common memory leak patterns in React:

// 1. Forgotten event listeners (window, document, WebSocket)
// 2. Uncleared timers (setInterval, setTimeout)
// 3. Closures capturing large state
// 4. Detached DOM nodes (refs to unmounted elements)
// 5. Growing state (unbounded arrays, caches without eviction)

// React's useEffect cleanup is the primary defense:
useEffect(() => {
  // Setup: allocate resources
  const subscription = eventBus.subscribe(handler);
  const timer = setInterval(tick, 1000);
  window.addEventListener('resize', handleResize);

  // Cleanup: release resources (runs on unmount or re-run)
  return () => {
    subscription.unsubscribe();
    clearInterval(timer);
    window.removeEventListener('resize', handleResize);
  };
}, []);

4. Event Listener Leaks

// вқҢ Leak: event listener not cleaned up
function ScrollTracker() {
  const [scrollY, setScrollY] = useState(0);

  useEffect(() => {
    window.addEventListener('scroll', () => {
      setScrollY(window.scrollY);  // anonymous function вҖ” can't remove!
    });
    // No cleanup! Listener persists after unmount
    // Each mount adds another listener
  }, []);
}

// вң… Fixed: named handler + cleanup
function ScrollTracker() {
  const [scrollY, setScrollY] = useState(0);

  useEffect(() => {
    const handleScroll = () => setScrollY(window.scrollY);
    window.addEventListener('scroll', handleScroll, { passive: true });
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);
}

// вқҢ Leak: WebSocket not closed
function ChatRoom({ roomId }) {
  useEffect(() => {
    const ws = new WebSocket(`wss://chat.example.com/${roomId}`);
    ws.onmessage = (e) => addMessage(JSON.parse(e.data));
    // No cleanup! WebSocket stays open after unmount
  }, [roomId]);
}

// вң… Fixed: close WebSocket on cleanup
useEffect(() => {
  const ws = new WebSocket(`wss://chat.example.com/${roomId}`);
  ws.onmessage = (e) => addMessage(JSON.parse(e.data));
  return () => ws.close();
}, [roomId]);

5. Timer & Interval Leaks

// вқҢ Leak: interval not cleared
function PollingComponent() {
  useEffect(() => {
    setInterval(async () => {
      const data = await fetchData();
      setData(data);  // setState on unmounted component!
    }, 5000);
    // Interval runs forever, even after unmount
  }, []);
}

// вң… Fixed: clear interval + abort fetch
function PollingComponent() {
  useEffect(() => {
    const controller = new AbortController();

    const intervalId = setInterval(async () => {
      try {
        const data = await fetchData({ signal: controller.signal });
        setData(data);
      } catch (e) {
        if (e.name !== 'AbortError') console.error(e);
      }
    }, 5000);

    return () => {
      clearInterval(intervalId);
      controller.abort();
    };
  }, []);
}

6. Closure Leaks

// Closures capture variables from their enclosing scope
// If a closure lives longer than expected, it keeps those variables alive

// вқҢ Closure captures entire component scope
function DataProcessor({ hugeDataset }) {
  useEffect(() => {
    const processedData = expensiveProcess(hugeDataset); // 100MB

    const handler = () => {
      // This closure captures processedData (100MB)
      // Even if we only use processedData.summary
      console.log(processedData.summary);
    };

    window.addEventListener('focus', handler);
    return () => window.removeEventListener('focus', handler);
  }, [hugeDataset]);
}

// вң… Extract only what the closure needs
function DataProcessor({ hugeDataset }) {
  useEffect(() => {
    const processedData = expensiveProcess(hugeDataset);
    const summary = processedData.summary;  // extract small value
    // processedData can now be GC'd

    const handler = () => {
      console.log(summary);  // closure only captures summary (small)
    };

    window.addEventListener('focus', handler);
    return () => window.removeEventListener('focus', handler);
  }, [hugeDataset]);
}

7. Detached DOM Nodes

// Detached DOM: nodes removed from the document but still referenced

// вқҢ Ref holds reference to unmounted DOM node
function DynamicList({ items }) {
  const itemRefs = useRef({});  // accumulates refs over time

  return items.map(item => (
    <div key={item.id} ref={el => { itemRefs.current[item.id] = el; }}>
      {item.name}
    </div>
  ));
  // When items are removed, their DOM nodes are detached
  // but itemRefs.current still holds references вҶ’ leak!
}

// вң… Clean up refs when items are removed
function DynamicList({ items }) {
  const itemRefs = useRef({});

  useEffect(() => {
    const currentIds = new Set(items.map(i => i.id));
    for (const id of Object.keys(itemRefs.current)) {
      if (!currentIds.has(id)) {
        delete itemRefs.current[id];  // clean up stale refs
      }
    }
  }, [items]);

  return items.map(item => (
    <div key={item.id} ref={el => {
      if (el) itemRefs.current[item.id] = el;
      else delete itemRefs.current[item.id];
    }}>
      {item.name}
    </div>
  ));
}

// Monitor memory growth over time
function useMemoryMonitor(intervalMs = 5000) {
  useEffect(() => {
    const readings = [];
    const id = setInterval(() => {
      if (performance.memory) {
        readings.push({
          time: Date.now(),
          heap: performance.memory.usedJSHeapSize,
        });
        if (readings.length > 2) {
          const growth = readings.at(-1).heap - readings[0].heap;
          if (growth > 50 * 1024 * 1024) {
            console.warn(`Memory grew ${(growth / 1024 / 1024).toFixed(1)}MB`);
          }
        }
      }
    }, intervalMs);
    return () => clearInterval(id);
  }, [intervalMs]);
}

9. WeakRef & FinalizationRegistry

// WeakRef: holds a reference that doesn't prevent GC
const cache = new Map();

function getCachedData(key, fetchFn) {
  const weakRef = cache.get(key);
  const cached = weakRef?.deref();

  if (cached) return cached;  // still alive in memory

  const data = fetchFn();
  cache.set(key, new WeakRef(data));
  return data;
}
// If memory pressure is high, GC can collect the cached data
// Next call to getCachedData will re-fetch

// FinalizationRegistry: callback when object is GC'd
const registry = new FinalizationRegistry((key) => {
  console.log(`Object for key "${key}" was garbage collected`);
  cache.delete(key);  // clean up the WeakRef entry
});

function cacheData(key, data) {
  cache.set(key, new WeakRef(data));
  registry.register(data, key);  // notify when data is GC'd
}

// WeakMap: keys are weakly held (GC'd when no other references)
const metadata = new WeakMap();
function attachMetadata(domNode, data) {
  metadata.set(domNode, data);
  // When domNode is GC'd, the entry is automatically removed
  // No manual cleanup needed
}

10. Memory Profiling Patterns

// Pattern 1: Repeated action test
// 1. Navigate to page A
// 2. Take heap snapshot
// 3. Navigate to page B
// 4. Navigate back to page A
// 5. Force GC
// 6. Take heap snapshot
// 7. Repeat steps 3-6 several times
// 8. If heap grows with each cycle вҶ’ leak

// Pattern 2: Component mount/unmount test
// 1. Render component
// 2. Take heap snapshot
// 3. Unmount component
// 4. Force GC
// 5. Take heap snapshot
// 6. Compare: look for objects from the component that persist

// Pattern 3: Automated leak detection in tests
async function detectLeaks(action, iterations = 10) {
  const measurements = [];

  for (let i = 0; i < iterations; i++) {
    await action();
    if (global.gc) global.gc();  // node --expose-gc
    measurements.push(process.memoryUsage().heapUsed);
  }

  const growth = measurements.at(-1) - measurements[0];
  const avgGrowthPerIteration = growth / iterations;

  if (avgGrowthPerIteration > 100 * 1024) {
    console.error(`Potential leak: ${(avgGrowthPerIteration / 1024).toFixed(0)}KB per iteration`);
  }
}
Architect Insight

Memory leaks in SPAs are insidious вҖ” they accumulate over time as users navigate between pages. A leak of 1MB per navigation means the app consumes 100MB after 100 navigations, eventually causing the browser tab to crash. Establish a memory profiling practice: (1) profile critical user flows in CI, (2) set memory budgets (e.g., heap should not grow more than 5MB per navigation cycle), (3) use the repeated action test pattern regularly, (4) review useEffect cleanup in code reviews.
