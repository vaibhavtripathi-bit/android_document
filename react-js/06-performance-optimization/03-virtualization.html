1. The Virtualization Concept

Virtualization (windowing) renders only the items visible in the viewport, plus a small buffer. Instead of creating 10,000 DOM nodes for 10,000 items, it creates ~20 DOM nodes and repositions them as the user scrolls.

Without Virtualization (10,000 items): ┌─────────────────┐ │ Item 1 │ ← DOM node │ Item 2 │ ← DOM node │ Item 3 │ ← DOM node │ ... │ │ Item 10,000 │ ← DOM node └─────────────────┘ 10,000 DOM nodes in memory Initial render: ~2-5 seconds Memory: ~50-200MB With Virtualization: ┌─────────────────┐ │ (spacer: 4900px)│ ← empty div for scroll height ┌─ overscan ──── │ Item 48 │ ← DOM node (buffer) │ │ Item 49 │ ← DOM node (buffer) │ ┌─ viewport ─ │ Item 50 │ ← DOM node (visible) │ │ │ Item 51 │ ← DOM node (visible) │ │ │ Item 52 │ ← DOM node (visible) │ └──────────── │ Item 53 │ ← DOM node (visible) │ │ Item 54 │ ← DOM node (buffer) └──────────────── │ Item 55 │ ← DOM node (buffer) │ (spacer: 4900px)│ ← empty div for remaining height └─────────────────┘ ~8 DOM nodes in memory (4 visible + 4 overscan) Initial render: ~5ms Memory: ~1-2MB
Architect Insight

The performance difference is dramatic: rendering 10,000 items without virtualization creates 10,000 DOM nodes, each requiring memory for the node itself, its layout, paint layers, and event listeners. With virtualization, you maintain ~20-50 DOM nodes regardless of list size. This is the difference between a 5-second initial render and a 5-millisecond one. Any list over ~100 items should be considered for virtualization.

2. How Virtual Scrolling Works

// Simplified virtual list implementation
function VirtualList({ items, itemHeight, containerHeight }) {
  const [scrollTop, setScrollTop] = useState(0);

  const totalHeight = items.length * itemHeight;
  const startIndex = Math.floor(scrollTop / itemHeight);
  const visibleCount = Math.ceil(containerHeight / itemHeight);
  const overscan = 3;

  const renderStart = Math.max(0, startIndex - overscan);
  const renderEnd = Math.min(items.length, startIndex + visibleCount + overscan);

  const visibleItems = [];
  for (let i = renderStart; i < renderEnd; i++) {
    visibleItems.push(
      <div
        key={items[i].id}
        style={{
          position: 'absolute',
          top: i * itemHeight,
          height: itemHeight,
          width: '100%',
        }}
      >
        {items[i].name}
      </div>
    );
  }

  return (
    <div
      style={{ height: containerHeight, overflow: 'auto', position: 'relative' }}
      onScroll={(e) => setScrollTop(e.currentTarget.scrollTop)}
    >
      <div style={{ height: totalHeight }}>
        {visibleItems}
      </div>
    </div>
  );
}
Virtual List Internals: Scroll Container (overflow: auto, height: 400px) │ └── Inner Container (height: totalHeight = 10000 * 50px = 500000px) │ ├── [absolute positioned items only in visible range] │ ├── Item at top: renderStart * itemHeight │ ├── ...visible items... │ └── Item at bottom: (renderEnd - 1) * itemHeight │ └── Empty space creates scrollbar proportional to total content The browser's native scrollbar works because the inner container has the correct total height. Only ~20 items are actually rendered.

// TanStack Virtual: headless virtualizer
import { useVirtualizer } from '@tanstack/react-virtual';

function VirtualList({ items }) {
  const parentRef = useRef(null);

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
    overscan: 5,
  });

  return (
    <div ref={parentRef} style={{ height: 600, overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualizer.getVirtualItems().map(virtualRow => (
          <div
            key={virtualRow.key}
            style={{
              position: 'absolute',
              top: virtualRow.start,
              height: virtualRow.size,
              width: '100%',
            }}
          >
            {items[virtualRow.index].name}
          </div>
        ))}
      </div>
    </div>
  );
}

4. Fixed-Size vs Variable-Size Lists

// Fixed-size: all items have the same height
// Calculation is trivial: item position = index * itemHeight
// No measurement needed — fastest option

// Variable-size: items have different heights
// Two approaches:

// 1. Known sizes (provide a size function)
const virtualizer = useVirtualizer({
  count: items.length,
  getScrollElement: () => parentRef.current,
  estimateSize: (index) => items[index].type === 'header' ? 80 : 50,
});

// 2. Dynamic measurement (measure after render)
const virtualizer = useVirtualizer({
  count: items.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 50,  // initial estimate
  measureElement: (el) => el.getBoundingClientRect().height,
});

// Each item must ref the measureElement:
{virtualizer.getVirtualItems().map(row => (
  <div
    key={row.key}
    ref={virtualizer.measureElement}
    data-index={row.index}
  >
    <DynamicContent item={items[row.index]} />
  </div>
))}
Variable-Size Challenges

Variable-size virtualization is significantly more complex: (1) scroll position calculation requires summing all previous item heights, (2) jumping to a specific index requires estimating heights for unmeasured items, (3) items changing size after measurement causes scroll position jumps, (4) ResizeObserver callbacks can cause layout thrashing. Use fixed-size lists whenever possible — they're simpler and faster.

5. Grid Virtualization

// TanStack Virtual: 2D grid virtualization
function VirtualGrid({ rows, columns }) {
  const parentRef = useRef(null);

  const rowVirtualizer = useVirtualizer({
    count: rows,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
    overscan: 5,
  });

  const colVirtualizer = useVirtualizer({
    horizontal: true,
    count: columns,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 150,
    overscan: 3,
  });

  return (
    <div ref={parentRef} style={{ height: 600, width: 800, overflow: 'auto' }}>
      <div style={{
        height: rowVirtualizer.getTotalSize(),
        width: colVirtualizer.getTotalSize(),
        position: 'relative',
      }}>
        {rowVirtualizer.getVirtualItems().map(virtualRow => (
          colVirtualizer.getVirtualItems().map(virtualCol => (
            <div
              key={`${virtualRow.index}-${virtualCol.index}`}
              style={{
                position: 'absolute',
                top: virtualRow.start,
                left: virtualCol.start,
                height: virtualRow.size,
                width: virtualCol.size,
              }}
            >
              Cell {virtualRow.index},{virtualCol.index}
            </div>
          ))
        ))}
      </div>
    </div>
  );
}

6. Overscan & Buffer Zones

Overscan renders extra items above and below the visible area to prevent blank flashes during fast scrolling.

Overscan Effect: overscan=0: overscan=5: ┌───────────────┐ ┌───────────────┐ │ │ ← blank │ Item 45 │ ← pre-rendered │ │ during │ Item 46 │ │ │ fast │ Item 47 │ │ │ scroll │ Item 48 │ ├─ viewport ────┤ │ Item 49 │ │ Item 50 │ ├─ viewport ────┤ │ Item 51 │ │ Item 50 │ │ Item 52 │ │ Item 51 │ │ Item 53 │ │ Item 52 │ ├───────────────┤ │ Item 53 │ │ │ ← blank ├───────────────┤ │ │ │ Item 54 │ └───────────────┘ │ Item 55 │ │ Item 56 │ │ Item 57 │ │ Item 58 │ ← pre-rendered └───────────────┘ Higher overscan = smoother scrolling but more DOM nodes Typical values: 3-10 for lists, 1-3 for grids

7. Infinite Scroll Implementation

// Infinite scroll with TanStack Virtual + TanStack Query
function InfiniteList() {
  const {
    data,
    fetchNextPage,
    hasNextPage,
    isFetchingNextPage,
  } = useInfiniteQuery({
    queryKey: ['items'],
    queryFn: ({ pageParam = 0 }) => fetchItems({ offset: pageParam, limit: 50 }),
    getNextPageParam: (lastPage) => lastPage.nextOffset,
  });

  const allItems = data?.pages.flatMap(page => page.items) ?? [];
  const parentRef = useRef(null);

  const virtualizer = useVirtualizer({
    count: hasNextPage ? allItems.length + 1 : allItems.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
    overscan: 5,
  });

  // Fetch next page when approaching the end
  useEffect(() => {
    const lastItem = virtualizer.getVirtualItems().at(-1);
    if (!lastItem) return;

    if (
      lastItem.index >= allItems.length - 1 &&
      hasNextPage &&
      !isFetchingNextPage
    ) {
      fetchNextPage();
    }
  }, [virtualizer.getVirtualItems(), hasNextPage, isFetchingNextPage, allItems.length]);

  return (
    <div ref={parentRef} style={{ height: 600, overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualizer.getVirtualItems().map(virtualRow => {
          const isLoaderRow = virtualRow.index >= allItems.length;
          return (
            <div key={virtualRow.key} style={{
              position: 'absolute',
              top: virtualRow.start,
              height: virtualRow.size,
              width: '100%',
            }}>
              {isLoaderRow ? 'Loading more...' : allItems[virtualRow.index].name}
            </div>
          );
        })}
      </div>
    </div>
  );
}

8. Scroll Restoration with Virtualization

// Challenge: scroll restoration requires the virtualizer to
// seek to a specific offset, but items may not be measured yet

// TanStack Virtual: scrollToIndex
const virtualizer = useVirtualizer({
  count: items.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 50,
});

// Restore scroll position
useEffect(() => {
  const savedIndex = sessionStorage.getItem('scroll-index');
  if (savedIndex) {
    virtualizer.scrollToIndex(Number(savedIndex), { align: 'start' });
  }
}, []);

// Save scroll position before navigation
useEffect(() => {
  return () => {
    const items = virtualizer.getVirtualItems();
    if (items.length > 0) {
      sessionStorage.setItem('scroll-index', String(items[0].index));
    }
  };
}, []);

9. DOM Node Recycling

DOM recycling reuses existing DOM nodes instead of creating and destroying them as items scroll in and out of view. This reduces garbage collection pressure and DOM allocation overhead.

// Most React virtualization libraries create/destroy DOM nodes
// as items enter/leave the viewport. This generates GC pressure.

// DOM recycling approach (used by some native virtualizers):
// Instead of unmounting Item A and mounting Item B,
// reuse Item A's DOM node and update its content to show Item B.

// In React, this is achieved via the key prop:
// Using stable keys (item.id) → React reuses fiber, updates props
// Using index keys → React reuses DOM nodes in order (recycling effect)

// For virtualization, index-based keys can actually be beneficial:
{virtualizer.getVirtualItems().map((virtualRow, i) => (
  <div
    key={i}  // index key → DOM node recycled as items scroll
    style={{ ... }}
  >
    {items[virtualRow.index].name}
  </div>
))}

// Trade-off: index keys prevent React from preserving component state
// across scroll positions. Use item.id keys when items have internal state.

10. Memory Management

// Memory considerations for virtualized lists:

// 1. Data array: 10,000 items × ~1KB each = ~10MB
//    The full data array must be in memory even if only 20 items render
//    For very large datasets, use pagination instead of loading all data

// 2. Measured sizes cache: variable-size lists store measured heights
//    10,000 items × 8 bytes = ~80KB (negligible)

// 3. DOM nodes: ~20 nodes × ~2KB each = ~40KB (vs 20MB without virtualization)

// 4. React fibers: ~20 fibers × ~1KB each = ~20KB

// Memory optimization for infinite lists:
function useWindowedData(fetchPage, pageSize = 50, maxPages = 10) {
  const [pages, setPages] = useState(new Map());

  // Keep only the most recent maxPages in memory
  // Evict oldest pages when limit exceeded
  const loadPage = useCallback(async (pageIndex) => {
    if (pages.has(pageIndex)) return;

    const data = await fetchPage(pageIndex);
    setPages(prev => {
      const next = new Map(prev);
      next.set(pageIndex, data);

      // Evict oldest pages if over limit
      while (next.size > maxPages) {
        const oldest = next.keys().next().value;
        next.delete(oldest);
      }
      return next;
    });
  }, [fetchPage, maxPages]);

  return { pages, loadPage };
}
Architect Insight

Virtualization is not free. It adds complexity: keyboard navigation, accessibility (screen readers can't see unmounted items), search/find-in-page doesn't work for off-screen items, and print stylesheets need special handling. Consider alternatives first: pagination (simplest), "show more" buttons, or server-side filtering to reduce the dataset. Use virtualization when you genuinely need to display thousands of items in a scrollable container.
