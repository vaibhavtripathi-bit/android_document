Series 2: Rendering & Reconciliation

From render phases and scheduler internals to concurrent features, Suspense, batching, hydration, and the react-dom host renderer — the complete rendering pipeline at Staff-level depth.

This series traces the complete journey of a React update from setState through the scheduler, render phase, commit phase, and finally to DOM mutations. We explore React 18's concurrent rendering engine, the Suspense mechanism, automatic batching, the react-dom host renderer internals, and server-side rendering with hydration. Each document targets the depth expected for Google L6 / Staff-level frontend interviews.

Documents in This Series

01 — Render Phases

Two-phase rendering model: render (pure, interruptible) and commit (synchronous, side-effects). beginWork, completeWork, commitWork internals. Phase diagram with fiber tree traversal.

02 — Scheduler Internals

React Scheduler package, priority lanes, MessageChannel scheduling, 5ms time slicing, task expiration, starvation prevention, shouldYield() mechanism, and scheduler vs browser task queue.

03 — Concurrent Features

startTransition, useTransition, useDeferredValue, concurrent rendering with interruptible render trees, selective hydration, transition lanes, and real-world use cases.

04 — Suspense Deep Dive

Suspense boundaries, thrown promises, use() hook, streaming SSR with renderToPipeableStream, nested Suspense, waterfall solutions, SuspenseList, and cache integration.

05 — Batching Updates

Automatic batching in React 18+, legacy batching behavior, flushSync, microtask scheduling, why setTimeout was unbatched pre-18, and multiple setState optimization.

06 — React DOM Internals

react-dom as host renderer, createRoot vs legacy render, property vs attribute handling, controlled vs uncontrolled components, event delegation, SyntheticEvent, and event priorities.

07 — Hydration

Server HTML to client hydration, hydrateRoot API, mismatch detection, selective hydration, progressive hydration, streaming SSR + selective hydration flow, and performance optimization.

08 — Interview Q&A

55+ L6-level interview questions with detailed answers covering render phases, scheduler internals, concurrent features, Suspense mechanics, batching, hydration edge cases.
