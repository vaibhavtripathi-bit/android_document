Series 5: Routing & Navigation

From the History API and URL parsing to React Router internals, data loading patterns, code splitting, and Next.js App Router — the complete routing story every Staff-level frontend engineer must command.

Routing is the backbone of any non-trivial web application. At the Staff level, you're expected to understand not just how to configure routes, but the underlying browser APIs that make client-side routing possible, the algorithms that match URLs to components, the data loading strategies that prevent waterfalls, and the code splitting techniques that keep bundles lean. This series covers the full routing stack from raw History API through React Router v6+ internals to Next.js App Router architecture.

Documents in This Series

01 — Client-Side Routing

History API (pushState, replaceState, popstate), hash vs browser routing, route matching algorithms, URL parsing, search params management, navigation interception, and scroll restoration.

02 — React Router Internals

React Router v6+ architecture, createBrowserRouter, route objects and matching algorithm, Outlet and nested routes, data router loaders/actions, route-level error boundaries, and index routes.

03 — Data Loading Patterns

Route-level data loading with loaders, parallel vs waterfall fetching, defer() for streaming, Suspense integration, optimistic navigation, prefetching, stale-while-revalidate, and race condition handling.

04 — Code Splitting & Routes

React.lazy for route-based splitting, dynamic import(), Webpack/Vite chunk strategies, prefetching route chunks, bundle analysis, Suspense fallbacks, retry logic, and module federation.

05 — Navigation Patterns

Programmatic navigation, protected routes, role-based access, breadcrumb generation, tab navigation state, modal routes, search param state management, deep linking, and URL-as-state encoding.

06 — Next.js App Router

App Router architecture, file-system routing, Server Components in routes, layout nesting, loading/error conventions, parallel and intercepting routes, route groups, middleware, and Server Actions.

07 — Interview Q&A

50+ L6-level interview questions with detailed answers covering routing internals, data loading, code splitting, navigation patterns, and Next.js routing architecture.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6
History API Internals Understanding pushState/popstate is the foundation — Staff engineers must reason about browser navigation at the API level, not just the framework level
Route Matching Algorithms Knowing how URLs map to components (ranked matching, specificity) enables debugging complex nested route configurations
Data Loading Architecture Waterfall elimination, parallel fetching, and streaming are critical for performance at scale — a key L6 design discussion
Code Splitting Strategy Route-based splitting is the highest-impact optimization — Staff engineers must design chunk strategies that balance load time and caching
URL as State Encoding application state in URLs enables deep linking, sharing, and back/forward navigation — an architectural decision with broad impact
Next.js App Router The convergence of routing, rendering, and data fetching in the App Router represents the future of React — essential knowledge for L6

Learning Objectives

  • Implement client-side routing from scratch using the History API
  • Explain React Router v6's ranked route matching algorithm and data router architecture
  • Design data loading strategies that eliminate waterfalls and handle race conditions
  • Architect route-based code splitting with prefetching and retry logic
  • Implement protected routes, breadcrumbs, and modal routes at scale
  • Compare Next.js Pages Router vs App Router and explain the architectural tradeoffs
  • Answer 50+ Staff-level interview questions on routing with confidence

Prerequisites

  • Strong React fundamentals (components, hooks, context, Suspense)
  • Familiarity with browser APIs (DOM, events, fetch)
  • Experience with React Router or similar routing library
  • Basic understanding of bundlers (webpack, Vite) and code splitting
