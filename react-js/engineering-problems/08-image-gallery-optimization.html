Building a High-Performance Image Gallery

Lazy loading, blur-up, responsive images, CDN transforms, virtualization, zoom/pan, preload strategy, and memory discipline.

Systems view

An image gallery stresses decode throughput, GPU memory, layout thrash, and network scheduling. Staff-level work separates when bytes arrive, when pixels decode, and when React commits—then proves it with traces on low-end Android.

Lazy loading: IntersectionObserver and native lazy

loading="lazy" is correct for simple below-the-fold images. For grids with complex placeholders or prefetch heuristics, IntersectionObserver offers root margins (anticipatory loading) and fine-grained callbacks.

// useLazyImage.ts
import { useEffect, useRef, useState } from 'react';

export function useInView<T extends Element>(options?: IntersectionObserverInit) {
  const ref = useRef<T | null>(null);
  const [inView, setInView] = useState(false);

  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    const obs = new IntersectionObserver(([e]) => {
      if (e.isIntersecting) {
        setInView(true);
        obs.disconnect();
      }
    }, options);
    obs.observe(el);
    return () => obs.disconnect();
  }, [options?.root, options?.rootMargin, options?.threshold]);

  return { ref, inView };
}

Progressive loading: blur-up and LQIP

Low Quality Image Placeholders (LQIP) are tiny Base64 blurs or separate 10–20px previews inlined in markup. On load, cross-fade to the full asset.

Blur-up component

// BlurUpImage.tsx
import { useState, type CSSProperties } from 'react';

type Props = {
  src: string;
  lqip: string; // data URL or CDN tiny variant
  alt: string;
  sizes: string;
  srcSet: string;
  style?: CSSProperties;
};

export function BlurUpImage({ src, lqip, alt, sizes, srcSet, style }: Props) {
  const [loaded, setLoaded] = useState(false);

  return (
    <span className="blur-up" style={{ position: 'relative', display: 'block', overflow: 'hidden', ...style }}>
      <img
        src={lqip}
        alt=""
        aria-hidden
        decoding="async"
        style={{
          filter: loaded ? 'blur(0px)' : 'blur(18px)',
          transform: loaded ? 'scale(1)' : 'scale(1.06)',
          transition: 'filter 320ms ease, transform 320ms ease',
          width: '100%',
          height: '100%',
          objectFit: 'cover',
        }}
      />
      <img
        src={src}
        srcSet={srcSet}
        sizes={sizes}
        alt={alt}
        loading="lazy"
        decoding="async"
        onLoad={() => setLoaded(true)}
        style={{
          position: 'absolute',
          inset: 0,
          width: '100%',
          height: '100%',
          objectFit: 'cover',
          opacity: loaded ? 1 : 0,
          transition: 'opacity 320ms ease',
        }}
      />
    </span>
  );
}
Double decode cost

Blur-up uses two images; on memory-constrained devices, prefer a single progressive JPEG/WebP or AVIF where the browser can render partial scans without a second element.

Responsive images: srcset, sizes, picture

srcset with w descriptors requires accurate sizes—otherwise the browser picks wrong widths and you ship megabytes. Art direction uses <picture> with media queries.

Responsive image component

// ResponsiveImg.tsx
type BreakSrc = { media: string; srcSet: string };

export function ResponsiveImg({
  alt,
  defaultSrcSet,
  sources,
  sizes,
}: {
  alt: string;
  defaultSrcSet: string;
  sources: BreakSrc[];
  sizes: string;
}) {
  return (
    <picture>
      {sources.map((s) => (
        <source key={s.media} media={s.media} srcSet={s.srcSet} sizes={sizes} />
      ))}
      <img srcSet={defaultSrcSet} sizes={sizes} alt={alt} loading="lazy" decoding="async" />
    </picture>
  );
}

Image CDN integration (Cloudinary / Imgix)

Centralize transforms in a builder so components never concatenate strings ad hoc—avoids cache fragmentation and XSS in URL params.

// cdnUrl.ts
const BASE = 'https://res.cloudinary.com/demo/image/upload';

export function galleryUrl(publicId: string, w: number, q = 'auto:good', fmt: 'auto' | 'webp' | 'avif' = 'auto') {
  // f_auto,q_auto,w_800 — keep ordered transforms stable for CDN caching
  return `${BASE}/f_${fmt},q_${q},w_${w},c_limit/${publicId}`;
}

Virtualized grid

Render only visible cells plus overscan. Pair with fixed row heights (or measured cache) for O(1) scroll-to-index.

Virtualized gallery grid

// VirtualMasonry.tsx — simplified column virtualization
import { useMemo, useRef, useState, useEffect, type UIEvent } from 'react';

type Item = { id: string; aspect: number; src: string };

export function VirtualMasonry({
  items,
  columnWidth,
  gap,
  height,
  renderCell,
}: {
  items: Item[];
  columnWidth: number;
  gap: number;
  height: number;
  renderCell: (item: Item, w: number) => JSX.Element;
}) {
  const scrollRef = useRef<HTMLDivElement>(null);
  const [scrollTop, setScrollTop] = useState(0);

  const layout = useMemo(() => {
    const cols: { h: number; items: { item: Item; top: number; left: number; col: number }[] }[] = [];
    const n = Math.max(1, Math.floor((scrollRef.current?.clientWidth ?? 900) / (columnWidth + gap)));
    for (let c = 0; c < n; c++) cols.push({ h: 0, items: [] });
    items.forEach((item, idx) => {
      const col = idx % n;
      const top = cols[col].h;
      const left = col * (columnWidth + gap);
      const h = columnWidth / item.aspect;
      cols[col].items.push({ item, top, left, col });
      cols[col].h += h + gap;
    });
    const totalHeight = Math.max(...cols.map((c) => c.h), 0);
    return { cols, totalHeight, n };
  }, [items, columnWidth, gap]);

  useEffect(() => {
    const el = scrollRef.current;
    if (!el) return;
    const onScroll = (e: Event) => {
      setScrollTop((e.target as HTMLDivElement).scrollTop);
    };
    el.addEventListener('scroll', onScroll);
    return () => el.removeEventListener('scroll', onScroll);
  }, []);

  const overscan = 600;
  const viewTop = scrollTop - overscan;
  const viewBottom = scrollTop + height + overscan;

  return (
    <div ref={scrollRef} style={{ height, overflowY: 'auto' }}>
      <div style={{ position: 'relative', height: layout.totalHeight }}>
        {layout.cols.flatMap((c) =>
          c.items
            .filter(({ top, item }) => {
              const cellH = columnWidth / item.aspect;
              const bottom = top + cellH;
              return bottom >= viewTop && top <= viewBottom;
            })
            .map(({ item, top, left }) => (
              <div
                key={item.id}
                style={{
                  position: 'absolute',
                  top,
                  left,
                  width: columnWidth,
                }}
              >
                {renderCell(item, columnWidth)}
              </div>
            )),
        )}
      </div>
    </div>
  );
}
Masonry caveat

True masonry with variable measured heights needs position caching and resize observers. For infinite feeds, prefer uniform tiles or CSS grid with dense packing plus virtualization libraries (TanStack Virtual) once requirements exceed hand-rolled math.

Zoom and pan

Use CSS transforms (translate + scale) on a GPU layer; avoid animating top/left. For pinch zoom, combine pointer events with a reference scale and translation clamped to image bounds.

Zoom/pan handler

// useZoomPan.ts
import { useCallback, useRef, useState } from 'react';

type Point = { x: number; y: number };

export function useZoomPan() {
  const [transform, setTransform] = useState({ x: 0, y: 0, k: 1 });
  const start = useRef<{ p: Point; t: typeof transform } | null>(null);

  const onPointerDown = useCallback((e: React.PointerEvent) => {
    (e.target as HTMLElement).setPointerCapture(e.pointerId);
    start.current = { p: { x: e.clientX, y: e.clientY }, t: transform };
  }, [transform]);

  const onPointerMove = useCallback((e: React.PointerEvent) => {
    if (!start.current) return;
    const dx = e.clientX - start.current.p.x;
    const dy = e.clientY - start.current.p.y;
    setTransform({ ...start.current.t, x: start.current.t.x + dx, y: start.current.t.y + dy });
  }, []);

  const onPointerUp = useCallback(() => {
    start.current = null;
  }, []);

  const zoom = useCallback((delta: number, origin: Point) => {
    setTransform((t) => {
      const k = Math.min(4, Math.max(1, t.k + delta));
      // anchor zoom to cursor (simplified)
      const factor = k / t.k;
      return {
        k,
        x: origin.x - (origin.x - t.x) * factor,
        y: origin.y - (origin.y - t.y) * factor,
      };
    });
  }, []);

  return { transform, onPointerDown, onPointerMove, onPointerUp, zoom };
}

Preloading adjacent images

When the lightbox opens on index i, inject <link rel="preload" as="image"> for i±1 with the exact srcset the viewer will use—duplicate URLs collapse in the network stack.

// preload.ts
export function preloadImage(href: string) {
  const l = document.createElement('link');
  l.rel = 'preload';
  l.as = 'image';
  l.href = href;
  document.head.appendChild(l);
}

Memory management

  • Revoke object URLs — URL.revokeObjectURL after blob previews are discarded.
  • Decode budget — for huge lists, set loading="lazy" and disconnect observers; optionally call img.decode() sequentially for hero images only.
  • Unmount heavy canvases — if using WebGL zoom, dispose textures on close.

Deep-dive Q&A

Why not only use background-image for galleries? Senior

<img> participates in accessibility, srcset, LCP, and native lazy loading. Background images complicate responsive selection and often hurt SEO and screen reader semantics unless carefully duplicated.

How does HTTP/3 help image-heavy pages? L6

Independent streams reduce head-of-line blocking versus HTTP/2 under loss—galleries with many parallel thumbnails benefit. You still must right-size images; transport cannot fix oversized pixels.

When is native lazy loading insufficient? Arch

When you need custom rootMargin, coordinated prefetch with scroll velocity, or integration with a media pipeline that swaps sources before paint. IntersectionObserver remains the flexible primitive.
