State Sync Across Tabs

BroadcastChannel pub/sub, SharedWorker coordination, storage events, leader election for single-flight API calls, CRDT-style conflict handling, auth and cart synchronization, and notification deduplication across browsing contexts.

Architecture lens

Cross-tab sync is a distributed systems problem with unreliable message ordering and partial failure. Choose a transport (BroadcastChannel vs SharedWorker vs storage) based on latency, payload size, and Safari support matrix, then layer version vectors or logical clocks for merge semantics.

BroadcastChannel API

BroadcastChannel delivers same-origin messages to every browsing context—tabs, windows, workers (if constructed with channel name). It is faster than localStorage ping-pong and avoids writing disk. Caveat: no message persistence; receivers not yet subscribed miss messages.

// broadcastSync.ts
const CHANNEL = 'app:state:v1';

export type TabMessage =
  | { type: 'STATE_PATCH'; payload: unknown; from: string; ts: number }
  | { type: 'PING' };

export function createTabChannel(tabId: string) {
  const bc = new BroadcastChannel(CHANNEL);

  return {
    post(patch: unknown) {
      const msg: TabMessage = {
        type: 'STATE_PATCH',
        payload: patch,
        from: tabId,
        ts: performance.now(),
      };
      bc.postMessage(msg);
    },
    subscribe(handler: (m: TabMessage) => void) {
      const fn = (e: MessageEvent<TabMessage>) => {
        if (e.data.from === tabId) return; // ignore self (optional)
        handler(e.data);
      };
      bc.addEventListener('message', fn);
      return () => bc.removeEventListener('message', fn);
    },
    close() {
      bc.close();
    },
  };
}
// useBroadcastSync.ts
import { useEffect, useId, useRef } from 'react';

export function useBroadcastSync<T>(
  state: T,
  applyRemote: (patch: Partial<T>) => void,
  makePatch: (prev: T, next: T) => Partial<T> | null
) {
  const id = useId();
  const prev = useRef(state);

  useEffect(() => {
    const ch = createTabChannel(id);
    const off = ch.subscribe((m) => {
      if (m.type === 'STATE_PATCH') applyRemote(m.payload as Partial<T>);
    });
    return () => {
      off();
      ch.close();
    };
  }, [applyRemote]);

  useEffect(() => {
    const patch = makePatch(prev.current, state);
    prev.current = state;
    if (!patch) return;
    const ch = createTabChannel(id);
    ch.post(patch);
    ch.close();
  }, [state, id]);
}

SharedWorker: centralized state

A SharedWorker holds canonical state; tabs connect via MessagePort. Benefits: single connection to WebSocket, deduplicated subscriptions, authoritative merge. Costs: worker lifecycle complexity and harder debugging.

// sharedWorkerState.ts (worker file)
const ports: MessagePort[] = [];
let state: Record<string, unknown> = {};

self.onconnect = (e: MessageEvent) => {
  const port = (e as any).ports[0] as MessagePort;
  ports.push(port);
  port.start();
  port.postMessage({ type: 'HYDRATE', state });

  port.onmessage = (ev) => {
    if (ev.data.type === 'PATCH') {
      state = { ...state, ...ev.data.payload };
      broadcast({ type: 'STATE', state }, port);
    }
  };
};

function broadcast(msg: unknown, except?: MessagePort) {
  for (const p of ports) {
    if (p !== except) p.postMessage(msg);
  }
}
Fallback

When SharedWorker is unavailable, degrade to BroadcastChannel with last-writer-wins and visible conflict UI for high-value entities (cart quantities).

localStorage and the storage event

Writing localStorage.setItem fires storage events in other tabs on the same origin. Use a single key with JSON payloads and throttle writes (e.g., requestAnimationFrame coalescing) to avoid main-thread jank.

export function useStorageSync<T>(key: string, value: T, onRemote: (v: T) => void) {
  useEffect(() => {
    const onStorage = (e: StorageEvent) => {
      if (e.key !== key || e.newValue == null) return;
      try {
        onRemote(JSON.parse(e.newValue) as T);
      } catch {
        /* ignore */
      }
    };
    window.addEventListener('storage', onStorage);
    return () => window.removeEventListener('storage', onStorage);
  }, [key, onRemote]);

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
}

Leader election: single API tab

Only the leader refreshes tokens, polls the inbox, or runs expensive sync. Use random backoff + BroadcastChannel or navigator.locks.request('leader', { ifAvailable: true }) when available.

// leaderElection.ts
const LEADER_KEY = 'app:leader';
const HEARTBEAT_MS = 2000;

export async function electLeader(tabId: string, onLeader: () => void, onFollower: () => void) {
  if (navigator.locks) {
    navigator.locks.request(LEADER_KEY, async () => {
      onLeader();
      await new Promise<void>(() => {}); // hold lock until tab dies
    });
    return;
  }

  // Storage-based lease fallback
  const tryAcquire = () => {
    const now = Date.now();
    const raw = localStorage.getItem(LEADER_KEY);
    const lease = raw ? JSON.parse(raw) as { id: string; until: number } : null;
    if (!lease || lease.until < now) {
      localStorage.setItem(LEADER_KEY, JSON.stringify({ id: tabId, until: now + HEARTBEAT_MS * 3 }));
      return true;
    }
    return lease.id === tabId;
  };

  const tick = () => {
    if (tryAcquire()) onLeader();
    else onFollower();
  };

  const iv = setInterval(tick, HEARTBEAT_MS);
  tick();
  return () => clearInterval(iv);
}
Tab1 acquires lock ──▶ LEADER (API refresh) Tab2 observes lease held ──▶ FOLLOWER (read-only cache) Tab1 crash ──▶ lease expires ──▶ Tab2 promotes
Clock skew

Storage leases depend on local clocks. Prefer navigator.locks when supported; it is tied to browser lifecycle, not wall clock.

export type Versioned<T> = { v: number; data: T };

export function mergeLWW<T>(a: Versioned<T>, b: Versioned<T>): Versioned<T> {
  if (a.v === b.v) return { v: a.v + 1, data: b.data }; // tie-break: prefer incoming
  return a.v > b.v ? a : b;
}

Authentication state sync

On logout, broadcast { type: 'LOGOUT_ALL' } and clear IndexedDB caches in every tab. Token refresh should be coordinated: leader refreshes, others listen for TOKEN_UPDATE patches. Avoid storing refresh tokens in localStorage if XSS is a concern—use HttpOnly cookies and still broadcast session_revoked via channel.

export function initAuthSync(
  onLogout: () => void,
  onToken: (t: string) => void
) {
  const bc = new BroadcastChannel('auth:v1');
  bc.onmessage = (e) => {
    if (e.data.type === 'LOGOUT_ALL') onLogout();
    if (e.data.type === 'TOKEN_UPDATE') onToken(e.data.token);
  };
  return {
    broadcastLogout() {
      bc.postMessage({ type: 'LOGOUT_ALL' });
    },
    broadcastToken(token: string) {
      bc.postMessage({ type: 'TOKEN_UPDATE', token });
    },
  };
}

Cart state sync

Maintain cart as Versioned<Cart> in worker or sessionStorage + BroadcastChannel. On add-to-cart racing across tabs, increment op counter client-side; server reconciles with idempotent line-item IDs (sku + priceSnapshotId).

Notification deduplication

When multiple tabs subscribe to push or SSE, only the leader shows OS notifications; others increment an in-app badge. Use a deterministic notificationId hash so duplicate events collapse.

export function dedupeNotification(id: string, ttlMs = 60_000) {
  const key = `notif:${id}`;
  const now = Date.now();
  const prev = sessionStorage.getItem(key);
  if (prev && now - Number(prev) < ttlMs) return false;
  sessionStorage.setItem(key, String(now));
  return true;
}
// showNotification.ts
export function showTabAwareNotification(title: string, body: string, id: string) {
  const bc = new BroadcastChannel('notify:v1');
  bc.postMessage({ type: 'CLAIM', id });

  let claimed = false;
  const onMsg = (e: MessageEvent) => {
    if (e.data.type === 'CLAIM_ACK' && e.data.id === id) {
      claimed = true;
    }
  };
  bc.addEventListener('message', onMsg);

  // naive leader: first tab to post CLAIM wins within 50ms window
  setTimeout(() => {
    bc.removeEventListener('message', onMsg);
    if (!claimed && dedupeNotification(id)) {
      new Notification(title, { body });
      bc.postMessage({ type: 'CLAIM_ACK', id });
    }
  }, 50);
}

Service worker fan-out

When tabs cannot use SharedWorker (policy or platform), a service worker can hold state and relay messages via clients.matchAll(). This pattern also enables background sync for queued mutations—at the cost of SW lifecycle complexity and extra latency on cold start.

// sw-sync.ts (service worker)
self.addEventListener('message', (event) => {
  if (event.data?.type === 'RELAY') {
    event.waitUntil(
      self.clients.matchAll({ type: 'window', includeUncontrolled: true }).then((clients) => {
        for (const c of clients) {
          if (c.id !== event.source?.id) {
            c.postMessage({ type: 'RELAY', payload: event.data.payload });
          }
        }
      })
    );
  }
});
Registration timing

Until the SW activates, fall back to BroadcastChannel. Race dual-publish during upgrade windows or you will miss messages on slow networks.

bfcache and pageshow reconciliation

When users navigate back, frozen pages may display stale cart or auth. Subscribe to window.addEventListener('pageshow', (e) => { if (e.persisted) resync(); }) and re-read canonical state from the worker or sessionStorage revision counter.

export function useBfCacheResync(resync: () => void) {
  useEffect(() => {
    const onShow = (e: PageTransitionEvent) => {
      if (e.persisted) resync();
    };
    window.addEventListener('pageshow', onShow);
    return () => window.removeEventListener('pageshow', onShow);
  }, [resync]);
}

Safari / ITP

Intelligent Tracking Prevention can partition storage for embedded contexts. Cross-tab sync for first-party top-level documents is usually fine; avoid relying on localStorage in iframes for authoritative state.

Versioned message schema

Prefix every message with v: number. On unknown version, drop or route to a compatibility shim. This avoids cascading failures when you add required fields during deploys.

export type TabEnvelope<T> = { v: 1; type: string; payload: T };

export function parseEnvelope(raw: unknown): TabEnvelope<unknown> | null {
  if (typeof raw !== 'object' || raw === null) return null;
  const v = (raw as { v?: unknown }).v;
  if (v !== 1) return null;
  return raw as TabEnvelope<unknown>;
}

Interview Q&A

Why not always use localStorage for cross-tab state? L6

Synchronous main-thread writes, quota limits, and lack of binary efficiency make it poor for high-frequency updates. It is excellent as a persistence anchor combined with BroadcastChannel for live propagation.

How does this interact with bfcache? Senior

Pages in back-forward cache freeze. On pageshow with persisted, revalidate snapshots from the worker or storage and reconcile. Otherwise users see stale UI after navigating back.
