Dynamic Form Builder

Schema-driven rendering, conditional visibility graphs, layered validation (sync/async, field vs form), wizard state machines, DnD Kit ordering, analytics instrumentation, server error mapping, and accessibility contracts at staff level.

Architecture lens

Treat the form as a directed acyclic graph of fields with validation as a monoidal layer (field errors compose to form errors). Persist wizard state as versioned snapshots so migrations do not corrupt in-flight user sessions.

Schema-driven forms: JSON Schema to React

At L6 depth, the renderer is not a giant switch on type; you register widget resolvers keyed by format, x-widget, or capability flags. Resolution order: explicit widget override → format mapping → type default → fallback text input.

// types.ts — minimal schema surface for a builder
export type FieldSchema = {
  id: string;
  type: 'string' | 'number' | 'boolean' | 'enum' | 'object' | 'array';
  title?: string;
  description?: string;
  enum?: string[];
  'x-widget'?: 'textarea' | 'select' | 'date' | 'file';
  'x-dependsOn'?: DependencyRule[];
  properties?: Record<string, FieldSchema>;
};

export type DependencyRule = {
  when: string; // JSON Pointer or dot path to watched field
  op: 'eq' | 'neq' | 'in' | 'truthy';
  value?: unknown;
};

// registry.ts — pluggable widget map (OCP-friendly)
type WidgetProps = {
  schema: FieldSchema;
  value: unknown;
  onChange: (v: unknown) => void;
  error?: string;
  disabled?: boolean;
};

const widgets: Record<string, React.FC<WidgetProps>> = {
  string: TextInput,
  textarea: Textarea,
  enum: Select,
  boolean: Checkbox,
};

export function resolveWidget(schema: FieldSchema) {
  if (schema['x-widget'] && widgets[schema['x-widget']]) {
    return widgets[schema['x-widget']];
  }
  if (schema.enum) return widgets.enum;
  return widgets[schema.type] ?? widgets.string;
}
// SchemaFieldRenderer.tsx — isolate subscriptions to avoid full-form re-renders
import { memo, useCallback } from 'react';
import { useFormContext, Controller } from 'react-hook-form';

export const SchemaFieldRenderer = memo(function SchemaFieldRenderer({
  schema,
  namePrefix = '',
}: {
  schema: FieldSchema;
  namePrefix?: string;
}) {
  const path = `${namePrefix}${schema.id}`;
  const { control, formState } = useFormContext();
  const Widget = resolveWidget(schema);
  const visible = useVisibility(schema, namePrefix);

  const err = formState.errors[path]?.message as string | undefined;

  if (!visible) return null;

  if (schema.type === 'object' && schema.properties) {
    return (
      <fieldset aria-labelledby={`${path}-legend`}>
        <legend id={`${path}-legend`}>{schema.title}</legend>
        {Object.values(schema.properties).map((child) => (
          <SchemaFieldRenderer key={child.id} schema={child} namePrefix={`${path}.`} />
        ))}
      </fieldset>
    );
  }

  return (
    <Controller
      name={path}
      control={control}
      render={({ field }) => (
        <Widget
          schema={schema}
          value={field.value}
          onChange={field.onChange}
          error={err}
          disabled={formState.isSubmitting}
        />
      )}
    />
  );
});
Performance note

Memoize per schema.id and pass stable onChange wrappers from context. For 200+ fields, consider splitting into micro-forms with a coordinator reducer or useFormState({ control, name: subset }) subscriptions.

Conditional fields: visibility as a pure function

Evaluate dependencies from a frozen snapshot of values (or RHF watch() with debounce for expensive rules). Cycle detection: build adjacency list from x-dependsOn and run DFS; staff systems fail CI if a cycle exists.

// visibilityEngine.ts
import get from 'lodash/get';

export function evaluateVisibility(
  schema: FieldSchema,
  values: Record<string, unknown>,
  namePrefix: string
): boolean {
  const rules = schema['x-dependsOn'];
  if (!rules?.length) return true;

  return rules.every((rule) => {
    const fullPath = `${namePrefix}${rule.when}`;
    const actual = get(values, fullPath.replace(/\./g, '.')); // or json-pointer lib
    switch (rule.op) {
      case 'eq':
        return actual === rule.value;
      case 'neq':
        return actual !== rule.value;
      case 'in':
        return Array.isArray(rule.value) && rule.value.includes(actual as never);
      case 'truthy':
        return Boolean(actual);
      default:
        return true;
    }
  });
}

// useVisibility.ts — subscribe only to dependency paths
import { useWatch, useFormContext } from 'react-hook-form';

export function useVisibility(schema: FieldSchema, namePrefix: string) {
  const { control } = useFormContext();
  const deps = schema['x-dependsOn']?.map((r) => `${namePrefix}${r.when}`) ?? [];
  const slice = useWatch({ control, name: deps as any }); // RHF supports array of names
  const all = useWatch({ control }); // fallback if deps empty
  const values = deps.length ? Object.fromEntries(deps.map((d, i) => [d, slice?.[i]])) : all;
  return evaluateVisibility(schema, values as Record<string, unknown>, namePrefix);
}

// validationEngine.ts — composable validators with async cancellation
export type FieldValidator = (value: unknown, ctx: FormCtx) =>
  | string | undefined | Promise<string | undefined>;

export type FormCtx = {
  getValues: () => Record<string, unknown>;
  signal: AbortSignal;
};

export async function runFieldAsync(
  validators: FieldValidator[],
  value: unknown,
  ctx: FormCtx
): Promise<string | undefined> {
  for (const v of validators) {
    const msg = await v(value, ctx);
    if (msg) return msg;
  }
  return undefined;
}

Async races

Always key async validation by fieldId + valueVersion. If the user changes input before the prior request resolves, discard the stale result. Expose isValidating per field for loading indicators without blocking submit globally.

Multi-step forms: state machine and persistence

Model steps as explicit states with events NEXT, BACK, SAVE_DRAFT, SUBMIT. Persist to IndexedDB or encrypted sessionStorage with a schema version header; on load, run a migrator. Hydrate RHF with reset(migrated) once.

// wizardMachine.ts — XState-style reducer (no lib required)
type WizardState = {
  step: number;
  data: Record<string, unknown>;
  lastSavedAt?: number;
};

type WizardEvt =
  | { type: 'NEXT'; payload?: Record<string, unknown> }
  | { type: 'BACK' }
  | { type: 'HYDRATE'; snapshot: WizardState };

export function wizardReducer(s: WizardState, e: WizardEvt): WizardState {
  switch (e.type) {
    case 'NEXT':
      return {
        ...s,
        step: s.step + 1,
        data: { ...s.data, ...e.payload },
        lastSavedAt: Date.now(),
      };
    case 'BACK':
      return { ...s, step: Math.max(0, s.step - 1) };
    case 'HYDRATE':
      return e.snapshot;
    default:
      return s;
  }
}

Drag-and-drop ordering with DnD Kit

Use @dnd-kit/core with collision detection tuned for nested lists (closestCenter vs pointerWithin). For accessibility, implement KeyboardSensor and announce moves via aria-live. Store order as an array of stable IDs, not array indices in the DOM.

import {
  DndContext,
  PointerSensor,
  KeyboardSensor,
  useSensor,
  useSensors,
  closestCenter,
} from '@dnd-kit/core';
import {
  SortableContext,
  verticalListSortingStrategy,
  arrayMove,
} from '@dnd-kit/sortable';

function FieldOrderEditor({
  ids,
  onReorder,
}: {
  ids: string[];
  onReorder: (next: string[]) => void;
}) {
  const sensors = useSensors(
    useSensor(PointerSensor, { activationConstraint: { distance: 6 } }),
    useSensor(KeyboardSensor)
  );

  return (
    <DndContext
      sensors={sensors}
      collisionDetection={closestCenter}
      onDragEnd={({ active, over }) => {
        if (!over || active.id === over.id) return;
        const oldIndex = ids.indexOf(String(active.id));
        const newIndex = ids.indexOf(String(over.id));
        onReorder(arrayMove(ids, oldIndex, newIndex));
      }}
    >
      <SortableContext items={ids} strategy={verticalListSortingStrategy}>
        {ids.map((id) => (
          <SortableFieldRow key={id} id={id} />
        ))}
      </SortableContext>
    </DndContext>
  );
}

// formAnalytics.ts — batch + sendBeacon on page hide
const queue: AnalyticsEvent[] = [];

export function trackFieldBlur(fieldId: string, dwellMs: number) {
  queue.push({ type: 'field_blur', fieldId, dwellMs, ts: Date.now() });
  if (queue.length >= 20) flush();
}

export function flush() {
  if (!queue.length) return;
  const body = JSON.stringify(queue.splice(0, queue.length));
  navigator.sendBeacon('/api/form-telemetry', body);
}

Server-side validation: mapping errors to fields

Standardize on RFC 7807 Problem+JSON or a custom envelope { errors: { path: message[] } }. Normalize paths to RHF names with a single serverPathToClient function shared by web and API teams.

type ServerErrorBody = {
  errors: Record<string, string[]>;
};

export function applyServerErrors(
  setError: UseFormSetError<FieldValues>,
  body: ServerErrorBody
) {
  for (const [path, messages] of Object.entries(body.errors)) {
    const name = path.replace(/^\//, '').replace(/\//g, '.'); // JSON Pointer → RHF
    setError(name as FieldPath<FieldValues>, {
      type: 'server',
      message: messages[0] ?? 'Invalid',
    });
  }
}

Accessibility: labels, live regions, focus

  • Associate every input with <label htmlFor> or aria-labelledby.
  • On submit failure, move focus to the first error with role="alert" container or aria-live="assertive" for screen readers.
  • When conditional fields mount, use useLayoutEffect to focus the first newly visible control if the user triggered the dependency change via keyboard.
function FormErrorSummary({ errors }: { errors: FieldErrors }) {
  const first = Object.keys(errors)[0];
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (first) ref.current?.focus();
  }, [first]);

  if (!first) return null;

  return (
    <div
      ref={ref}
      tabIndex={-1}
      role="alert"
      aria-live="assertive"
      className="error-summary"
    >
      <h2>There are errors in the form</h2>
      <ul>
        {Object.entries(errors).map(([name, e]) => (
          <li key={name}>
            <a href={`#${name}`}>{e?.message as string}</a>
          </li>
        ))}
      </ul>
    </div>
  );
}
Staff takeaway

Invest in a schema IR shared with the backend (OpenAPI/JSON Schema codegen) so validation, visibility, and analytics keys stay aligned across services. The alternative is endless stringly-typed path drift.

Interview Q&A

How do you unregister hidden conditional fields? L6

Use RHF unregister when visibility flips false, or keep values but strip at submit with a transform layer. Unregister avoids sending stale data; keeping values preserves UX if the user toggles back. Document the product decision explicitly.

Where does Zod outperform AJV for React forms? Senior

Zod integrates ergonomically with TypeScript inference and zodResolver. AJJ shines when the canonical schema is JSON Schema and must be identical on server and client—then use @hookform/resolvers/ajv or convert schemas carefully.
