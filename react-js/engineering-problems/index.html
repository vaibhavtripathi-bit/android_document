The subset of indices rendered to DOM at a given scroll offset.
Overscan
Extra rows above/below the viewport to smooth fast scroll.
Cursor (pagination)
Opaque token tying a fetch to a stable position in the server ordering.
Compaction
Merging tombstones / snapshots in CRDT or op logs to bound metadata size.

Interview Q&A

How do you decide between CRDT and OT for a collaborative editor? L6

OT historically dominated centralized servers with total ordering: transforms are defined against a single stream of operations; correctness hinges on transformation functions satisfying TP1/TP2 (intention preservation). It couples you to an authoritative sequencer.

CRDTs target replication without a central total order: local operations apply immediately; merges are algebraic. Text often uses sequences (RGA, YATA) or Fugue-like structures; rich data uses registers, maps, and counters with defined semirings.

Choose OT when you already have a strong central service, need minimal storage of tombstones, and can accept operational transformation complexity. Choose CRDT when you want peer-to-peer, offline-first, or multi-region with lazy sync—accepting metadata overhead and sometimes more subtle UX (e.g., concurrent insert tie-breaking).

What breaks when virtualization meets browser find-in-page and accessibility? Architect

Off-screen rows are not in the DOM (or are absolutely positioned placeholders), so native Cmd+F may skip content unless you integrate with the browser’s content-visibility patterns or virtualize within an aria-friendly list pattern. Screen readers need stable aria-posinset/aria-setsize or a roving tabindex strategy. Mitigations: optional “full list” mode for assistive tech, custom search that queries the data model, or hybrid rendering for small result sets.

How do you test a design system migration without blocking fifty teams? L6

Use layered gates: (1) automated codemods with AST fixtures in CI, (2) visual regression on a representative story matrix, (3) bundle-size budgets per package with historical baselines, (4) canary consumers in low-risk apps before default bumps. Expose an adoption dashboard (version histogram + top importers) so platform can chase long tails rather than big-bang upgrades.

What is the difference between vector clocks and version vectors for CRDT sync? L6

Version vectors (one counter per replica) summarize “what this replica has seen” from each other replica; they are ideal for tracking replica-level causality in state-based CRDT synchronization.

Vector clocks generalize Lamport time for event ordering; in distributed systems literature the terms are often conflated, but in CRDT engineering you care whether your metadata can detect concurrent updates (incomparable vectors) and whether merges are commutative/associative.

In interviews, tie this to your wire protocol: do you send operations (op-based) or states (state-based), and how do you bound metadata growth (truncation, periodic compaction, dotted version vectors).

How would you implement scroll restoration for a virtualized route in React Router? Senior

Disable automatic browser restoration with history.scrollRestoration = 'manual'. Persist a tuple { scrollTop, firstVisibleId, offsetWithinRow } keyed by location key. On return navigation, remount the list, seek the virtualizer to firstVisibleId (binary search in item metadata), then apply offsetWithinRow for sub-pixel accuracy. Use requestAnimationFrame double buffering to avoid measuring before paint.
