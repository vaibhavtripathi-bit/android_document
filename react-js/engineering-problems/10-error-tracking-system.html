Error Tracking System

Source maps, deterministic fingerprints, breadcrumb trails, session replay foundations, performance correlation, alert policy design, impact scoring, and incident tool integrations—built the way observability vendors do it internally.

Architecture lens

Ingestion must be idempotent and partitionable by project key + release. The hot path is normalize → fingerprint → dedupe → enrich → store; everything else (source map expansion, PII scrubbing) can be async workers.

Source maps: upload, association, and stack resolution

Production bundles reference //# sourceMappingURL=...—often hidden behind auth. SaaS trackers upload maps out-of-band during CI with the same release and dist artifact hashes. Resolution uses the sourcesContent embedded in the map when available to avoid fetching original files.

// symbolicateFrame.ts — simplified; real impl uses map consumer + source-map library
import { SourceMapConsumer } from 'source-map';

export async function symbolicate(
  rawFrame: { filename: string; lineno: number; colno: number },
  mapJson: string
): Promise<{ file: string; line: number; column: number; name?: string }> {
  await SourceMapConsumer.initialize?.(); // browser build
  const consumer = await new SourceMapConsumer(JSON.parse(mapJson));
  const pos = consumer.originalPositionFor({
    line: rawFrame.lineno,
    column: rawFrame.colno,
  });
  consumer.destroy();
  return {
    file: pos.source ?? rawFrame.filename,
    line: pos.line ?? rawFrame.lineno,
    column: pos.column ?? rawFrame.colno,
    name: pos.name ?? undefined,
  };
}
Debug IDs

Modern tooling injects a debug id into both bundle and map so the server can pair them even when filenames collide. Prefer that over fragile path matching across CDNs.

Error deduplication: fingerprinting

Fingerprint = hash of normalized exception identity: exception type, top in-app frames (after stripping library frames), and optionally transaction name. Two users hitting the same null deref should collapse to one issue with rising event count.

import { createHash } from 'crypto';

type Frame = { filename: string; function?: string; lineno?: number };

function normalizeFrame(f: Frame): string {
  const file = f.filename.replace(/\?.*$/, ''); // strip query
  const fn = (f.function ?? '').replace(/\s+/g, '');
  return `${file}:${fn}:${f.lineno ?? 0}`;
}

export function fingerprint(
  exceptionValue: string,
  type: string,
  frames: Frame[],
  appRoot: RegExp
): string {
  const inApp = frames.filter((f) => appRoot.test(f.filename));
  const top = inApp.slice(0, 5).map(normalizeFrame).join('|');
  const payload = `${type}|${exceptionValue}|${top}`;
  return createHash('sha256').update(payload).digest('hex').slice(0, 32);
}
// errorCollector.ts — browser SDK core
type CapturedError = {
  message: string;
  name: string;
  stack?: string;
  frames?: Frame[];
  breadcrumbs?: Breadcrumb[];
  release?: string;
};

export function captureException(err: unknown, ctx?: Partial<CapturedError>) {
  const normalized = normalizeError(err);
  const frames = parseStack(normalized.stack);
  const fp = fingerprint(
    normalized.message,
    normalized.name,
    frames,
    /https:\/\/(app\.example\.com\/static\/js)/
  );

  const body = {
    fingerprint: fp,
    exception: normalized,
    breadcrumbs: breadcrumbBuffer.snapshot(),
    ...ctx,
  };

  transport.enqueue(body);
}
Over-grouping

If you hash only message, dynamic strings (e.g., Failed for id 123) fragment issues. Strip numbers/UUIDs with caution—over-stripping merges unrelated bugs. Allow fingerprint overrides from catch blocks for known noisy sites.

Breadcrumb trails: ordering and PII

Implement a fixed-size ring buffer (e.g., 100 items) with monotonic sequence numbers. Categories: navigation, ui.click, fetch, console, custom. Scrub URLs and headers at capture time.

export type Breadcrumb = {
  ts: number;
  seq: number;
  category: string;
  message: string;
  data?: Record<string, unknown>;
  level: 'debug' | 'info' | 'warning' | 'error';
};

class BreadcrumbBuffer {
  private items: Breadcrumb[] = [];
  private seq = 0;
  constructor(private max = 100) {}

  add(b: Omit<Breadcrumb, 'ts' | 'seq'>) {
    const crumb: Breadcrumb = { ...b, ts: Date.now(), seq: ++this.seq };
    this.items.push(crumb);
    if (this.items.length > this.max) this.items.shift();
  }

  snapshot() {
    return [...this.items];
  }
}

export const breadcrumbBuffer = new BreadcrumbBuffer();

Session replay: DOM mutations and events

// replayRecorderSketch.ts — illustrative, not full rrweb
export function startReplayRecording({ maskSelectors }: { maskSelectors: string[] }) {
  const events: unknown[] = [];

  const mo = new MutationObserver((muts) => {
    for (const m of muts) {
      events.push(serializeMutation(m));
    }
    maybeFlush(events);
  });

  mo.observe(document.documentElement, {
    subtree: true,
    childList: true,
    attributes: true,
    characterData: true,
  });

  document.addEventListener(
    'input',
    (e) => {
      const t = e.target as HTMLElement;
      if (matchesAny(t, maskSelectors)) return;
      events.push({ type: 'input', target: cssPath(t), value: '***' });
    },
    true
  );

  return () => mo.disconnect();
}

Performance correlation

Attach trace_id / span_id from OpenTelemetry to error events when the error surfaces inside a traced operation. On the backend, join issues to p95 latency regression windows. React: wrap router transitions and data loaders with spans; on rejection, call captureException with active span context.

// correlateErrorWithWebVitals.ts
export function initPerfCorrelation() {
  new PerformanceObserver((list) => {
    for (const e of list.getEntries()) {
      if (e.entryType === 'largest-contentful-paint') {
        window.__LAST_LCP__ = e.startTime;
      }
    }
  }).observe({ type: 'largest-contentful-paint', buffered: true });

  window.addEventListener('error', () => {
    captureMessage('window.error', {
      extra: { lcp: window.__LAST_LCP__ },
    });
  });
}

Alert fatigue: grouping, rate limits, severity

  • Grouping: one issue = one fingerprint; notifications reference issue, not every event.
  • Rate limiting: suppress duplicate alerts within a time window; escalate only on new releases or user growth spikes.
  • Severity: map unhandled rejections in payment flow to P1; cosmetic UI to P4 via rules on tags and transaction names.

Error prioritization: impact score

Define impact = unique_users_affected × log(1 + event_count) × severity_weight × revenue_weight. Cap event_count contribution so one bot loop does not dominate. Recompute hourly in a warehouse or stream processor.

export function impactScore(input: {
  users: number;
  events: number;
  severity: 1 | 2 | 3 | 4 | 5;
  inCheckout: boolean;
}): number {
  const sevW = input.severity ** 2;
  const revW = input.inCheckout ? 3 : 1;
  const freq = Math.log1p(input.events);
  return input.users * freq * sevW * revW;
}

Incident management: PagerDuty and Slack

Use a single orchestration service that receives webhooks from the tracker, applies policies, then calls vendor APIs. Include dedupe keys (fingerprint + environment) so PagerDuty merges incidents.

// webhookToPagerDuty.ts
export async function routeIssueToPagerDuty(issue: IssuePayload) {

      dedup_key: `${issue.fingerprint}:${issue.environment}`,

        severity: issue.severity,
        source: issue.project,
        custom_details: {
          users: issue.affectedUsers,
          release: issue.release,
          url: issue.permalink,
        },
      },
    }),
  });
}

export async function postSlackDigest(channel: string, issues: IssuePayload[]) {
  const text = issues
    .map((i) => `*${i.title}* — ${i.affectedUsers} users — <${i.permalink}|open>`)

SDK transport: batching, backoff, and offline queues

Browser SDKs should never block the main thread on the network path. Use an in-memory queue flushed on visibilitychange, beforeunload via sendBeacon, and exponential backoff with jitter on 429/503. Cap queue size and drop lowest-severity events first under memory pressure.

// transport.ts
type Envelope = { event_id: string; body: unknown; attempts: number };

export class EventTransport {
  private q: Envelope[] = [];
  private timer: ReturnType<typeof setTimeout> | null = null;

  constructor(private endpoint: string, private maxBatch = 20) {
    document.addEventListener('visibilitychange', () => {
      if (document.visibilityState === 'hidden') this.flushBeacon();
    });
  }

  enqueue(body: unknown) {
    this.q.push({ event_id: crypto.randomUUID(), body, attempts: 0 });
    this.schedule();
  }

  private schedule() {
    if (this.timer) return;
    this.timer = setTimeout(() => this.flushFetch(), 200);
  }

  private async flushFetch() {
    this.timer = null;
    if (!this.q.length) return;
    const batch = this.q.splice(0, this.maxBatch);
    try {
      const res = await fetch(this.endpoint, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ events: batch }),
        keepalive: true,
      });
      if (!res.ok) throw new Error(String(res.status));
    } catch {
      for (const e of batch) {
        e.attempts += 1;
        if (e.attempts < 5) this.q.push(e);
      }
      setTimeout(() => this.schedule(), 2 ** batch[0].attempts * 200 + Math.random() * 100);
    }
  }

  private flushBeacon() {
    const batch = this.q.splice(0, this.maxBatch);
    if (!batch.length) return;
    navigator.sendBeacon(this.endpoint, JSON.stringify({ events: batch }));
  }
}

Sampling, quotas, and cardinality control

High-traffic SPAs must sample benign errors (e.g., third-party ad block noise) while always capturing $level:fatal and payment flows. Implement dynamic sampling keyed by fingerprint so rare issues are not starved.

Ingress rate R │ ├── fingerprint seen often? ──yes──▶ sample at 1% │ └── novel fingerprint? ────────────▶ capture at 100% │ └── override: user in allowlist ───▶ capture at 100%
export function shouldSample(
  fingerprint: string,
  counts: Map<string, number>,
  baseRate: number
): boolean {
  const c = counts.get(fingerprint) ?? 0;
  if (c < 10) return false; // always keep early signal
  const p = Math.min(0.99, baseRate * Math.log10(1 + c));
  return Math.random() < p;
}

CSP and third-party scripts

Inline stack frames from extensions and tag managers pollute fingerprints. Strip frames whose filename matches chrome-extension: or known tag hosts before hashing; optionally attach raw stack as a non-indexed attachment for support engineers.

Interview Q&A

How do you avoid leaking PII in breadcrumbs? L6

Scrub at the edge: URL query keys, email regex in messages, and configurable deny-lists. Never log full GraphQL bodies by default. For replay, mask inputs and block entire subtrees (chat widgets, medical forms).

Why separate issue from event? Senior

Events are immutable facts; issues are aggregated, mutable workflow objects (assigned, resolved, muted). Storage and alerting efficiencies depend on that split—billions of events, thousands of issues.
