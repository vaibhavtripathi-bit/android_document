Series 10: Security & Web

XSS prevention, CSRF protection, authentication patterns, authorization models, secure data handling, dependency security, and the browser security model — security architecture for React applications at Staff-level depth.

Security is not a feature — it's a property of the entire system. A Staff-level frontend engineer must understand attack vectors, defense-in-depth strategies, and the browser security model that underpins all web applications. This series covers the full security landscape for React applications: from React's built-in XSS protections and their limitations, through authentication and authorization architecture, to supply chain security and the browser's same-origin policy. Each document focuses on the "why" behind security decisions, not just the "how."

Documents in This Series

01 — XSS Prevention

Cross-Site Scripting types (reflected, stored, DOM-based), React's built-in protection (JSX escaping, $$typeof Symbol), dangerouslySetInnerHTML risks, CSP headers, Trusted Types API, and DOMPurify sanitization.

02 — CSRF Protection

Cross-Site Request Forgery attacks, SameSite cookies, CSRF tokens, double-submit cookie pattern, Origin header checking, CORS and CSRF interaction, and Server Actions CSRF protection in Next.js.

03 — Authentication Patterns

JWT structure and verification, session-based auth, OAuth 2.0 / OpenID Connect, cookie-based auth, token storage strategies, auth state management in React, protected routes, and NextAuth.js architecture.

04 — Authorization

RBAC, ABAC, permission checking in React components, route-level and API-level authorization, CASL library, server-side authorization in RSC, defense in depth, and feature flags.

05 — Secure Data Handling

Sensitive data in React apps, environment variable exposure risks, API key protection, secrets management, data encryption, PII handling, GDPR compliance, secure forms, and clipboard security.

06 — Dependency Security

npm supply chain attacks, dependency auditing, lock files and integrity hashes, typosquatting, malicious packages, Subresource Integrity, automated updates, and license compliance.

07 — Browser Security Model

Same-Origin Policy, CORS, Content Security Policy, Permissions Policy, Referrer Policy, iframe sandboxing, postMessage security, Web Crypto API, and secure contexts.

08 — Interview Q&A

55+ L6/Staff-level questions covering XSS, CSRF, authentication, authorization, secure data handling, dependency security, and the browser security model.
