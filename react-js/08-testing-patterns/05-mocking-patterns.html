jest.mock and jest.spyOn

jest.mock — Module-Level Mocking

// jest.mock replaces an entire module with a mock implementation
// It's hoisted to the top of the file automatically

// Mock an entire module
jest.mock('./analytics', () => ({
  trackEvent: jest.fn(),
  trackPageView: jest.fn(),
  identifyUser: jest.fn(),
}));

import { trackEvent } from './analytics';

test('tracks button click', async () => {
  render(<TrackableButton label="Buy" />);
  await userEvent.click(screen.getByRole('button', { name: 'Buy' }));

  expect(trackEvent).toHaveBeenCalledWith('button_click', {
    label: 'Buy',
  });
  expect(trackEvent).toHaveBeenCalledTimes(1);
});

// Mock with partial implementation
jest.mock('@/lib/auth', () => ({
  ...jest.requireActual('@/lib/auth'), // Keep real implementations
  getCurrentUser: jest.fn(), // Override just this function
}));

// Mock default export
jest.mock('./Logger', () => {
  return jest.fn().mockImplementation(() => ({
    log: jest.fn(),
    error: jest.fn(),
    warn: jest.fn(),
  }));
});

jest.spyOn — Method-Level Spying

// jest.spyOn wraps an existing method with a spy
// The original implementation is preserved unless overridden

test('logs errors to console', () => {
  // Spy on console.error without replacing it
  const spy = jest.spyOn(console, 'error').mockImplementation();

  render(<ComponentThatLogs />);

  expect(spy).toHaveBeenCalledWith(
    expect.stringContaining('deprecation warning')
  );

  spy.mockRestore(); // Restore original console.error
});

// Spy on object methods
test('calls save method', () => {
  const storage = new StorageService();
  const saveSpy = jest.spyOn(storage, 'save');

  render(<Form storage={storage} />);
  // ... fill and submit form ...

  expect(saveSpy).toHaveBeenCalledWith({
    name: 'Alice',
    email: 'alice@example.com',
  });
});

// Spy on prototype methods
test('fetch is called with correct URL', () => {
  const fetchSpy = jest.spyOn(global, 'fetch').mockResolvedValue(
    new Response(JSON.stringify({ data: 'test' }))
  );

  render(<DataLoader url="/api/data" />);

  expect(fetchSpy).toHaveBeenCalledWith('/api/data', expect.any(Object));

  fetchSpy.mockRestore();
});

Mock Return Values and Implementations

const mockFn = jest.fn();

// Return value (all calls)
mockFn.mockReturnValue(42);

// Return value (once, then default)
mockFn.mockReturnValueOnce('first').mockReturnValueOnce('second');
mockFn(); // 'first'
mockFn(); // 'second'
mockFn(); // undefined

// Async return values
mockFn.mockResolvedValue({ data: 'async result' });
mockFn.mockRejectedValue(new Error('API failed'));

// Custom implementation
mockFn.mockImplementation((x, y) => x + y);

// Assertions
expect(mockFn).toHaveBeenCalled();
expect(mockFn).toHaveBeenCalledTimes(3);
expect(mockFn).toHaveBeenCalledWith('arg1', 'arg2');
expect(mockFn).toHaveBeenLastCalledWith('final');
expect(mockFn).toHaveBeenNthCalledWith(1, 'first call');

// Reset mocks
mockFn.mockClear();  // Clear call history, keep implementation
mockFn.mockReset();  // Clear everything, returns undefined
mockFn.mockRestore(); // Restore original (only for spyOn)

Manual Mocks

// Manual mocks live in __mocks__ directory next to the module
// src/
// ├── lib/
// │   ├── analytics.ts
// │   └── __mocks__/
// │       └── analytics.ts  ← manual mock

// src/lib/__mocks__/analytics.ts
export const trackEvent = jest.fn();
export const trackPageView = jest.fn();
export const identifyUser = jest.fn();

export function resetAllMocks() {
  trackEvent.mockClear();
  trackPageView.mockClear();
  identifyUser.mockClear();
}

// In test file — just declare jest.mock, manual mock is used automatically
jest.mock('@/lib/analytics');

// For node_modules, __mocks__ goes at project root
// __mocks__/
// └── axios.ts  ← mocks 'axios' package

// __mocks__/axios.ts
const axios = {
  get: jest.fn().mockResolvedValue({ data: {} }),
  post: jest.fn().mockResolvedValue({ data: {} }),
  put: jest.fn().mockResolvedValue({ data: {} }),
  delete: jest.fn().mockResolvedValue({ data: {} }),
  create: jest.fn().mockReturnThis(),
  interceptors: {
    request: { use: jest.fn() },
    response: { use: jest.fn() },
  },
};

export default axios;

Timer Mocking

// jest.useFakeTimers replaces setTimeout, setInterval, Date.now, etc.

test('debounced search triggers after delay', async () => {
  jest.useFakeTimers();
  const user = userEvent.setup({ advanceTimers: jest.advanceTimersByTime });

  render(<SearchInput onSearch={mockSearch} debounceMs={300} />);

  await user.type(screen.getByRole('searchbox'), 'react');

  // Search hasn't fired yet (within debounce window)
  expect(mockSearch).not.toHaveBeenCalled();

  // Advance time past debounce
  act(() => jest.advanceTimersByTime(300));

  expect(mockSearch).toHaveBeenCalledWith('react');
  expect(mockSearch).toHaveBeenCalledTimes(1);

  jest.useRealTimers();
});

// Testing intervals
test('auto-refresh polls every 30 seconds', () => {
  jest.useFakeTimers();
  const fetchData = jest.fn();

  render(<AutoRefresh onRefresh={fetchData} intervalMs={30000} />);

  // Initial fetch
  expect(fetchData).toHaveBeenCalledTimes(1);

  // After 30s
  act(() => jest.advanceTimersByTime(30000));
  expect(fetchData).toHaveBeenCalledTimes(2);

  // After another 30s
  act(() => jest.advanceTimersByTime(30000));
  expect(fetchData).toHaveBeenCalledTimes(3);

  jest.useRealTimers();
});

// Testing setTimeout cleanup
test('cleanup cancels pending timers', () => {
  jest.useFakeTimers();
  const callback = jest.fn();

  const { unmount } = render(
    <DelayedAction onAction={callback} delayMs={5000} />
  );

  // Unmount before timer fires
  unmount();

  // Advance past the delay
  act(() => jest.advanceTimersByTime(10000));

  // Callback should NOT have been called
  expect(callback).not.toHaveBeenCalled();

  jest.useRealTimers();
});

Network Mocking with MSW

// MSW provides the most realistic network mocking
// It intercepts at the service worker/node level

import { http, HttpResponse, delay } from 'msw';
import { setupServer } from 'msw/node';

const server = setupServer(
  // REST handlers
  http.get('/api/users', () => {
    return HttpResponse.json([
      { id: '1', name: 'Alice' },
      { id: '2', name: 'Bob' },
    ]);
  }),

  // Dynamic handlers with request inspection
  http.post('/api/users', async ({ request }) => {
    const body = await request.json();

    // Validate request
    if (!body.name || !body.email) {
      return HttpResponse.json(
        { error: 'Missing required fields' },
        { status: 400 }
      );
    }

    return HttpResponse.json(
      { id: '3', ...body },
      { status: 201 }
    );
  }),

  // Simulate network errors
  http.get('/api/flaky', () => {
    return HttpResponse.error(); // Network error (no response)
  }),

  // Simulate slow responses
  http.get('/api/slow', async () => {
    await delay(2000);
    return HttpResponse.json({ data: 'finally' });
  }),

  // GraphQL handlers
  http.post('/graphql', async ({ request }) => {
    const { query, variables } = await request.json();

    if (query.includes('GetUser')) {
      return HttpResponse.json({
        data: {
          user: { id: variables.id, name: 'Alice' }
        }
      });
    }
  }),
);

// Request assertions
test('sends correct headers', async () => {
  let capturedHeaders;

  server.use(
    http.get('/api/data', ({ request }) => {
      capturedHeaders = Object.fromEntries(request.headers);
      return HttpResponse.json({ data: 'test' });
    })
  );

  render(<AuthenticatedComponent />);
  await screen.findByText('test');

  expect(capturedHeaders['authorization']).toBe('Bearer mock-token');
  expect(capturedHeaders['content-type']).toBe('application/json');
});

Mocking Browser APIs

// IntersectionObserver mock
const mockIntersectionObserver = jest.fn();
mockIntersectionObserver.mockReturnValue({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
});
window.IntersectionObserver = mockIntersectionObserver;

// Trigger intersection
test('lazy loads image when visible', () => {
  render(<LazyImage src="/photo.jpg" />);

  // Get the callback passed to IntersectionObserver
  const [callback] = mockIntersectionObserver.mock.calls[0];

  // Simulate element becoming visible
  callback([{ isIntersecting: true, target: screen.getByRole('img') }]);

  expect(screen.getByRole('img')).toHaveAttribute('src', '/photo.jpg');
});

// ResizeObserver mock
class MockResizeObserver {
  callback;
  constructor(callback) { this.callback = callback; }
  observe() {}
  unobserve() {}
  disconnect() {}

  // Helper to trigger resize
  triggerResize(entries) { this.callback(entries); }
}
window.ResizeObserver = MockResizeObserver;

// matchMedia mock
Object.defineProperty(window, 'matchMedia', {
  writable: true,
  value: jest.fn().mockImplementation(query => ({
    matches: query === '(prefers-color-scheme: dark)',
    media: query,
    onchange: null,
    addListener: jest.fn(),
    removeListener: jest.fn(),
    addEventListener: jest.fn(),
    removeEventListener: jest.fn(),
    dispatchEvent: jest.fn(),
  })),
});

// Geolocation mock
const mockGeolocation = {
  getCurrentPosition: jest.fn().mockImplementation(success =>
    success({
      coords: { latitude: 37.7749, longitude: -122.4194 },
    })
  ),
  watchPosition: jest.fn(),
  clearWatch: jest.fn(),
};
Object.defineProperty(navigator, 'geolocation', {
  value: mockGeolocation,
});

// Clipboard mock
Object.assign(navigator, {
  clipboard: {
    writeText: jest.fn().mockResolvedValue(undefined),
    readText: jest.fn().mockResolvedValue('copied text'),
  },
});

test('copies text to clipboard', async () => {
  render(<CopyButton text="Hello" />);
  await userEvent.click(screen.getByRole('button', { name: /copy/i }));

  expect(navigator.clipboard.writeText).toHaveBeenCalledWith('Hello');
  expect(screen.getByText('Copied!')).toBeInTheDocument();
});

Mocking React Hooks

// PREFERRED: Test hooks through components, not by mocking them
// Mocking hooks is usually a sign of testing implementation details

// AVOID: Mocking useAuth directly
jest.mock('./useAuth', () => ({
  useAuth: () => ({ user: { name: 'Alice' }, isAuthenticated: true }),
}));

// PREFER: Provide auth context in test
test('shows user name when authenticated', () => {
  render(
    <AuthProvider initialUser={{ name: 'Alice' }}>
      <UserGreeting />
    </AuthProvider>
  );
  expect(screen.getByText('Hello, Alice')).toBeInTheDocument();
});

// When you MUST mock a hook (third-party, complex setup):
jest.mock('next/navigation', () => ({
  useRouter: () => ({
    push: jest.fn(),
    back: jest.fn(),
    pathname: '/current-page',
  }),
  useSearchParams: () => new URLSearchParams('?q=test'),
  usePathname: () => '/current-page',
}));

// Mocking custom hooks for specific test scenarios
jest.mock('./useMediaQuery', () => ({
  useMediaQuery: jest.fn(),
}));

import { useMediaQuery } from './useMediaQuery';

test('renders mobile layout on small screens', () => {
  (useMediaQuery as jest.Mock).mockReturnValue(true); // matches mobile
  render(<ResponsiveLayout />);
  expect(screen.getByTestId('mobile-nav')).toBeInTheDocument();
});

test('renders desktop layout on large screens', () => {
  (useMediaQuery as jest.Mock).mockReturnValue(false); // doesn't match
  render(<ResponsiveLayout />);
  expect(screen.getByTestId('desktop-nav')).toBeInTheDocument();
});

When to Mock vs Real Implementations

Mocking Decision Tree: Is it a network request? ├─ YES → Use MSW (mock at network boundary) └─ NO │ Is it a browser API not in jsdom? ├─ YES → Mock the API (IntersectionObserver, etc.) └─ NO │ Is it a timer/date? ├─ YES → Use jest.useFakeTimers() └─ NO │ Is it a third-party service (analytics, auth)? ├─ YES → Mock the module (jest.mock) └─ NO │ Is it your own code? ├─ YES → DON'T MOCK — use real implementation └─ Reconsider your test design
MockDon't Mock
Network requests (use MSW)Your own components
Browser APIs not in jsdomYour own utility functions
Timers and datesYour own hooks (test via components)
Third-party analytics/trackingState management (Redux, Zustand)
External services (Stripe, Auth0)React Router (use MemoryRouter)
Random number generationCSS/styling (use visual regression)
File system (in Node.js tests)React Context (provide real context)
Architect Insight

Over-mocking is the #1 cause of tests that pass but don't catch bugs. Every mock is a point where your test diverges from reality. The ideal test mocks only at the system boundary (network, browser APIs, external services) and uses real implementations for everything else. If you find yourself mocking 5+ things for a single test, the component under test likely has too many dependencies — consider refactoring the component before adding more mocks.
