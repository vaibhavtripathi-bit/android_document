Testing Component Interactions

Integration tests verify that multiple components work together correctly. The key principle: render the highest-level component that encompasses the interaction you're testing, and mock only at the network boundary.

// Testing a todo app — multiple components interacting
test('complete todo workflow: add, toggle, delete', async () => {
  const user = userEvent.setup();
  render(<TodoApp />);

  // Add a todo
  await user.type(screen.getByPlaceholderText('What needs to be done?'), 'Buy milk');
  await user.keyboard('{Enter}');
  expect(screen.getByText('Buy milk')).toBeInTheDocument();

  // Add another
  await user.type(screen.getByPlaceholderText('What needs to be done?'), 'Walk dog');
  await user.keyboard('{Enter}');
  expect(screen.getByText('Walk dog')).toBeInTheDocument();

  // Toggle completion
  await user.click(screen.getByRole('checkbox', { name: /buy milk/i }));
  expect(screen.getByRole('checkbox', { name: /buy milk/i })).toBeChecked();

  // Filter to active
  await user.click(screen.getByRole('button', { name: /active/i }));
  expect(screen.queryByText('Buy milk')).not.toBeInTheDocument();
  expect(screen.getByText('Walk dog')).toBeInTheDocument();

  // Delete
  await user.hover(screen.getByText('Walk dog'));
  await user.click(screen.getByRole('button', { name: /delete/i }));
  expect(screen.queryByText('Walk dog')).not.toBeInTheDocument();
  expect(screen.getByText('No active todos')).toBeInTheDocument();
});

MSW (Mock Service Worker)

MSW intercepts network requests at the service worker level, providing realistic API mocking without modifying application code. It's the gold standard for integration test network mocking.

// mocks/handlers.ts — default API handlers
import { http, HttpResponse } from 'msw';

export const handlers = [
  http.get('/api/users', () => {
    return HttpResponse.json([
      { id: '1', name: 'Alice', role: 'admin' },
      { id: '2', name: 'Bob', role: 'user' },
    ]);
  }),

  http.get('/api/users/:id', ({ params }) => {
    const users = {
      '1': { id: '1', name: 'Alice', role: 'admin' },
      '2': { id: '2', name: 'Bob', role: 'user' },
    };
    const user = users[params.id];
    if (!user) {
      return new HttpResponse(null, { status: 404 });
    }
    return HttpResponse.json(user);
  }),

  http.post('/api/users', async ({ request }) => {
    const body = await request.json();
    return HttpResponse.json(
      { id: '3', ...body },
      { status: 201 }
    );
  }),

  http.delete('/api/users/:id', () => {
    return new HttpResponse(null, { status: 204 });
  }),
];

// mocks/server.ts — test server setup
import { setupServer } from 'msw/node';
import { handlers } from './handlers';

export const server = setupServer(...handlers);

// jest.setup.ts or vitest.setup.ts
beforeAll(() => server.listen({ onUnhandledRequest: 'error' }));
afterEach(() => server.resetHandlers());
afterAll(() => server.close());
// Override handlers per test for specific scenarios
test('shows error when API fails', async () => {
  // Override the default handler for this test only
  server.use(
    http.get('/api/users', () => {
      return new HttpResponse(null, { status: 500 });
    })
  );

  render(<UserList />);
  expect(await screen.findByText(/failed to load users/i)).toBeInTheDocument();
});

test('shows empty state when no users', async () => {
  server.use(
    http.get('/api/users', () => {
      return HttpResponse.json([]);
    })
  );

  render(<UserList />);
  expect(await screen.findByText(/no users found/i)).toBeInTheDocument();
});

// Test network delay
test('shows loading state during fetch', async () => {
  server.use(
    http.get('/api/users', async () => {
      await delay(200); // Simulate slow network
      return HttpResponse.json([{ id: '1', name: 'Alice' }]);
    })
  );

  render(<UserList />);

  // Loading state visible immediately
  expect(screen.getByText(/loading/i)).toBeInTheDocument();

  // Data appears after fetch completes
  expect(await screen.findByText('Alice')).toBeInTheDocument();
  expect(screen.queryByText(/loading/i)).not.toBeInTheDocument();
});
Why MSW Over jest.mock

MSW intercepts at the network level, so your application code (fetch, axios, react-query) runs exactly as in production. jest.mock replaces modules, which means you're not testing the actual data fetching code. MSW also works in the browser (Storybook, dev mode), providing a single mocking layer for all environments.

Testing Data Fetching Flows

// Testing React Query integration
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

function createTestQueryClient() {
  return new QueryClient({
    defaultOptions: {
      queries: {
        retry: false,
        gcTime: 0, // Don't cache between tests
      },
    },
  });
}

function renderWithQuery(ui) {
  const queryClient = createTestQueryClient();
  return render(
    <QueryClientProvider client={queryClient}>
      {ui}
    </QueryClientProvider>
  );
}

test('fetches and displays product list', async () => {
  server.use(
    http.get('/api/products', () => {
      return HttpResponse.json({
        products: [
          { id: '1', name: 'Widget', price: 9.99 },
          { id: '2', name: 'Gadget', price: 19.99 },
        ],
        total: 2,
      });
    })
  );

  renderWithQuery(<ProductList />);

  // Wait for data to load
  expect(await screen.findByText('Widget')).toBeInTheDocument();
  expect(screen.getByText('$9.99')).toBeInTheDocument();
  expect(screen.getByText('Gadget')).toBeInTheDocument();
  expect(screen.getByText('Showing 2 products')).toBeInTheDocument();
});

// Testing pagination
test('loads next page on scroll', async () => {
  server.use(
    http.get('/api/products', ({ request }) => {
      const url = new URL(request.url);
      const page = url.searchParams.get('page') ?? '1';

      if (page === '1') {
        return HttpResponse.json({
          products: [{ id: '1', name: 'Widget' }],
          hasMore: true,
        });
      }
      return HttpResponse.json({
        products: [{ id: '2', name: 'Gadget' }],
        hasMore: false,
      });
    })
  );

  renderWithQuery(<InfiniteProductList />);

  expect(await screen.findByText('Widget')).toBeInTheDocument();

  // Trigger load more
  await userEvent.click(screen.getByRole('button', { name: /load more/i }));

  expect(await screen.findByText('Gadget')).toBeInTheDocument();
  expect(screen.queryByRole('button', { name: /load more/i }))
    .not.toBeInTheDocument();
});

Testing Routing

import { MemoryRouter, Route, Routes } from 'react-router-dom';

// MemoryRouter for testing — no real browser history
function renderWithRouter(ui, { initialEntries = ['/'] } = {}) {
  return render(
    <MemoryRouter initialEntries={initialEntries}>
      {ui}
    </MemoryRouter>
  );
}

test('navigates between pages', async () => {
  renderWithRouter(
    <Routes>
      <Route path="/" element={<HomePage />} />
      <Route path="/about" element={<AboutPage />} />
      <Route path="/products/:id" element={<ProductPage />} />
    </Routes>
  );

  // Start on home page
  expect(screen.getByRole('heading', { name: /home/i })).toBeInTheDocument();

  // Navigate to about
  await userEvent.click(screen.getByRole('link', { name: /about/i }));
  expect(screen.getByRole('heading', { name: /about/i })).toBeInTheDocument();
});

// Test with initial route
test('renders product page for specific route', () => {
  server.use(
    http.get('/api/products/42', () => {
      return HttpResponse.json({ id: '42', name: 'Widget' });
    })
  );

  renderWithRouter(
    <Routes>
      <Route path="/products/:id" element={<ProductPage />} />
    </Routes>,
    { initialEntries: ['/products/42'] }
  );

  expect(screen.findByText('Widget')).resolves.toBeInTheDocument();
});

// Test redirect
test('redirects unauthenticated users to login', () => {
  renderWithRouter(
    <AuthProvider user={null}>
      <Routes>
        <Route path="/login" element={<LoginPage />} />
        <Route path="/dashboard" element={
          <ProtectedRoute><Dashboard /></ProtectedRoute>
        } />
      </Routes>

});

Testing Form Submissions

test('submits registration form and shows success', async () => {
  const user = userEvent.setup();
  let capturedBody;

  server.use(
    http.post('/api/register', async ({ request }) => {
      capturedBody = await request.json();
      return HttpResponse.json({ id: '1', ...capturedBody });
    })
  );

  render(<RegistrationForm />);

  // Fill out form
  await user.type(screen.getByLabelText(/name/i), 'Alice Smith');
  await user.type(screen.getByLabelText(/email/i), 'alice@example.com');
  await user.type(screen.getByLabelText(/password/i), 'SecureP@ss123');
  await user.type(screen.getByLabelText(/confirm password/i), 'SecureP@ss123');
  await user.click(screen.getByRole('checkbox', { name: /terms/i }));

  // Submit
  await user.click(screen.getByRole('button', { name: /register/i }));

  // Verify success
  expect(await screen.findByText(/registration successful/i)).toBeInTheDocument();

  // Verify correct data was sent
  expect(capturedBody).toEqual({
    name: 'Alice Smith',
    email: 'alice@example.com',
    password: 'SecureP@ss123',
  });
});

// Test form validation
test('shows validation errors for invalid input', async () => {
  const user = userEvent.setup();
  render(<RegistrationForm />);

  // Submit empty form
  await user.click(screen.getByRole('button', { name: /register/i }));

  // Check validation messages
  expect(screen.getByText(/name is required/i)).toBeInTheDocument();
  expect(screen.getByText(/email is required/i)).toBeInTheDocument();

  // Fix one error
  await user.type(screen.getByLabelText(/name/i), 'Alice');
  await user.click(screen.getByRole('button', { name: /register/i }));

  // Name error gone, email error remains
  expect(screen.queryByText(/name is required/i)).not.toBeInTheDocument();
  expect(screen.getByText(/email is required/i)).toBeInTheDocument();
});

Testing Error Boundaries

// Testing error boundary behavior
function BuggyComponent({ shouldThrow }) {
  if (shouldThrow) throw new Error('Component crashed!');
  return <div>Working fine</div>;
}

test('error boundary catches rendering errors', () => {
  // Suppress console.error for expected errors
  const consoleSpy = jest.spyOn(console, 'error').mockImplementation();

  render(
    <ErrorBoundary fallback={<div>Something went wrong</div>}>
      <BuggyComponent shouldThrow={true} />
    </ErrorBoundary>
  );

  expect(screen.getByText('Something went wrong')).toBeInTheDocument();
  consoleSpy.mockRestore();
});

// Testing error boundary recovery
test('error boundary recovers on retry', async () => {
  const consoleSpy = jest.spyOn(console, 'error').mockImplementation();
  let shouldThrow = true;

  function UnstableComponent() {
    if (shouldThrow) throw new Error('Temporary failure');
    return <div>Recovered!</div>;
  }

  render(
    <ErrorBoundary>
      <UnstableComponent />
    </ErrorBoundary>
  );

  expect(screen.getByText(/something went wrong/i)).toBeInTheDocument();

  // Fix the error condition
  shouldThrow = false;

  // Click retry
  await userEvent.click(screen.getByRole('button', { name: /retry/i }));
  expect(screen.getByText('Recovered!')).toBeInTheDocument();

  consoleSpy.mockRestore();
});

Testing Suspense Boundaries

// Testing Suspense loading states
test('shows loading fallback then content', async () => {
  server.use(
    http.get('/api/data', async () => {
      await delay(100);
      return HttpResponse.json({ message: 'Hello' });
    })
  );

  render(
    <Suspense fallback={<div>Loading...</div>}>
      <AsyncComponent />
    </Suspense>
  );

  // Fallback shown immediately
  expect(screen.getByText('Loading...')).toBeInTheDocument();

  // Content appears after data loads
  expect(await screen.findByText('Hello')).toBeInTheDocument();
  expect(screen.queryByText('Loading...')).not.toBeInTheDocument();
});

// Testing nested Suspense
test('nested suspense boundaries load independently', async () => {
  server.use(
    http.get('/api/fast', () => {
      return HttpResponse.json({ data: 'Fast data' });
    }),
    http.get('/api/slow', async () => {
      await delay(500);
      return HttpResponse.json({ data: 'Slow data' });
    })
  );

  render(
    <div>
      <Suspense fallback={<div>Loading fast...</div>}>
        <FastComponent />
      </Suspense>
      <Suspense fallback={<div>Loading slow...</div>}>
        <SlowComponent />
      </Suspense>
    </div>
  );

  // Fast component loads first
  expect(await screen.findByText('Fast data')).toBeInTheDocument();
  expect(screen.getByText('Loading slow...')).toBeInTheDocument();

  // Slow component loads later
  expect(await screen.findByText('Slow data')).toBeInTheDocument();
});

Testing Authentication Flows

test('complete login flow', async () => {
  const user = userEvent.setup();

  server.use(
    http.post('/api/login', async ({ request }) => {
      const { email, password } = await request.json();
      if (email === 'alice@example.com' && password === 'password123') {
        return HttpResponse.json({
          user: { id: '1', name: 'Alice' },
          token: 'mock-jwt-token',
        });
      }
      return HttpResponse.json(
        { error: 'Invalid credentials' },
        { status: 401 }
      );
    }),
    http.get('/api/dashboard', () => {
      return HttpResponse.json({ stats: { users: 100 } });
    })
  );

  render(<App />);

  // Should be on login page
  expect(screen.getByRole('heading', { name: /login/i })).toBeInTheDocument();

  // Enter credentials
  await user.type(screen.getByLabelText(/email/i), 'alice@example.com');
  await user.type(screen.getByLabelText(/password/i), 'password123');
  await user.click(screen.getByRole('button', { name: /sign in/i }));

  // Should redirect to dashboard
  expect(await screen.findByRole('heading', { name: /dashboard/i }))
    .toBeInTheDocument();
  expect(screen.getByText('Welcome, Alice')).toBeInTheDocument();
});

Test Organization Patterns

// Arrange-Act-Assert pattern
test('adds item to cart', async () => {
  // ARRANGE: Set up the test environment
  server.use(
    http.get('/api/products/1', () =>
      HttpResponse.json({ id: '1', name: 'Widget', price: 9.99 })
    )
  );
  render(<ProductPage productId="1" />);
  await screen.findByText('Widget');

  // ACT: Perform the user action
  await userEvent.click(screen.getByRole('button', { name: /add to cart/i }));

  // ASSERT: Verify the outcome
  expect(screen.getByText('1 item in cart')).toBeInTheDocument();
  expect(screen.getByText('$9.99')).toBeInTheDocument();
});

Architect Insight

For large codebases, organize integration tests by user flow rather than by component. A test file named checkout-flow.test.tsx that tests the entire checkout process is more valuable than separate tests for CartButton, CartDrawer, CheckoutForm, and PaymentInput. Flow-based tests catch integration bugs that component-level tests miss.
