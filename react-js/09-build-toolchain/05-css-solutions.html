2. CSS Modules

CSS Modules provide locally-scoped class names by default, preventing global namespace collisions. The bundler transforms class names to unique hashes at build time.

/* Button.module.css */
.button {
  padding: 8px 16px;
  border-radius: 4px;
  font-weight: 600;
}

.button:hover {
  opacity: 0.9;
}

.primary {
  composes: button;  /* Composition — reuse styles */
  background: #3b82f6;
  color: white;
}

.secondary {
  composes: button;
  background: transparent;
  border: 1px solid #3b82f6;
  color: #3b82f6;
}

/* Global escape hatch */
:global(.external-class) {
  color: red;
}
// Button.tsx
import styles from './Button.module.css';

function Button({ variant = 'primary', children }) {
  return (
    <button className={styles[variant]}>
      {children}
    </button>
  );
}

// Rendered HTML:
// <button class="Button_primary_x7yz2 Button_button_a3bc1">Click</button>
// Class names are hashed — no collisions possible

// TypeScript: typed CSS modules
// With typescript-plugin-css-modules or typed-css-modules:
// Button.module.css.d.ts (auto-generated)
declare const styles: {
  readonly button: string;
  readonly primary: string;
  readonly secondary: string;
};
export default styles;

CSS Modules Configuration

// Webpack CSS Modules config
module.exports = {
  module: {
    rules: [{
      test: /\.module\.css$/,
      use: [
        'style-loader',
        {
          loader: 'css-loader',
          options: {
            modules: {
              localIdentName: '[name]__[local]--[hash:base64:5]',
              exportLocalsConvention: 'camelCase',
            },
          },
        },
      ],
    }],
  },
};

// Vite: CSS Modules work out of the box
// Any .module.css file is automatically treated as CSS Modules
// vite.config.ts
export default defineConfig({
  css: {
    modules: {
      localsConvention: 'camelCaseOnly',
      generateScopedName: '[name]__[local]___[hash:base64:5]',
    },
  },
});

3. CSS-in-JS: styled-components

styled-components uses tagged template literals to create React components with attached styles. Styles are generated at runtime and injected into the DOM.

import styled, { css, keyframes } from 'styled-components';

// Basic styled component
const Button = styled.button<{ $primary?: boolean }>`
  padding: 8px 16px;
  border-radius: 4px;
  font-weight: 600;
  cursor: pointer;
  transition: all 0.2s;

  ${({ $primary }) => $primary
    ? css`
        background: #3b82f6;
        color: white;
        border: none;
      `
    : css`
        background: transparent;
        color: #3b82f6;
        border: 1px solid #3b82f6;
      `
  }

  &:hover {
    opacity: 0.9;
    transform: translateY(-1px);
  }

  &:disabled {
    opacity: 0.5;
    cursor: not-allowed;
  }
`;

// Extending styles
const SubmitButton = styled(Button)`
  min-width: 120px;
  font-size: 1rem;
`;

// Animations
const fadeIn = keyframes`
  from { opacity: 0; transform: translateY(10px); }
  to { opacity: 1; transform: translateY(0); }
`;

const AnimatedCard = styled.div`
  animation: ${fadeIn} 0.3s ease-out;
`;

// Theme usage
const ThemedButton = styled.button`
  background: ${({ theme }) => theme.colors.primary};
  color: ${({ theme }) => theme.colors.text};
  font-size: ${({ theme }) => theme.fontSizes.md};
`;

styled-components SSR

// Server-side rendering with styled-components
import { ServerStyleSheet, StyleSheetManager } from 'styled-components';

// In your SSR handler:
const sheet = new ServerStyleSheet();
try {
  const html = renderToString(
    <StyleSheetManager sheet={sheet.instance}>
      <App />
    </StyleSheetManager>
  );
  const styleTags = sheet.getStyleTags();
  // Insert styleTags into <head> of HTML response

  return `
    <html>
      <head>${styleTags}</head>
      <body><div id="root">${html}</div></body>
    </html>
  `;
} finally {
  sheet.seal();
}

Runtime CSS-in-JS Performance Concerns

styled-components and Emotion (runtime mode) generate CSS at render time, which means: (1) JavaScript must parse template literals and compute styles on every render, (2) styles are injected into the DOM dynamically, causing style recalculation, (3) SSR requires collecting styles during rendering and rehydrating on the client. For performance-critical applications, consider zero-runtime alternatives.

4. Emotion

// Emotion — two APIs: styled and css prop
/** @jsxImportSource @emotion/react */
import { css } from '@emotion/react';
import styled from '@emotion/styled';

// css prop (requires JSX pragma or babel plugin)
function Card({ title, children }) {
  return (
    <div css={css`
      padding: 16px;
      border-radius: 8px;
      box-shadow: 0 2px 8px rgba(0,0,0,0.1);
    `}>
      <h2 css={css`
        margin-bottom: 8px;
        color: #1a1a1a;
      `}>{title}</h2>
      {children}
    </div>
  );
}

// Object styles (better TypeScript support)
const containerStyle = css({
  display: 'flex',
  flexDirection: 'column',
  gap: '16px',
  maxWidth: '800px',
  margin: '0 auto',

// Emotion styled (same API as styled-components)
const Badge = styled.span<{ variant: 'success' | 'error' | 'warning' }>`
  padding: 2px 8px;
  border-radius: 12px;
  font-size: 0.75rem;
  background: ${({ variant }) => ({
    success: '#dcfce7',
    error: '#fce4ec',
    warning: '#fff3e0',
  }[variant])};
`;

5. Linaria (Zero-Runtime CSS-in-JS)

// Linaria extracts CSS at build time — zero runtime cost
import { styled } from '@linaria/react';
import { css } from '@linaria/core';

// Styles are extracted to .css files at build time
const Button = styled.button`
  padding: 8px 16px;
  border-radius: 4px;
  background: ${({ primary }) => primary ? '#3b82f6' : 'transparent'};
  color: ${({ primary }) => primary ? 'white' : '#3b82f6'};
`;

// css tag for class names
const header = css`
  font-size: 2rem;
  font-weight: 700;
  color: #1a1a1a;
`;

function Header() {
  return <h1 className={header}>Title</h1>;
}

// Build output: styles extracted to separate .css files
// No JavaScript runtime for style computation

6. Tailwind CSS

// Tailwind — utility-first CSS framework
// tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ['./src/**/*.{js,jsx,ts,tsx}'],
  theme: {
    extend: {
      colors: {
        brand: {
          50: '#eff6ff',
          500: '#3b82f6',
          900: '#1e3a5f',
        },
      },
      spacing: {
        '18': '4.5rem',
      },
    },
  },
  plugins: [
    require('@tailwindcss/forms'),
    require('@tailwindcss/typography'),
  ],
};

// Component with Tailwind classes
function Card({ title, children }) {
  return (
    <div className="rounded-lg shadow-md p-6 bg-white
                    hover:shadow-lg transition-shadow duration-200">
      <h2 className="text-xl font-semibold text-gray-900 mb-3">
        {title}
      </h2>
      <div className="text-gray-600 leading-relaxed">
        {children}
      </div>
    </div>
  );
}

// Conditional classes with clsx/cn
import { clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

function cn(...inputs) {
  return twMerge(clsx(inputs));
}

function Button({ variant, size, className, ...props }) {
  return (
    <button
      className={cn(
        'rounded font-medium transition-colors',
        variant === 'primary' && 'bg-blue-500 text-white hover:bg-blue-600',
        variant === 'secondary' && 'bg-gray-100 text-gray-900 hover:bg-gray-200',
        size === 'sm' && 'px-3 py-1.5 text-sm',
        size === 'lg' && 'px-6 py-3 text-lg',
        className
      )}
      {...props}
    />
  );
}

Tailwind JIT and Purging

Tailwind's JIT (Just-In-Time) engine generates only the CSS classes you actually use by scanning your source files. This means: (1) development builds are as small as production, (2) arbitrary values like w-[347px] work without configuration, (3) the full CSS output is typically 5-15KB gzipped regardless of how many utilities exist. The content array in config tells Tailwind which files to scan for class usage.

7. Vanilla Extract

// Vanilla Extract — type-safe, zero-runtime CSS
// styles.css.ts (note the .css.ts extension)
import { style, styleVariants, createTheme, globalStyle } from '@vanilla-extract/css';

// Type-safe theme
export const [themeClass, vars] = createTheme({
  color: {
    brand: '#3b82f6',
    text: '#1a1a1a',
    background: '#ffffff',
  },
  space: {
    sm: '8px',
    md: '16px',
    lg: '24px',
  },
});

// Styles are fully type-checked
export const button = style({
  padding: vars.space.md,
  borderRadius: '4px',
  fontWeight: 600,
  cursor: 'pointer',
  ':hover': {
    opacity: 0.9,
  },
  '@media': {
    '(min-width: 768px)': {
      padding: vars.space.lg,
    },
  },
});

// Style variants
export const buttonVariant = styleVariants({
  primary: [button, { background: vars.color.brand, color: 'white' }],
  secondary: [button, { background: 'transparent', border: `1px solid ${vars.color.brand}` }],
});

// Usage in component
import { buttonVariant } from './styles.css';

function Button({ variant = 'primary', children }) {
  return <button className={buttonVariant[variant]}>{children}</button>;
}

8. CSS Cascade Layers

/* CSS Cascade Layers (@layer) — control specificity ordering */
/* Layers are ordered by declaration — later layers win */

@layer reset, base, components, utilities;

@layer reset {
  *, *::before, *::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
  }
}

@layer base {
  body {
    font-family: system-ui, sans-serif;
    line-height: 1.6;
  }
  a { color: #3b82f6; }
}

@layer components {
  .button {
    padding: 8px 16px;
    border-radius: 4px;
  }
  .card {
    padding: 16px;
    border-radius: 8px;
    box-shadow: 0 2px 8px rgba(0,0,0,0.1);
  }
}

@layer utilities {
  .sr-only { /* screen reader only */ }
  .hidden { display: none; }
}

/* Unlayered styles always win over layered styles */

9. Container Queries

/* Container queries — responsive to parent, not viewport */
.card-container {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
    gap: 16px;
  }
}

@container card (min-width: 600px) {
  .card {
    grid-template-columns: 300px 1fr;
  }
  .card-title {
    font-size: 1.5rem;
  }
}
// React component using container queries
function ResponsiveCard({ title, image, description }) {
  return (
    <div style={{ containerType: 'inline-size' }}>
      <div className="card">
        <img src={image} alt={title} />
        <div>
          <h3 className="card-title">{title}</h3>
          <p>{description}</p>
        </div>
      </div>
    </div>
  );
}

10. CSS Custom Properties with React

// Dynamic theming with CSS custom properties
function ThemeProvider({ theme, children }) {
  const style = {
    '--color-primary': theme.primary,
    '--color-secondary': theme.secondary,
    '--color-background': theme.background,
    '--color-text': theme.text,
    '--spacing-unit': `${theme.spacingUnit}px`,
    '--border-radius': `${theme.borderRadius}px`,
  } as React.CSSProperties;

  return <div style={style}>{children}</div>;
}

// CSS consumes the variables
/* styles.css */
.button {
  background: var(--color-primary);
  color: white;
  padding: calc(var(--spacing-unit) * 2);
  border-radius: var(--border-radius);
}

Architect Insight: Choosing a CSS Strategy

For new React projects at scale: (1) Tailwind CSS for rapid prototyping and consistent design systems with utility classes, (2) CSS Modules for teams that prefer traditional CSS with scoping, (3) Vanilla Extract for type-safe, zero-runtime CSS in TypeScript-heavy codebases, (4) styled-components/Emotion only if you need highly dynamic styles that depend on props at render time. The industry trend is moving away from runtime CSS-in-JS toward zero-runtime solutions, especially with React Server Components where runtime CSS-in-JS doesn't work.
