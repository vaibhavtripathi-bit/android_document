Files, npm, and Async Work

A real backend application does more than return text from one route. It reads files, handles errors, uses packages, and deals with work that takes time. This lesson connects those pieces.

Reading a File

Node.js can read files directly from your machine. A clean beginner approach is to use the promise-based API. 

const fs = require("fs/promises");

async function readFileExample() {
  const text = await fs.readFile("notes.txt", "utf8");
  console.log(text);
}

readFileExample();

Here, await means “wait for the file-reading result before moving to the next line inside this function.”

Writing a File

const fs = require("fs/promises");

async function writeFileExample() {
  await fs.writeFile("notes.txt", "Learning Node.js step by step");
  console.log("File written");
}

writeFileExample();

This is useful for logs, small prototypes, generated files, and simple local data experiments.

Why try/catch Matters

Files may not exist, permissions may fail, or the path may be wrong. That is why async code often sits inside try/catch. 

const fs = require("fs/promises");

async function safeRead() {
  try {
    const text = await fs.readFile("missing-file.txt", "utf8");
    console.log(text);
  } catch (error) {
    console.error("Could not read file:", error.message);
  }
}

safeRead();
Backend rule: assume file reads, database calls, network requests, and external services can fail. Your code should not behave as if the happy path is the only path.

Use npm Scripts Instead of Repeating Commands

Update your package.json:

{
  "name": "nodejs-learning",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": {
    "start": "node server.js",
    "dev": "node server.js"
  }
}

Now you can run:

npm run start

Later, many projects use tools such as nodemon for restart-on-save during development, but you do not need that before understanding the basics.

Built-In Module vs External Package

Built-in example

fs, path, and http come from Node.js itself. No installation is required.

External example

Packages such as chalk, express, or dotenv come from npm and must be installed.

Environment Variables in Real Projects

const PORT = process.env.PORT || 3000;
const APP_NAME = process.env.APP_NAME || "My Node App";

console.log(APP_NAME, "running on", PORT);

Backend applications often avoid hardcoding important configuration like ports, API keys, or database URLs. Those values usually come from the environment.

Async Example With Two Tasks

function wait(ms) {
  return new Promise((resolve) => {
    setTimeout(resolve, ms);
  });
}

async function run() {
  console.log("Start");
  await wait(1000);
  console.log("One second later");
}

run();

This small example is useful because it makes async behavior feel visible. You start work, wait, and continue later.

One Small Practical Project Idea

Add a route to your server that reads a local file and returns its content as JSON. For example, create a file named profile.json: 

{
  "name": "Asha",
  "role": "Node learner"
}

Then add server code that reads the file and returns it.

const http = require("http");
const fs = require("fs/promises");

const server = http.createServer(async (request, response) => {
  if (request.url === "/profile") {
    try {
      const text = await fs.readFile("profile.json", "utf8");
      response.statusCode = 200;
      response.setHeader("Content-Type", "application/json");
      response.end(text);
    } catch (error) {
      response.statusCode = 500;
      response.end("Could not read profile");
    }
    return;
  }

  response.statusCode = 404;
  response.end("Not found");
});

This combines a backend route, file reading, JSON, and error handling in one beginner-sized example.
