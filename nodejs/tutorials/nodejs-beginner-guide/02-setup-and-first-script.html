Setup and Your First Script

This lesson gets you from “I heard of Node.js” to “I can run Node on my machine and create a simple file.” The target is practical comfort, not just installing software.

Install Node.js

Install the current LTS release of Node.js from the official Node.js website. The LTS release is the beginner-friendly choice because it is the stable line most backend projects want.

Recommendation: start with the LTS version, not the newest experimental release. Stability matters more than novelty when you are learning.

Verify the Installation

Open a terminal and run:

node -v
npm -v

If both commands print version numbers, your installation is working. node is the runtime. npm is the package manager that comes with Node.js.

Create a Workspace

In your terminal, create a new folder for learning:

mkdir nodejs-learning
cd nodejs-learning

Keeping a separate folder for experiments is useful. When you learn backend development, you will create many small files, and it helps to keep that work contained.

Run Your First Node Script

Create a file named app.js with this content:

console.log("Hello from Node.js");

Then run:

node app.js

You should see Hello from Node.js printed in the terminal. This is your first proof that JavaScript is running outside the browser.

Try the Node REPL

REPL means Read-Eval-Print Loop. It is an interactive mode where you can type JavaScript directly into Node. 

node

After that, try:

2 + 3
const name = "Node learner"
name.toUpperCase()

To exit the REPL, press Ctrl + C twice.

Why this matters: the REPL is a fast place to test small language ideas, string methods, arrays, objects, or little code experiments.

Useful Built-In Globals

Create or update app.js:

console.log("Node version:", process.version);
console.log("Current folder:", process.cwd());
console.log("Current file:", __filename);
console.log("Current folder of this file:", __dirname);

This shows that Node gives your code access to process information and file location information. Those things do not exist the same way in normal browser JavaScript.

Initialize a Project With npm

Inside your learning folder, run:

npm init -y

This creates a package.json file. That file describes your project. It usually contains the project name, version, scripts, and dependencies.

package.json

The main file that describes a Node.js project.

Dependencies

External packages your project needs to run.

Scripts

Short commands you can run with npm run.

Small JavaScript Refresher

You do not need to master all of JavaScript before learning Node, but you should be comfortable with these shapes: 

const user = {
  name: "Asha",
  role: "learner"
};

function greet(person) {
  return `Hello, ${person.name}`;
}

console.log(greet(user));

If objects, arrays, and functions still feel confusing, spend a bit of time with them now. They show up everywhere in backend code.

Quick Exercise

  1. Create a file called intro.js.
  2. Print your name, the Node version, and your current folder.
  3. Run the file from the terminal.
  4. Open the REPL and test one string method and one array method.
