Build Your First Node.js Server

This page is the first real backend step. You will create an HTTP server with Node’s built-in module, respond to requests, and learn the meaning of route, status code, JSON response, and port.

The Built-In HTTP Module

You do not need Express to create a server. Node’s built-in http module can already accept requests and send responses. Frameworks are useful later, but building one server by hand makes backend concepts much easier to understand.

Create server.js

const http = require("http");

const PORT = 3000;

const server = http.createServer((request, response) => {
  response.statusCode = 200;
  response.setHeader("Content-Type", "text/plain");
  response.end("Hello from my first Node.js server");
});

server.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`);
});

Run it with:

node server.js

Then open http://localhost:3000 in your browser. You should see the response text.

What Is Happening Here?

http.createServer()

Creates a server and gives you a callback for every incoming request.

request

Represents information about the incoming client request.

response

Represents what your server will send back to the client.

server.listen()

Starts the server and makes it listen on a specific port.

What Is a Port?

A port is a communication endpoint. When you run the server on port 3000, you are telling the operating system to accept requests for that application at localhost:3000.

Common beginner confusion: the port is not part of your code logic. It is how the server process exposes itself to the machine and network.

Add Simple Routes

A route means “if the client asks for this path, send this kind of response.” Update server.js: 

const http = require("http");

const PORT = 3000;

const server = http.createServer((request, response) => {
  if (request.url === "/") {
    response.statusCode = 200;
    response.setHeader("Content-Type", "text/plain");
    response.end("Welcome to the home page");
    return;
  }

  if (request.url === "/health") {
    response.statusCode = 200;
    response.setHeader("Content-Type", "application/json");
    response.end(JSON.stringify({ status: "ok" }));
    return;
  }

  response.statusCode = 404;
  response.setHeader("Content-Type", "text/plain");
  response.end("Route not found");
});

server.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`);
});

Now test these URLs:

  • http://localhost:3000/
  • http://localhost:3000/health
  • http://localhost:3000/anything-else

What Is JSON and Why Is It Everywhere?

JSON is a text format for sending structured data. Backend APIs use it heavily because it is easy to produce and easy for clients to read. 

{
  "status": "ok"
}

In Node, JSON.stringify() turns a JavaScript object into a JSON string before sending it in the response.

Add Method Checks

Clients can send different HTTP methods such as GET and POST. For a beginner server, it is enough to understand that method and path together decide the meaning of a request. 

if (request.method === "GET" && request.url === "/health") {
  response.statusCode = 200;
  response.setHeader("Content-Type", "application/json");
  response.end(JSON.stringify({ status: "ok" }));
  return;
}

Use an Environment Variable for the Port

const PORT = process.env.PORT || 3000;

This is a normal backend pattern. In production, the environment may decide the port. Locally, you can fall back to 3000.
