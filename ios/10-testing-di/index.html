Testing & Dependency Injection

A comprehensive deep-dive into iOS testing frameworks, mocking strategies, dependency injection patterns, and CI/CD pipelines — targeting Google L6 / Staff-level interview mastery.

This series covers the full spectrum of iOS testing and dependency injection: from XCTest fundamentals and the new Swift Testing framework, through protocol-oriented mocking and UI testing, to enterprise-grade DI architectures and CI/CD pipelines that scale to hundreds of developers. Each document is written for engineers who need to design test infrastructure, not just write individual tests.

The Test Pyramid at Staff Level

┌───────────┐ │ E2E/UI │ ← Slow, flaky, expensive │ Tests │ ~5-10% of total tests ├─────────────┤ │ Integration │ ← Module boundaries, API contracts │ Tests │ ~15-25% of total tests ├─────────────────┤ │ Unit Tests │ ← Fast, isolated, deterministic │ (XCTest + │ ~65-80% of total tests │ Swift Testing) │ └──────────────────────┘ Staff-level insight: The pyramid is a guideline, not a rule. Some features (navigation, gestures) need more UI tests. Some features (pure logic) need only unit tests. The key metric is: confidence-per-minute-of-CI-time.
