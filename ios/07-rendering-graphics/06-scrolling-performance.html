1. The Frame Budget

Frame Budget During Scroll: ═══════════════════════════ 60Hz Display: 16.67ms per frame 120Hz Display: 8.33ms per frame Within each frame during scroll: ┌──────────────────────────────────────────────────────────────┐ │ │ │ ┌───────────┐ ┌──────────┐ ┌────────┐ ┌──────────────┐│ │ │ Cell │ │ Layout │ │ Draw │ │ Commit to ││ │ │ Dequeue / │ │ Subviews │ │ (if │ │ Render ││ │ │ Configure │ │ │ │ needed)│ │ Server ││ │ │ │ │ │ │ │ │ ││ │ │ ~2-4ms │ │ ~1-2ms │ │ ~0-2ms │ │ ~1-2ms ││ │ └───────────┘ └──────────┘ └────────┘ └──────────────┘│ │ │ │ Total app work: ~4-10ms │ │ Remaining for render server: ~6-12ms │ │ If app takes > 10ms → FRAME DROP │ └──────────────────────────────────────────────────────────────┘

During scrolling, the system must dequeue a cell, configure it with data, lay it out, and commit the layer tree — all within the frame budget. Any work that exceeds this budget causes a visible hitch (jank).

2. The 60fps Scroll Checklist

#CheckImpactPriority
1 Images decoded on background thread Eliminates 5-50ms main thread decode Critical
2 Images downsampled to display size Reduces memory 10-100x, faster decode Critical

4 shadowPath set for all shadows Eliminates shadow outline computation High

3. Cell Preparation Cost Reduction

Expensive Cell Configuration (Before)

// ❌ Expensive: doing too much work in cellForItemAt
func collectionView(
    _ collectionView: UICollectionView,
    cellForItemAt indexPath: IndexPath
) -> UICollectionViewCell {
    let cell = collectionView.dequeueReusableCell(
        withReuseIdentifier: "Cell", for: indexPath
    ) as! PostCell

    let post = posts[indexPath.item]

    // ❌ Decoding full-resolution image on main thread
    cell.imageView.image = UIImage(contentsOfFile: post.imagePath)

    // ❌ Creating attributed string every time
    let attributed = NSAttributedString(
        string: post.caption,
        attributes: [.font: UIFont.systemFont(ofSize: 14)]
    )
    cell.captionLabel.attributedText = attributed

    // ❌ Date formatting every time
    let formatter = DateFormatter()
    formatter.dateStyle = .medium
    cell.dateLabel.text = formatter.string(from: post.date)

    // ❌ Corner radius + masksToBounds on image
    cell.imageView.layer.cornerRadius = 8
    cell.imageView.clipsToBounds = true

    return cell
}

Optimized Cell Configuration (After)

// ✅ Optimized: minimal work in cellForItemAt

// Pre-computed view models with cached formatted strings
struct PostViewModel {
    let imageURL: URL
    let caption: NSAttributedString  // Pre-computed
    let dateString: String            // Pre-formatted
    let cellHeight: CGFloat           // Pre-calculated
}

class OptimizedFeedController: UICollectionViewController {
    private var viewModels: [PostViewModel] = []
    private let imagePipeline = ImagePipeline.shared
    private static let dateFormatter: DateFormatter = {
        let f = DateFormatter()
        f.dateStyle = .medium
        return f
    }()

    func prepareViewModels(from posts: [Post]) {
        DispatchQueue.global(qos: .userInitiated).async { [weak self] in
            let vms = posts.map { post -> PostViewModel in
                let caption = NSAttributedString(
                    string: post.caption,
                    attributes: [.font: UIFont.systemFont(ofSize: 14)]
                )
                return PostViewModel(
                    imageURL: post.imageURL,
                    caption: caption,
                    dateString: Self.dateFormatter.string(from: post.date),
                    cellHeight: Self.calculateHeight(for: caption, width: 375)
                )
            }
            DispatchQueue.main.async {
                self?.viewModels = vms
                self?.collectionView.reloadData()
            }
        }
    }

    override func collectionView(
        _ collectionView: UICollectionView,
        cellForItemAt indexPath: IndexPath
    ) -> UICollectionViewCell {
        let cell = collectionView.dequeueReusableCell(
            withReuseIdentifier: "Cell", for: indexPath
        ) as! PostCell

        let vm = viewModels[indexPath.item]

        // ✅ Pre-computed attributed string
        cell.captionLabel.attributedText = vm.caption
        cell.dateLabel.text = vm.dateString

        // ✅ Async image loading with downsampling
        let targetSize = cell.imageView.bounds.size
        imagePipeline.loadImage(url: vm.imageURL, targetSize: targetSize) { image in
            cell.imageView.image = image
        }

        return cell
    }
}

4. Image Decoding on Background Thread

class AsyncImageView: UIImageView {
    private var currentURL: URL?
    private static let decodingQueue = DispatchQueue(
        label: "com.app.image.decoding",
        qos: .userInitiated,
        attributes: .concurrent
    )

    func loadImage(from url: URL, targetSize: CGSize) {
        currentURL = url
        image = nil

        Self.decodingQueue.async { [weak self] in
            guard let data = try? Data(contentsOf: url) else { return }

            // Downsample: only decode pixels we need
            let options: [CFString: Any] = [
                kCGImageSourceShouldCache: false
            ]
            guard let source = CGImageSourceCreateWithData(
                data as CFData, options as CFDictionary
            ) else { return }

            let maxPixel = max(targetSize.width, targetSize.height) *
                           UIScreen.main.scale
            let thumbOptions: [CFString: Any] = [
                kCGImageSourceCreateThumbnailFromImageAlways: true,
                kCGImageSourceShouldCacheImmediately: true,
                kCGImageSourceCreateThumbnailWithTransform: true,
                kCGImageSourceThumbnailMaxPixelSize: maxPixel
            ]

            guard let cgImage = CGImageSourceCreateThumbnailAtIndex(
                source, 0, thumbOptions as CFDictionary
            ) else { return }

            let decoded = UIImage(cgImage: cgImage)

            DispatchQueue.main.async {
                guard self?.currentURL == url else { return }
                self?.image = decoded
            }
        }
    }

    func cancelLoading() {
        currentURL = nil
    }
}

5. Prefetching API

class FeedController: UICollectionViewController,
                       UICollectionViewDataSourcePrefetching {

    private let imagePipeline = ImagePipeline.shared
    private var prefetchTasks: [IndexPath: URLSessionDataTask] = [:]

    override func viewDidLoad() {
        super.viewDidLoad()
        collectionView.prefetchDataSource = self
    }

    // Called when cells are about to come into view
    func collectionView(
        _ collectionView: UICollectionView,
        prefetchItemsAt indexPaths: [IndexPath]
    ) {
        for indexPath in indexPaths {
            let post = posts[indexPath.item]
            // Start loading image before cell is visible
            imagePipeline.prefetch(url: post.imageURL)
        }
    }

    // Called when prefetched cells are no longer needed
    func collectionView(
        _ collectionView: UICollectionView,
        cancelPrefetchingForItemsAt indexPaths: [IndexPath]
    ) {
        for indexPath in indexPaths {
            let post = posts[indexPath.item]
            imagePipeline.cancelPrefetch(url: post.imageURL)
        }
    }
}
How Prefetching Works Internally

UICollectionView predicts which cells will become visible based on scroll velocity and direction. It calls prefetchItemsAt for cells that are ~1-2 screens away. If the user changes scroll direction, it calls cancelPrefetchingForItemsAt for cells that are no longer approaching the viewport. This gives you a window to start network requests and image decoding before the cell is needed.

6. shouldRasterize for Complex Cells

class ComplexCell: UICollectionViewCell {
    override init(frame: CGRect) {
        super.init(frame: frame)

        // For cells with many sublayers that don't change often:
        // - Multiple labels, images, badges, separators
        // - Shadow effects
        // - Gradient backgrounds
        layer.shouldRasterize = true
        layer.rasterizationScale = UIScreen.main.scale
    }

    override func prepareForReuse() {
        super.prepareForReuse()
        // Temporarily disable rasterization during configuration
        // to avoid rebuilding the cache for every property change
        layer.shouldRasterize = false
    }

    func configure(with viewModel: CellViewModel) {
        // ... set all properties ...

        // Re-enable after configuration is complete
        // Cache will be built once on next render
        layer.shouldRasterize = true
    }

    required init?(coder: NSCoder) {
        fatalError("init(coder:) has not been implemented")
    }
}
When shouldRasterize Hurts

If the cell content changes frequently (e.g., a progress bar, countdown timer, or live-updating content), shouldRasterize forces the cache to rebuild every frame — making performance worse. Use Instruments' "Color Hits Green and Misses Red" to verify: green means the cache is being reused (good), red means it's being rebuilt (bad).

7. Blended Layers Detection

Blended layers occur when the GPU must read the destination pixel and blend it with the source pixel (alpha compositing). This happens when a layer is not fully opaque.

Opaque Layer (Fast): ════════════════════ Source pixel ──▶ Framebuffer (overwrite) GPU just writes the pixel. Done. Blended Layer (Slow): ═════════════════════ Source pixel ──┐ ├──▶ Blend equation ──▶ Framebuffer Dest pixel ◀──┘ (read + compute + write) GPU must READ existing pixel, COMPUTE blend, WRITE result. 
// ✅ Eliminate blending: set opaque backgrounds
label.backgroundColor = .systemBackground  // Opaque
label.isOpaque = true

// ❌ Causes blending: clear or transparent background
label.backgroundColor = .clear  // Forces blending

// ✅ For labels on colored backgrounds, match the background
let cellBackground = UIColor.systemBackground
label.backgroundColor = cellBackground

// ✅ For UIImageView, set opaque if image has no transparency
imageView.isOpaque = true
imageView.backgroundColor = .systemBackground

8. Misaligned Images

Misaligned images occur when an image's pixel grid doesn't align with the screen's pixel grid. The GPU must perform anti-aliasing interpolation, which is expensive.

// ❌ Misaligned: fractional point values
imageView.frame = CGRect(x: 10.3, y: 20.7, width: 50.5, height: 50.5)

// ✅ Pixel-aligned: snap to pixel boundaries
extension CGRect {
    var pixelAligned: CGRect {
        let scale = UIScreen.main.scale
        return CGRect(
            x: round(origin.x * scale) / scale,
            y: round(origin.y * scale) / scale,
            width: round(size.width * scale) / scale,
            height: round(size.height * scale) / scale
        )
    }
}

imageView.frame = CGRect(x: 10.3, y: 20.7, width: 50.5, height: 50.5).pixelAligned

// ❌ Misaligned: image size doesn't match view size
// A 200×200 image in a 100×100 @3x view = 200px image in 300px view
// GPU must resample every frame

// ✅ Downsample image to exact display pixel size
let displayPixelSize = CGSize(
    width: imageView.bounds.width * UIScreen.main.scale,
    height: imageView.bounds.height * UIScreen.main.scale
)

9. Compositing Cost

Every visible layer adds to the compositing cost. The GPU must process layers back-to-front, applying transforms, blending, and effects for each one.

FactorImpactMitigation
Layer count per cell Linear — more layers = more GPU work Flatten hierarchy; use shouldRasterize
Layer size Larger layers = more pixels to composite Avoid oversized layers; clip to visible area
Transparency Each transparent layer requires read-blend-write Set isOpaque = true; use opaque backgrounds
Off-screen passes Each pass requires context switch + temp buffer Eliminate triggers; use shadowPath
Overdraw Overlapping opaque layers waste GPU cycles Minimize overlapping views 
// Measure layer count for a cell
func countLayers(in view: UIView) -> Int {
    var count = 1  // This view's layer
    count += view.layer.sublayers?.count ?? 0
    for subview in view.subviews {
        count += countLayers(in: subview)
    }
    return count
}

// Complex cells can easily reach 100+ layers

}

11. Systematic Jank Diagnosis

Architect Must-Know: Systematic Scroll Jank Diagnosis

At Staff level, you need a repeatable methodology for diagnosing and fixing scroll performance. Here's the systematic approach:

Phase 1: Measure

  1. Profile on the oldest supported device (not the latest iPhone Pro)
  2. Use Instruments Core Animation template to measure baseline FPS
  3. Record a 30-second scroll session through the problematic screen
  4. Note frame drops: how many, how severe (1 frame vs 5 frames), and when they occur

Phase 2: Identify

  1. Enable all Core Animation debug overlays one at a time
  2. Use Time Profiler to identify CPU bottlenecks in cellForItemAt
  3. Check GPU utilization — if high, the problem is compositing; if low, it's CPU
  4. Categorize issues: CPU-bound (cell configuration) vs GPU-bound (compositing)

Phase 3: Fix (Priority Order)

PriorityFixExpected Impact
1 Move image decoding off main thread Eliminates largest single hitch source
2 Downsample images to display size Reduces decode time + memory pressure
3 Add shadowPath to all shadows Eliminates expensive GPU outline tracing
4 Set opaque backgrounds Eliminates blending cost
5 Pre-compute cell heights Eliminates layout computation during scroll
6 Enable prefetching Hides data loading latency
7 Consider shouldRasterize for complex cells Reduces per-frame compositing cost

Phase 4: Verify & Prevent

  1. Re-measure with Instruments to confirm improvement
  2. Set up MetricKit monitoring for scroll hitch rate in production
  3. Add performance assertions in debug builds
  4. Document team guidelines for cell implementation
// Debug-only performance assertion for cell configuration time
#if DEBUG
func assertCellConfigTime(_ block: () -> Void) {
    let start = CACurrentMediaTime()
    block()
    let elapsed = (CACurrentMediaTime() - start) * 1000
    assert(elapsed < 4.0,
           "Cell configuration took \(elapsed)ms — exceeds 4ms budget")
}
#endif
Best Practice Summary
  • Always profile on the oldest supported device — that's where users experience jank
  • Move all heavy work (image decode, text measurement, date formatting) off the main thread
  • Pre-compute everything possible: cell heights, attributed strings, formatted dates
  • Use prefetching to start loading data before cells become visible
  • Eliminate every off-screen rendering trigger in scroll view cells
  • Set opaque backgrounds on all cell subviews to eliminate blending
  • Monitor scroll hitch rate in production with MetricKit
  • Establish team coding standards for cell implementation performance
