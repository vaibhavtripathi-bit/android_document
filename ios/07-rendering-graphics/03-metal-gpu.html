1. Metal Architecture Overview

Metal is Apple's low-level GPU programming framework, providing near-direct access to the GPU hardware. It replaces OpenGL ES (deprecated since iOS 12) and serves as the foundation for Core Animation compositing, Core Image filtering, and all GPU work on Apple platforms.

2. Device & MTKView Setup

import MetalKit

class MetalViewController: UIViewController {
    var metalView: MTKView!
    var device: MTLDevice!
    var commandQueue: MTLCommandQueue!

    override func viewDidLoad() {
        super.viewDidLoad()

        // 1. Get the default GPU device
        guard let device = MTLCreateSystemDefaultDevice() else {
            fatalError("Metal is not supported on this device")
        }
        self.device = device

        // 2. Create command queue (one per app, typically)
        guard let queue = device.makeCommandQueue() else {
            fatalError("Failed to create command queue")
        }
        self.commandQueue = queue

        // 3. Configure MTKView
        metalView = MTKView(frame: view.bounds, device: device)
        metalView.delegate = self
        metalView.preferredFramesPerSecond = 60
        metalView.colorPixelFormat = .bgra8Unorm
        metalView.clearColor = MTLClearColor(
            red: 0.0, green: 0.0, blue: 0.0, alpha: 1.0
        )

        // For on-demand rendering (not continuous)
        // metalView.isPaused = true
        // metalView.enableSetNeedsDisplay = true

        view.addSubview(metalView)
    }
}

extension MetalViewController: MTKViewDelegate {
    func mtkView(_ view: MTKView, drawableSizeWillChange size: CGSize) {
        // Handle resize — update projection matrices, etc.
    }

    func draw(in view: MTKView) {
        guard let drawable = view.currentDrawable,
              let descriptor = view.currentRenderPassDescriptor,
              let commandBuffer = commandQueue.makeCommandBuffer(),
              let encoder = commandBuffer.makeRenderCommandEncoder(
                  descriptor: descriptor
              ) else { return }

        // Encode draw calls here
        encoder.endEncoding()

        commandBuffer.present(drawable)
        commandBuffer.commit()
    }
}
Deep Dive: Unified Memory Architecture

Apple Silicon uses unified memory — the CPU and GPU share the same physical memory. This means MTLBuffer and MTLTexture data doesn't need to be copied between CPU and GPU memory (unlike discrete GPUs). You can create buffers with .storageModeShared for zero-copy access from both CPU and GPU. This is a significant performance advantage for iOS/Apple Silicon compared to traditional GPU architectures.

3. Command Model: Buffers, Queues, Encoders

Command Submission Pipeline: ════════════════════════════ MTLDevice │ ├──▶ MTLCommandQueue (serial, one per app) │ │ │ ├──▶ MTLCommandBuffer #1 │ │ ├──▶ MTLRenderCommandEncoder │ │ │ ├─ setRenderPipelineState() │ │ │ ├─ setVertexBuffer() │ │ │ ├─ drawPrimitives() │ │ │ └─ endEncoding() │ │ ├──▶ MTLComputeCommandEncoder │ │ │ ├─ setComputePipelineState() │ │ │ ├─ setTexture() │ │ │ ├─ dispatchThreadgroups() │ │ │ └─ endEncoding() │ │ ├──▶ MTLBlitCommandEncoder │ │ │ ├─ copy(from:to:) │ │ │ └─ endEncoding() │ │ │ │ │ ├─ present(drawable) │ │ └─ commit() ──▶ GPU executes │ │ │ └──▶ MTLCommandBuffer #2 ... │ ├──▶ MTLLibrary (compiled shaders) ├──▶ MTLBuffer (vertex data, uniforms) └──▶ MTLTexture (images, render targets)

Triple Buffering Pattern

class TripleBufferedRenderer {
    static let maxFramesInFlight = 3
    let frameSemaphore = DispatchSemaphore(value: maxFramesInFlight)
    var uniformBuffers: [MTLBuffer] = []
    var currentBufferIndex = 0

    func setup(device: MTLDevice) {
        for _ in 0..<TripleBufferedRenderer.maxFramesInFlight {
            let buffer = device.makeBuffer(
                length: MemoryLayout<Uniforms>.stride,
                options: .storageModeShared
            )!
            uniformBuffers.append(buffer)
        }
    }

    func draw(in view: MTKView) {
        frameSemaphore.wait()

        let bufferIndex = currentBufferIndex % TripleBufferedRenderer.maxFramesInFlight
        let uniformBuffer = uniformBuffers[bufferIndex]

        // Update uniforms for this frame
        let uniforms = uniformBuffer.contents().bindMemory(
            to: Uniforms.self, capacity: 1
        )
        uniforms.pointee.modelViewProjection = calculateMVP()

        guard let commandBuffer = commandQueue.makeCommandBuffer() else { return }

        commandBuffer.addCompletedHandler { [weak self] _ in
            self?.frameSemaphore.signal()
        }

        // Encode and submit...
        currentBufferIndex += 1
    }
}

4. Render Pipeline

func buildRenderPipeline(device: MTLDevice, view: MTKView) -> MTLRenderPipelineState {
    let library = device.makeDefaultLibrary()!

    let descriptor = MTLRenderPipelineDescriptor()
    descriptor.vertexFunction = library.makeFunction(name: "vertexShader")
    descriptor.fragmentFunction = library.makeFunction(name: "fragmentShader")
    descriptor.colorAttachments[0].pixelFormat = view.colorPixelFormat

    // Enable alpha blending
    descriptor.colorAttachments[0].isBlendingEnabled = true
    descriptor.colorAttachments[0].sourceRGBBlendFactor = .sourceAlpha
    descriptor.colorAttachments[0].destinationRGBBlendFactor = .oneMinusSourceAlpha

    // Vertex descriptor for vertex buffer layout
    let vertexDescriptor = MTLVertexDescriptor()
    vertexDescriptor.attributes[0].format = .float3  // position
    vertexDescriptor.attributes[0].offset = 0
    vertexDescriptor.attributes[0].bufferIndex = 0
    vertexDescriptor.attributes[1].format = .float2  // texCoord
    vertexDescriptor.attributes[1].offset = MemoryLayout<SIMD3<Float>>.stride
    vertexDescriptor.attributes[1].bufferIndex = 0
    vertexDescriptor.layouts[0].stride = MemoryLayout<Vertex>.stride
    descriptor.vertexDescriptor = vertexDescriptor

    return try! device.makeRenderPipelineState(descriptor: descriptor)
}

5. Metal Shading Language Basics

Metal shaders are written in Metal Shading Language (MSL), a C++14-based language compiled at build time into GPU bytecode.

// Shaders.metal — compiled at build time by Xcode

#include <metal_stdlib>
using namespace metal;

struct VertexIn {
    float3 position [[attribute(0)]];
    float2 texCoord [[attribute(1)]];
};

struct VertexOut {
    float4 position [[position]];
    float2 texCoord;
};

struct Uniforms {
    float4x4 modelViewProjection;
};

// Vertex shader: runs once per vertex
vertex VertexOut vertexShader(
    VertexIn in [[stage_in]],
    constant Uniforms &uniforms [[buffer(1)]]
) {
    VertexOut out;
    out.position = uniforms.modelViewProjection * float4(in.position, 1.0);
    out.texCoord = in.texCoord;
    return out;
}

// Fragment shader: runs once per pixel
fragment float4 fragmentShader(
    VertexOut in [[stage_in]],
    texture2d<float> texture [[texture(0)]],
    sampler textureSampler [[sampler(0)]]
) {
    return texture.sample(textureSampler, in.texCoord);
}

6. Compute Shaders for Image Processing

Metal compute shaders run massively parallel operations on the GPU, ideal for image processing, physics simulations, and machine learning inference.

// Compute shader: grayscale conversion
// Runs one thread per pixel — thousands of pixels processed simultaneously

kernel void grayscaleKernel(
    texture2d<float, access::read> inTexture [[texture(0)]],
    texture2d<float, access::write> outTexture [[texture(1)]],
    uint2 gid [[thread_position_in_grid]]
) {
    if (gid.x >= inTexture.get_width() || gid.y >= inTexture.get_height()) {
        return;
    }

    float4 color = inTexture.read(gid);
    float gray = dot(color.rgb, float3(0.2126, 0.7152, 0.0722));
    outTexture.write(float4(gray, gray, gray, color.a), gid);
}

Dispatching Compute Work from Swift

func applyGrayscale(to texture: MTLTexture) -> MTLTexture {
    let library = device.makeDefaultLibrary()!
    let kernel = library.makeFunction(name: "grayscaleKernel")!
    let pipelineState = try! device.makeComputePipelineState(function: kernel)

    // Create output texture
    let descriptor = MTLTextureDescriptor.texture2DDescriptor(
        pixelFormat: texture.pixelFormat,
        width: texture.width,
        height: texture.height,
        mipmapped: false
    )
    descriptor.usage = [.shaderRead, .shaderWrite]
    let outputTexture = device.makeTexture(descriptor: descriptor)!

    let commandBuffer = commandQueue.makeCommandBuffer()!
    let encoder = commandBuffer.makeComputeCommandEncoder()!

    encoder.setComputePipelineState(pipelineState)
    encoder.setTexture(texture, index: 0)
    encoder.setTexture(outputTexture, index: 1)

    // Calculate threadgroup sizes
    let threadgroupSize = MTLSize(
        width: min(pipelineState.threadExecutionWidth, texture.width),
        height: min(pipelineState.maxTotalThreadsPerThreadgroup /
                    pipelineState.threadExecutionWidth, texture.height),
        depth: 1
    )
    let threadgroups = MTLSize(
        width: (texture.width + threadgroupSize.width - 1) / threadgroupSize.width,
        height: (texture.height + threadgroupSize.height - 1) / threadgroupSize.height,
        depth: 1
    )

    encoder.dispatchThreadgroups(threadgroups,
                                  threadsPerThreadgroup: threadgroupSize)
    encoder.endEncoding()
    commandBuffer.commit()
    commandBuffer.waitUntilCompleted()

    return outputTexture
}

7. Metal Performance Shaders (MPS)

MPS provides highly optimized GPU kernels for common operations: image processing, matrix math, neural networks, and ray tracing. Always prefer MPS over custom shaders for standard operations.

import MetalPerformanceShaders

// Gaussian blur using MPS
func blurImage(_ texture: MTLTexture, sigma: Float) -> MTLTexture {
    let blur = MPSImageGaussianBlur(device: device, sigma: sigma)

    let descriptor = MTLTextureDescriptor.texture2DDescriptor(
        pixelFormat: texture.pixelFormat,
        width: texture.width,
        height: texture.height,
        mipmapped: false
    )
    descriptor.usage = [.shaderRead, .shaderWrite]
    let output = device.makeTexture(descriptor: descriptor)!

    let commandBuffer = commandQueue.makeCommandBuffer()!
    blur.encode(commandBuffer: commandBuffer,
               sourceTexture: texture,
               destinationTexture: output)
    commandBuffer.commit()
    commandBuffer.waitUntilCompleted()

    return output
}

// Image scaling with Lanczos resampling
let scale = MPSImageLanczosScale(device: device)

// Histogram computation
let histogram = MPSImageHistogram(device: device,
                                   histogramInfo: &histogramInfo)

// Neural network inference
let convolution = MPSCNNConvolution(device: device, weights: dataSource)

8. Core Image with Metal Backend

Core Image is a high-level image processing framework that uses Metal as its GPU backend. It provides a graph-based filter pipeline with lazy evaluation and automatic optimization.

import CoreImage

class CoreImageProcessor {
    let context: CIContext

    init(device: MTLDevice) {
        // Create CIContext backed by Metal for GPU acceleration
        context = CIContext(mtlDevice: device, options: [
            .workingColorSpace: CGColorSpace(name: CGColorSpace.sRGB)!,
            .cacheIntermediates: false  // Reduce memory for one-shot processing
        ])
    }

    func applyChain(to image: CIImage) -> CIImage {
        // Core Image builds a filter graph — evaluated lazily
        var output = image

        // Bloom effect chain
        output = output.applyingFilter("CIBloom", parameters: [
            kCIInputRadiusKey: 10.0,
            kCIInputIntensityKey: 0.5
        ])

        // Color adjustment
        output = output.applyingFilter("CIColorControls", parameters: [
            kCIInputSaturationKey: 1.2,
            kCIInputContrastKey: 1.1
        ])

        // Sharpen
        output = output.applyingFilter("CISharpenLuminance", parameters: [
            kCIInputSharpnessKey: 0.4
        ])

        return output
    }

    func render(_ ciImage: CIImage, to texture: MTLTexture,
               commandBuffer: MTLCommandBuffer) {
        let bounds = CGRect(x: 0, y: 0,
                            width: texture.width, height: texture.height)
        let colorSpace = CGColorSpaceCreateDeviceRGB()

        context.render(ciImage,
                      to: texture,
                      commandBuffer: commandBuffer,
                      bounds: bounds,
                      colorSpace: colorSpace)
    }
}

9. GPU Frame Capture in Xcode

Xcode's GPU Frame Capture lets you inspect every GPU command, texture, and shader execution for a single frame. Essential for debugging rendering issues and optimizing GPU performance.

How to Capture

  1. Run your app from Xcode on a device or simulator
  2. Click the camera icon in the debug bar (or Debug → Capture GPU Frame)
  3. Xcode captures the next frame and opens the GPU debugger

What You Can Inspect

FeatureWhat It Shows
Command Buffer Timeline Every GPU command in execution order
Shader Debugger Step through vertex/fragment/compute shaders line by line
Texture Viewer Inspect every texture at every pipeline stage
Memory Viewer GPU memory allocation and usage
Performance Counters GPU utilization, bandwidth, occupancy
Dependency Viewer Resource dependencies between render passes

11. MetalFX Upscaling

MetalFX (iOS 16+) provides GPU-accelerated upscaling — render at lower resolution and upscale to native resolution with minimal quality loss. Useful for games and GPU-intensive apps.

import MetalFX

func setupMetalFXUpscaling(device: MTLDevice) -> MTLFXSpatialScaler? {
    let descriptor = MTLFXSpatialScalerDescriptor()
    descriptor.inputWidth = 960   // Render at half resolution
    descriptor.inputHeight = 540
    descriptor.outputWidth = 1920  // Upscale to full resolution
    descriptor.outputHeight = 1080
    descriptor.colorTextureFormat = .bgra8Unorm
    descriptor.outputTextureFormat = .bgra8Unorm
    descriptor.colorProcessingMode = .perceptual

    return descriptor.makeSpatialScaler(device: device)
}

12. When to Drop Down to Metal

Architect Must-Know: When Metal Is Justified

Most iOS apps never need direct Metal code. Here's the decision framework:

Use CaseRecommended ApproachWhy
Standard UI UIKit / SwiftUI + Core Animation Already Metal-backed; no benefit from going lower
Image filters Core Image 200+ built-in filters, automatic GPU optimization
Standard blur/resize MPS Apple-optimized for each GPU generation
Custom real-time effects Metal compute shaders Core Image can't express your algorithm
3D rendering SceneKit / RealityKit, then Metal Higher-level APIs handle 90% of cases
Games Metal directly Need full control over render pipeline
ML inference Core ML → MPS → Metal Core ML handles most models; MPS for custom ops
Video processing AVFoundation + Core Image, then Metal AVFoundation handles decode; CI for filters

Rule of thumb: Use the highest-level API that meets your requirements. Drop to Metal only when you've exhausted higher-level options or need guaranteed frame-level control.

Best Practice Summary
  • Metal is the foundation of all GPU work on Apple platforms — even Core Animation uses it
  • Prefer MPS for standard operations (blur, resize, ML) over custom shaders
  • Use Core Image for filter chains — it optimizes the graph automatically
  • Create MTLCommandQueue once and reuse — it's expensive to create
  • Use triple buffering to overlap CPU and GPU work
  • Profile with GPU Frame Capture before optimizing
  • Apple Silicon's unified memory means zero-copy CPU↔GPU data sharing
