1. Keychain Services Architecture

The iOS Keychain is an encrypted SQLite database managed by the securityd daemon, a system-level process running outside your app's sandbox. Unlike UserDefaults, keychain data is encrypted at rest using hardware-backed keys derived from the device passcode and the Secure Enclave.

Your app never directly reads or writes the keychain database. Instead, it communicates through the Security framework, which marshals requests to securityd via XPC (inter-process communication).

┌─────────────────────────────────────────────────────────┐ │ Your Application │ │ │ │ ┌──────────────┐ ┌──────────────────────────────┐ │ │ │ SecItemAdd() │ │ SecItemCopyMatching() │ │ │ │ SecItemUpdate│ │ SecItemDelete() │ │ │ └──────┬───────┘ └──────────────┬───────────────┘ │ │ │ │ │ │ └───────────┬───────────────┘ │ │ ▼ │ │ ┌───────────────────────┐ │ │ │ Security.framework │ │ │ └───────────┬───────────┘ │ └──────────────────────┼──────────────────────────────────┘ │ XPC (inter-process communication) ▼ ┌────────────────────────┐ │ securityd daemon │ │ (system-level process │ │ outside app sandbox) │ └────────────┬───────────┘ │ ┌────────────▼───────────┐ │ Keychain Database │ │ (encrypted SQLite) │ │ │ │ Protected by: │ │ • Device passcode │ │ • Secure Enclave key │ │ • Data Protection │ └────────────────────────┘

Keychain Item Classes

Every keychain item belongs to exactly one class, specified by the kSecClass key in the query dictionary:

Class Constant Purpose Typical Use
kSecClassGenericPassword Generic password storage Auth tokens, API keys, user credentials
kSecClassInternetPassword Internet password with server/protocol metadata Website logins, HTTP auth credentials
kSecClassCertificate X.509 certificates SSL/TLS client certificates
kSecClassKey Cryptographic keys RSA/EC keys for signing or encryption
kSecClassIdentity Certificate + private key pair Client certificate authentication
Deep Dive — Keychain Uniqueness

For kSecClassGenericPassword, uniqueness is determined by the combination of kSecAttrService and kSecAttrAccount. For kSecClassInternetPassword, uniqueness includes kSecAttrServer, kSecAttrProtocol, kSecAttrPort, kSecAttrPath, and kSecAttrAccount. Attempting to add a duplicate item returns errSecDuplicateItem (-25299).

2. CRUD Operations

All keychain operations use query dictionaries — [String: Any] dictionaries cast to CFDictionary. Each operation returns an OSStatus code indicating success or failure.

Add — SecItemAdd

import Security

func saveToKeychain(
    service: String,
    account: String,
    data: Data
) throws {
    let query: [String: Any] = [
        kSecClass as String:           kSecClassGenericPassword,
        kSecAttrService as String:     service,
        kSecAttrAccount as String:     account,
        kSecValueData as String:       data,
        kSecAttrAccessible as String:  kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly
    ]

    let status = SecItemAdd(query as CFDictionary, nil)

    guard status == errSecSuccess else {
        if status == errSecDuplicateItem {
            // Item already exists — update instead
            try updateInKeychain(service: service, account: account, data: data)
            return
        }
        throw KeychainError.unhandled(status: status)
    }
}

Read — SecItemCopyMatching

func readFromKeychain(
    service: String,
    account: String
) throws -> Data {
    let query: [String: Any] = [
        kSecClass as String:           kSecClassGenericPassword,
        kSecAttrService as String:     service,
        kSecAttrAccount as String:     account,
        kSecReturnData as String:      true,
        kSecMatchLimit as String:      kSecMatchLimitOne
    ]

    var result: AnyObject?
    let status = SecItemCopyMatching(query as CFDictionary, &result)

    guard status == errSecSuccess,
          let data = result as? Data else {
        if status == errSecItemNotFound {
            throw KeychainError.itemNotFound
        }
        throw KeychainError.unhandled(status: status)
    }

    return data
}

Update — SecItemUpdate

func updateInKeychain(
    service: String,
    account: String,
    data: Data
) throws {
    let query: [String: Any] = [
        kSecClass as String:       kSecClassGenericPassword,
        kSecAttrService as String: service,
        kSecAttrAccount as String: account
    ]

    let attributes: [String: Any] = [
        kSecValueData as String: data
    ]

    let status = SecItemUpdate(
        query as CFDictionary,
        attributes as CFDictionary
    )

    guard status == errSecSuccess else {
        throw KeychainError.unhandled(status: status)
    }
}

Delete — SecItemDelete

func deleteFromKeychain(
    service: String,
    account: String
) throws {
    let query: [String: Any] = [
        kSecClass as String:       kSecClassGenericPassword,
        kSecAttrService as String: service,
        kSecAttrAccount as String: account
    ]

    let status = SecItemDelete(query as CFDictionary)

    guard status == errSecSuccess || status == errSecItemNotFound else {
        throw KeychainError.unhandled(status: status)
    }
}

Error Handling

enum KeychainError: Error, LocalizedError {
    case itemNotFound
    case duplicateItem
    case authFailed
    case unhandled(status: OSStatus)

    var errorDescription: String? {
        switch self {
        case .itemNotFound:
            return "Keychain item not found"
        case .duplicateItem:
            return "Keychain item already exists"
        case .authFailed:
            return "Authentication failed"
        case .unhandled(let status):
            let msg = SecCopyErrorMessageString(status, nil) as String? ?? "Unknown"
            return "Keychain error \(status): \(msg)"
        }
    }
}
Pitfall — SecItemUpdate Query Must Not Include kSecValueData

A common mistake is including kSecValueData in the query dictionary of SecItemUpdate. The query identifies which item to update; the attributes dictionary specifies the new values. Including value data in the query causes errSecItemNotFound because no existing item matches a query containing the new data.

3. kSecAttrAccessible Options

The kSecAttrAccessible attribute controls when a keychain item's data is decryptable. Choosing the right level is critical for balancing security and usability.

Constant Available When Backup Migrates? Best For
kSecAttrAccessibleWhenUnlocked Device unlocked Yes General credentials
kSecAttrAccessibleWhenUnlockedThisDeviceOnly Device unlocked No Session tokens, device-bound secrets
kSecAttrAccessibleAfterFirstUnlock After first unlock (even if locked) Yes Background fetch tokens, push certs
kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly After first unlock (even if locked) No Background tasks needing device-bound tokens
kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly Device unlocked + passcode set No Highest security — deleted if passcode removed
Deep Dive — "ThisDeviceOnly" Implications

Items with ThisDeviceOnly are encrypted with a key tied to the specific hardware. They are excluded from iCloud and iTunes backups, meaning they will not transfer when a user migrates to a new device. This is desirable for device-bound tokens (e.g., push notification tokens, device attestation keys) but problematic for credentials the user expects to carry over. Always consider the user experience when choosing this option.

Best Practice — Default Accessibility

Use kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly as your default for auth tokens and API keys. It allows background operations (network refreshes, silent push handling) while keeping data device-bound. Only use WhenUnlocked variants if you truly need to restrict access to foreground-only scenarios.

Pitfall — WhenPasscodeSet and Passcode Removal

If you use kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly and the user removes their device passcode, all items with this accessibility are permanently deleted. Your app must handle this gracefully — detect the missing item on next access and prompt the user to re-authenticate.

4. Biometric Protection

Beyond accessibility timing, you can require biometric authentication (Face ID / Touch ID) or device passcode before a keychain item is decrypted. This uses SecAccessControl combined with LAContext from the LocalAuthentication framework.

SecAccessControl Flags

Flag Behavior
.biometryAny Any enrolled biometric (including newly added fingerprints/faces)
.biometryCurrentSet Only the biometric set enrolled at time of item creation
.devicePasscode Device passcode only
.userPresence Biometric OR passcode (most flexible)
.or Combine flags with OR logic
.and Combine flags with AND logic

Complete Biometric-Protected Keychain Item

import Security
import LocalAuthentication

func saveBiometricProtected(
    service: String,
    account: String,
    data: Data
) throws {
    var error: Unmanaged<CFError>?

    guard let accessControl = SecAccessControlCreateWithFlags(
        kCFAllocatorDefault,
        kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly,
        [.biometryCurrentSet, .or, .devicePasscode],
        &error
    ) else {
        throw error!.takeRetainedValue() as Error
    }

    let query: [String: Any] = [
        kSecClass as String:            kSecClassGenericPassword,
        kSecAttrService as String:      service,
        kSecAttrAccount as String:      account,
        kSecValueData as String:        data,
        kSecAttrAccessControl as String: accessControl
    ]

    let status = SecItemAdd(query as CFDictionary, nil)
    guard status == errSecSuccess else {
        throw KeychainError.unhandled(status: status)
    }
}

Reading with LAContext

func readBiometricProtected(
    service: String,
    account: String,
    reason: String
) throws -> Data {
    let context = LAContext()
    context.localizedReason = reason
    context.localizedCancelTitle = "Use Password"

    let query: [String: Any] = [
        kSecClass as String:            kSecClassGenericPassword,
        kSecAttrService as String:      service,
        kSecAttrAccount as String:      account,
        kSecReturnData as String:       true,
        kSecMatchLimit as String:       kSecMatchLimitOne,
        kSecUseAuthenticationContext as String: context
    ]

    var result: AnyObject?
    let status = SecItemCopyMatching(query as CFDictionary, &result)

    guard status == errSecSuccess,
          let data = result as? Data else {
        switch status {
        case errSecItemNotFound:
            throw KeychainError.itemNotFound
        case errSecUserCanceled, errSecAuthFailed:
            throw KeychainError.authFailed
        default:
            throw KeychainError.unhandled(status: status)
        }
    }

    return data
}
Pitfall — biometryCurrentSet vs biometryAny

Using .biometryAny means a keychain item remains accessible even if the user adds a new fingerprint or Face ID appearance. An attacker who gains physical access and adds their own biometric can then unlock the item. Use .biometryCurrentSet for high-security items — the item becomes inaccessible if the biometric enrollment changes, requiring re-authentication through another channel (e.g., password login).

Deep Dive — LAContext Reuse

An LAContext instance caches the authentication result for a configurable duration via touchIDAuthenticationAllowableReuseDuration (max 5 minutes). Reusing the same context across multiple keychain reads avoids prompting the user repeatedly. Set to 0 to require authentication on every access, or LATouchIDAuthenticationMaximumAllowableReuseDuration for maximum reuse.

5. Keychain Sharing via App Groups

By default, each app has its own isolated keychain namespace. To share keychain items between apps from the same development team, you use Keychain Access Groups configured through entitlements.

┌──────────────────────────────────────────────────────┐ │ Keychain Database │ │ │ │ ┌────────────────────────────────────────────────┐ │ │ │ Access Group: TEAM_ID.com.company.shared │ │ │ │ │ │ │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ │ │ App A │ │ App B │ │ App Ext │ │ │ │ │ │ (read/ │ │ (read/ │ │ (read/ │ │ │ │ │ │ write) │ │ write) │ │ write) │ │ │ │ │ └──────────┘ └──────────┘ └──────────┘ │ │ │ └────────────────────────────────────────────────┘ │ │ │ │ ┌─────────────────────┐ ┌─────────────────────┐ │ │ │ TEAM_ID.com.app-a │ │ TEAM_ID.com.app-b │ │ │ │ (App A private) │ │ (App B private) │ │ │ └─────────────────────┘ └─────────────────────┘ │ └──────────────────────────────────────────────────────┘

Setup Steps

  1. In Xcode, enable Keychain Sharing capability for each app.
  2. Add the same keychain access group identifier (e.g., com.company.shared) to all apps.
  3. Xcode generates the entitlements file with the TEAM_ID prefix automatically.

    guard status == errSecSuccess else {
        throw KeychainError.unhandled(status: status)
    }

// Reading from a shared keychain group
func readFromSharedKeychain(
    account: String,
    accessGroup: String

        kSecClass as String:           kSecClassGenericPassword,
        kSecAttrService as String:     "com.company.shared-auth",

    guard status == errSecSuccess,
          let data = result as? Data else {

}
Pitfall — Keychain Access Group Configuration

The access group string must be prefixed with your Team ID (e.g., ABC123DEF.com.company.shared). A mismatch between the entitlements and the kSecAttrAccessGroup value silently fails — SecItemAdd returns errSecMissingEntitlement (-34018). Always verify your entitlements file and ensure the Team ID prefix matches your provisioning profile. This is the #1 cause of "keychain sharing doesn't work" bugs.

6. Keychain vs UserDefaults vs File Protection

Aspect Keychain UserDefaults File Protection
Encryption AES-256 via Secure Enclave None (plain plist) AES-256 per-file (Data Protection API)
Backup Behavior Configurable (ThisDeviceOnly excluded) Always included in backups Included unless excluded via resource values
Accessibility Control Fine-grained (6 levels + biometric) None 4 levels (Complete, UntilFirstAuth, etc.)
Performance Slower (IPC to securityd) Fast (in-memory cache) Fast (filesystem I/O)
Data Size Small items (< few KB recommended) Moderate (plist serializable) Unlimited (filesystem)
Sharing Keychain Access Groups App Groups (suite name) App Groups (shared container)
Persists Across Reinstall Yes (survives app deletion) No (deleted with app) No (deleted with app)
Appropriate Use Cases Passwords, tokens, keys, certificates Preferences, feature flags, non-sensitive settings Documents, cached data, large encrypted files
Best Practice — Storage Decision Matrix

Is it a secret? → Keychain. Is it a user preference? → UserDefaults. Is it a large file needing encryption? → File Protection with .completeFileProtection. Never store tokens, passwords, or API keys in UserDefaults — it's a plain plist readable by anyone with device access or a backup.

Deep Dive — Keychain Persistence Across Reinstall

Keychain items survive app uninstallation and reinstallation. This is by design — it allows SSO-like experiences where a user reinstalls an app and is still logged in. However, it can also cause bugs: stale tokens from a previous install may conflict with fresh state. Always implement a "first launch" check (e.g., a UserDefaults flag) and clear keychain items on first launch if your app requires a clean slate.

7. CryptoKit

Apple's CryptoKit framework (iOS 13+) provides a modern, Swift-native API for cryptographic operations. It wraps hardware-accelerated primitives and integrates with the Secure Enclave.

AES-GCM Symmetric Encryption

import CryptoKit

func encryptAESGCM(data: Data, key: SymmetricKey) throws -> Data {
    let sealedBox = try AES.GCM.seal(data, using: key)

    // Combined representation: nonce + ciphertext + tag
    guard let combined = sealedBox.combined else {
        throw CryptoError.sealingFailed
    }
    return combined
}

func decryptAESGCM(combined: Data, key: SymmetricKey) throws -> Data {
    let sealedBox = try AES.GCM.SealedBox(combined: combined)
    return try AES.GCM.open(sealedBox, using: key)
}

// Generate a 256-bit symmetric key
let key = SymmetricKey(size: .bits256)

Hashing — SHA256 / SHA384 / SHA512

import CryptoKit

let data = "Hello, Keychain!".data(using: .utf8)!

let sha256 = SHA256.hash(data: data)
let sha384 = SHA384.hash(data: data)
let sha512 = SHA512.hash(data: data)

// Convert to hex string
let hexString = sha256.compactMap { String(format: "%02x", $0) }.joined()
print(hexString) // 64-character hex string

HMAC — Message Authentication

import CryptoKit

let key = SymmetricKey(size: .bits256)
let message = "Verify this message".data(using: .utf8)!

// Create HMAC
let authCode = HMAC<SHA256>.authenticationCode(for: message, using: key)

// Verify HMAC
let isValid = HMAC<SHA256>.isValidAuthenticationCode(
    authCode,
    authenticating: message,
    using: key
)
print("HMAC valid: \(isValid)") // true

Elliptic Curve Operations — P256 Signing

import CryptoKit

// Generate P-256 key pair
let privateKey = P256.Signing.PrivateKey()
let publicKey = privateKey.publicKey

// Sign data
let dataToSign = "Critical transaction".data(using: .utf8)!
let signature = try privateKey.signature(for: dataToSign)

// Verify signature
let isSignatureValid = publicKey.isValidSignature(signature, for: dataToSign)
print("Signature valid: \(isSignatureValid)") // true

// Export public key for server
let publicKeyData = publicKey.rawRepresentation

Curve25519 Key Agreement

import CryptoKit

// Alice generates her key pair
let alicePrivate = Curve25519.KeyAgreement.PrivateKey()
let alicePublic = alicePrivate.publicKey

// Bob generates his key pair
let bobPrivate = Curve25519.KeyAgreement.PrivateKey()
let bobPublic = bobPrivate.publicKey

// Both derive the same shared secret
let aliceShared = try alicePrivate.sharedSecretFromKeyAgreement(with: bobPublic)
let bobShared = try bobPrivate.sharedSecretFromKeyAgreement(with: alicePublic)

// Derive a symmetric key from the shared secret
let symmetricKey = aliceShared.hkdfDerivedSymmetricKey(
    using: SHA256.self,
    salt: "salt".data(using: .utf8)!,
    sharedInfo: "MyApp-E2E".data(using: .utf8)!,
    outputByteCount: 32
)

Secure Enclave Key Generation

import CryptoKit

// P-256 key stored in the Secure Enclave — never leaves hardware
let sePrivateKey = try SecureEnclave.P256.Signing.PrivateKey()

// Sign with the SE-backed key
let payload = "Authenticate me".data(using: .utf8)!
let seSignature = try sePrivateKey.signature(for: payload)

// Verify with the public key (can be exported)
let sePublicKey = sePrivateKey.publicKey
let valid = sePublicKey.isValidSignature(seSignature, for: payload)

// Persist the SE key reference in the keychain
let keyData = sePrivateKey.dataRepresentation
Best Practice — Secure Enclave for Signing Keys

Always use SecureEnclave.P256 for signing keys that protect high-value operations (payments, authentication challenges). The private key material never leaves the Secure Enclave hardware — even a kernel-level compromise cannot extract it. Store the key's dataRepresentation in the keychain so you can reconstruct the reference after app restart.

8. App Attest

App Attest (iOS 14+) allows your server to verify that requests come from a legitimate, unmodified instance of your app running on genuine Apple hardware. It uses the DCAppAttestService API and Apple's attestation servers.

┌──────────┐ ┌──────────────┐ ┌─────────────────┐ │ Your │ 1. Generate key │ │ │ │ App │─────────────────────► │ Secure │ │ │ │ │ Enclave │ │ │ │ 2. attestKey(keyId, │ │ │ │ │ clientDataHash) │ │ │ │ │─────────────────────► │ │ │ │ │ │─────────┤ Apple Attest │ │ │ │ 3.Attest│ Service │ │ │ 4. Attestation object │ request│ │ │ │◄───────────────────── │◄────────┤ │ │ │ │ │ │ │ │ 5. Send attestation │ │ │ │ │ to your server │ │ │ │ │──────────────────┐ │ │ │ └──────────┘ │ └─────────┘ └──────────────┘ ▼ ┌──────────────────┐ │ Your Server │ │ │ │ 6. Verify with │ │ Apple root cert │ │ 7. Extract │ │ public key │ │ 8. Store key │ │ for assertions │ └──────────────────┘

Implementation

import DeviceCheck
import CryptoKit

final class AppAttestManager {
    private let service = DCAppAttestService.shared

    func generateKeyAndAttest() async throws -> (String, Data) {
        // 1. Check if App Attest is supported
        guard service.isSupported else {
            throw AttestError.notSupported
        }

        // 2. Generate a key pair in the Secure Enclave
        let keyId = try await service.generateKey()

        // 3. Create a challenge hash from your server's challenge
        let serverChallenge = try await fetchChallengeFromServer()
        let challengeHash = Data(SHA256.hash(data: serverChallenge))

        // 4. Attest the key — Apple verifies device + app integrity
        let attestation = try await service.attestKey(keyId, clientDataHash: challengeHash)

        // 5. Send keyId + attestation to your server for verification
        return (keyId, attestation)
    }

    func generateAssertion(
        keyId: String,
        requestData: Data
    ) async throws -> Data {
        let clientDataHash = Data(SHA256.hash(data: requestData))
        return try await service.generateAssertion(keyId, clientDataHash: clientDataHash)
    }

    private func fetchChallengeFromServer() async throws -> Data {
        let url = URL(string: "https://api.example.com/attest/challenge")!
        let (data, _) = try await URLSession.shared.data(from: url)
        return data
    }
}
Deep Dive — Attestation vs Assertion

Attestation is a one-time operation that proves the key was generated on genuine Apple hardware running your unmodified app. Your server verifies the attestation object against Apple's root certificate and extracts the public key. Assertions are subsequent requests signed with the attested key — your server verifies these using the stored public key. Attestation is expensive (network round-trip to Apple); assertions are cheap (local signing).

9. DeviceCheck

DeviceCheck (iOS 11+) provides a server-side mechanism to store two bits of data per device on Apple's servers. Unlike App Attest, it doesn't verify app integrity — it tracks device state across app reinstalls and even factory resets (tied to hardware, not software).

DCDevice API

import DeviceCheck

final class DeviceCheckManager {
    func generateToken() async throws -> Data {
        guard DCDevice.current.isSupported else {
            throw DeviceCheckError.notSupported
        }

        return try await DCDevice.current.generateToken()
    }

    func sendTokenToServer(token: Data) async throws {
        let tokenBase64 = token.base64EncodedString()

        var request = URLRequest(
            url: URL(string: "https://api.example.com/devicecheck/verify")!
        )
        request.httpMethod = "POST"
        request.httpBody = try JSONEncoder().encode(
            ["device_token": tokenBase64]
        )
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")

        let (_, response) = try await URLSession.shared.data(for: request)

        guard let httpResponse = response as? HTTPURLResponse,
              httpResponse.statusCode == 200 else {
            throw DeviceCheckError.serverRejected
        }
    }
}

Server-Side: Query and Update Bits

Production-Ready KeychainManager

import Foundation
import Security
import LocalAuthentication

protocol SecureStorable {
    func save(_ data: Data, for key: CredentialKey) throws
    func read(_ key: CredentialKey) throws -> Data
    func delete(_ key: CredentialKey) throws
    func deleteAll() throws
}

enum CredentialKey: String, CaseIterable {
    case accessToken
    case refreshToken
    case apiKey
    case userPin
    case encryptionKey

    var requiresBiometric: Bool {
        switch self {
        case .refreshToken, .userPin, .encryptionKey: return true
        default: return false
        }
    }

    var accessibility: CFString {
        switch self {
        case .accessToken:
            return kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly
        case .refreshToken, .userPin, .encryptionKey:
            return kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly
        case .apiKey:
            return kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly
        }
    }
}

final class KeychainManager: SecureStorable {
    static let shared = KeychainManager()

    private let service: String
    private let accessGroup: String?

    init(
        service: String = Bundle.main.bundleIdentifier ?? "com.app.keychain",
        accessGroup: String? = nil
    ) {
        self.service = service
        self.accessGroup = accessGroup
    }

    // MARK: - Save

    func save(_ data: Data, for key: CredentialKey) throws {
        try delete(key)

        var query = baseQuery(for: key)
        query[kSecValueData as String] = data

        if key.requiresBiometric {
            query[kSecAttrAccessControl as String] = try biometricAccessControl(for: key)
        } else {
            query[kSecAttrAccessible as String] = key.accessibility
        }

        let status = SecItemAdd(query as CFDictionary, nil)
        guard status == errSecSuccess else {
            throw KeychainError.unhandled(status: status)
        }
    }

    // MARK: - Read

    func read(_ key: CredentialKey) throws -> Data {
        var query = baseQuery(for: key)
        query[kSecReturnData as String] = true
        query[kSecMatchLimit as String] = kSecMatchLimitOne

        if key.requiresBiometric {
            let context = LAContext()
            context.localizedReason = "Access secure credentials"
            query[kSecUseAuthenticationContext as String] = context
        }

        var result: AnyObject?
        let status = SecItemCopyMatching(query as CFDictionary, &result)

        guard status == errSecSuccess,
              let data = result as? Data else {
            switch status {
            case errSecItemNotFound:
                throw KeychainError.itemNotFound
            case errSecUserCanceled, errSecAuthFailed:
                throw KeychainError.authFailed
            default:
                throw KeychainError.unhandled(status: status)
            }
        }
        return data
    }

    // MARK: - Delete

    func delete(_ key: CredentialKey) throws {
        let query = baseQuery(for: key)
        let status = SecItemDelete(query as CFDictionary)
        guard status == errSecSuccess || status == errSecItemNotFound else {
            throw KeychainError.unhandled(status: status)
        }
    }

    func deleteAll() throws {
        for key in CredentialKey.allCases {
            try delete(key)
        }
    }

    // MARK: - Private Helpers

    private func baseQuery(for key: CredentialKey) -> [String: Any] {
        var query: [String: Any] = [
            kSecClass as String:       kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: key.rawValue
        ]
        if let group = accessGroup {
            query[kSecAttrAccessGroup as String] = group
        }
        return query
    }

    private func biometricAccessControl(
        for key: CredentialKey
    ) throws -> SecAccessControl {
        var error: Unmanaged<CFError>?
        guard let access = SecAccessControlCreateWithFlags(
            kCFAllocatorDefault,
            key.accessibility,
            [.biometryCurrentSet, .or, .devicePasscode],
            &error
        ) else {
            throw error!.takeRetainedValue() as Error
        }
        return access
    }
}

CredentialManager — High-Level Facade

final class CredentialManager {
    private let store: SecureStorable
    private let encoder = JSONEncoder()
    private let decoder = JSONDecoder()

    init(store: SecureStorable = KeychainManager.shared) {
        self.store = store
    }

    // MARK: - Token Management

    func storeTokens(access: String, refresh: String) throws {
        guard let accessData = access.data(using: .utf8),
              let refreshData = refresh.data(using: .utf8) else {
            throw KeychainError.unhandled(status: errSecParam)
        }
        try store.save(accessData, for: .accessToken)
        try store.save(refreshData, for: .refreshToken)
    }

    func getAccessToken() throws -> String {
        let data = try store.read(.accessToken)
        guard let token = String(data: data, encoding: .utf8) else {
            throw KeychainError.unhandled(status: errSecDecode)
        }
        return token
    }

    func getRefreshToken() throws -> String {
        let data = try store.read(.refreshToken)
        guard let token = String(data: data, encoding: .utf8) else {
            throw KeychainError.unhandled(status: errSecDecode)
        }
        return token
    }

    func clearAllCredentials() throws {
        try store.deleteAll()
    }
}

Migration from UserDefaults to Keychain

final class KeychainMigrationService {
    private static let migrationKey = "keychain_migration_v1_complete"

    static func migrateIfNeeded(credentialManager: CredentialManager) {
        let defaults = UserDefaults.standard

        guard !defaults.bool(forKey: migrationKey) else { return }

        // Read legacy tokens from UserDefaults
        if let legacyAccess = defaults.string(forKey: "access_token"),
           let legacyRefresh = defaults.string(forKey: "refresh_token") {

            do {
                try credentialManager.storeTokens(
                    access: legacyAccess,
                    refresh: legacyRefresh
                )

                // Remove from UserDefaults after successful migration
                defaults.removeObject(forKey: "access_token")
                defaults.removeObject(forKey: "refresh_token")
            } catch {
                // Log but don't crash — retry on next launch
                print("Migration failed: \(error)")
                return
            }
        }

        defaults.set(true, forKey: migrationKey)
    }
}
Best Practice — Architect's Checklist for Secure Storage
  • Protocol-based abstraction — SecureStorable protocol enables testing with mock keychain
  • Delete-before-add pattern — avoids errSecDuplicateItem without try-update-then-add complexity
  • Biometric per credential type — not all credentials need biometric; access tokens used in background must not require it
  • Atomic migration — only mark migration complete after successful write; remove old data after confirmed migration
  • Access group support — optional shared keychain for multi-app architectures
  • Centralized error handling — KeychainError enum maps OSStatus to meaningful errors
  • First-launch cleanup — detect fresh install via UserDefaults flag and clear stale keychain items from previous installs
Pitfall — Keychain on Simulator

The iOS Simulator does not have a Secure Enclave, and keychain behavior differs from physical devices. Biometric-protected items may behave unexpectedly, and kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly has no effect since the Simulator has no passcode. Always test keychain code on a physical device before shipping. Use #if targetEnvironment(simulator) guards to provide fallback behavior during development.

Architect Must-Know — Interview Talking Points

When discussing secure credential storage at L6 level, demonstrate understanding of:

  • Threat modeling — What are you protecting against? Jailbreak? Physical access? Network MITM? Backup extraction?
  • Defense in depth — Keychain encryption + biometric + certificate pinning + App Attest form layered security
  • Token lifecycle — Short-lived access tokens (15 min), longer refresh tokens (30 days), rotation on use, revocation on logout
  • Migration strategy — How to move from insecure storage without data loss or requiring re-login for all users
  • Testability — Protocol-based keychain abstraction allows unit testing without hitting real keychain
  • Failure modes — What happens when biometric fails? Passcode removed? Device restored from backup? Keychain corrupted?
  • Compliance — GDPR right-to-erasure requires clearing all keychain items; HIPAA requires encryption at rest (keychain satisfies this)
