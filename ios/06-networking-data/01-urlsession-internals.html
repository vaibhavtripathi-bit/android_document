1. URLSession Architecture Overview

Deep Dive: The Shared Session

URLSession.shared is a singleton with no delegate and uses URLSessionConfiguration.default. It is suitable only for simple, fire-and-forget requests. You cannot customize caching, authentication challenges, or background transfers with the shared session. For any production app, always create a custom session.

The key insight is that URLSession copies the configuration at init time. Mutating the configuration object after session creation has no effect. This is a common interview pitfall.

let config = URLSessionConfiguration.default
config.timeoutIntervalForRequest = 30

let session = URLSession(configuration: config)

// This has NO effect on the session — config was copied
config.timeoutIntervalForRequest = 60

2. URLSessionConfiguration Deep Dive

URLSessionConfiguration defines the behavior and policies for a URLSession. There are three factory configurations, each with distinct characteristics.

Creating Each Configuration Type

// 1. Default — persistent storage (cookies, cache, credentials)
let defaultConfig = URLSessionConfiguration.default

// 2. Ephemeral — in-memory only, no persistent storage
let ephemeralConfig = URLSessionConfiguration.ephemeral

// 3. Background — survives app termination, system-managed
let backgroundConfig = URLSessionConfiguration.background(
    withIdentifier: "com.myapp.background-download"
)

Best Practice: When to Use Each Configuration
  • Default — Most API calls, where you want persistent cookies and caching.
  • Ephemeral — Sensitive operations (login, payment), incognito browsing, or when you need a clean slate per session.
  • Background — Large file downloads/uploads that must complete even if the user kills the app. Think podcast downloads, photo uploads, OS update-style transfers.

3. URLSessionTask Hierarchy

URLSessionTask is the abstract base class for all work performed by a session. The class hierarchy reflects the different transfer semantics.

URLSessionTask (abstract) ├── state: .running | .suspended | .canceling | .completed ├── priority: 0.0 ... 1.0 ├── countOfBytesReceived / countOfBytesExpectedToReceive ├── progress: Progress │ ├── URLSessionDataTask │ └── Receives data in memory via delegate/completion │ ├── URLSessionUploadTask (extends DataTask) │ └── Sends body data (file, stream, Data) │ ├── URLSessionDownloadTask │ └── Downloads to temp file on disk │ └── Supports resume data │ ├── URLSessionStreamTask │ └── Raw TCP/TLS streams │ └── Read/write byte buffers │ └── URLSessionWebSocketTask (extends DataTask) └── WebSocket frames (text/binary) └── Ping/pong support

Task State Machine

┌───────────┐ │ suspended │ ◀── initial state after creation └─────┬─────┘ │ resume() ▼ ┌───────────┐ ┌────▶│ running │◀────┐ │ └─────┬─────┘ │ │ │ │ │ resume() │ suspend()│ │ ▼ │ │ ┌───────────┐ │ └─────│ suspended │─────┘ └─────┬─────┘ │ cancel() ▼ ┌───────────┐ │ canceling │ └─────┬─────┘ │ (system completes) ▼ ┌───────────┐ │ completed │ ── terminal state └───────────┘
Pitfall: Tasks Start Suspended

All newly created tasks start in the suspended state. You must call resume() to begin execution. This is the #1 beginner mistake with URLSession. If you forget resume(), the request never fires and no error is thrown.

DataTask — In-Memory Response

// Completion handler style
let url = URL(string: "https://api.example.com/users")!
let task = session.dataTask(with: url) { data, response, error in
    guard let httpResponse = response as? HTTPURLResponse,
          (200...299).contains(httpResponse.statusCode),
          let data = data else {
        return
    }
    let users = try? JSONDecoder().decode([User].self, from: data)
}
task.resume()

// async/await style (iOS 15+)
func fetchUsers() async throws -> [User] {
    let (data, response) = try await session.data(from: url)
    guard let httpResponse = response as? HTTPURLResponse,
          httpResponse.statusCode == 200 else {
        throw NetworkError.invalidResponse
    }
    return try JSONDecoder().decode([User].self, from: data)
}

UploadTask — Sending Data

func uploadImage(_ imageData: Data) async throws -> UploadResponse {
    var request = URLRequest(url: URL(string: "https://api.example.com/upload")!)
    request.httpMethod = "POST"
    request.setValue("image/jpeg", forHTTPHeaderField: "Content-Type")

    let (data, response) = try await session.upload(for: request, from: imageData)
    guard let httpResponse = response as? HTTPURLResponse,
          httpResponse.statusCode == 201 else {
        throw NetworkError.uploadFailed
    }
    return try JSONDecoder().decode(UploadResponse.self, from: data)
}

// Upload from file (better for large files — avoids loading into memory)
func uploadFile(at fileURL: URL) async throws -> UploadResponse {
    var request = URLRequest(url: URL(string: "https://api.example.com/upload")!)
    request.httpMethod = "PUT"

    let (data, _) = try await session.upload(for: request, fromFile: fileURL)
    return try JSONDecoder().decode(UploadResponse.self, from: data)
}

DownloadTask — File Downloads with Resume

func downloadFile(from url: URL) async throws -> URL {
    let (tempURL, response) = try await session.download(from: url)

    guard let httpResponse = response as? HTTPURLResponse,
          httpResponse.statusCode == 200 else {
        throw NetworkError.downloadFailed
    }

    // tempURL is deleted after this scope — move it
    let documentsURL = FileManager.default
        .urls(for: .documentDirectory, in: .userDomainMask).first!
    let destinationURL = documentsURL.appendingPathComponent(url.lastPathComponent)

    try FileManager.default.moveItem(at: tempURL, to: destinationURL)
    return destinationURL
}

// Resumable download with delegate for progress
class DownloadManager: NSObject, URLSessionDownloadDelegate {
    private var resumeData: Data?
    private var session: URLSession!

    func urlSession(
        _ session: URLSession,
        downloadTask: URLSessionDownloadTask,
        didFinishDownloadingTo location: URL
    ) {
        // Move file from temp location before this method returns
        let dest = FileManager.default.temporaryDirectory
            .appendingPathComponent(UUID().uuidString)
        try? FileManager.default.moveItem(at: location, to: dest)
    }

    func urlSession(
        _ session: URLSession,
        downloadTask: URLSessionDownloadTask,
        didWriteData bytesWritten: Int64,
        totalBytesWritten: Int64,
        totalBytesExpectedToWrite: Int64
    ) {
        let progress = Double(totalBytesWritten) / Double(totalBytesExpectedToWrite)
        print("Download progress: \(Int(progress * 100))%")
    }

    func urlSession(
        _ session: URLSession,
        task: URLSessionTask,
        didCompleteWithError error: Error?
    ) {
        if let error = error as? NSError,
           let data = error.userInfo[NSURLSessionDownloadTaskResumeData] as? Data {
            self.resumeData = data
        }
    }

    func resumeDownload() {
        guard let resumeData = resumeData else { return }
        let task = session.downloadTask(withResumeData: resumeData)
        task.resume()
    }
}

4. HTTP/2 Multiplexing

URLSession transparently negotiates HTTP/2 via ALPN (Application-Layer Protocol Negotiation) during the TLS handshake. No code changes are required — the same API works for HTTP/1.1 and HTTP/2.

HTTP/1.1 vs HTTP/2 Connection Model

HTTP/1.1 — One request per connection (head-of-line blocking) ═══════════════════════════════════════════════════════════ Connection 1: ──[Request A]────────[Response A]──────────────────▶ Connection 2: ──[Request B]────────[Response B]──────────────────▶ Connection 3: ──[Request C]────────[Response C]──────────────────▶ Connection 4: ──[Request D]──(waiting for connection)────────────▶ Connection 5: ──[Request E]──(waiting for connection)────────────▶ Connection 6: ──[Request F]──(waiting for connection)────────────▶ ⚠ Limited to httpMaximumConnectionsPerHost (default: 6 on iOS) ⚠ Each connection = separate TCP handshake + TLS negotiation HTTP/2 — Multiplexed streams over single connection ═══════════════════════════════════════════════════ Single Connection: Stream 1: ──[Req A]──[Resp A frames]──────────────────▶ Stream 3: ──[Req B]────[Resp B frames]────────────────▶ Stream 5: ──[Req C]──────[Resp C frames]──────────────▶ Stream 7: ──[Req D]────────[Resp D frames]────────────▶ Stream 9: ──[Req E]──────────[Resp E frames]──────────▶ Stream 11:──[Req F]────────────[Resp F frames]────────▶ ✓ Single TCP connection, single TLS handshake ✓ Frames from different streams interleaved ✓ Server can prioritize streams ✓ Header compression (HPACK)

Connection Coalescing

HTTP/2 connection coalescing allows URLSession to reuse a single TCP connection for multiple hostnames if they resolve to the same IP and share a TLS certificate (via SAN — Subject Alternative Name). For example, api.example.com and cdn.example.com can share one connection if the certificate covers both.

Deep Dive: Head-of-Line Blocking

HTTP/2 solves HTTP-level head-of-line blocking (multiple requests on one connection), but TCP-level HOL blocking remains: if a TCP packet is lost, all streams on that connection stall until retransmission. This is why HTTP/3 (QUIC) uses UDP — each stream has independent loss recovery. iOS 15+ supports HTTP/3 experimentally via assumesHTTP3Capable on URLSessionConfiguration.

// Opt into HTTP/3 (QUIC) — iOS 15+
let config = URLSessionConfiguration.default
config.assumesHTTP3Capable = true

// Check which protocol was negotiated
let (data, response) = try await session.data(from: url)
if let httpResponse = response as? HTTPURLResponse {
    // Access via task metrics for protocol details
}

// Collect detailed metrics including protocol version
func urlSession(
    _ session: URLSession,
    task: URLSessionTask,
    didFinishCollecting metrics: URLSessionTaskMetrics
) {
    for tx in metrics.transactionMetrics {
        print("Protocol: \(tx.networkProtocolName ?? "unknown")")
        // "h2" for HTTP/2, "h3" for HTTP/3
        print("Reused connection: \(tx.isReusedConnection)")
        print("DNS: \(tx.domainLookupStartDate) → \(tx.domainLookupEndDate)")
        print("TLS: \(tx.secureConnectionStartDate) → \(tx.secureConnectionEndDate)")
    }
}

5. URLProtocol Interception

URLProtocol is an abstract class that lets you intercept, modify, or completely replace the loading behavior of URL requests. Registered protocols are checked in LIFO order — the last registered protocol gets first chance to handle a request.

URLProtocol Lifecycle

URLSession creates a request │ ▼ ┌───────────────────────────────────┐ │ For each registered URLProtocol │ │ (LIFO order): │ │ │ │ canInit(with: URLRequest) ? │──── false ──▶ try next protocol │ │ │ │ true │ │ │ │ │ ▼ │ │ canonicalRequest(for:) │ │ (normalize the request) │ │ │ │ │ ▼ │ │ init(request:cachedResponse: │ │ client:) │ │ │ │ │ ▼ │ │ startLoading() │ │ ┌─ client.urlProtocol( │ │ │ didReceive: response) │ │ │ client.urlProtocol( │ │ │ didLoad: data) │ │ │ client.urlProtocolDidFinish │ │ │ Loading() │ │ └──────────────────────────────│ │ │ │ │ ▼ │ │ stopLoading() │ │ (cancel/cleanup) │ └───────────────────────────────────┘

Complete Mock URLProtocol for Testing

final class MockURLProtocol: URLProtocol {

    static var requestHandler: ((URLRequest) throws -> (HTTPURLResponse, Data))?

    override class func canInit(with request: URLRequest) -> Bool {
        return true
    }

    override class func canonicalRequest(for request: URLRequest) -> URLRequest {
        return request
    }

    override func startLoading() {
        guard let handler = MockURLProtocol.requestHandler else {
            client?.urlProtocol(self, didFailWithError: URLError(.badURL))
            return
        }

        do {
            let (response, data) = try handler(request)
            client?.urlProtocol(self, didReceive: response, cacheStoragePolicy: .notAllowed)
            client?.urlProtocol(self, didLoad: data)
            client?.urlProtocolDidFinishLoading(self)
        } catch {
            client?.urlProtocol(self, didFailWithError: error)
        }
    }

    override func stopLoading() {
        // No-op for mock
    }
}

// Usage in tests
final class APIClientTests: XCTestCase {

    var session: URLSession!

    override func setUp() {
        let config = URLSessionConfiguration.ephemeral
        config.protocolClasses = [MockURLProtocol.self]
        session = URLSession(configuration: config)
    }

    func testFetchUsersReturnsDecodedData() async throws {
        let mockUsers = [User(id: 1, name: "Alice")]
        let mockData = try JSONEncoder().encode(mockUsers)

        MockURLProtocol.requestHandler = { request in
            let response = HTTPURLResponse(
                url: request.url!,
                statusCode: 200,
                httpVersion: nil,
                headerFields: ["Content-Type": "application/json"]
            )!
            return (response, mockData)
        }

        let client = APIClient(session: session)
        let users = try await client.fetchUsers()
        XCTAssertEqual(users.count, 1)
        XCTAssertEqual(users.first?.name, "Alice")
    }
}
Pitfall: URLProtocol Registration Scope

Global registration via URLProtocol.registerClass(_:) only works with URLSession.shared and legacy NSURLConnection. For custom URLSession instances, you must set protocolClasses on the URLSessionConfiguration before creating the session. This is a frequent source of bugs where a mock protocol "doesn't work" in tests.

Also beware: URLProtocol callbacks happen on an arbitrary background thread. If your protocol interacts with UI or non-thread-safe state, you must dispatch to the correct queue.

Pitfall: URLProtocol and Background Sessions

URLProtocol subclasses are not supported with background session configurations. The system handles background transfers in a separate daemon process (nsurlsessiond) that cannot load your custom protocol classes.

6. URLCache

URLCache provides a composite in-memory and on-disk cache for URL responses. It sits between your code and the network, transparently caching responses based on HTTP cache semantics.

Cache Architecture

┌──────────────┐ miss ┌──────────────┐ miss ┌──────────┐ │ Memory Cache │─────────────▶│ Disk Cache │─────────────▶│ Network │ │ (fast, RAM) │ │ (SQLite DB) │ │ │ │ │◀─────────────│ │◀─────────────│ │ │ │ populate │ │ populate │ │ └──────────────┘ └──────────────┘ └──────────┘ Default sizes: Memory: 512 KB (URLSession.shared) or 4 MB (.default config) Disk: 10 MB (URLSession.shared) or 20 MB (.default config) Location: <app-sandbox>/Library/Caches/<bundle-id>/Cache.db

CachePolicy Enum

Policy Behavior Use Case
.useProtocolCachePolicy Follows HTTP cache headers (Cache-Control, Expires, ETag, Last-Modified). Default. Standard API calls
.reloadIgnoringLocalCacheData Always fetches from network. Ignores local cache entirely. Pull-to-refresh, force reload
.returnCacheDataElseLoad Returns cached data regardless of age. Only goes to network if no cache exists. Offline-first, stale-while-revalidate
.returnCacheDataDontLoad Returns cached data or fails. Never goes to network. Strict offline mode
.reloadRevalidatingCacheData Validates cached data with server (conditional GET with ETag/If-Modified-Since). Ensure freshness with minimal bandwidth

Custom Cache Configuration

// Custom cache with generous limits
let cache = URLCache(
    memoryCapacity: 50 * 1024 * 1024,   // 50 MB memory
    diskCapacity: 500 * 1024 * 1024,    // 500 MB disk
    directory: FileManager.default
        .urls(for: .cachesDirectory, in: .userDomainMask).first?
        .appendingPathComponent("APICache")
)

let config = URLSessionConfiguration.default
config.urlCache = cache
config.requestCachePolicy = .useProtocolCachePolicy

// Per-request cache policy override
var request = URLRequest(url: url)
request.cachePolicy = .reloadIgnoringLocalCacheData

// Manually store a cached response
let cachedResponse = CachedURLResponse(
    response: httpResponse,
    data: responseData,
    storagePolicy: .allowed
)
cache.storeCachedResponse(cachedResponse, for: request)

// Query cache without making a network request
if let cached = cache.cachedResponse(for: request) {
    let data = cached.data
    let response = cached.response
    print("Cache hit: \(data.count) bytes")
}

// Clear specific or all cached data
cache.removeCachedResponse(for: request)
cache.removeAllCachedResponses()
cache.removeCachedResponses(since: Date().addingTimeInterval(-3600))
Deep Dive: Cache-Control Headers

The default .useProtocolCachePolicy respects standard HTTP cache headers:

  • Cache-Control: max-age=3600 — cache for 1 hour
  • Cache-Control: no-cache — must revalidate before using
  • Cache-Control: no-store — never cache (sensitive data)
  • ETag / If-None-Match — conditional GET, server returns 304 if unchanged
  • Last-Modified / If-Modified-Since — date-based conditional GET

If the server sends no-store, URLCache will not cache the response regardless of your local cache policy. POST requests are never cached by default.

Best Practice: Cache Strategy

For production apps, configure generous disk cache sizes (200–500 MB) and use .useProtocolCachePolicy as the default. Override per-request for pull-to-refresh (.reloadIgnoringLocalCacheData) and offline mode (.returnCacheDataElseLoad). Monitor cache hit rates via URLSessionTaskMetrics to tune server cache headers.

7. TLS & App Transport Security

App Transport Security (ATS) is a security feature introduced in iOS 9 that enforces best-practice HTTPS connections. It is enabled by default and requires all network connections to use TLS 1.2+ with forward secrecy.

ATS Requirements

Requirement Minimum Notes
Protocol TLS 1.2 TLS 1.0/1.1 rejected. TLS 1.3 preferred when available.
Cipher suites Forward secrecy (ECDHE) AES-128-GCM, AES-256-GCM, ChaCha20-Poly1305
Certificate SHA-256+ signature RSA 2048-bit or ECC 256-bit minimum key size
Scheme HTTPS only Plain HTTP blocked unless explicitly excepted

        <key>localhost</key>

            <key>NSExceptionAllowsInsecureHTTPLoads</key>

        </dict>

Pitfall: ATS and App Review

Apple requires justification for ATS exceptions during App Store review. Using NSAllowsArbitraryLoads = true will trigger review scrutiny and may result in rejection. Always use per-domain exceptions with the minimum necessary relaxation. The App Store review team specifically checks for blanket ATS disabling.

// Programmatic TLS configuration via URLSessionConfiguration
let config = URLSessionConfiguration.default

// Require TLS 1.2 minimum (matches ATS default)
config.tlsMinimumSupportedProtocolVersion = .TLSv12

// Prefer TLS 1.3 when available
config.tlsMaximumSupportedProtocolVersion = .TLSv13
Deep Dive: Forward Secrecy

Forward secrecy (via ECDHE key exchange) ensures that even if the server's private key is compromised in the future, past recorded traffic cannot be decrypted. Each session uses a unique ephemeral key pair. ATS requires forward secrecy by default because without it, a single key compromise exposes all historical traffic — a catastrophic failure mode for any app handling user data.

8. Certificate Pinning

Certificate pinning restricts which certificates your app trusts for a given domain, protecting against man-in-the-middle attacks even if a rogue CA issues a valid certificate for your domain.

Certificate Pinning vs Public Key Pinning

Approach Pins Rotation Pros Cons
Certificate pinning Entire X.509 certificate Must update app when cert rotates Simple to implement Brittle — cert rotation breaks app
Public key pinning Subject Public Key Info (SPKI) Survives cert renewal if key unchanged More resilient to rotation Slightly more complex extraction

Complete Certificate Pinning Implementation

final class PinnedSessionDelegate: NSObject, URLSessionDelegate {

    private let pinnedCertificates: [SecCertificate]

    init(pinnedCertificateNames: [String]) {
        self.pinnedCertificates = pinnedCertificateNames.compactMap { name in
            guard let url = Bundle.main.url(forResource: name, withExtension: "cer"),
                  let data = try? Data(contentsOf: url),
                  let cert = SecCertificateCreateWithData(nil, data as CFData) else {
                return nil
            }
            return cert
        }
        super.init()
    }

    func urlSession(
        _ session: URLSession,
        didReceive challenge: URLAuthenticationChallenge,
        completionHandler: @escaping (URLSession.AuthChallengeDisposition, URLCredential?) -> Void
    ) {
        guard challenge.protectionSpace.authenticationMethod
                == NSURLAuthenticationMethodServerTrust,
              let serverTrust = challenge.protectionSpace.serverTrust else {
            completionHandler(.performDefaultHandling, nil)
            return
        }

        // Set pinned certificates as the only trusted anchors
        SecTrustSetAnchorCertificates(serverTrust, pinnedCertificates as CFArray)
        SecTrustSetAnchorCertificatesOnly(serverTrust, true)

        // Evaluate the trust chain
        var error: CFError?
        let isTrusted = SecTrustEvaluateWithError(serverTrust, &error)

        if isTrusted {
            let credential = URLCredential(trust: serverTrust)
            completionHandler(.useCredential, credential)
        } else {
            completionHandler(.cancelAuthenticationChallenge, nil)
        }
    }
}

Public Key Pinning (More Resilient)

final class PublicKeyPinningDelegate: NSObject, URLSessionDelegate {

    // SHA-256 hashes of the Subject Public Key Info (SPKI)
    private let pinnedKeyHashes: Set<String> = [
        "BBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBB=",  // primary
        "CCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCC=",  // backup
    ]

    func urlSession(
        _ session: URLSession,
        didReceive challenge: URLAuthenticationChallenge,
        completionHandler: @escaping (URLSession.AuthChallengeDisposition, URLCredential?) -> Void
    ) {
        guard challenge.protectionSpace.authenticationMethod
                == NSURLAuthenticationMethodServerTrust,
              let serverTrust = challenge.protectionSpace.serverTrust else {
            completionHandler(.performDefaultHandling, nil)
            return
        }

        // Standard trust evaluation first
        var error: CFError?
        guard SecTrustEvaluateWithError(serverTrust, &error) else {
            completionHandler(.cancelAuthenticationChallenge, nil)
            return
        }

        // Extract and verify public key from the leaf certificate
        let certCount = SecTrustGetCertificateCount(serverTrust)
        guard certCount > 0,
              let serverCert = SecTrustCopyCertificateChain(serverTrust)
                as? [SecCertificate],
              let leafCert = serverCert.first,
              let publicKey = SecCertificateCopyKey(leafCert),
              let publicKeyData = SecKeyCopyExternalRepresentation(publicKey, nil)
                as? Data else {
            completionHandler(.cancelAuthenticationChallenge, nil)
            return
        }

        let keyHash = publicKeyData.sha256Base64()

        if pinnedKeyHashes.contains(keyHash) {
            let credential = URLCredential(trust: serverTrust)
            completionHandler(.useCredential, credential)
        } else {
            completionHandler(.cancelAuthenticationChallenge, nil)
        }
    }
}

extension Data {
    func sha256Base64() -> String {
        var hash = [UInt8](repeating: 0, count: Int(CC_SHA256_DIGEST_LENGTH))
        self.withUnsafeBytes { buffer in
            _ = CC_SHA256(buffer.baseAddress, CC_LONG(self.count), &hash)
        }
        return Data(hash).base64EncodedString()
    }
}
Best Practice: Pinning Strategy
  • Always pin public keys, not certificates — survives certificate renewal.
  • Pin at least two keys — a primary and a backup. If you lose the primary key, the backup prevents a total lockout.
  • Pin the intermediate CA, not the leaf — more stable and survives leaf rotation.
  • Include a kill switch — a remote config flag to disable pinning if you need to rotate keys in an emergency.
  • Monitor failures — log pinning failures to detect MITM attacks vs. legitimate cert changes.

Background sessions allow downloads and uploads to continue even after the user switches apps or the system terminates the app. The system daemon nsurlsessiond manages the actual transfer.

Pitfall: Background Session Rules
  • Unique identifier required — each background session must have a globally unique identifier. If you create two sessions with the same ID, behavior is undefined.
  • Only download and upload tasks — data tasks are not supported in background sessions (they are automatically converted to download tasks in iOS 13+).
  • Delegate required — completion handler-based tasks are not supported. You must use the delegate pattern.
  • System scheduling — the system may delay discretionary transfers based on battery, network conditions, and thermal state. Set isDiscretionary = false for user-initiated transfers.
  • Recreate with same ID — when the app relaunches, you must recreate the session with the exact same identifier to reconnect to pending transfers.
  • No URLProtocol — custom URLProtocol subclasses are not supported in background sessions.
  • Call the completion handler — failing to call the system-provided completion handler in urlSessionDidFinishEvents causes the app snapshot to not update and wastes system resources.

10. URLSessionWebSocketTask

URLSessionWebSocketTask (iOS 13+) provides native WebSocket support without third-party dependencies. It handles the HTTP upgrade handshake, frame encoding/decoding, ping/pong, and close handshake.

WebSocket API

final class WebSocketClient {

    private var webSocketTask: URLSessionWebSocketTask?
    private let session: URLSession

    init() {
        self.session = URLSession(configuration: .default)
    }

    // MARK: - Connect

    func connect(to url: URL) {
        webSocketTask = session.webSocketTask(with: url)
        webSocketTask?.resume()
        listenForMessages()
        schedulePing()
    }

    // MARK: - Send

    func send(text: String) async throws {
        let message = URLSessionWebSocketTask.Message.string(text)
        try await webSocketTask?.send(message)
    }

    func send(data: Data) async throws {
        let message = URLSessionWebSocketTask.Message.data(data)
        try await webSocketTask?.send(message)
    }

    // MARK: - Receive (recursive listener)

    private func listenForMessages() {
        webSocketTask?.receive { [weak self] result in
            switch result {
            case .success(let message):
                switch message {
                case .string(let text):
                    print("Received text: \(text)")
                case .data(let data):
                    print("Received data: \(data.count) bytes")
                @unknown default:
                    break
                }
                self?.listenForMessages()

            case .failure(let error):
                print("WebSocket error: \(error)")
            }
        }
    }

    // MARK: - Ping/Pong

    private func schedulePing() {
        webSocketTask?.sendPing { [weak self] error in
            if let error = error {
                print("Ping failed: \(error)")
                return
            }
            // Schedule next ping in 30 seconds
            DispatchQueue.main.asyncAfter(deadline: .now() + 30) {
                self?.schedulePing()
            }
        }
    }

    // MARK: - Disconnect

    func disconnect() {
        webSocketTask?.cancel(with: .goingAway, reason: nil)
        webSocketTask = nil
    }
}

Deep Dive: WebSocket Connection Lifecycle

WebSocket connections do not survive app backgrounding. When iOS suspends your app, the TCP connection is torn down. You must implement reconnection logic. For real-time features, consider using APNs (push notifications) as a fallback when the WebSocket is disconnected. The URLSessionWebSocketTask does not provide automatic reconnection — you must detect disconnection via the receive callback failure and reconnect manually.

Token Manager with Thread-Safe Refresh

actor TokenManager {

    private var accessToken: String?
    private var refreshToken: String?
    private var refreshTask: Task<String, Error>?

    func setTokens(access: String, refresh: String) {
        self.accessToken = access
        self.refreshToken = refresh
    }

    func currentAccessToken() -> String? {
        return accessToken
    }

    func refreshAccessToken(using session: URLSession) async throws -> String {
        // Coalesce concurrent refresh requests into a single network call
        if let existingTask = refreshTask {
            return try await existingTask.value
        }

        let task = Task { [refreshToken] () -> String in
            guard let refreshToken = refreshToken else {
                throw NetworkError.tokenRefreshFailed
            }

            var request = URLRequest(
                url: URL(string: "https://api.example.com/auth/refresh")!
            )
            request.httpMethod = "POST"
            request.setValue("application/json", forHTTPHeaderField: "Content-Type")
            request.httpBody = try JSONEncoder().encode(
                ["refresh_token": refreshToken]
            )

            let (data, response) = try await session.data(for: request)
            guard let httpResponse = response as? HTTPURLResponse,
                  httpResponse.statusCode == 200 else {
                throw NetworkError.tokenRefreshFailed
            }

            let tokenResponse = try JSONDecoder().decode(
                TokenResponse.self, from: data
            )
            return tokenResponse.accessToken
        }

        self.refreshTask = task

        do {
            let newToken = try await task.value
            self.accessToken = newToken
            self.refreshTask = nil
            return newToken
        } catch {
            self.refreshTask = nil
            throw error
        }
    }

    func clearTokens() {
        accessToken = nil
        refreshToken = nil
        refreshTask = nil
    }
}

struct TokenResponse: Decodable {
    let accessToken: String
    let refreshToken: String
    let expiresIn: Int
}

Request Deduplication

actor RequestDeduplicator {

    private var inFlightRequests: [String: Task<Data, Error>] = [:]

    func deduplicated(
        key: String,
        work: @escaping () async throws -> Data
    ) async throws -> Data {
        if let existing = inFlightRequests[key] {
            return try await existing.value
        }

        let task = Task {
            try await work()
        }

        inFlightRequests[key] = task

        do {
            let result = try await task.value
            inFlightRequests.removeValue(forKey: key)
            return result
        } catch {
            inFlightRequests.removeValue(forKey: key)
            throw error
        }
    }
}

Comprehensive NetworkManager

final class NetworkManager: @unchecked Sendable {

    static let shared = NetworkManager()

    private let session: URLSession
    private let tokenManager = TokenManager()
    private let deduplicator = RequestDeduplicator()
    private let decoder: JSONDecoder

    private let maxRetries = 3
    private let baseDelay: TimeInterval = 1.0

    private init() {
        let config = URLSessionConfiguration.default
        config.waitsForConnectivity = true
        config.timeoutIntervalForRequest = 30
        config.timeoutIntervalForResource = 300
        config.httpAdditionalHeaders = [
            "Accept": "application/json",
            "Accept-Encoding": "gzip, deflate, br"
        ]
        config.urlCache = URLCache(
            memoryCapacity: 50 * 1024 * 1024,
            diskCapacity: 200 * 1024 * 1024
        )

        self.session = URLSession(configuration: config)

        self.decoder = JSONDecoder()
        self.decoder.keyDecodingStrategy = .convertFromSnakeCase
        self.decoder.dateDecodingStrategy = .iso8601
    }

    // MARK: - Public API

    func request<T: Decodable>(
        _ endpoint: Endpoint,
        deduplicate: Bool = false
    ) async throws -> T {
        let urlRequest = try endpoint.asURLRequest()

        let data: Data
        if deduplicate {
            let key = endpoint.deduplicationKey
            data = try await deduplicator.deduplicated(key: key) {
                try await self.performRequest(urlRequest, attempt: 0)
            }
        } else {
            data = try await performRequest(urlRequest, attempt: 0)
        }

        do {
            return try decoder.decode(T.self, from: data)
        } catch let error as DecodingError {
            throw NetworkError.decodingError(error)
        }
    }

    // MARK: - Core Request with Retry + Token Refresh

    private func performRequest(
        _ request: URLRequest,
        attempt: Int
    ) async throws -> Data {
        var mutableRequest = request

        // Inject auth token
        if let token = await tokenManager.currentAccessToken() {
            mutableRequest.setValue(
                "Bearer \(token)",
                forHTTPHeaderField: "Authorization"
            )
        }

        do {
            let (data, response) = try await session.data(for: mutableRequest)

            guard let httpResponse = response as? HTTPURLResponse else {
                throw NetworkError.invalidResponse
            }

            switch httpResponse.statusCode {
            case 200...299:
                return data

            case 401:
                return try await handleUnauthorized(
                    originalRequest: request,
                    attempt: attempt
                )

            case 429:
                let retryAfter = httpResponse.value(
                    forHTTPHeaderField: "Retry-After"
                ).flatMap(TimeInterval.init)
                return try await retryWithBackoff(
                    request: request,
                    attempt: attempt,
                    overrideDelay: retryAfter
                )

            case 500...599:
                return try await retryWithBackoff(
                    request: request,
                    attempt: attempt
                )

            default:
                throw NetworkError.httpError(
                    statusCode: httpResponse.statusCode,
                    data: data
                )
            }

        } catch let error as URLError {
            switch error.code {
            case .notConnectedToInternet, .networkConnectionLost:
                throw NetworkError.networkUnavailable
            case .timedOut:
                return try await retryWithBackoff(
                    request: request,
                    attempt: attempt
                )
            case .cancelled:
                throw NetworkError.cancelled
            default:
                throw error
            }
        }
    }

    // MARK: - 401 Handler: Refresh Token + Retry

    private func handleUnauthorized(
        originalRequest: URLRequest,
        attempt: Int
    ) async throws -> Data {
        guard attempt == 0 else {
            await tokenManager.clearTokens()
            throw NetworkError.unauthorized
        }

        do {
            _ = try await tokenManager.refreshAccessToken(using: session)
            return try await performRequest(originalRequest, attempt: attempt + 1)
        } catch {
            await tokenManager.clearTokens()
            throw NetworkError.unauthorized
        }
    }

    // MARK: - Exponential Backoff with Jitter

    private func retryWithBackoff(
        request: URLRequest,
        attempt: Int,
        overrideDelay: TimeInterval? = nil
    ) async throws -> Data {
        guard attempt < maxRetries else {
            throw NetworkError.maxRetriesExceeded
        }

        let delay: TimeInterval
        if let overrideDelay = overrideDelay {
            delay = overrideDelay
        } else {
            // Exponential backoff: 1s, 2s, 4s + random jitter
            let exponential = baseDelay * pow(2.0, Double(attempt))
            let jitter = Double.random(in: 0...0.5)
            delay = exponential + jitter
        }

        try await Task.sleep(nanoseconds: UInt64(delay * 1_000_000_000))

        try Task.checkCancellation()

        return try await performRequest(request, attempt: attempt + 1)
    }
}

Endpoint Abstraction

struct Endpoint {
    let path: String
    let method: HTTPMethod
    let queryItems: [URLQueryItem]?
    let body: Data?
    let headers: [String: String]

    static let baseURL = "https://api.example.com/v2"

    var deduplicationKey: String {
        return "\(method.rawValue):\(path):\(queryItems?.description ?? "")"
    }

    func asURLRequest() throws -> URLRequest {
        guard var components = URLComponents(
            string: Endpoint.baseURL + path
        ) else {
            throw NetworkError.invalidURL
        }
        components.queryItems = queryItems

        guard let url = components.url else {
            throw NetworkError.invalidURL
        }

        var request = URLRequest(url: url)
        request.httpMethod = method.rawValue
        request.httpBody = body

        for (key, value) in headers {
            request.setValue(value, forHTTPHeaderField: key)
        }

        if body != nil {
            request.setValue("application/json", forHTTPHeaderField: "Content-Type")
        }

        return request
    }
}

enum HTTPMethod: String {
    case GET, POST, PUT, PATCH, DELETE
}

Architect Must-Know: Production Network Layer Design

At Staff/L6 level, you are expected to design a network layer that handles these concerns:

  • Auth token lifecycle — automatic injection, 401 detection, token refresh with request coalescing (multiple concurrent 401s should trigger only one refresh), and retry of the original request. The actor-based TokenManager above guarantees thread safety and single-flight refresh.
  • Exponential backoff with jitter — prevents thundering herd on server recovery. Jitter randomizes retry timing so thousands of clients don't all retry at the same instant. Respect Retry-After headers from 429 responses.
  • Request deduplication — identical GET requests fired within a short window should share a single network call. This prevents redundant traffic when multiple UI components request the same data simultaneously. Never deduplicate mutations (POST/PUT/DELETE).
  • Structured error handling — map URLError codes and HTTP status codes to domain-specific errors. Distinguish between retryable errors (timeout, 5xx, 429) and terminal errors (4xx, decoding failures).
  • Cancellation propagation — use Task.checkCancellation() between retries to respect Swift concurrency's cooperative cancellation model.
  • Observability — collect URLSessionTaskMetrics for every request. Log DNS time, TLS handshake time, request/response sizes, protocol version, and whether the connection was reused. Feed these into your monitoring system.
  • Configuration as a first-class concern — base URL, timeouts, cache sizes, and retry policies should be injectable, not hardcoded. This enables different configurations for debug/staging/production and makes the layer testable via MockURLProtocol.
