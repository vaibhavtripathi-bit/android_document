Series 2: UIKit View System

Deep internals of UIView, CALayer, touch handling, layout, rendering, drawing, animations, and collection views — the foundation of every iOS app. Target: Google L6 / Staff-level iOS interviews.

The UIKit view system is the backbone of every iOS application. Even with SwiftUI's rise, understanding the view hierarchy, the rendering pipeline, and the touch delivery system remains the most frequently tested domain in Staff-level iOS interviews. This series covers everything from the hardware touch event originating in IOKit all the way through the GPU compositing pass that puts pixels on screen.

What This Series Covers

Eight deep-dive documents plus a comprehensive Q&A covering 70+ interview questions. Each document targets the depth expected at Google L6 / Staff level — not just "how to use" but "how it works internally and why."

01 — View Hierarchy Internals

UIView/CALayer duality, the AppKit history behind the split, UIWindow tree structure, hitTest reverse pre-order traversal, coordinate systems, and z-ordering.

02 — Touch & Event Flow

The full touch pipeline from IOKit HID through SpringBoard to UIResponder chain. Gesture recognizer state machine. UIScrollView's 150ms delay. Classic Staff-level question territory.

03 — Layout System

Auto Layout's Cassowary simplex solver, constraint priorities, intrinsicContentSize, the three-pass layout cycle, UIStackView internals, and constraint performance at scale.

04 — Rendering Pipeline

Core Animation commit transactions, the render server (backboardd), GPU compositing, off-screen rendering triggers, CADisplayLink, and achieving 120fps on ProMotion.

05 — Drawing & Core Graphics

UIGraphicsImageRenderer vs legacy APIs, CGContext drawing, UIBezierPath, PDF rendering, TextKit 1 vs TextKit 2, and when to choose Core Graphics vs CAShapeLayer vs Metal.

06 — Custom Views

Building production-quality UIView subclasses, intrinsicContentSize contracts, state invalidation, accessibility with UIAccessibility, Dynamic Type, and designing reusable component libraries.

07 — Animations

UIView.animate to UIViewPropertyAnimator, Core Animation layer model, presentation vs model layer, interactive transitions, hero animations, and gesture-driven interruptible transitions.

08 — Table & Collection Views

Cell reuse internals, UICollectionViewCompositionalLayout, DiffableDataSource, CellRegistration, prefetching, self-sizing cells, and building generic list infrastructure at scale.

09 — Interview Q&A

70+ L6-level interview questions with detailed toggle answers covering every topic in this series. Tagged by difficulty: Senior, Architect, L6.

Why This Matters at L6

Architect Must-Know

At Staff level, interviewers expect you to reason about systems, not APIs. You should be able to:

  • Trace a touch event from the hardware digitizer through the kernel, through SpringBoard, through your app's responder chain, and explain every decision point
  • Explain why a particular UI is dropping frames by reasoning about the Core Animation commit/render/GPU pipeline
  • Design a layout system that scales to hundreds of screen types without constraint explosion
  • Build an animation system that supports interruptible, gesture-driven transitions at 120fps
  • Architect a collection view infrastructure that handles heterogeneous content with diffable data sources
