1. URL Schemes vs Universal Links vs App Clips

Feature URL Schemes Universal Links App Clips
Format myapp://path https://example.com/path https://example.com/clip
Uniqueness Not guaranteed (any app can claim) Verified via AASA file Verified via AASA file
Fallback Error dialog if app not installed Opens in Safari Downloads App Clip card
User confirmation Alert: "Open in App?" None (direct open) App Clip card
Works from Safari Yes (with confirmation) Yes (banner or direct) Yes
Works cross-app Yes Yes Limited
Security Low (no verification) High (domain verification) High (domain verification)
Requires server No Yes (AASA file) Yes (AASA file)
Common Pitfall: URL Scheme Hijacking

Custom URL schemes have no ownership verification. Any app can register the same scheme. If two apps register myapp://, the system's behavior is undefined — it may open either app. This is a security vulnerability: a malicious app could intercept OAuth callbacks. Always use Universal Links for security-sensitive flows (authentication, payments).

3. Universal Links & AASA

3.1 apple-app-site-association (AASA) File

The AASA file proves your app owns the domain. It must be served at https://example.com/.well-known/apple-app-site-association with Content-Type: application/json.

{
    "applinks": {
        "details": [
            {
                "appIDs": [
                    "TEAMID.com.example.myapp"
                ],
                "components": [
                    {
                        "/": "/products/*",
                        "comment": "Product detail pages"
                    },
                    {
                        "/": "/users/*",
                        "comment": "User profiles"
                    },
                    {
                        "/": "/invite/*",
                        "comment": "Invitation links"
                    },
                    {
                        "/": "/auth/callback",
                        "comment": "OAuth callback"
                    },
                    {
                        "/": "/download",
                        "exclude": true,
                        "comment": "Don't open in app"

            }

    },
    "appclips": {
        "apps": [
            "TEAMID.com.example.myapp.Clip"

    }

3.2 AASA Requirements and Gotchas

Requirement Detail
HTTPS Must be served over HTTPS with valid certificate
No redirects Must be served directly (no 301/302)
Content-Type application/json (not application/pkcs7-mime for unsigned)
Size limit 128 KB maximum
CDN caching Apple's CDN caches the file; updates can take 24-48 hours
Signing No longer required (was required pre-iOS 9)
Entitlement App must have com.apple.developer.associated-domains

3.3 Apple's CDN and Debugging

Deep Dive: How Apple Fetches AASA

Since iOS 14, Apple uses its own CDN (app-site-association.cdn-apple.com) to fetch and cache AASA files. The device never contacts your server directly. This means:

  • Updates to your AASA file propagate through Apple's CDN with a delay (hours to days)
  • You cannot use IP-based access control to restrict AASA access
  • Apple's bot fetches the file periodically; you can't force a refresh
  • For development, use the ?mode=developer associated domains entitlement to bypass the CDN
// Debugging: check if AASA is valid
// curl https://app-site-association.cdn-apple.com/a/v1/example.com

// In entitlements for development (bypasses CDN):
// applinks:example.com?mode=developer

// Xcode: Associated Domains entitlement
// applinks:example.com
// applinks:*.example.com  (wildcard subdomain)

5. Handling in Scene-Based Apps

5.1 UIOpenURLContext

class SceneDelegate: UIResponder, UIWindowSceneDelegate {
    
    // URL arrives when scene is being created
    func scene(_ scene: UIScene,
        willConnectTo session: UISceneSession,
        options: UIScene.ConnectionOptions) {
        
        // Check for URL contexts
        if let urlContext = options.urlContexts.first {
            handle(urlContext: urlContext)
        }
        
        // Check for user activities (Universal Links)
        if let activity = options.userActivities.first {
            handle(activity: activity)
        }
    }
    
    // URL arrives when scene is already running
    func scene(_ scene: UIScene, openURLContexts contexts: Set<UIOpenURLContext>) {
        for context in contexts {
            handle(urlContext: context)
        }
    }
    
    // Universal Link arrives when scene is already running
    func scene(_ scene: UIScene, continue userActivity: NSUserActivity) {
        handle(activity: userActivity)
    }
    
    private func handle(urlContext: UIOpenURLContext) {
        let url = urlContext.url
        let sourceApp = urlContext.options.sourceApplication
        let openInPlace = urlContext.options.openInPlace
        
        DeepLinkRouter.shared.handle(url: url)
    }
}

6. NSUserActivity & Handoff

// Advertise an activity for Handoff
class ArticleViewController: UIViewController {
    var article: Article!
    
    override func viewDidAppear(_ animated: Bool) {
        super.viewDidAppear(animated)
        
        let activity = NSUserActivity(activityType: "com.app.viewArticle")
        activity.title = article.title
        activity.webpageURL = URL(string: "https://example.com/articles/\(article.id)")
        activity.isEligibleForHandoff = true
        activity.isEligibleForSearch = true
        activity.isEligibleForPrediction = true
        activity.userInfo = ["articleID": article.id]
        
        // Required keys for Spotlight indexing
        let attributes = CSSearchableItemAttributeSet(contentType: .text)
        attributes.title = article.title
        attributes.contentDescription = article.summary
        activity.contentAttributeSet = attributes
        
        self.userActivity = activity
        activity.becomeCurrent()
    }
}

7. Deferred Deep Links

Deferred deep links handle the case where a user clicks a link but doesn't have the app installed. After installing from the App Store, the app should still navigate to the linked content.

7.1 The Problem

User clicks link → App not installed → App Store → Install → Launch │ How does the app know which link was clicked?

7.2 Solutions

Approach Mechanism Reliability
Clipboard Write URL to pasteboard; read on first launch Low (iOS 16+ paste permission)
Server fingerprinting Match device fingerprint (IP, UA) on server Medium (privacy concerns)
SKAdNetwork Apple's attribution framework Limited (no deep link data)
App Clips → Full App App Clip stores data; full app reads it High (shared Keychain)
Third-party SDKs Branch, Firebase Dynamic Links, etc. Medium-High (various techniques) 
// Deferred deep link via App Clip → Full App handoff
// In App Clip: save context to shared Keychain
func saveDeepLinkContext(url: URL) {
    let query: [String: Any] = [
        kSecClass as String: kSecClassGenericPassword,
        kSecAttrService as String: "com.app.deeplink",
        kSecAttrAccount as String: "deferred_link",
        kSecValueData as String: url.absoluteString.data(using: .utf8)!,
        kSecAttrAccessGroup as String: "TEAMID.com.app.shared"
    ]
    SecItemAdd(query as CFDictionary, nil)
}

// In Full App: check for deferred link on first launch
func checkDeferredDeepLink() -> URL? {
    let query: [String: Any] = [
        kSecClass as String: kSecClassGenericPassword,
        kSecAttrService as String: "com.app.deeplink",
        kSecAttrAccount as String: "deferred_link",
        kSecReturnData as String: true,
        kSecAttrAccessGroup as String: "TEAMID.com.app.shared"
    ]
    var result: AnyObject?
    let status = SecItemCopyMatching(query as CFDictionary, &result)
    
    guard status == errSecSuccess,
          let data = result as? Data,
          let urlString = String(data: data, encoding: .utf8)
    else { return nil }
    
    // Clean up after reading
    SecItemDelete(query as CFDictionary)
    
    return URL(string: urlString)
}

8. Architect: Routing Architecture at Scale

Architect Must-Know: Building a Deep Link Router for 100+ Screens

At scale, deep link routing is a system design problem. A naive switch statement over URL paths becomes unmaintainable. You need a registration-based router with pattern matching, middleware, authentication guards, and analytics hooks.

8.1 Router Architecture

Incoming URL/Activity │ ▼ ┌───────────────┐ │ URL Parser │ Normalize, extract components └───────┬───────┘ │ ▼ ┌───────────────┐ │ Middleware │ Auth check, feature flags, A/B test │ Pipeline │ analytics tracking, rate limiting └───────┬───────┘ │ ▼ ┌───────────────┐ │ Route Matcher │ Pattern matching against registered routes └───────┬───────┘ │ ▼ ┌───────────────┐ │ Route Handler │ Create VC/View, extract params └───────┬───────┘ │ ▼ ┌───────────────┐ │ Navigator │ Push, present, or reset navigation stack └───────────────┘

8.2 Implementation

// Route definition with pattern matching
struct Route {
    let pattern: String           // "/products/:id"
    let requiresAuth: Bool
    let handler: (RouteMatch) -> UIViewController?
}

struct RouteMatch {
    let params: [String: String]    // ["id": "123"]
    let queryItems: [String: String] // ["ref": "push"]
    let url: URL
}

// Registration-based router
final class DeepLinkRouter {
    static let shared = DeepLinkRouter()
    
    private var routes: [Route] = []
    private var middleware: [RouteMiddleware] = []
    private var fallbackHandler: ((URL) -> Void)?
    
    // Feature modules register their routes
    func register(pattern: String,
                  requiresAuth: Bool = false,
                  handler: @escaping (RouteMatch) -> UIViewController?) {
        routes.append(Route(
            pattern: pattern,
            requiresAuth: requiresAuth,
            handler: handler
        ))
    }
    
    func addMiddleware(_ middleware: RouteMiddleware) {
        self.middleware.append(middleware)
    }
    
    func handle(url: URL) -> Bool {
        // Normalize URL
        let path = normalizePath(url)
        
        // Run middleware pipeline
        for mw in middleware {
            guard mw.shouldContinue(url: url) else { return true }
        }
        
        // Find matching route
        for route in routes {
            if let match = match(path: path, against: route.pattern, url: url) {
                // Auth guard
                if route.requiresAuth && !AuthManager.shared.isAuthenticated {
                    queueForPostAuth(url: url)
                    presentLogin()
                    return true
                }
                
                if let vc = route.handler(match) {
                    navigate(to: vc)
                    return true
                }
            }
        }
        
        // No route matched
        fallbackHandler?(url)
        return false
    }
    
    private func match(path: String, against pattern: String, url: URL) -> RouteMatch? {
        let pathComponents = path.split(separator: "/")
        let patternComponents = pattern.split(separator: "/")
        
        guard pathComponents.count == patternComponents.count else { return nil }
        
        var params: [String: String] = [:]
        
        for (pathComp, patternComp) in zip(pathComponents, patternComponents) {
            if patternComp.hasPrefix(":") {
                let paramName = String(patternComp.dropFirst())
                params[paramName] = String(pathComp)
            } else if pathComp != patternComp {
                return nil
            }
        }
        
        let queryItems = URLComponents(url: url, resolvingAgainstBaseURL: false)?
            .queryItems?
            .reduce(into: [:]) { $0[$1.name] = $1.value } ?? [:]
        
        return RouteMatch(params: params, queryItems: queryItems, url: url)
    }
}

// Middleware protocol
protocol RouteMiddleware {
    func shouldContinue(url: URL) -> Bool
}

// Analytics middleware
struct AnalyticsMiddleware: RouteMiddleware {
    func shouldContinue(url: URL) -> Bool {
        Analytics.track("deep_link_opened", properties: [
            "url": url.absoluteString,
            "source": detectSource(url)
        ])
        return true
    }
}

// Feature flag middleware
struct FeatureFlagMiddleware: RouteMiddleware {
    func shouldContinue(url: URL) -> Bool {
        if url.path.hasPrefix("/new-feature") && !FeatureFlags.newFeatureEnabled {
            return false
        }
        return true
    }
}

8.3 Module Registration Pattern

// Each feature module registers its own routes
protocol DeepLinkRegistrable {
    static func registerRoutes(with router: DeepLinkRouter)
}

// Products module
enum ProductsDeepLinks: DeepLinkRegistrable {
    static func registerRoutes(with router: DeepLinkRouter) {
        router.register(pattern: "/products/:id") { match in
            guard let id = match.params["id"] else { return nil }
            return ProductDetailViewController(productID: id)
        }
        
        router.register(pattern: "/products/:id/reviews") { match in
            guard let id = match.params["id"] else { return nil }
            return ReviewsViewController(productID: id)
        }
        
        router.register(pattern: "/categories/:slug") { match in
            guard let slug = match.params["slug"] else { return nil }
            return CategoryViewController(slug: slug)
        }
    }
}

// At app launch, register all modules
func registerAllRoutes() {
    let router = DeepLinkRouter.shared
    router.addMiddleware(AnalyticsMiddleware())
    router.addMiddleware(FeatureFlagMiddleware())
    
    ProductsDeepLinks.registerRoutes(with: router)
    ProfileDeepLinks.registerRoutes(with: router)
    MessagingDeepLinks.registerRoutes(with: router)
    SettingsDeepLinks.registerRoutes(with: router)
}
Best Practice: Deep Link Testing
  • Test with xcrun simctl openurl booted "myapp://path"
  • Test Universal Links with xcrun simctl openurl booted "https://example.com/path"
  • Validate AASA with Apple's validator: https://search.developer.apple.com/appsearch-validation-tool/
  • Write unit tests for your route matcher with all URL patterns
  • Test cold launch (app not running) vs warm launch (app backgrounded) — different code paths
  • Test with authentication required — does the deferred navigation work after login?
