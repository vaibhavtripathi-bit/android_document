Custom Views & ViewGroups

Building production-grade custom Views — measurement, layout, drawing, state, accessibility, RTL, compound views, and performance

1. When to Create a Custom View

Before writing a custom View, choose the right base class. The wrong choice leads to unnecessary complexity or re-inventing what the framework already provides.

ApproachBase ClassUse Case
Custom drawingViewCharts, gauges, signatures, progress indicators
Modified behaviorExisting subclass (e.g., AppCompatEditText)Phone-number EditText, auto-clear Button
Custom layoutViewGroupFlow layout, circular layout, staggered grid
Compound ViewFrameLayout / LinearLayoutLabeled input, star rating bar, search header

2. Custom Attributes

The Four Constructors

Every View has up to four constructors, each called in a different context:

// 1. Created from code: val view = MyView(context)
constructor(context: Context)

// 2. Inflated from XML: <com.example.MyView ... />
constructor(context: Context, attrs: AttributeSet?)

// 3. Inflated from XML with a default style attribute from the theme
constructor(context: Context, attrs: AttributeSet?, defStyleAttr: Int)

// 4. API 21+: adds defStyleRes for a default style resource
constructor(context: Context, attrs: AttributeSet?, defStyleAttr: Int, defStyleRes: Int)

The attribute resolution order: XML attributes → style attribute on the View → defStyleAttr (theme-level default) → defStyleRes (hardcoded default style) → theme attributes.

Reading Attributes with @JvmOverloads

class CircularGauge @JvmOverloads constructor(
    context: Context,
    attrs: AttributeSet? = null,
    defStyleAttr: Int = 0
) : View(context, attrs, defStyleAttr) {

    private var gaugeColor = Color.BLUE
    private var gaugeMax = 100
    private var gaugeValue = 0
    private var strokeWidth = 12f
    private var label = ""

    init {
        val ta = context.obtainStyledAttributes(attrs, R.styleable.CircularGauge, defStyleAttr, 0)
        try {
            gaugeColor = ta.getColor(R.styleable.CircularGauge_gaugeColor, Color.BLUE)
            gaugeMax = ta.getInt(R.styleable.CircularGauge_gaugeMax, 100)
            gaugeValue = ta.getInt(R.styleable.CircularGauge_gaugeValue, 0)
            strokeWidth = ta.getDimension(R.styleable.CircularGauge_gaugeStrokeWidth, 12f)
            label = ta.getString(R.styleable.CircularGauge_gaugeLabel) ?: ""
        } finally {
            ta.recycle() // MUST recycle — TypedArray is pooled
        }
    }
}
Common Mistake — Forgetting recycle()

TypedArray objects are pooled for performance. Failing to call recycle() leaks the array back to the pool, eventually exhausting it and forcing new allocations. Use try/finally to guarantee cleanup. Android Lint will flag this, but it's a common interview gotcha.

3. Implementing onMeasure()

The measurement contract: the parent tells you constraints via MeasureSpec, and you must call setMeasuredDimension() with your final size.

MeasureSpec Modes

ModeMeaningYour Response
EXACTLYParent dictates exact size (match_parent or fixed dp)Use the spec size
AT_MOSTYou can be up to this size (wrap_content)Use min(desired, specSize)
UNSPECIFIEDNo constraint (ScrollView child, or measuring pass)Use your desired size

Complete onMeasure() for a Circular Gauge

override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
    val desiredSize = (200 * resources.displayMetrics.density).toInt()

    val width = resolveSize(desiredSize, widthMeasureSpec)
    val height = resolveSize(desiredSize, heightMeasureSpec)

    // Force square: use the smaller dimension
    val size = minOf(width, height)
    setMeasuredDimension(size, size)
}

Manual MeasureSpec Handling (When resolveSize Isn't Enough)

override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
    val widthMode = MeasureSpec.getMode(widthMeasureSpec)
    val widthSize = MeasureSpec.getSize(widthMeasureSpec)
    val heightMode = MeasureSpec.getMode(heightMeasureSpec)
    val heightSize = MeasureSpec.getSize(heightMeasureSpec)

    // Calculate desired size based on content
    val contentWidth = calculateContentWidth() + paddingLeft + paddingRight
    val contentHeight = calculateContentHeight() + paddingTop + paddingBottom

    val width = when (widthMode) {
        MeasureSpec.EXACTLY -> widthSize
        MeasureSpec.AT_MOST -> minOf(contentWidth, widthSize)
        else -> contentWidth
    }
    val height = when (heightMode) {
        MeasureSpec.EXACTLY -> heightSize
        MeasureSpec.AT_MOST -> minOf(contentHeight, heightSize)
        else -> contentHeight
    }

    setMeasuredDimension(width, height)
}
Staff-Level Insight — The Default onMeasure() Trap

The default View.onMeasure() calls getDefaultSize(), which returns the spec size for both EXACTLY and AT_MOST. This means a custom View without an onMeasure() override will fill its parent even with wrap_content — a bug that trips up many developers. In AOSP, getDefaultSize() was designed for simplicity, not correctness for custom Views. Always override onMeasure().

4. Implementing onLayout() for Custom ViewGroups

A custom ViewGroup must override onLayout() to position each child. You also need onMeasure() to measure children and compute your own size.

FlowLayout — Wraps Children to the Next Line

class FlowLayout @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null, defStyleAttr: Int = 0
) : ViewGroup(context, attrs, defStyleAttr) {

    private var horizontalSpacing = 8.dp
    private var verticalSpacing = 8.dp

    override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
        val maxWidth = MeasureSpec.getSize(widthMeasureSpec)
        val availableWidth = maxWidth - paddingLeft - paddingRight

        var currentLineWidth = 0
        var currentLineHeight = 0
        var totalWidth = 0
        var totalHeight = 0

        for (i in 0 until childCount) {
            val child = getChildAt(i)
            if (child.visibility == GONE) continue

            measureChildWithMargins(child, widthMeasureSpec, 0, heightMeasureSpec, 0)
            val lp = child.layoutParams as MarginLayoutParams
            val childWidth = child.measuredWidth + lp.leftMargin + lp.rightMargin
            val childHeight = child.measuredHeight + lp.topMargin + lp.bottomMargin

            if (currentLineWidth + childWidth > availableWidth && currentLineWidth > 0) {
                // Wrap to next line
                totalHeight += currentLineHeight + verticalSpacing
                totalWidth = maxOf(totalWidth, currentLineWidth)
                currentLineWidth = childWidth + horizontalSpacing
                currentLineHeight = childHeight
            } else {
                currentLineWidth += childWidth + if (currentLineWidth > 0) horizontalSpacing else 0
                currentLineHeight = maxOf(currentLineHeight, childHeight)
            }
        }
        totalHeight += currentLineHeight
        totalWidth = maxOf(totalWidth, currentLineWidth)

        setMeasuredDimension(
            resolveSize(totalWidth + paddingLeft + paddingRight, widthMeasureSpec),
            resolveSize(totalHeight + paddingTop + paddingBottom, heightMeasureSpec)
        )
    }

    override fun onLayout(changed: Boolean, l: Int, t: Int, r: Int, b: Int) {
        val availableWidth = r - l - paddingLeft - paddingRight
        var x = paddingLeft
        var y = paddingTop
        var lineHeight = 0

        for (i in 0 until childCount) {
            val child = getChildAt(i)
            if (child.visibility == GONE) continue

            val lp = child.layoutParams as MarginLayoutParams
            val childWidth = child.measuredWidth
            val childHeight = child.measuredHeight

            if (x + lp.leftMargin + childWidth + lp.rightMargin > availableWidth + paddingLeft
                && x > paddingLeft) {
                x = paddingLeft
                y += lineHeight + verticalSpacing
                lineHeight = 0
            }

            val childLeft = x + lp.leftMargin
            val childTop = y + lp.topMargin
            child.layout(childLeft, childTop, childLeft + childWidth, childTop + childHeight)

            x += lp.leftMargin + childWidth + lp.rightMargin + horizontalSpacing
            lineHeight = maxOf(lineHeight, childHeight + lp.topMargin + lp.bottomMargin)
        }
    }

    override fun generateLayoutParams(attrs: AttributeSet) = MarginLayoutParams(context, attrs)
    override fun generateDefaultLayoutParams() = MarginLayoutParams(LayoutParams.WRAP_CONTENT, LayoutParams.WRAP_CONTENT)
    override fun checkLayoutParams(p: LayoutParams) = p is MarginLayoutParams

    private val Int.dp get() = (this * resources.displayMetrics.density).toInt()
}

CircularLayout — Positions Children in a Circle

class CircularLayout @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null, defStyleAttr: Int = 0
) : ViewGroup(context, attrs, defStyleAttr) {

    override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
        measureChildren(widthMeasureSpec, heightMeasureSpec)
        val size = resolveSize(400.dp, widthMeasureSpec)
        setMeasuredDimension(size, resolveSize(size, heightMeasureSpec))
    }

    override fun onLayout(changed: Boolean, l: Int, t: Int, r: Int, b: Int) {
        val centerX = (r - l) / 2f
        val centerY = (b - t) / 2f
        val visibleChildren = (0 until childCount).map { getChildAt(it) }
            .filter { it.visibility != GONE }
        val radius = minOf(centerX, centerY) * 0.65f
        val angleStep = (2 * Math.PI / visibleChildren.size).toFloat()

        visibleChildren.forEachIndexed { index, child ->
            val angle = angleStep * index - Math.PI.toFloat() / 2 // Start from top
            val cx = centerX + radius * kotlin.math.cos(angle)
            val cy = centerY + radius * kotlin.math.sin(angle)

            val halfW = child.measuredWidth / 2
            val halfH = child.measuredHeight / 2
            child.layout(
                (cx - halfW).toInt(), (cy - halfH).toInt(),
                (cx + halfW).toInt(), (cy + halfH).toInt()
            )
        }
    }

    private val Int.dp get() = (this * resources.displayMetrics.density).toInt()
}
Key Detail — generateLayoutParams()

If your ViewGroup reads child margins, you must override generateLayoutParams() to return MarginLayoutParams. The default returns plain LayoutParams, which has no margin fields — calling child.layoutParams as MarginLayoutParams will throw a ClassCastException. Also override generateDefaultLayoutParams() and checkLayoutParams().

5. Implementing onDraw()

The Golden Rule: No Allocations in onDraw()

onDraw() is called every frame during animation (up to 120 fps on modern devices). Allocating objects here triggers GC pauses that cause jank. Pre-allocate everything in init {}.

Circular Progress Bar — Complete Implementation

class CircularProgressBar @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null, defStyleAttr: Int = 0
) : View(context, attrs, defStyleAttr) {

    var progress: Float = 0f
        set(value) {
            field = value.coerceIn(0f, maxProgress)
            invalidate()
        }
    var maxProgress: Float = 100f
    var progressColor: Int = Color.BLUE
        set(value) { field = value; progressPaint.color = value; invalidate() }

    // Pre-allocate all drawing objects
    private val backgroundPaint = Paint(Paint.ANTI_ALIAS_FLAG).apply {
        style = Paint.Style.STROKE
        color = Color.LTGRAY
        strokeWidth = 16f
        strokeCap = Paint.Cap.ROUND
    }
    private val progressPaint = Paint(Paint.ANTI_ALIAS_FLAG).apply {
        style = Paint.Style.STROKE
        color = progressColor
        strokeWidth = 16f
        strokeCap = Paint.Cap.ROUND
    }
    private val textPaint = Paint(Paint.ANTI_ALIAS_FLAG).apply {
        color = Color.BLACK
        textSize = 48f
        textAlign = Paint.Align.CENTER
        typeface = Typeface.DEFAULT_BOLD
    }
    private val arcRect = RectF()
    private var gradient: SweepGradient? = null

    override fun onSizeChanged(w: Int, h: Int, oldw: Int, oldh: Int) {
        super.onSizeChanged(w, h, oldw, oldh)
        val strokeHalf = progressPaint.strokeWidth / 2
        arcRect.set(
            paddingLeft + strokeHalf,
            paddingTop + strokeHalf,
            w - paddingRight - strokeHalf,
            h - paddingBottom - strokeHalf
        )
        // Rebuild gradient when size changes
        gradient = SweepGradient(
            arcRect.centerX(), arcRect.centerY(),
            intArrayOf(progressColor, Color.argb(255, 100, 200, 255), progressColor),
            null
        )
        progressPaint.shader = gradient
    }

    override fun onDraw(canvas: Canvas) {
        super.onDraw(canvas)

        // Background circle
        canvas.drawArc(arcRect, 0f, 360f, false, backgroundPaint)

        // Progress arc: starts at -90° (12 o'clock position)
        val sweepAngle = (progress / maxProgress) * 360f
        canvas.drawArc(arcRect, -90f, sweepAngle, false, progressPaint)

        // Center text
        val text = "${(progress / maxProgress * 100).toInt()}%"
        val textY = arcRect.centerY() - (textPaint.descent() + textPaint.ascent()) / 2
        canvas.drawText(text, arcRect.centerX(), textY, textPaint)
    }

    override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
        val desired = (160 * resources.displayMetrics.density).toInt()
        val size = minOf(resolveSize(desired, widthMeasureSpec),
                         resolveSize(desired, heightMeasureSpec))
        setMeasuredDimension(size, size)
    }
}

Bar Chart View — Data-Driven Drawing

class BarChartView @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null, defStyleAttr: Int = 0
) : View(context, attrs, defStyleAttr) {

    data class BarData(val label: String, val value: Float, val color: Int)

    var data: List<BarData> = emptyList()
        set(value) { field = value; invalidate() }

    private val barPaint = Paint(Paint.ANTI_ALIAS_FLAG)
    private val labelPaint = Paint(Paint.ANTI_ALIAS_FLAG).apply {
        textSize = 28f; textAlign = Paint.Align.CENTER; color = Color.DKGRAY
    }
    private val barRect = RectF()

    override fun onDraw(canvas: Canvas) {
        if (data.isEmpty()) return

        val drawWidth = width - paddingLeft - paddingRight
        val drawHeight = height - paddingTop - paddingBottom
        val barWidth = drawWidth / (data.size * 2f)
        val maxValue = data.maxOf { it.value }
        val labelSpace = 40f

        data.forEachIndexed { i, bar ->
            val barHeight = (bar.value / maxValue) * (drawHeight - labelSpace)
            val left = paddingLeft + barWidth * (2 * i + 0.5f)
            barRect.set(left, paddingTop + drawHeight - labelSpace - barHeight,
                        left + barWidth, paddingTop.toFloat() + drawHeight - labelSpace)

            barPaint.color = bar.color
            canvas.drawRoundRect(barRect, 8f, 8f, barPaint)
            canvas.drawText(bar.label, barRect.centerX(),
                            paddingTop + drawHeight.toFloat(), labelPaint)
        }
    }
}

6. Saving and Restoring State

Custom Views must survive configuration changes and process death. The framework calls onSaveInstanceState() / onRestoreInstanceState() on every View that has an android:id.

The SavedState Pattern

class CircularProgressBar ... {

    // ... (fields and drawing code from above)

    override fun onSaveInstanceState(): Parcelable {
        val superState = super.onSaveInstanceState()
        return SavedState(superState).apply {
            savedProgress = progress
            savedMax = maxProgress
            savedColor = progressColor
        }
    }

    override fun onRestoreInstanceState(state: Parcelable?) {
        if (state !is SavedState) {
            super.onRestoreInstanceState(state)
            return
        }
        super.onRestoreInstanceState(state.superState)
        progress = state.savedProgress
        maxProgress = state.savedMax
        progressColor = state.savedColor
    }

    private class SavedState : BaseSavedState {
        var savedProgress = 0f
        var savedMax = 100f
        var savedColor = Color.BLUE

        constructor(superState: Parcelable?) : super(superState)

        constructor(parcel: Parcel) : super(parcel) {
            savedProgress = parcel.readFloat()
            savedMax = parcel.readFloat()
            savedColor = parcel.readInt()
        }

        override fun writeToParcel(out: Parcel, flags: Int) {
            super.writeToParcel(out, flags)
            out.writeFloat(savedProgress)
            out.writeFloat(savedMax)
            out.writeInt(savedColor)
        }

        companion object CREATOR : Parcelable.Creator<SavedState> {
            override fun createFromParcel(parcel: Parcel) = SavedState(parcel)
            override fun newArray(size: Int) = arrayOfNulls<SavedState>(size)
        }
    }
}

Critical Requirements

  • View must have an android:id — the framework uses the ID as the key in the sparse array of saved states. No ID = no save/restore.
  • Duplicate IDs — if two Views in the same layout have the same ID, their saved states collide. The second View restores the first View's state. This is a subtle bug in compound Views that inflate sub-layouts with hardcoded IDs.
  • setSaveEnabled(false) — opt out of state saving for Views whose state is derived from the data layer (e.g., a chart that re-fetches data).
Staff-Level Insight — Parcelize vs BaseSavedState

Modern Kotlin code can use @Parcelize for simpler Parcelable generation, but BaseSavedState is still the correct pattern for View saved state because it chains the superclass state. If you use a plain @Parcelize data class, you must manually handle super.onSaveInstanceState() wrapping. In interviews, showing the BaseSavedState pattern demonstrates you understand the View state restoration chain — the framework restores state top-down through the View hierarchy, and each View must pass unrecognized state to its super.

7. Accessibility

Custom Views are invisible to accessibility services by default. You must explicitly describe your View's semantics.

Making a Custom View Accessible

class CircularProgressBar ... {

    init {
        // Announce as a ProgressBar to screen readers
        importantForAccessibility = IMPORTANT_FOR_ACCESSIBILITY_YES
    }

    override fun onInitializeAccessibilityNodeInfo(info: AccessibilityNodeInfo) {
        super.onInitializeAccessibilityNodeInfo(info)
        info.className = ProgressBar::class.java.name
        info.contentDescription = "$label: ${(progress / maxProgress * 100).toInt()} percent"

        // Range info for progress-type Views
        info.rangeInfo = AccessibilityNodeInfo.RangeInfo.obtain(
            AccessibilityNodeInfo.RangeInfo.RANGE_TYPE_FLOAT,
            0f, maxProgress, progress
        )

        // Custom actions for non-standard interactions
        info.addAction(AccessibilityNodeInfo.AccessibilityAction(
            AccessibilityNodeInfo.AccessibilityAction.ACTION_SCROLL_FORWARD.id,
            "Increase progress"
        ))
        info.addAction(AccessibilityNodeInfo.AccessibilityAction(
            AccessibilityNodeInfo.AccessibilityAction.ACTION_SCROLL_BACKWARD.id,
            "Decrease progress"
        ))
    }

    override fun performAccessibilityAction(action: Int, arguments: Bundle?): Boolean {
        return when (action) {
            AccessibilityNodeInfo.ACTION_SCROLL_FORWARD -> {
                progress = minOf(progress + maxProgress * 0.1f, maxProgress)
                sendAccessibilityEvent(AccessibilityEvent.TYPE_VIEW_SELECTED)
                true
            }
            AccessibilityNodeInfo.ACTION_SCROLL_BACKWARD -> {
                progress = maxOf(progress - maxProgress * 0.1f, 0f)
                sendAccessibilityEvent(AccessibilityEvent.TYPE_VIEW_SELECTED)
                true
            }
            else -> super.performAccessibilityAction(action, arguments)
        }
    }

    // Notify TalkBack when progress changes
    fun setProgressAnimated(target: Float) {
        ValueAnimator.ofFloat(progress, target).apply {
            addUpdateListener { progress = it.animatedValue as Float }
            doOnEnd {
                announceForAccessibility(
                    "Progress updated to ${(progress / maxProgress * 100).toInt()} percent"
                )
            }
            start()
        }
    }
}

AccessibilityDelegate — Without Subclassing

ViewCompat.setAccessibilityDelegate(existingView, object : AccessibilityDelegateCompat() {
    override fun onInitializeAccessibilityNodeInfo(
        host: View, info: AccessibilityNodeInfoCompat
    ) {
        super.onInitializeAccessibilityNodeInfo(host, info)
        info.roleDescription = "Rating control"
        info.stateDescription = "3 out of 5 stars"
    }
})

importantForAccessibility Values

ValueBehavior
IMPORTANT_FOR_ACCESSIBILITY_YESAlways reported to accessibility services
IMPORTANT_FOR_ACCESSIBILITY_NOHidden from accessibility (decorative Views)
IMPORTANT_FOR_ACCESSIBILITY_NO_HIDE_DESCENDANTSHides this View and all children

Staff-Level Insight — Accessibility Is Not Optional at L6

Google's accessibility team participates in L6 domain interviews. If you present a custom View design and don't mention accessibility, it's a red flag. The bar: every interactive custom View must have contentDescription or stateDescription, respond to accessibility actions, and send events on state changes. For compound Views, ensure the accessibility tree is logical — use NO_HIDE_DESCENDANTS on decorative containers and expose a single meaningful node. Test with TalkBack and the Accessibility Scanner before shipping.

8. RTL (Right-to-Left) Support

Layout Direction in Custom ViewGroups

override fun onLayout(changed: Boolean, l: Int, t: Int, r: Int, b: Int) {
    val isRtl = layoutDirection == LAYOUT_DIRECTION_RTL
    val width = r - l

    for (i in 0 until childCount) {
        val child = getChildAt(i)
        if (child.visibility == GONE) continue

        val childLeft = if (isRtl) {
            width - paddingRight - accumulatedX - child.measuredWidth
        } else {
            paddingLeft + accumulatedX
        }
        child.layout(childLeft, childTop,
                     childLeft + child.measuredWidth,
                     childTop + child.measuredHeight)
        // ...
    }
}

Drawing Direction in Custom Views

override fun onDraw(canvas: Canvas) {
    val isRtl = layoutDirection == LAYOUT_DIRECTION_RTL

    if (isRtl) {
        // Mirror the entire canvas for simple cases
        canvas.save()
        canvas.scale(-1f, 1f, width / 2f, height / 2f)
    }

    // Draw as if LTR — the canvas flip handles RTL
    drawProgressArc(canvas)
    drawLabels(canvas)

    if (isRtl) {
        canvas.restore()
    }
}

// For text that shouldn't be mirrored (numbers, labels):
private fun drawLabels(canvas: Canvas) {
    if (layoutDirection == LAYOUT_DIRECTION_RTL) {
        canvas.save()
        canvas.scale(-1f, 1f, width / 2f, height / 2f) // Un-mirror for text
        canvas.drawText(label, x, y, textPaint)
        canvas.restore()
    } else {
        canvas.drawText(label, x, y, textPaint)
    }
}

RTL Checklist

  • android:supportsRtl="true" in the manifest — required for any RTL support
  • Use start/end instead of left/right in XML attributes
  • Use paddingStart/paddingEnd and marginStart/marginEnd
  • In custom onLayout(): check layoutDirection and swap positioning
  • In custom onDraw(): flip coordinates or use canvas mirroring
  • Drawables: provide mirrored versions or use android:autoMirrored="true"
  • Test with adb shell settings put global debug.force_rtl 1

Compound View Class

class LabeledInputView @JvmOverloads constructor(
    context: Context, attrs: AttributeSet? = null, defStyleAttr: Int = 0
) : LinearLayout(context, attrs, defStyleAttr) {

    private val labelView: TextView
    private val inputLayout: TextInputLayout
    private val inputView: TextInputEditText
    private val errorView: TextView

    var labelText: String
        get() = labelView.text.toString()
        set(value) { labelView.text = value }

    var inputText: String
        get() = inputView.text.toString()
        set(value) { inputView.setText(value) }

    var error: String?
        get() = errorView.text?.toString()
        set(value) {
            if (value.isNullOrEmpty()) {
                errorView.visibility = GONE
                inputLayout.error = null
            } else {
                errorView.text = value
                errorView.visibility = VISIBLE
                inputLayout.error = value
            }
        }

    init {
        orientation = VERTICAL
        LayoutInflater.from(context).inflate(R.layout.view_labeled_input, this, true)

        labelView = findViewById(R.id.label)
        inputLayout = findViewById(R.id.inputLayout)
        inputView = findViewById(R.id.input)
        errorView = findViewById(R.id.errorText)

        val ta = context.obtainStyledAttributes(attrs, R.styleable.LabeledInputView)
        try {
            labelView.text = ta.getString(R.styleable.LabeledInputView_labelText) ?: ""
            inputView.hint = ta.getString(R.styleable.LabeledInputView_hintText)
            inputView.inputType = ta.getInt(
                R.styleable.LabeledInputView_android_inputType,
                android.text.InputType.TYPE_CLASS_TEXT
            )
        } finally {
            ta.recycle()
        }
    }

    fun addTextChangedListener(watcher: TextWatcher) {
        inputView.addTextChangedListener(watcher)
    }
}

Why <merge> Matters

Without <merge>, inflating into this creates a redundant ViewGroup:

// Without <merge>: LinearLayout → LinearLayout → children (extra nesting!)
// With <merge>:    LinearLayout → children (flat, correct)

The tools:parentTag attribute tells the layout editor what the parent will be, so preview works correctly even with <merge>.

Measurement Caching

class ExpensiveView ... {
    private var cachedContentWidth = -1
    private var cachedContentHeight = -1
    private var dataDirty = true

    var data: List<Item> = emptyList()
        set(value) {
            field = value
            dataDirty = true
            requestLayout() // Triggers re-measure
            invalidate()
        }

    override fun onMeasure(widthMeasureSpec: Int, heightMeasureSpec: Int) {
        if (dataDirty) {
            cachedContentWidth = computeExpensiveWidth()
            cachedContentHeight = computeExpensiveHeight()
            dataDirty = false
        }
        setMeasuredDimension(
            resolveSize(cachedContentWidth + paddingLeft + paddingRight, widthMeasureSpec),
            resolveSize(cachedContentHeight + paddingTop + paddingBottom, heightMeasureSpec)
        )
    }
}
Performance Trap — invalidate() vs requestLayout()

invalidate() triggers only a redraw (onDraw()). requestLayout() triggers the full measure → layout → draw pipeline. Calling requestLayout() when only visual state changed (e.g., color) wastes an entire layout pass. Conversely, calling only invalidate() when size changed means the View draws at the wrong size. Know which to call: data that affects size → requestLayout(); data that affects appearance only → invalidate().

Profiling Custom Views

  • GPU Profiling: adb shell setprop debug.hwui.profile true — shows per-frame GPU render time
  • Systrace / Perfetto: trace view and gfx categories to see measure/layout/draw timing
  • Layout Inspector: verify your View hierarchy depth and identify redundant nesting
  • Overdraw visualization: Developer Options → Debug GPU Overdraw — custom Views should minimize layered drawing
  • Allocation tracking: Android Studio Profiler → record allocations during scroll/animation to catch onDraw() allocations
Staff-Level Insight — When Custom Views Beat Compose

In an L6 interview, you might be asked when a custom View is preferable to a Compose custom layout. The answer: (1) when you need pixel-precise Canvas control with complex shader/path operations that don't map cleanly to Compose's DrawScope, (2) when integrating with legacy View-based libraries (MapView, WebView, SurfaceView), (3) when the View is used in a RemoteViews context (widgets, notifications) where Compose isn't supported. For everything else, Compose's Layout composable and Canvas composable are the modern equivalent with less boilerplate.
