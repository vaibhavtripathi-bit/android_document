Series 1: App & Scene Lifecycle

From pre-main dyld loading to multi-window scene management — everything a Staff iOS engineer must know about how an iOS app starts, runs, backgrounds, and dies.

This series covers the complete lifecycle of an iOS application at the depth expected for a Google L6 / Staff-level interview. We trace the journey from the moment a user taps an app icon through dyld loading, UIKit bootstrap, scene creation, backgrounding, process suspension, and eventual termination by Jetsam. Each document dives into internals that distinguish a Staff engineer from a Senior one.

Documents in This Series

01 — AppDelegate Evolution

UIApplicationDelegate from iOS 2 to SwiftUI @main. Pre-main boot sequence, dyld, +load/+initialize, UIApplicationMain internals, and the full path from icon tap to first frame.

02 — Scene Lifecycle

UIScene, UISceneSession, UIWindowScene, and multi-window architecture on iPad. Scene activation states, configuration, connection/disconnection semantics, and state restoration via NSUserActivity.

03 — Process Lifecycle

Background execution model, beginBackgroundTask, BGTaskScheduler, background modes, Jetsam OOM killer, process suspension vs termination, and designing reliable background work.

04 — State Restoration

NSCoder-based legacy restoration, NSUserActivity-based restoration, SwiftUI @SceneStorage, restorationIdentifier chains, and building robust restoration across app updates.

05 — Deep Links & Universal Links

URL Schemes vs Universal Links vs App Clips. AASA file format, UIOpenURLContext, deferred deep links, and building a scalable deep link router for 100+ screens.

06 — Extensions Lifecycle

App Extension process model, memory limits, App Groups for IPC, shared containers, and sharing code between app and extensions without bloating the extension binary.

07 — Interview Q&A

60+ L6-level interview questions with detailed answers covering AppDelegate evolution, scene lifecycle edge cases, process death, background execution, multi-scene architecture, state restoration, deep linking, and extensions.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6

Scene-based lifecycle Multi-window broke singleton patterns; architects must design shared state for multi-scene

State restoration Three different APIs across eras; must work across app updates without data loss
Deep link routing At scale (100+ screens), routing architecture is a system design problem, not a feature
Extension architecture Separate processes, strict memory limits, shared data via App Groups — real IPC constraints
