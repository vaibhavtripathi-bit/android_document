1. UIGraphicsImageRenderer vs Legacy API

Apple introduced UIGraphicsImageRenderer in iOS 10 as a replacement for the legacy UIGraphicsBeginImageContext family. The new API is superior in every way.

Why the Legacy API is Bad

Issue Legacy API UIGraphicsImageRenderer
Color space sRGB only Wide color (Display P3) automatic
Scale Must specify manually Automatic (@1x, @2x, @3x)
Pixel format Always 32-bit RGBA Automatic optimal format
Memory Always allocates bitmap Can use extended sRGB, optimized formats
API safety Must balance begin/end calls Block-based, auto-cleanup
HDR support No Yes (iOS 17+) 
// LEGACY (DON'T USE):
UIGraphicsBeginImageContextWithOptions(size, false, 0)
// ... drawing code ...
let image = UIGraphicsGetImageFromCurrentImageContext()
UIGraphicsEndImageContext()
// Problems:
// - Must remember to call End (crash if forgotten)
// - Scale 0 means "use screen scale" but doesn't work in extensions
// - Always sRGB — loses wide color from P3 displays
// - Always 32-bit — wastes memory for grayscale content

// MODERN (USE THIS):
let renderer = UIGraphicsImageRenderer(size: size)
let image = renderer.image { context in
    // context.cgContext is the underlying CGContext
    // context.currentImage gives the image so far
    // context.format has the pixel format info

    UIColor.systemBlue.setFill()
    context.fill(CGRect(origin: .zero, size: size))
}
// Automatic cleanup, wide color, correct scale, optimal format

// Custom format:
let format = UIGraphicsImageRendererFormat()
format.scale = 2.0                    // Force @2x
format.opaque = true                   // No alpha channel (saves memory)
format.preferredRange = .standard      // sRGB (or .extended for wide color)

let renderer = UIGraphicsImageRenderer(size: size, format: format)

// Getting PNG/JPEG data directly:
let pngData = renderer.pngData { context in
    // drawing code
}

let jpegData = renderer.jpegData(withCompressionQuality: 0.8) { context in
    // drawing code
}

2. CGContext Drawing

// CGContext is the Core Graphics drawing context
// It represents a 2D drawing destination (bitmap, PDF, etc.)

class ChartView: UIView {
    var dataPoints: [CGFloat] = []

    override func draw(_ rect: CGRect) {
        guard let context = UIGraphicsGetCurrentContext() else { return }

        // State management — save/restore is a stack
        context.saveGState()

        // Coordinate system: origin is top-left in UIKit
        // (Core Graphics native origin is bottom-left)
        // UIKit flips the context for you in draw(_:)

        // Fill background
        context.setFillColor(UIColor.systemBackground.cgColor)
        context.fill(rect)

        // Draw grid lines
        context.setStrokeColor(UIColor.separator.cgColor)
        context.setLineWidth(0.5)
        for y in stride(from: 0, through: rect.height, by: 40) {
            context.move(to: CGPoint(x: 0, y: y))
            context.addLine(to: CGPoint(x: rect.width, y: y))
        }
        context.strokePath()

        // Draw data line
        context.setStrokeColor(UIColor.systemBlue.cgColor)
        context.setLineWidth(2)
        context.setLineCap(.round)
        context.setLineJoin(.round)

        guard dataPoints.count > 1 else { return }

        let stepX = rect.width / CGFloat(dataPoints.count - 1)
        let maxValue = dataPoints.max() ?? 1

        context.move(to: CGPoint(
            x: 0,
            y: rect.height - (dataPoints[0] / maxValue) * rect.height
        ))

        for (index, value) in dataPoints.enumerated().dropFirst() {
            context.addLine(to: CGPoint(
                x: CGFloat(index) * stepX,
                y: rect.height - (value / maxValue) * rect.height
            ))
        }
        context.strokePath()

        // Draw gradient fill under the line
        let colors = [
            UIColor.systemBlue.withAlphaComponent(0.3).cgColor,
            UIColor.systemBlue.withAlphaComponent(0).cgColor
        ] as CFArray
        let gradient = CGGradient(
            colorsSpace: CGColorSpaceCreateDeviceRGB(),
            colors: colors,
            locations: [0, 1]
        )!

        context.clip()  // Clip to the path we just drew
        context.drawLinearGradient(
            gradient,
            start: CGPoint(x: 0, y: 0),
            end: CGPoint(x: 0, y: rect.height),
            options: []
        )

        context.restoreGState()
    }
}

CGContext State Stack

// CGContext maintains a state stack
// saveGState() pushes, restoreGState() pops
// State includes: CTM, clip region, line width, stroke/fill color,
// blend mode, font, alpha, shadow, color space, etc.

context.saveGState()       // Push state
context.setAlpha(0.5)      // Modify state
context.translateBy(x: 50, y: 50)
// ... draw with modified state ...
context.restoreGState()    // Pop — back to original state

// IMPORTANT: Always balance save/restore calls
// Unbalanced calls lead to undefined behavior

3. draw(_:) vs draw(in:) Methods

// UIView.draw(_ rect: CGRect)
// Called by the system when the view needs to redraw
// The rect parameter is the dirty rect (area that needs redrawing)
// A CGContext is already set up for you

class CustomView: UIView {
    override func draw(_ rect: CGRect) {
        // rect is the dirty rect — optimize by only drawing in this area
        // For most views, rect == bounds (full redraw)
        // For partial redraws: setNeedsDisplay(rect) limits the dirty rect
    }
}

// CALayer.draw(in ctx: CGContext)
// Called on the layer's delegate when the layer needs content
// You must draw into the provided context

class LayerDelegate: NSObject, CALayerDelegate {
    func draw(_ layer: CALayer, in ctx: CGContext) {
        // Draw directly into the context
        ctx.setFillColor(UIColor.red.cgColor)
        ctx.fill(layer.bounds)
    }
}

// CALayer.display(_ layer: CALayer)
// Alternative delegate method — you provide the contents directly
// More efficient when you already have a bitmap

class EfficientDelegate: NSObject, CALayerDelegate {
    func display(_ layer: CALayer) {
        layer.contents = preRenderedImage.cgImage
    }
}

// UIImage drawing methods:
let image = UIImage(named: "photo")!

// draw(at:) — draws at natural size at the specified point
image.draw(at: CGPoint(x: 10, y: 10))

// draw(in:) — scales to fill the specified rect
image.draw(in: CGRect(x: 0, y: 0, width: 100, height: 100))

// draw(in:blendMode:alpha:) — with blending and transparency
image.draw(in: rect, blendMode: .multiply, alpha: 0.8)

4. PDF Rendering

// Creating PDF documents with UIGraphicsPDFRenderer
let pdfRenderer = UIGraphicsPDFRenderer(
    bounds: CGRect(x: 0, y: 0, width: 612, height: 792)  // US Letter
)

let pdfData = pdfRenderer.pdfData { context in
    // Page 1
    context.beginPage()

    let title = "Report"
    let attrs: [NSAttributedString.Key: Any] = [
        .font: UIFont.boldSystemFont(ofSize: 24),
        .foregroundColor: UIColor.black
    ]
    title.draw(at: CGPoint(x: 72, y: 72), withAttributes: attrs)

    // Draw chart
    let chartRect = CGRect(x: 72, y: 120, width: 468, height: 300)
    drawChart(in: chartRect, context: context.cgContext)

    // Page 2
    context.beginPage()
    // ... more content ...
}

// Save to file
try? pdfData.write(to: fileURL)

// Rendering an existing PDF page
guard let document = CGPDFDocument(pdfURL as CFURL),
      let page = document.page(at: 1) else { return }

let pageRect = page.getBoxRect(.mediaBox)

let renderer = UIGraphicsImageRenderer(size: pageRect.size)
let pageImage = renderer.image { context in
    UIColor.white.setFill()
    context.fill(pageRect)

    let ctx = context.cgContext
    ctx.translateBy(x: 0, y: pageRect.height)
    ctx.scaleBy(x: 1, y: -1)  // Flip for PDF coordinate system
    ctx.drawPDFPage(page)
}

5. UIBezierPath

// UIBezierPath is a UIKit wrapper around CGPath
// It provides a simpler API for common path operations

// Basic shapes
let rect = UIBezierPath(rect: bounds)
let oval = UIBezierPath(ovalIn: bounds)
let roundedRect = UIBezierPath(
    roundedRect: bounds,
    byRoundingCorners: [.topLeft, .topRight],
    cornerRadii: CGSize(width: 12, height: 12)
)

// Custom path
let path = UIBezierPath()
path.move(to: CGPoint(x: 0, y: 50))
path.addLine(to: CGPoint(x: 50, y: 0))
path.addLine(to: CGPoint(x: 100, y: 50))
path.addCurve(
    to: CGPoint(x: 0, y: 50),
    controlPoint1: CGPoint(x: 80, y: 100),
    controlPoint2: CGPoint(x: 20, y: 100)
)
path.close()

// Drawing in a view
override func draw(_ rect: CGRect) {
    UIColor.systemBlue.setFill()
    UIColor.systemBlue.setStroke()
    path.lineWidth = 2
    path.fill()
    path.stroke()
}

// Using as a CAShapeLayer path (more efficient for animations)
let shapeLayer = CAShapeLayer()
shapeLayer.path = path.cgPath
shapeLayer.fillColor = UIColor.systemBlue.cgColor
shapeLayer.strokeColor = UIColor.white.cgColor
shapeLayer.lineWidth = 2
view.layer.addSublayer(shapeLayer)

// Path operations
let contains = path.contains(CGPoint(x: 50, y: 50))  // Hit testing
path.apply(CGAffineTransform(scaleX: 2, y: 2))       // Transform
path.append(otherPath)                                 // Combine paths

6. TextKit 1 vs TextKit 2

TextKit 1 Architecture (iOS 7-16): NSTextStorage ──▶ NSLayoutManager ──▶ NSTextContainer ──▶ UITextView (model) (layout engine) (geometry) (display) • NSTextStorage: subclass of NSMutableAttributedString • NSLayoutManager: converts text + attributes → glyph positions • NSTextContainer: defines the region where text is laid out • One-to-many: 1 storage → N layout managers → N containers TextKit 2 Architecture (iOS 16+): NSTextContentStorage ──▶ NSTextLayoutManager ──▶ NSTextContainer (model) (layout engine) (geometry) │ │ ▼ ▼ NSTextElement NSTextLayoutFragment NSTextParagraph (viewport-based layout) Key difference: TextKit 2 uses VIEWPORT-BASED layout Only lays out text visible on screen (lazy layout) TextKit 1 lays out ALL text eagerly (expensive for long documents) 
// TextKit 1 — still widely used
let textStorage = NSTextStorage(attributedString: attributedText)
let layoutManager = NSLayoutManager()
let textContainer = NSTextContainer(size: containerSize)

textStorage.addLayoutManager(layoutManager)
layoutManager.addTextContainer(textContainer)

// Measuring text size
layoutManager.ensureLayout(for: textContainer)
let usedRect = layoutManager.usedRect(for: textContainer)

// Finding glyph at point (for tap-on-link detection)
let glyphIndex = layoutManager.glyphIndex(
    for: tapPoint,
    in: textContainer,
    fractionOfDistanceThroughGlyph: nil
)
let charIndex = layoutManager.characterIndexForGlyph(at: glyphIndex)

// TextKit 2 — iOS 16+
let contentStorage = NSTextContentStorage()
contentStorage.attributedString = attributedText

let textLayoutManager = NSTextLayoutManager()
contentStorage.addTextLayoutManager(textLayoutManager)

let container = NSTextContainer(size: containerSize)
textLayoutManager.textContainer = container

// Viewport-based enumeration (only visible fragments)
textLayoutManager.enumerateTextLayoutFragments(
    from: textLayoutManager.documentRange.location,
    options: [.ensuresLayout]
) { fragment in
    let frame = fragment.layoutFragmentFrame
    // Only fragments in the visible viewport are laid out
    return true  // continue enumeration
}
Feature TextKit 1 TextKit 2
Layout model Eager (all text) Viewport-based (lazy)
Performance for long docs O(n) — lays out everything O(visible) — only visible text
Glyph access Direct glyph manipulation No direct glyph access
Custom layout NSLayoutManagerDelegate NSTextLayoutManagerDelegate
Non-contiguous layout Optional (buggy) Default (core design)
Availability iOS 7+ iOS 16+

7. NSAttributedString Performance

// NSAttributedString creation is expensive — cache when possible

// BAD: Creating attributed strings in cellForRow
func tableView(_ tableView: UITableView,
               cellForRowAt indexPath: IndexPath) -> UITableViewCell {
    let cell = tableView.dequeueReusableCell(withIdentifier: "cell")!
    // This is called every time a cell is dequeued — expensive!
    cell.textLabel?.attributedText = NSAttributedString(
        string: items[indexPath.row].title,
        attributes: [
            .font: UIFont.systemFont(ofSize: 16, weight: .bold),
            .foregroundColor: UIColor.label
        ]
    )
    return cell
}

// GOOD: Pre-compute attributed strings
class ViewModel {
    let attributedTitle: NSAttributedString

    init(title: String) {
        attributedTitle = NSAttributedString(
            string: title,
            attributes: [
                .font: UIFont.systemFont(ofSize: 16, weight: .bold),
                .foregroundColor: UIColor.label
            ]
        )
    }
}

// Measuring attributed string size
let size = attributedString.boundingRect(
    with: CGSize(width: maxWidth, height: .greatestFiniteMagnitude),
    options: [.usesLineFragmentOrigin, .usesFontLeading],
    context: nil
).size

// IMPORTANT: Always use .usesLineFragmentOrigin for multi-line text
// Without it, you get single-line measurement

8. Architect: Custom Drawing at Scale

Architect Must-Know

When to use Core Graphics vs CAShapeLayer vs Metal:

Approach Best For Avoid When
Core Graphics (draw(_:)) Static complex content, charts, custom text layout Content changes every frame (expensive redraw)
CAShapeLayer Animated vector paths, progress indicators, simple shapes Very complex paths with thousands of points
CATextLayer Simple text rendering without UILabel overhead Complex text layout (use TextKit instead)
Metal (CAMetalLayer) Real-time rendering, particle systems, image processing Simple UI elements (massive overkill)
UIGraphicsImageRenderer Off-screen image generation, thumbnails, exports On-screen real-time drawing

Key principle: Prefer compositing (layers) over drawing (Core Graphics). Compositing is GPU-accelerated and doesn't allocate backing stores. Only use draw(_:) when you need pixel-level control that layers can't provide.

Best Practice

For a charting library at scale: render the chart data path as a CAShapeLayer (GPU-accelerated, animatable), use CATextLayer for axis labels (no backing store), and only use Core Graphics draw(_:) for the final export-to-image feature. This gives you smooth 120fps interactions with minimal memory overhead.
