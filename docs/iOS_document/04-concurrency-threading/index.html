Series 4: Concurrency & Threading

From libdispatch internals and thread pools to Swift's actor model and structured concurrency — the complete concurrency story every Staff iOS engineer must command.

Concurrency is the single most dangerous area of iOS development. Data races, priority inversions, deadlocks, and thread explosions have brought down production apps at every scale. This series covers the full concurrency stack: the legacy GCD and OperationQueue foundations, the modern async/await and actor model, reactive streams via Combine and AsyncSequence, and the often-misunderstood RunLoop that underpins UIKit's main thread. Each document targets the depth expected at a Google L6 / Staff-level interview — not just "how to use it" but "how it works under the hood and when it breaks."

Decision Framework: Which Concurrency Tool?

Scenario Best Tool Why
Simple background work async/await + Task Structured, cancellable, no thread explosion
Parallel fan-out TaskGroup Bounded concurrency, automatic child cancellation
Complex dependency graph OperationQueue Explicit dependencies, cancellation propagation, KVO
Protecting mutable state actor Compile-time isolation, no manual locking
Reactive event streams Combine / AsyncSequence Backpressure, operator composition, cancellation
Fire-and-forget (careful!) Task.detached No parent context — use sparingly, loses structured benefits
Low-level system events DispatchSource File descriptors, signals, timers at kernel level
UI updates @MainActor Compile-time guarantee of main thread execution

Architect Must-Know

At the Staff level, you're expected to make concurrency architecture decisions for entire teams. The key insight: there is no single "right" concurrency tool. Modern iOS apps use GCD, OperationQueue, async/await, actors, and Combine simultaneously. Your job is to define clear boundaries — which layer uses which tool, how they interop, and how to prevent the combinatorial complexity from becoming a maintenance nightmare. The best architects establish a "concurrency contract" for their codebase: e.g., "all new code uses async/await + actors; GCD is only in the networking layer; Combine is only in the ViewModel binding layer."
