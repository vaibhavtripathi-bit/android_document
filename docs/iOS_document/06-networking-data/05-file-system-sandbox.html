File System & App Sandbox

Every iOS app lives inside a sandbox — a private directory structure that isolates its data from other apps and the system. Understanding the sandbox layout, file protection classes, and cross-process sharing mechanisms is essential for building secure, performant, and data-resilient applications at scale.

1. App Sandbox Structure

When iOS installs an app it creates a unique container directory with a UUID-based path. The container is the app's entire universe — it cannot read or write outside of it (with narrow exceptions like App Groups and Keychain access groups).

/var/mobile/Containers/ ├── Bundle/ │ └── Application/ │ └── <UUID>/ │ └── MyApp.app/ ← READ-ONLY app bundle │ ├── MyApp (executable) │ ├── Info.plist │ ├── Assets.car │ ├── Base.lproj/ │ └── embedded.mobileprovision │ └── Data/ └── Application/ └── <UUID>/ ← READ-WRITE data container ├── Documents/ ← User data, backed up ├── Library/ │ ├── Caches/ ← Purgeable, NOT backed up │ ├── Preferences/ ← UserDefaults .plist, backed up │ └── Application Support/← App data, backed up ├── tmp/ ← Temporary, NOT backed up └── SystemData/ ← System-managed

Backup Behavior Summary

Directory Backed Up (iTunes/iCloud) Purged by System Typical Contents
Documents/ Yes No User-created files, exported data
Library/Application Support/ Yes No App-generated databases, config
Library/Preferences/ Yes No UserDefaults plist
Library/Caches/ No Yes Downloaded images, API responses
tmp/ No Yes Transient scratch files

Getting Each Directory Path

import Foundation

let fm = FileManager.default

// Documents
let docs = fm.urls(for: .documentDirectory, in: .userDomainMask).first!

// Library/Caches
let caches = fm.urls(for: .cachesDirectory, in: .userDomainMask).first!

// Library/Application Support
let appSupport = fm.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!

// Library (parent)
let library = fm.urls(for: .libraryDirectory, in: .userDomainMask).first!

// Preferences (Library/Preferences)
let prefs = library.appendingPathComponent("Preferences")

// Temporary directory
let tmp1 = fm.temporaryDirectory
let tmp2 = NSTemporaryDirectory()  // returns String

// App bundle
let bundle = Bundle.main.bundleURL
Deep Dive: Container UUIDs Change

The UUID in the container path changes every time the app is reinstalled and can change during updates. Never persist absolute paths — always resolve directory URLs at runtime via FileManager.

2. Bundle Directory

The .app bundle is a signed, read-only directory containing the compiled executable, storyboards, asset catalogs, localized strings, embedded frameworks, and the Info.plist. Any attempt to write to the bundle directory will fail at runtime and violate code signing.

Accessing Bundle Resources

// Path to a JSON file shipped with the app
guard let url = Bundle.main.url(
    forResource: "defaults",
    withExtension: "json"
) else {
    fatalError("Missing defaults.json in bundle")
}

let data = try Data(contentsOf: url)
let config = try JSONDecoder().decode(AppConfig.self, from: data)

// Image from asset catalog
let image = UIImage(named: "onboarding-hero")

// Localized string
let greeting = NSLocalizedString(
    "welcome_message",
    comment: "Greeting shown on first launch"
)

Localized Resources

// Explicitly request a specific localization
if let path = Bundle.main.path(
    forResource: "Localizable",
    ofType: "strings",
    inDirectory: nil,
    forLocalization: "ja"
) {
    let jaStrings = NSDictionary(contentsOfFile: path) as? [String: String]
}

// List all localizations the bundle supports
let localizations = Bundle.main.localizations  // ["en", "ja", "de", ...]
let preferred = Bundle.main.preferredLocalizations  // ordered by user prefs
Pitfall: Writing to the Bundle

Attempting to write files into Bundle.main.bundlePath will crash on device (the directory is code-signed and mounted read-only). On the Simulator it may appear to work, masking bugs that only surface in production. Always write to Documents or Application Support instead.

3. Documents Directory

Documents/ is for user-generated or user-facing content that cannot be re-created by the app. It is backed up by iTunes and iCloud, and can optionally be exposed to the Files app and iTunes File Sharing.

Enabling Files App & iTunes File Sharing

Two Info.plist keys control visibility:

Key Effect
UIFileSharingEnabled Exposes Documents/ in iTunes/Finder file sharing panel
LSSupportsOpeningDocumentsInPlace Lets other apps open files directly from your Documents/ via the Files app (no copy)

Working with the Documents Directory

func saveUserDocument(_ text: String, filename: String) throws {
    let docs = FileManager.default.urls(
        for: .documentDirectory,
        in: .userDomainMask
    ).first!

    let fileURL = docs.appendingPathComponent(filename)

    try text.write(to: fileURL, atomically: true, encoding: .utf8)
}

func loadUserDocument(_ filename: String) throws -> String {
    let docs = FileManager.default.urls(
        for: .documentDirectory,
        in: .userDomainMask
    ).first!

    let fileURL = docs.appendingPathComponent(filename)
    return try String(contentsOf: fileURL, encoding: .utf8)
}

// Exclude a file from iCloud backup
func excludeFromBackup(_ url: URL) throws {
    var resourceURL = url
    var values = URLResourceValues()
    values.isExcludedFromBackup = true
    try resourceURL.setResourceValues(values)
}
Best Practice

Only store genuinely user-created content in Documents/. App-generated databases, caches, and configuration belong in Library/. Apple's App Review may reject apps that bloat iCloud backups with re-downloadable data stored in Documents/.

4. Library Directory

Library/ contains three critical subdirectories, each with distinct backup and purging semantics.

Library/Caches

Store data that can be re-downloaded or regenerated. The system may purge this directory when the device is low on storage — even while your app is suspended. Data here is not backed up. 

func cacheImageData(_ data: Data, key: String) throws {
    let caches = FileManager.default.urls(
        for: .cachesDirectory,
        in: .userDomainMask
    ).first!

    let imageDir = caches.appendingPathComponent("ImageCache")
    try FileManager.default.createDirectory(
        at: imageDir,
        withIntermediateDirectories: true
    )

    let fileURL = imageDir.appendingPathComponent(key)
    try data.write(to: fileURL, options: [.atomic])
}

Library/Preferences

Managed by UserDefaults. Do not write files here directly — the system synchronizes the .plist file on its own schedule.

Library/Application Support

The right place for app-generated data that is not user-facing but cannot be re-created (or is expensive to re-create). This directory is backed up. You must create it yourself — it does not exist by default. 

func appSupportDirectory() throws -> URL {
    let fm = FileManager.default
    let url = fm.urls(
        for: .applicationSupportDirectory,
        in: .userDomainMask
    ).first!

    if !fm.fileExists(atPath: url.path) {
        try fm.createDirectory(at: url, withIntermediateDirectories: true)
    }
    return url
}

// Store a Core Data SQLite database here
let storeURL = try appSupportDirectory()
    .appendingPathComponent("Model.sqlite")
Pitfall: Caches Purging While Backgrounded

The system can purge Library/Caches/ at any time when the device is low on storage — even while your app is in the background. Always code defensively: check for file existence before reading and be prepared to re-download or regenerate cached data.

5. Temporary Directory

tmp/ is for truly transient data — files you need for seconds or minutes, not across app launches. The system periodically purges this directory when the app is not running. It is not backed up. 

// Two equivalent ways to get the tmp directory
let tmp1 = FileManager.default.temporaryDirectory
let tmp2 = URL(fileURLWithPath: NSTemporaryDirectory())

// Create a unique temporary file for a video export
func temporaryVideoURL() -> URL {
    let filename = UUID().uuidString + ".mp4"
    return FileManager.default.temporaryDirectory
        .appendingPathComponent(filename)
}

// Clean up temp files proactively
func cleanupTempFiles() {
    let fm = FileManager.default
    let tmpDir = fm.temporaryDirectory

    guard let contents = try? fm.contentsOfDirectory(
        at: tmpDir,
        includingPropertiesForKeys: [.creationDateKey]
    ) else { return }

    let cutoff = Date().addingTimeInterval(-3600)  // 1 hour ago

    for url in contents {
        let values = try? url.resourceValues(forKeys: [.creationDateKey])
        if let created = values?.creationDate, created < cutoff {
            try? fm.removeItem(at: url)
        }
    }
}
Deep Dive: tmp vs Caches

Use tmp/ for data needed only during the current session (e.g., intermediate video frames, upload staging). Use Library/Caches/ for data that should survive across launches but can be regenerated (e.g., thumbnail caches, API response caches). The system purges tmp/ more aggressively than Caches/.

6. File Protection Classes

iOS encrypts files on disk using hardware-backed keys tied to the user's passcode. File protection classes control when decryption keys are available, letting you balance security against background-access needs.

Protection Class Key Available Use Case
NSFileProtectionComplete Only while device is unlocked Highly sensitive data (health records, financial data)
NSFileProtectionCompleteUnlessOpen Unlocked to create/open; remains accessible if file handle is open when device locks Large downloads that continue in background
NSFileProtectionCompleteUntilFirstUserAuthentication After first unlock until reboot (default for most files) General app data, databases
NSFileProtectionNone Always (even before first unlock) Data needed during boot (e.g., VPN config) — avoid if possible
Device State Timeline & Key Availability BOOT ──────── FIRST UNLOCK ──────── LOCK ──────── UNLOCK ──────── LOCK │ │ │ │ │ │ None: ✅ │ None: ✅ │ None: ✅ │ None: ✅ │ None: ✅ │ UntilFirst: ❌ │ UntilFirst: ✅ │ Until: ✅ │ Until: ✅ │ Until: ✅ │ UnlessOpen: ❌ │ UnlessOpen: ✅ │ Open: ⚠️* │ Unless: ✅ │ Open: ⚠️* │ Complete: ❌ │ Complete: ✅ │ Comp: ❌ │ Comp: ✅ │ Comp: ❌ │ │ │ │ │ * ⚠️ = accessible only if file handle was opened before lock

Setting File Protection

// Set protection when writing
let sensitiveData = "SSN: 123-45-6789".data(using: .utf8)!
try sensitiveData.write(
    to: fileURL,
    options: [.atomic, .completeFileProtection]
)

// Set protection via FileManager attributes
try FileManager.default.setAttributes(
    [.protectionKey: FileProtectionType.complete],
    ofItemAtPath: fileURL.path
)

// Set protection on a directory (applies to new files created within)
try FileManager.default.createDirectory(
    at: secureDir,
    withIntermediateDirectories: true,
    attributes: [.protectionKey: FileProtectionType.complete]
)

// Read current protection level
let attrs = try FileManager.default.attributesOfItem(atPath: fileURL.path)
let protection = attrs[.protectionKey] as? FileProtectionType
Pitfall: Background Access with Complete Protection

If your app uses NSFileProtectionComplete and attempts to read files during a background fetch or push notification while the device is locked, the read will fail with a permission error. Use CompleteUntilFirstUserAuthentication for data that background tasks need, or use CompleteUnlessOpen and keep a file handle open before the device locks.

Deep Dive: Data Protection Entitlement

The Data Protection entitlement in your provisioning profile sets the default protection class for all files your app creates. Most apps should use NSFileProtectionComplete as the default and selectively downgrade specific files that need background access. Enable it in Xcode under Signing & Capabilities → Data Protection.

7. FileManager Operations

FileManager is the primary API for file system operations. While the .default singleton is safe to use from any thread for individual operations, delegate-based enumeration requires a dedicated instance on its own thread.

Creating Directories

let fm = FileManager.default

// Create nested directory structure
let nestedDir = fm.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!
    .appendingPathComponent("Data/v2/migrations")

try fm.createDirectory(
    at: nestedDir,
    withIntermediateDirectories: true,  // creates parent dirs
    attributes: nil
)

Copying, Moving, and Removing

// Copy a file
try fm.copyItem(at: sourceURL, to: destinationURL)

// Move (rename) a file
try fm.moveItem(at: oldURL, to: newURL)

// Remove a file or directory
try fm.removeItem(at: fileURL)

// Safe removal — don't throw if file doesn't exist
if fm.fileExists(atPath: fileURL.path) {
    try fm.removeItem(at: fileURL)
}

File Attributes

let attrs = try fm.attributesOfItem(atPath: fileURL.path)
let fileSize = attrs[.size] as? UInt64 ?? 0
let modified = attrs[.modificationDate] as? Date
let created = attrs[.creationDate] as? Date

// Using URL resource values (preferred modern API)
let resourceValues = try fileURL.resourceValues(forKeys: [
    .fileSizeKey,
    .contentModificationDateKey,
    .isDirectoryKey,
    .totalFileAllocatedSizeKey
])
let size = resourceValues.fileSize
let isDir = resourceValues.isDirectory ?? false

Directory Enumeration

// Shallow enumeration
let contents = try fm.contentsOfDirectory(
    at: directoryURL,
    includingPropertiesForKeys: [.fileSizeKey, .contentModificationDateKey],
    options: [.skipsHiddenFiles]
)

// Deep enumeration with enumerator (memory-efficient for large trees)
if let enumerator = fm.enumerator(
    at: directoryURL,
    includingPropertiesForKeys: [.fileSizeKey, .isDirectoryKey],
    options: [.skipsHiddenFiles]
) {
    var totalSize: UInt64 = 0
    for case let fileURL as URL in enumerator {
        let values = try fileURL.resourceValues(forKeys: [.fileSizeKey])
        totalSize += UInt64(values.fileSize ?? 0)
    }
    print("Total: \(totalSize) bytes")
}

Atomic Writes

// Atomic write ensures the file is either fully written or not at all
let jsonData = try JSONEncoder().encode(model)
try jsonData.write(to: fileURL, options: [.atomic])

// For strings
try csvString.write(to: fileURL, atomically: true, encoding: .utf8)

// Under the hood: writes to a temp file, then renames (POSIX rename is atomic)
Deep Dive: Thread Safety

FileManager.default is safe to call from multiple threads for individual operations (copy, move, remove, attributes). However, if you use the delegate property for callbacks during operations, you must create a dedicated FileManager() instance and use it from a single thread. For high-throughput file I/O, dispatch work to a serial queue with a dedicated FileManager instance.

8. App Groups Shared Container

App Groups enable data sharing between your main app and its extensions (widgets, share extensions, notification service extensions, etc.). Each group gets a shared container directory outside the normal sandbox.

┌─────────────────────────┐ ┌─────────────────────────┐ │ Main App │ │ Widget Extension │ │ Sandbox: /Data/App/A/ │ │ Sandbox: /Data/App/B/ │ └────────────┬────────────┘ └────────────┬────────────┘ │ │ └───────────┬───────────────────┘ │ ┌──────────▼──────────┐ │ Shared Container │ │ group.com.app.data │ │ │ │ ├── Library/ │ │ ├── Documents/ │ │ └── shared.sqlite │ └─────────────────────┘

Setup and Usage

// 1. Add App Groups capability in Xcode
//    Signing & Capabilities → + App Groups
//    Group ID: "group.com.mycompany.myapp"

// 2. Access the shared container
let sharedContainer = FileManager.default.containerURL(
    forSecurityApplicationGroupIdentifier: "group.com.mycompany.myapp"
)!

// 3. Write shared data
let sharedFile = sharedContainer.appendingPathComponent("shared-state.json")
let data = try JSONEncoder().encode(widgetData)
try data.write(to: sharedFile, options: [.atomic])

// 4. Shared UserDefaults
let sharedDefaults = UserDefaults(suiteName: "group.com.mycompany.myapp")!
sharedDefaults.set(42, forKey: "widgetRefreshCount")

// 5. Shared Core Data store
let storeURL = sharedContainer.appendingPathComponent("Model.sqlite")
let description = NSPersistentStoreDescription(url: storeURL)
Pitfall: Data Consistency in Shared Containers

The main app and extensions can run simultaneously. If both write to the same file or SQLite database concurrently, you risk data corruption. Use NSFileCoordinator for file-level coordination, or enable WAL mode with proper locking for SQLite. For Core Data, use NSPersistentStoreRemoteChangeNotification with persistent history tracking to merge changes safely.

Coordinated File Access

func writeSharedData(_ data: Data, to url: URL) throws {
    let coordinator = NSFileCoordinator()
    var coordError: NSError?
    var writeError: Error?

    coordinator.coordinate(
        writingItemAt: url,
        options: [.forReplacing],
        error: &coordError
    ) { coordURL in
        do {
            try data.write(to: coordURL, options: [.atomic])
        } catch {
            writeError = error
        }
    }

    if let error = coordError { throw error }
    if let error = writeError { throw error }
}

9. iCloud Document Sync

iOS offers multiple levels of iCloud integration, from simple key-value storage to full document sync with conflict resolution.

NSUbiquitousKeyValueStore

A simple key-value store synced across devices via iCloud. Limited to 1 MB total and 1024 keys. Ideal for lightweight preferences and state. 

let kvStore = NSUbiquitousKeyValueStore.default

// Write
kvStore.set(true, forKey: "hasCompletedOnboarding")
kvStore.set(5, forKey: "currentLevel")
kvStore.synchronize()  // hint to sync (not guaranteed immediate)

// Observe external changes
NotificationCenter.default.addObserver(
    forName: NSUbiquitousKeyValueStore.didChangeExternallyNotification,
    object: kvStore,
    queue: .main
) { notification in
    guard let userInfo = notification.userInfo,
          let reason = userInfo[NSUbiquitousKeyValueStoreChangeReasonKey] as? Int,
          let keys = userInfo[NSUbiquitousKeyValueStoreChangedKeysKey] as? [String]
    else { return }

    switch reason {
    case NSUbiquitousKeyValueStoreServerChange:
        // Merge server changes
        for key in keys {
            let value = kvStore.object(forKey: key)
            // Update local state...
        }
    case NSUbiquitousKeyValueStoreQuotaViolationChange:
        // Over 1 MB limit — prune data
        break
    default: break
    }
}

NSFileCoordinator & NSFilePresenter

class DocumentPresenter: NSObject, NSFilePresenter {
    let presentedItemURL: URL?

    init(url: URL) {

    }

        // File was modified by another process (e.g., iCloud sync)
        // Reload content on main queue
        DispatchQueue.main.async {
            self.reloadDocument()
        }
    }

    func presentedItemDidMoveToURL(_ newURL: URL) {
        // File was renamed or moved
    }

    deinit {
        NSFileCoordinator.removeFilePresenter(self)
    }

    private func reloadDocument() { /* ... */ }
}

10. UIDocument

UIDocument provides a document lifecycle with automatic background reading/writing, autosave, conflict detection, and iCloud integration. It is the foundation for document-based apps on iOS.

Complete UIDocument Subclass

class NoteDocument: UIDocument {

    var text: String = ""
    var metadata: NoteMetadata = NoteMetadata()

    // MARK: - Serialization (called on background queue)

    override func contents(forType typeName: String) throws -> Any {
        // Called when saving — return data to write to disk
        let wrapper = FileWrapper(directoryWithFileWrappers: [:])

        let textData = text.data(using: .utf8)!
        wrapper.addRegularFile(
            withContents: textData,
            preferredFilename: "content.txt"
        )

        let metaData = try JSONEncoder().encode(metadata)
        wrapper.addRegularFile(
            withContents: metaData,
            preferredFilename: "metadata.json"
        )

        return wrapper
    }

    override func load(fromContents contents: Any, ofType typeName: String?) throws {
        // Called when opening — parse data from disk
        guard let wrapper = contents as? FileWrapper,
              let textWrapper = wrapper.fileWrappers?["content.txt"],
              let textData = textWrapper.regularFileContents
        else {
            throw CocoaError(.fileReadCorruptFile)
        }

        self.text = String(data: textData, encoding: .utf8) ?? ""

        if let metaWrapper = wrapper.fileWrappers?["metadata.json"],
           let metaData = metaWrapper.regularFileContents {
            self.metadata = (try? JSONDecoder().decode(
                NoteMetadata.self, from: metaData
            )) ?? NoteMetadata()
        }
    }

    // MARK: - Conflict Resolution

    override func handleError(_ error: Error, userInteractionPermitted: Bool) {
        if userInteractionPermitted {
            // Show alert to user
        }
        super.handleError(error, userInteractionPermitted: userInteractionPermitted)
    }

    func resolveConflicts() {
        let versions = NSFileVersion.unresolvedConflictVersionsOfItem(at: fileURL) ?? []
        for version in versions {
            version.isResolved = true
        }
        try? NSFileVersion.removeOtherVersionsOfItem(at: fileURL)
    }
}

struct NoteMetadata: Codable {
    var title: String = "Untitled"
    var createdAt: Date = Date()
    var wordCount: Int = 0
}

Document Lifecycle

// Open a document
let doc = NoteDocument(fileURL: fileURL)
doc.open { success in
    guard success else { return }
    // doc.text is now loaded
    print(doc.text)
}

// Trigger autosave after edits
doc.text = "Updated content"
doc.updateChangeCount(.done)
// UIDocument will autosave within ~10 seconds

// Explicit save
doc.save(to: fileURL, for: .forOverwriting) { success in
    print("Saved: \(success)")
}

// Close the document
doc.close { success in
    print("Closed: \(success)")
}
Best Practice

Always call updateChangeCount(.done) after modifying document properties rather than calling save(to:for:) directly. This lets UIDocument coalesce multiple rapid edits into a single background save, avoiding disk thrashing and improving battery life.

11. On-Demand Resources

On-Demand Resources (ODR) let you tag assets in your Xcode project and download them only when needed. This reduces initial download size and lets the system purge unused resources when storage is low.

Asset Tags

Tag Category Behavior Example
Initial Install Tags Downloaded with the app Core UI assets, level 1 data
Prefetch Tag Order Downloaded after install completes Level 2-3 data, tutorial videos
On-Demand Tags Downloaded only when requested Level 50+ data, seasonal content

Requesting and Using Resources

class LevelLoader {
    private var resourceRequest: NSBundleResourceRequest?

    func loadLevel(_ level: Int) async throws {
        let tag = "level-\(level)"
        let request = NSBundleResourceRequest(tags: [tag])
        self.resourceRequest = request  // retain to keep resources

        request.loadingPriority = NSBundleResourceRequestLoadingPriorityUrgent

        // Check if already available
        let available = await request.conditionallyBeginAccessingResources()

        if !available {
            // Download required
            try await request.beginAccessingResources()
        }

        // Resources are now available in the main bundle
        guard let levelURL = Bundle.main.url(
            forResource: "level-\(level)",
            withExtension: "json"
        ) else {
            throw LevelError.missingData
        }

        let data = try Data(contentsOf: levelURL)
        // Parse level data...
    }

    func unloadLevel() {
        resourceRequest?.endAccessingResources()
        resourceRequest = nil
    }

    // Monitor download progress
    func downloadWithProgress(tag: String) -> AsyncStream<Double> {
        let request = NSBundleResourceRequest(tags: [tag])
        self.resourceRequest = request

        return AsyncStream { continuation in
            let observation = request.progress.observe(
                \.fractionCompleted
            ) { progress, _ in
                continuation.yield(progress.fractionCompleted)
            }

            Task {
                do {
                    try await request.beginAccessingResources()
                    continuation.yield(1.0)
                } catch {
                    // Handle error
                }
                continuation.finish()
                observation.invalidate()
            }
        }
    }
}

enum LevelError: Error {
    case missingData
}
Deep Dive: ODR Purging

The system can purge on-demand resources when storage is low. You must call beginAccessingResources() every time before using tagged assets — even if they were previously downloaded. Holding a reference to the NSBundleResourceRequest prevents purging while your code is actively using those resources. Call endAccessingResources() when done.

Best Practice

Set preservationPriority (0.0 to 1.0) on your resource requests to hint which tags the system should purge first. Give high priority to resources the user is likely to need next (e.g., the next game level) and low priority to resources they've already consumed.

12. File Organization Strategy for Large Local Data

Apps that manage gigabytes of local data — offline maps, media libraries, ML models — need a disciplined storage architecture. This section presents a production-grade FileStorageManager that handles directory layout, cache eviction, storage monitoring, background cleanup, and schema migration.

Directory Layout

Application Support/ ├── v2/ ← Schema version │ ├── db/ │ │ └── app.sqlite ← Core database │ ├── media/ │ │ ├── images/ ← Organized by hash prefix │ │ │ ├── 0a/ │ │ │ ├── 1b/ │ │ │ └── .../ │ │ └── video/ │ └── models/ │ └── ml-v3.mlmodelc │ Library/Caches/ ├── thumbnails/ ← Purgeable image thumbnails │ ├── 0a/ │ └── .../ ├── api-responses/ ← HTTP cache └── processed/ ← Intermediate processing output tmp/ ├── uploads/ ← Staging for uploads └── exports/ ← Temporary export files

Complete FileStorageManager

import Foundation
import os

final class FileStorageManager: @unchecked Sendable {

    static let shared = FileStorageManager()

    private let logger = Logger(subsystem: "com.app", category: "Storage")
    private let queue = DispatchQueue(label: "com.app.storage", qos: .utility)
    private let fm = FileManager()
    private let schemaVersion = "v2"

    // MARK: - Directory Roots

    private lazy var appSupport: URL = {
        let url = FileManager.default.urls(
            for: .applicationSupportDirectory, in: .userDomainMask
        ).first!.appendingPathComponent(schemaVersion)
        try? fm.createDirectory(at: url, withIntermediateDirectories: true)
        return url
    }()

    private lazy var cachesRoot: URL = {
        let url = FileManager.default.urls(
            for: .cachesDirectory, in: .userDomainMask
        ).first!
        return url
    }()

    var databaseURL: URL { appSupport.appendingPathComponent("db/app.sqlite") }
    var mediaRoot: URL    { appSupport.appendingPathComponent("media") }
    var thumbnailCache: URL { cachesRoot.appendingPathComponent("thumbnails") }

    // MARK: - Storage Monitoring

    struct StorageReport {
        let appSupportBytes: UInt64
        let cacheBytes: UInt64
        let tmpBytes: UInt64
        let availableDiskBytes: UInt64

        var totalAppBytes: UInt64 { appSupportBytes + cacheBytes + tmpBytes }
        var isLowDisk: Bool { availableDiskBytes < 500_000_000 }  // < 500 MB
    }

    func generateStorageReport() async -> StorageReport {
        await withCheckedContinuation { continuation in
            queue.async { [self] in
                let appSize = directorySize(appSupport)
                let cacheSize = directorySize(cachesRoot)
                let tmpSize = directorySize(fm.temporaryDirectory)
                let available = availableDiskSpace()

                continuation.resume(returning: StorageReport(
                    appSupportBytes: appSize,
                    cacheBytes: cacheSize,
                    tmpBytes: tmpSize,
                    availableDiskBytes: available
                ))
            }
        }
    }

    private func directorySize(_ url: URL) -> UInt64 {
        guard let enumerator = fm.enumerator(
            at: url,
            includingPropertiesForKeys: [.totalFileAllocatedSizeKey],
            options: [.skipsHiddenFiles]
        ) else { return 0 }

        var total: UInt64 = 0
        for case let fileURL as URL in enumerator {
            let values = try? fileURL.resourceValues(
                forKeys: [.totalFileAllocatedSizeKey]
            )
            total += UInt64(values?.totalFileAllocatedSize ?? 0)
        }
        return total
    }

    private func availableDiskSpace() -> UInt64 {
        let home = URL(fileURLWithPath: NSHomeDirectory())
        let values = try? home.resourceValues(
            forKeys: [.volumeAvailableCapacityForImportantUsageKey]
        )
        return UInt64(values?.volumeAvailableCapacityForImportantUsage ?? 0)
    }

    // MARK: - Cache Eviction (LRU)

    func evictCaches(targetFreeBytes: UInt64 = 200_000_000) async {
        await withCheckedContinuation { continuation in
            queue.async { [self] in
                defer { continuation.resume() }

                guard let enumerator = fm.enumerator(
                    at: cachesRoot,
                    includingPropertiesForKeys: [
                        .contentAccessDateKey,
                        .totalFileAllocatedSizeKey,
                        .isDirectoryKey
                    ],
                    options: [.skipsHiddenFiles]
                ) else { return }

                var files: [(URL, Date, UInt64)] = []
                for case let url as URL in enumerator {
                    guard let v = try? url.resourceValues(forKeys: [
                        .contentAccessDateKey, .totalFileAllocatedSizeKey, .isDirectoryKey
                    ]),
                    v.isDirectory != true
                    else { continue }

                    let accessed = v.contentAccessDate ?? Date.distantPast
                    let size = UInt64(v.totalFileAllocatedSize ?? 0)
                    files.append((url, accessed, size))
                }

                // Sort oldest-accessed first (LRU)
                files.sort { $0.1 < $1.1 }

                var freed: UInt64 = 0
                for (url, _, size) in files {
                    guard freed < targetFreeBytes else { break }
                    do {
                        try fm.removeItem(at: url)
                        freed += size
                        logger.debug("Evicted \(url.lastPathComponent) (\(size) bytes)")
                    } catch {
                        logger.error("Eviction failed: \(error)")
                    }
                }
                logger.info("Cache eviction freed \(freed) bytes")
            }
        }
    }

    // MARK: - Background Cleanup

    func performBackgroundCleanup() async {
        let report = await generateStorageReport()

        if report.isLowDisk {
            logger.warning("Low disk detected — aggressive eviction")
            await evictCaches(targetFreeBytes: 500_000_000)
        } else {
            // Routine cleanup: remove tmp files older than 1 hour
            cleanTmpDirectory(olderThan: 3600)
        }
    }

    private func cleanTmpDirectory(olderThan seconds: TimeInterval) {
        let cutoff = Date().addingTimeInterval(-seconds)
        guard let contents = try? fm.contentsOfDirectory(
            at: fm.temporaryDirectory,
            includingPropertiesForKeys: [.creationDateKey]
        ) else { return }

        for url in contents {
            let created = (try? url.resourceValues(
                forKeys: [.creationDateKey]
            ))?.creationDate
            if let created, created < cutoff {
                try? fm.removeItem(at: url)
            }
        }
    }

    // MARK: - Schema Migration

    func migrateIfNeeded() throws {
        let base = FileManager.default.urls(
            for: .applicationSupportDirectory, in: .userDomainMask
        ).first!

        let v1Dir = base.appendingPathComponent("v1")
        let v2Dir = base.appendingPathComponent("v2")

        guard fm.fileExists(atPath: v1Dir.path),
              !fm.fileExists(atPath: v2Dir.path)
        else { return }

        logger.info("Migrating storage from v1 → v2")

        try fm.createDirectory(at: v2Dir, withIntermediateDirectories: true)

        // Move database
        let oldDB = v1Dir.appendingPathComponent("app.sqlite")
        let newDB = v2Dir.appendingPathComponent("db/app.sqlite")
        try fm.createDirectory(
            at: newDB.deletingLastPathComponent(),
            withIntermediateDirectories: true
        )
        if fm.fileExists(atPath: oldDB.path) {
            try fm.moveItem(at: oldDB, to: newDB)
        }

        // Move media
        let oldMedia = v1Dir.appendingPathComponent("media")
        let newMedia = v2Dir.appendingPathComponent("media")
        if fm.fileExists(atPath: oldMedia.path) {
            try fm.moveItem(at: oldMedia, to: newMedia)
        }

        // Clean up v1
        try? fm.removeItem(at: v1Dir)
        logger.info("Migration v1 → v2 complete")
    }

    // MARK: - Hashed Subdirectory Storage

    func mediaURL(forKey key: String, extension ext: String) -> URL {
        let prefix = String(key.prefix(2))
        let dir = mediaRoot
            .appendingPathComponent("images")
            .appendingPathComponent(prefix)
        try? fm.createDirectory(at: dir, withIntermediateDirectories: true)
        return dir.appendingPathComponent("\(key).\(ext)")
    }
}
Pitfall: Ignoring Storage Pressure

Apps that accumulate unbounded cache data without eviction will be terminated by the system or flagged in Settings → Storage. Monitor disk usage with volumeAvailableCapacityForImportantUsageKey and subscribe to UIApplication.didReceiveMemoryWarningNotification to trigger cleanup. Apple's review guidelines require apps to provide a way to clear cached data.

Best Practice

Use hash-prefix subdirectories (e.g., images/0a/, images/1b/) when storing thousands of files. File systems degrade with too many entries in a single directory. A two-character hex prefix distributes files across 256 subdirectories, keeping directory enumeration fast even at scale.

Architect Must-Know: File Storage Strategy

At L6/Staff level, you own the storage architecture for the entire app. Key decisions:

  • Directory taxonomy: Separate backed-up data (Application Support) from purgeable data (Caches) from transient data (tmp). Misclassification bloats backups or loses user data.
  • Schema versioning: Version your directory layout (e.g., v1/, v2/) so migrations can run atomically — copy data to the new layout, validate, then delete the old one.
  • Cache eviction policy: Implement LRU or LFU eviction with a configurable budget. Monitor volumeAvailableCapacityForImportantUsageKey and trigger aggressive eviction below a threshold (e.g., 500 MB free).
  • Background cleanup: Use BGProcessingTask to schedule periodic cleanup. Respect the system's CPU and energy budgets.
  • File protection: Default to NSFileProtectionComplete and selectively downgrade files that background extensions need. Audit protection levels during security reviews.
  • Shared containers: Use NSFileCoordinator or persistent history tracking for any data shared between the main app and extensions. Never assume exclusive access.
  • Testing: Simulate low-disk conditions in the Simulator (Settings → General → Storage). Write integration tests that verify migration paths from every shipped schema version.
