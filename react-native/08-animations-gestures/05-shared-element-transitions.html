Shared Element Transitions

Shared element transitions create visual continuity between screens by animating a common element from its position on one screen to its position on the next. Implementing these in React Native requires understanding navigation integration, cross-screen measurement, and platform-specific rendering differences.

1. Shared Element Concept

Shared Element Transition Flow ════════════════════════════════════════════════════ Screen A (List) Screen B (Detail) ┌─────────────────┐ ┌─────────────────┐ │ ┌───┐ │ │ │ │ │img│ Title 1 │ │ ┌───────────┐ │ │ └───┘ │ │ │ │ │ │ ┌───┐ │ navigate │ │ image │ │ │ │img│ Title 2 │ ──────────→ │ │ │ │ │ └───┘ │ │ └───────────┘ │ │ ┌───┐ │ │ Title 2 │ │ │img│ Title 3 │ │ Description... │ └─────────────────┘ └─────────────────┘ The image appears to "fly" from its position in the list to its position in the detail screen. Under the hood: 1. Measure source element position/size on Screen A 2. Measure target element position/size on Screen B 3. Clone the element to an overlay layer 4. Animate clone from source → target position/size 5. Hide source and target during animation 6. Remove clone, show target when animation completes

2. Reanimated Shared Transitions

Reanimated v3 provides built-in shared element transition support through the sharedTransitionTag prop, which pairs elements across screens by matching tags.

import Animated, { SharedTransition, withSpring } from 'react-native-reanimated';

// Screen A: List item with shared element tag
function ListScreen({ navigation }) {
  return (
    <FlatList
      data={items}
      renderItem={({ item }) => (
        <Pressable onPress={() => navigation.navigate('Detail', { item })}>
          <Animated.Image
            source={{ uri: item.imageUrl }}
            style={styles.thumbnail}
            sharedTransitionTag={`image-${item.id}`}
          />
          <Animated.Text
            style={styles.title}
            sharedTransitionTag={`title-${item.id}`}
          >
            {item.title}
          </Animated.Text>
        </Pressable>
      )}
    />
  );
}

// Screen B: Detail with matching shared element tags
function DetailScreen({ route }) {
  const { item } = route.params;

  return (
    <ScrollView>
      <Animated.Image
        source={{ uri: item.imageUrl }}
        style={styles.heroImage}
        sharedTransitionTag={`image-${item.id}`}
      />
      <Animated.Text
        style={styles.detailTitle}
        sharedTransitionTag={`title-${item.id}`}
      >
        {item.title}
      </Animated.Text>
      <Text>{item.description}</Text>
    </ScrollView>
  );
}

2.1 Custom Shared Transition

const customTransition = SharedTransition.custom((values) => {
  'worklet';
  return {
    originX: withSpring(values.targetOriginX),
    originY: withSpring(values.targetOriginY),
    width: withSpring(values.targetWidth),
    height: withSpring(values.targetHeight),
    borderRadius: withSpring(values.targetBorderRadius ?? 0),
  };
});

<Animated.Image
  source={{ uri: item.imageUrl }}
  style={styles.thumbnail}
  sharedTransitionTag={`image-${item.id}`}
  sharedTransitionStyle={customTransition}
/>
Key Insight

Reanimated's shared transitions work by matching sharedTransitionTag strings across screens. When navigation occurs, Reanimated measures both elements, creates a transition layer, and animates between the two positions/sizes. The animation runs entirely on the UI thread via worklets. Tags must be unique across the entire app — if two visible elements share the same tag, behavior is undefined.

3. Navigation Integration

import { createNativeStackNavigator } from '@react-navigation/native-stack';

const Stack = createNativeStackNavigator();

function AppNavigator() {
  return (
    <Stack.Navigator>
      <Stack.Screen
        name="List"
        component={ListScreen}
      />
      <Stack.Screen
        name="Detail"
        component={DetailScreen}
        options={{
          animation: 'fade',
          // Shared transitions work best with native stack
          // and simple transition animations
        }}
      />
    </Stack.Navigator>
  );
}

// For react-native-shared-element library approach:
import { createSharedElementStackNavigator } from 'react-navigation-shared-element';

const SharedStack = createSharedElementStackNavigator();

<SharedStack.Screen
  name="Detail"
  component={DetailScreen}
  sharedElements={(route) => {
    const { item } = route.params;
    return [
      { id: `image.${item.id}`, animation: 'move', resize: 'clip' },
      { id: `title.${item.id}`, animation: 'fade' },
    ];
  }}
/>

// Hero animation: image expands from thumbnail to full-screen
function PhotoGallery() {
  const [selectedPhoto, setSelectedPhoto] = useState<string | null>(null);

  return (
    <View>
      <FlatList
        data={photos}
        numColumns={3}
        renderItem={({ item }) => (
          <Pressable onPress={() => setSelectedPhoto(item.id)}>
            <Animated.Image
              source={{ uri: item.thumbnailUrl }}
              style={styles.gridItem}
              sharedTransitionTag={`photo-${item.id}`}
            />
          </Pressable>
        )}
      />

      {selectedPhoto && (
        <Pressable
          style={StyleSheet.absoluteFill}
          onPress={() => setSelectedPhoto(null)}
        >
          <Animated.Image
            source={{ uri: getFullUrl(selectedPhoto) }}
            style={styles.fullScreenImage}
            sharedTransitionTag={`photo-${selectedPhoto}`}
          />
        </Pressable>
      )}
    </View>
  );
}

5. Custom Page Transitions

import { TransitionPresets } from '@react-navigation/stack';

// Custom card-style transition
const cardTransition = {
  gestureDirection: 'horizontal',
  transitionSpec: {
    open: { animation: 'spring', config: { stiffness: 1000, damping: 500, mass: 3 } },
    close: { animation: 'spring', config: { stiffness: 1000, damping: 500, mass: 3 } },
  },
  cardStyleInterpolator: ({ current, next, layouts }) => ({
    cardStyle: {
      transform: [
        {
          translateX: current.progress.interpolate({
            inputRange: [0, 1],
            outputRange: [layouts.screen.width, 0],
          }),
        },
        {
          scale: next
            ? next.progress.interpolate({
                inputRange: [0, 1],
                outputRange: [1, 0.95],
              })
            : 1,
        },
      ],
    },
    overlayStyle: {
      opacity: current.progress.interpolate({
        inputRange: [0, 1],
        outputRange: [0, 0.5],
      }),
    },
  }),
};

6. iOS vs Android Differences

Aspect iOS Android
Native support UIKit transition coordinators ActivityTransition / Fragment transitions
Clipping Smooth corner radius animation May show clipping artifacts
Text transitions Font size animation possible Font size changes may flash
Image transitions Smooth resize with content mode May need explicit resizeMode handling
Back gesture Interactive swipe-back with shared elements System back button (no gesture)
Performance Generally smoother (Core Animation) May need hardware layer hints
Warning

Shared element transitions with text are particularly challenging cross-platform. Font rendering differs between iOS and Android — the same text at the same font size can have different pixel dimensions. Animating text size during a shared transition can cause visible reflow. For text shared elements, prefer fade transitions over move transitions, or use fixed-size text containers.

8. Architect Considerations

Staff-Level Mental Model

Shared element transitions are the most visible indicator of a polished mobile app, but they're also the most fragile animation pattern in React Native. The core challenge is cross-screen element measurement: you need the exact position and size of an element on two different screens, both of which may not be fully laid out simultaneously. Reanimated's sharedTransitionTag approach handles this by measuring elements during the navigation transition. A Staff engineer understands the limitations: shared transitions work best with images and simple shapes, struggle with text and complex layouts, and have platform-specific rendering differences. The pragmatic approach is to use shared transitions for hero images (high visual impact, simple geometry) and fall back to screen-level transitions (fade, slide) for everything else.
