Architect Insight

Reanimated's core innovation is running a second JavaScript VM on the UI thread. This VM executes worklet functions — small JS functions that have been serialized and transferred from the main JS thread. Because they run on the UI thread, worklets can read gesture state, compute styles, and update views at 60fps without any cross-thread communication. This is fundamentally different from the Animated API's approach of pre-computing animation frames on the JS thread and sending them to native.

2. Worklet Concept & Threading

A worklet is a JavaScript function that can be executed on the UI thread. Reanimated's Babel plugin identifies worklet functions (marked with 'worklet' directive or used in worklet contexts) and serializes them for the UI thread VM.

2.1 Worklet Compilation

// Source code (what you write)
function clamp(value, min, max) {
  'worklet';
  return Math.min(Math.max(value, min), max);
}

// After Reanimated Babel plugin transforms it:
// 1. The function body is serialized to a string
// 2. Closure variables are captured
// 3. A __workletHash is generated for caching
// 4. The serialized worklet is sent to the UI thread VM via JSI

// Conceptually, the plugin produces:
const clamp = {
  __closure: {},
  __workletHash: 12345,
  __initData: {
    code: 'function clamp(value,min,max){return Math.min(Math.max(value,min),max);}',
  },
  __stackDetails: { /* source map info */ },
};

2.2 runOnUI and runOnJS

import { runOnUI, runOnJS } from 'react-native-reanimated';

// Run a function on the UI thread from JS thread
const updateNative = () => {
  'worklet';
  // This runs on UI thread — can access shared values directly
  sharedValue.value = 42;
};
runOnUI(updateNative)();

// Run a function on JS thread from UI thread (worklet)
const showAlert = (message: string) => {
  Alert.alert('Animation Complete', message);
};

const onAnimationEnd = () => {
  'worklet';
  // Can't call Alert.alert directly in a worklet
  // Must bounce back to JS thread
  runOnJS(showAlert)('Done!');
};
Warning

Worklets have a limited execution environment. They can't access React state, call React hooks, use most Node.js APIs, or access variables from the JS thread closure unless those variables are serializable or are shared values. The runOnJS bridge is asynchronous — there's no guarantee when the JS thread will execute the callback. Design worklets to be self-contained with all data passed through shared values.

3. Shared Values

Shared values are the communication channel between the JS thread and the UI thread. They're accessible from both threads via JSI, with no serialization overhead.

import { useSharedValue, withTiming, withSpring } from 'react-native-reanimated';

function AnimatedComponent() {
  const offset = useSharedValue(0);
  const isActive = useSharedValue(false);
  const color = useSharedValue('#FF0000');

  // Read/write from JS thread
  const handlePress = () => {
    offset.value = withSpring(100);  // Animate to 100
    isActive.value = true;
  };

  // Read/write from UI thread (worklet)
  const animatedStyle = useAnimatedStyle(() => {
    // This worklet runs on UI thread every frame
    return {
      transform: [{ translateX: offset.value }],
      backgroundColor: isActive.value ? 'blue' : 'gray',
    };
  });

  return (
    <Animated.View style={[styles.box, animatedStyle]}>
      <Pressable onPress={handlePress}>
        <Text>Tap me</Text>
      </Pressable>
    </Animated.View>
  );
}

3.1 Animation Modifiers

import {
  withTiming, withSpring, withDecay,
  withDelay, withSequence, withRepeat,
  Easing,
} from 'react-native-reanimated';

// Timing animation
offset.value = withTiming(100, {
  duration: 300,
  easing: Easing.bezier(0.25, 0.1, 0.25, 1),
});

// Spring animation
offset.value = withSpring(100, {
  damping: 15,
  stiffness: 150,
  mass: 1,
});

// Sequence: one after another
offset.value = withSequence(
  withTiming(50, { duration: 200 }),
  withTiming(100, { duration: 200 }),
);

// Repeat with reverse (bounce effect)
scale.value = withRepeat(
  withTiming(1.2, { duration: 500 }),
  -1,    // infinite repetitions
  true,  // reverse on each iteration
);

// Delay
offset.value = withDelay(500, withSpring(100));

// Callback on completion
offset.value = withTiming(100, {}, (finished) => {
  'worklet';
  if (finished) {
    runOnJS(onComplete)();
  }
});

4. useAnimatedStyle

import { useAnimatedStyle, interpolate, interpolateColor } from 'react-native-reanimated';

function Card({ index }: { index: number }) {
  const progress = useSharedValue(0);

  const animatedStyle = useAnimatedStyle(() => {
    // Full JS expressiveness — conditionals, math, interpolation
    const scale = interpolate(
      progress.value,
      [0, 0.5, 1],
      [1, 1.1, 1]
    );

    const backgroundColor = interpolateColor(
      progress.value,
      [0, 1],
      ['#FFFFFF', '#4A90D9']
    );

    const rotate = `${progress.value * 360}deg`;

    return {
      transform: [
        { scale },
        { rotate },
        { translateY: progress.value * -20 },
      ],
      backgroundColor,
      opacity: progress.value > 0.5 ? 1 : 0.5 + progress.value,
      shadowOpacity: interpolate(progress.value, [0, 1], [0, 0.3]),
    };
  });

  return <Animated.View style={[styles.card, animatedStyle]} />;
}

5. useAnimatedProps & Reactions

import { useAnimatedProps, useDerivedValue, useAnimatedReaction } from 'react-native-reanimated';
import { Circle, Svg } from 'react-native-svg';

const AnimatedCircle = Animated.createAnimatedComponent(Circle);

function ProgressRing({ progress }: { progress: SharedValue<number> }) {
  const animatedProps = useAnimatedProps(() => ({
    strokeDashoffset: (1 - progress.value) * 2 * Math.PI * 40,
  }));

  return (
    <Svg>
      <AnimatedCircle
        cx={50} cy={50} r={40}
        strokeDasharray={2 * Math.PI * 40}
        animatedProps={animatedProps}
      />
    </Svg>
  );
}

// Derived values: computed from other shared values
const x = useSharedValue(0);
const y = useSharedValue(0);
const distance = useDerivedValue(() => {
  return Math.sqrt(x.value ** 2 + y.value ** 2);
});

// Reactions: side effects when shared values change
useAnimatedReaction(
  () => progress.value,
  (current, previous) => {
    if (current >= 1 && (previous ?? 0) < 1) {
      runOnJS(onComplete)();
    }
  }
);

6. Layout Animations

import Animated, {
  FadeIn, FadeOut, SlideInRight, SlideOutLeft,
  Layout, LinearTransition,
} from 'react-native-reanimated';

// Entering and exiting animations
<Animated.View
  entering={FadeIn.duration(300).delay(100)}
  exiting={FadeOut.duration(200)}
>
  <Text>I fade in and out</Text>
</Animated.View>

// Layout transitions (when position/size changes)
<Animated.View layout={LinearTransition.springify()}>
  <Text>I animate when my layout changes</Text>
</Animated.View>

// Custom entering animation
const customEntering = (targetValues) => {
  'worklet';
  const animations = {
    opacity: withTiming(1, { duration: 300 }),
    transform: [
      { translateY: withSpring(targetValues.targetOriginY) },
      { scale: withSpring(1) },
    ],
  };
  const initialValues = {
    opacity: 0,
    transform: [
      { translateY: targetValues.targetOriginY + 50 },
      { scale: 0.8 },
    ],
  };
  return { initialValues, animations };
};

7. Reanimated vs Animated API

Aspect Animated API Reanimated v3
Execution model Static config sent to native JS worklets on UI thread
JS on UI thread No — only pre-computed frames Yes — full JS in worklets
Conditional logic Not possible during animation Full if/else/switch in worklets
Gesture integration Animated.event (limited) Gesture Handler composable API
Layout animations Not supported Entering/exiting/layout transitions
Dependencies Built-in (0 extra deps) ~200KB native + Babel plugin
Debugging Standard JS debugging Worklet debugging is harder
Performance ceiling 60fps for supported properties 60fps with full JS expressiveness
Best for Simple transitions, scroll-driven Complex gestures, interactive animations

8. Architect Considerations

Staff-Level Mental Model

Reanimated v3 is architecturally a second JavaScript runtime running on the UI thread. The Babel plugin extracts worklet functions at build time, serializes them, and the Reanimated runtime instantiates them in a lightweight Hermes/JSC VM on the UI thread. Shared values are C++ objects accessible via JSI from both threads — no serialization, no bridge, no async. This architecture enables the holy grail of mobile animation: full JavaScript expressiveness at native frame rates. The trade-off is complexity: worklets have a restricted execution environment, debugging is harder, and the Babel plugin adds build-time overhead. A Staff engineer uses Reanimated for gesture-driven interactions and complex animations, and falls back to the simpler Animated API for basic transitions.
