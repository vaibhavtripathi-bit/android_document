React Native Gesture Handler

Gesture Handler replaces React Native's built-in touch system with native gesture recognizers, providing reliable gesture detection that doesn't compete with the JS thread. The composable Gesture API enables complex multi-gesture interactions with direct Reanimated integration.

1. Why Gesture Handler?

RN Built-in Touch vs Gesture Handler ════════════════════════════════════════════════════ Built-in (PanResponder / Touchable) Gesture Handler ──────────────────────────────────── ───────────────── Touch Event Touch Event │ │ ▼ ▼ Native Touch System Native Gesture Recognizer │ (UIGestureRecognizer / Android) ▼ │ Bridge (async serialization) │ (native processing) │ │ ▼ ▼ JS Thread Gesture State Machine (onPanResponderMove, etc.) (BEGAN → ACTIVE → END) │ │ ▼ ▼ setState / Animated.event Worklet callback (UI thread) or JS callback Problems: • JS thread busy → dropped gestures Benefits: • No native gesture recognition • Native gesture recognition • Can't compose gestures • Works when JS is busy • Platform-inconsistent • Composable (simultaneous, exclusive) • Direct Reanimated integration
Key Insight

The fundamental problem with RN's built-in touch system is that touch events must cross the bridge to the JS thread for processing. If the JS thread is busy (rendering a complex list, running a network callback), touch events are delayed or dropped, causing janky gestures. Gesture Handler processes gestures on the native side using platform gesture recognizers (UIGestureRecognizer on iOS, GestureDetector on Android), only notifying JS/worklets of state changes.

2. Composable Gesture API

import { Gesture, GestureDetector } from 'react-native-gesture-handler';
import Animated, { useSharedValue, useAnimatedStyle, withSpring } from 'react-native-reanimated';

function DraggableBox() {
  const translateX = useSharedValue(0);
  const translateY = useSharedValue(0);
  const savedX = useSharedValue(0);
  const savedY = useSharedValue(0);

  const panGesture = Gesture.Pan()
    .onStart(() => {
      savedX.value = translateX.value;
      savedY.value = translateY.value;
    })
    .onUpdate((event) => {
      translateX.value = savedX.value + event.translationX;
      translateY.value = savedY.value + event.translationY;
    })
    .onEnd((event) => {
      // Snap back with spring if not dragged far enough
      if (Math.abs(translateX.value) < 100) {
        translateX.value = withSpring(0);
        translateY.value = withSpring(0);
      }
    });

  const animatedStyle = useAnimatedStyle(() => ({
    transform: [
      { translateX: translateX.value },
      { translateY: translateY.value },
    ],
  }));

  return (
    <GestureDetector gesture={panGesture}>
      <Animated.View style={[styles.box, animatedStyle]} />
    </GestureDetector>
  );
}

3.2 Pinch & Rotation

const scale = useSharedValue(1);
const rotation = useSharedValue(0);
const savedScale = useSharedValue(1);
const savedRotation = useSharedValue(0);

const pinch = Gesture.Pinch()
  .onStart(() => { savedScale.value = scale.value; })
  .onUpdate((e) => {
    scale.value = savedScale.value * e.scale;
  });

const rotate = Gesture.Rotation()
  .onStart(() => { savedRotation.value = rotation.value; })
  .onUpdate((e) => {
    rotation.value = savedRotation.value + e.rotation;
  });

// Simultaneous: pinch + rotate + pan on a photo
const composed = Gesture.Simultaneous(
  panGesture,
  pinchGesture,
  rotationGesture
);

// Exclusive: double-tap takes priority over single-tap
const doubleTap = Gesture.Tap().numberOfTaps(2).onEnd(() => { /* zoom */ });
const singleTap = Gesture.Tap().onEnd(() => { /* toggle UI */ });
const tapGesture = Gesture.Exclusive(doubleTap, singleTap);

// Race: first gesture to activate wins
const horizontalSwipe = Gesture.Pan().activeOffsetX([-20, 20]);
const verticalScroll = Gesture.Pan().activeOffsetY([-20, 20]);
const raceGesture = Gesture.Race(horizontalSwipe, verticalScroll);

<GestureDetector gesture={Gesture.Simultaneous(composed, tapGesture)}>
  <Animated.View style={animatedStyle} />
</GestureDetector>

5. Reanimated Integration

// Gesture callbacks run as worklets on the UI thread by default
// when used with Reanimated — no 'worklet' directive needed

const pan = Gesture.Pan()
  .onUpdate((event) => {
    // This is a worklet — runs on UI thread
    // Can directly update shared values
    translateX.value = event.translationX;
    translateY.value = event.translationY;
  })
  .onEnd((event) => {
    // Can use Reanimated animation modifiers
    translateX.value = withDecay({ velocity: event.velocityX });
    translateY.value = withDecay({ velocity: event.velocityY });
  });

// Gesture state tracking
const isPressed = useSharedValue(false);

const tap = Gesture.Tap()
  .onBegin(() => { isPressed.value = true; })
  .onFinalize(() => { isPressed.value = false; });

const animatedStyle = useAnimatedStyle(() => ({
  transform: [{ scale: withTiming(isPressed.value ? 0.95 : 1) }],
}));

7. Advanced Patterns

7.1 Swipeable Card

function SwipeableCard({ onSwipeLeft, onSwipeRight }) {
  const translateX = useSharedValue(0);
  const SWIPE_THRESHOLD = SCREEN_WIDTH * 0.3;

  const pan = Gesture.Pan()
    .onUpdate((e) => {
      translateX.value = e.translationX;
    })
    .onEnd((e) => {
      if (translateX.value > SWIPE_THRESHOLD) {
        translateX.value = withTiming(SCREEN_WIDTH, {}, () => {
          runOnJS(onSwipeRight)();
        });
      } else if (translateX.value < -SWIPE_THRESHOLD) {
        translateX.value = withTiming(-SCREEN_WIDTH, {}, () => {
          runOnJS(onSwipeLeft)();
        });
      } else {
        translateX.value = withSpring(0);
      }
    });

  const animatedStyle = useAnimatedStyle(() => ({
    transform: [
      { translateX: translateX.value },
      { rotate: `${interpolate(translateX.value, [-SCREEN_WIDTH, 0, SCREEN_WIDTH], [-15, 0, 15])}deg` },
    ],
  }));

  return (
    <GestureDetector gesture={pan}>
      <Animated.View style={[styles.card, animatedStyle]} />
    </GestureDetector>
  );
}

8. Architect Considerations

Staff-Level Mental Model

Gesture Handler solves the fundamental problem of RN's touch system: gestures should not depend on the JS thread. By using platform-native gesture recognizers, gestures are detected reliably even when JS is busy. The composable Gesture API (Simultaneous, Exclusive, Race) maps directly to how native gesture recognizers interact on iOS (UIGestureRecognizer delegate methods). Combined with Reanimated, gesture callbacks run as worklets on the UI thread — the entire gesture → animation pipeline never touches the JS thread. This is the architecture that enables Instagram-quality swipe interactions in React Native.
