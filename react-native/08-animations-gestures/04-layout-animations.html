Layout Animations

Layout animations automatically animate components as they enter, exit, or change position in the layout. Reanimated provides a declarative API for entering/exiting animations, layout transitions, and keyframe animations — plus the legacy LayoutAnimation API for simpler cases.

1. Entering Animations

Entering animations run when a component is first mounted. Reanimated provides a rich set of predefined entering animations that can be customized with duration, delay, easing, and spring physics.

Predefined Entering Animations ════════════════════════════════════════════════════ Fade: FadeIn, FadeInUp, FadeInDown, FadeInLeft, FadeInRight Slide: SlideInLeft, SlideInRight, SlideInUp, SlideInDown Zoom: ZoomIn, ZoomInRotate, ZoomInEasyUp, ZoomInEasyDown Bounce: BounceIn, BounceInUp, BounceInDown, BounceInLeft, BounceInRight Flip: FlipInXUp, FlipInXDown, FlipInYLeft, FlipInYRight Stretch: StretchInX, StretchInY Roll: RollInLeft, RollInRight Rotate: RotateInUpLeft, RotateInUpRight, RotateInDownLeft, RotateInDownRight LightSpeed: LightSpeedInLeft, LightSpeedInRight Pinwheel: PinwheelIn All support: .duration(ms), .delay(ms), .springify(), .damping(n), .stiffness(n), .withCallback(fn) 
import Animated, {
  FadeIn, FadeInUp, SlideInRight,
  BounceIn, ZoomIn, FlipInYLeft,
} from 'react-native-reanimated';

// Basic entering animation
<Animated.View entering={FadeIn}>
  <Text>I fade in on mount</Text>
</Animated.View>

// Customized entering
<Animated.View
  entering={FadeInUp
    .duration(400)
    .delay(200)
    .springify()
    .damping(15)
    .withCallback((finished) => {
      'worklet';
      if (finished) {
        runOnJS(onAnimationComplete)();
      }
    })
  }
>
  <Text>I slide up with spring physics</Text>
</Animated.View>

// Staggered list entrance
{items.map((item, index) => (
  <Animated.View
    key={item.id}
    entering={FadeInUp.delay(index * 100)}
  >
    <ListItem item={item} />
  </Animated.View>
))}

2. Exiting Animations

import Animated, {
  FadeOut, FadeOutDown, SlideOutLeft,
  ZoomOut, BounceOut,
} from 'react-native-reanimated';

function DismissableCard({ visible }: { visible: boolean }) {
  if (!visible) return null;

  return (
    <Animated.View
      entering={FadeIn.duration(300)}
      exiting={FadeOutDown.duration(200)}
    >
      <Text>I animate out when removed from tree</Text>
    </Animated.View>
  );
}

// Exiting animations keep the component mounted during the animation
// even though React has already removed it from the virtual tree.
// Reanimated intercepts the unmount and delays it until the
// exiting animation completes.
Key Insight

Exiting animations work by intercepting React's unmount lifecycle. When a component with an exiting prop is removed from the React tree, Reanimated keeps the native view alive and runs the exit animation. Only after the animation completes is the native view actually removed. This is handled entirely on the native side — React has already forgotten about the component.

3. Layout Transitions

Layout transitions animate changes in a component's position or size when the surrounding layout changes (e.g., a sibling is added/removed, causing this component to shift).

import Animated, {
  LinearTransition, SequencedTransition,
  FadingTransition, JumpingTransition,
  CurvedTransition, EntryExitTransition,
} from 'react-native-reanimated';

// Linear transition: smooth position change
<Animated.View layout={LinearTransition.springify()}>
  <Text>I smoothly move when layout changes</Text>
</Animated.View>

// Sequenced: animate width first, then height
<Animated.View layout={SequencedTransition.duration(500)}>
  <Text>Width animates, then height</Text>
</Animated.View>

// Practical example: reorderable list
function ReorderableList({ items }: { items: Item[] }) {
  return (
    <View>
      {items.map((item) => (
        <Animated.View
          key={item.id}
          layout={LinearTransition.springify().damping(15)}
          entering={FadeIn}
          exiting={FadeOut}
        >
          <ListItem item={item} />
        </Animated.View>
      ))}
    </View>
  );
}
Layout Transition Flow ════════════════════════════════════════════════════ Before: After (item B removed): ┌──────────────────┐ ┌──────────────────┐ │ Item A (y: 0) │ │ Item A (y: 0) │ ├──────────────────┤ ├──────────────────┤ │ Item B (y: 60) │ ──remove─→│ Item C (y: 60) │ ← animated from y:120 ├──────────────────┤ ├──────────────────┤ │ Item C (y: 120) │ │ Item D (y: 120) │ ← animated from y:180 ├──────────────────┤ └──────────────────┘ │ Item D (y: 180) │ └──────────────────┘ With layout={LinearTransition}: Items C and D smoothly animate to their new positions instead of jumping instantly.

5. Animating FlatList Items

import Animated, { FadeInUp, FadeOut, LinearTransition } from 'react-native-reanimated';

function AnimatedList({ data }: { data: Item[] }) {
  const renderItem = useCallback(({ item, index }: { item: Item; index: number }) => (
    <Animated.View
      entering={FadeInUp.delay(index * 50).springify()}
      exiting={FadeOut.duration(200)}
      layout={LinearTransition.springify()}
    >
      <ItemCard item={item} />
    </Animated.View>
  ), []);

  return (
    <FlatList
      data={data}
      renderItem={renderItem}
      keyExtractor={(item) => item.id}
      itemLayoutAnimation={LinearTransition.springify()}
    />
  );
}
Warning

Entering animations with .delay(index * n) in FlatList can cause performance issues for large lists. The delay is based on the render index, not the visible index — items far down the list will have large delays. For long lists, use itemLayoutAnimation on the FlatList itself instead of per-item entering animations, or calculate delay based on visible position rather than data index.

6. Legacy LayoutAnimation API

React Native's built-in LayoutAnimation API provides a simpler but less controllable way to animate layout changes. It animates the next layout change globally.

import { LayoutAnimation, UIManager, Platform } from 'react-native';

// Enable on Android (required)
if (Platform.OS === 'android') {
  UIManager.setLayoutAnimationEnabledExperimental?.(true);
}

function ExpandableSection() {
  const [expanded, setExpanded] = useState(false);

  const toggle = () => {
    // Configure animation BEFORE the state change
    LayoutAnimation.configureNext(
      LayoutAnimation.Presets.easeInEaseOut
    );
    setExpanded(!expanded);
  };

  return (
    <View>
      <Pressable onPress={toggle}>
        <Text>Toggle</Text>
      </Pressable>
      {expanded && (
        <View style={{ height: 200 }}>
          <Text>Expanded content</Text>
        </View>
      )}
    </View>
  );
}

// Custom LayoutAnimation config
const customConfig = {
  duration: 300,
  create: {
    type: LayoutAnimation.Types.easeInEaseOut,
    property: LayoutAnimation.Properties.opacity,
  },
  update: {
    type: LayoutAnimation.Types.spring,
    springDamping: 0.7,
  },
  delete: {
    type: LayoutAnimation.Types.easeOut,
    property: LayoutAnimation.Properties.opacity,
  },
};

7. Custom Animations

// Custom entering animation using worklet
const customEntering = (targetValues: EntryAnimationsValues) => {
  'worklet';
  const animations = {
    originX: withTiming(targetValues.targetOriginX, { duration: 300 }),
    originY: withSpring(targetValues.targetOriginY),
    opacity: withTiming(1, { duration: 200 }),
    transform: [
      { rotate: withTiming('0deg', { duration: 400 }) },
      { scale: withSpring(1) },
    ],
  };

  const initialValues = {
    originX: targetValues.targetOriginX - 100,
    originY: targetValues.targetOriginY + 200,
    opacity: 0,
    transform: [
      { rotate: '45deg' },
      { scale: 0.5 },
    ],
  };

  return { initialValues, animations };
};

// Custom exiting animation
const customExiting = (values: ExitAnimationsValues) => {
  'worklet';
  const animations = {
    opacity: withTiming(0, { duration: 200 }),
    transform: [
      { scale: withTiming(0.8, { duration: 200 }) },
      { translateY: withTiming(50, { duration: 200 }) },
    ],
  };

  const initialValues = {
    opacity: 1,
    transform: [{ scale: 1 }, { translateY: 0 }],
  };

  return { initialValues, animations };
};

<Animated.View entering={customEntering} exiting={customExiting}>
  <Text>Custom animated component</Text>
</Animated.View>

8. Architect Considerations

Staff-Level Mental Model

Layout animations solve the "flash of unstyled content" problem in mobile UIs. Without them, adding/removing items from a list causes remaining items to jump to their new positions instantly — a jarring experience. Reanimated's layout animations intercept the native layout system: they capture the before and after positions of each component, then animate the transition on the UI thread. The key architectural decision is granularity: use per-component entering/exiting for individual element animations, use layout prop for position transitions, and use itemLayoutAnimation on FlatList for list-level coordination. Avoid the legacy LayoutAnimation API in new code — its global scope makes it unpredictable in complex UIs.
