Image Handling

Deep dive into React Native's image loading pipeline — from source resolution through download, decode, and display. Understanding platform-specific caching (NSURLCache, Fresco), memory management, and optimization libraries like FastImage is essential for performant media-rich apps.

1. Image Component Internals

React Native's <Image> component wraps platform-specific image views with a unified API that handles source resolution, loading states, and display.

React Native <Image> │ ├──── iOS ──────────────────────────────────────── │ RCTImageView (UIImageView subclass) │ └─ RCTImageLoader (centralized loading) │ └─ NSURLSession for network requests │ └─ ImageIO framework for decoding │ └─ NSURLCache for HTTP caching │ └──── Android ──────────────────────────────────── ReactImageView (extends GenericDraweeView) └─ Fresco library (Facebook's image pipeline) └─ OkHttp for network requests └─ Android BitmapFactory for decoding └─ Multi-tier cache (bitmap, encoded, disk)

1.1 Image Source Types

// 1. Network images — loaded from URL
<Image
  source={{ uri: 'https://example.com/photo.jpg' }}
  style={{ width: 200, height: 200 }}
/>
// MUST specify width and height for network images
// React Native cannot know dimensions before download

// 2. Local/bundled images — resolved at build time
<Image source={require('./photo.png')} />
// Metro bundler resolves to platform-specific asset
// Dimensions known at build time — width/height optional
// @1x, @2x, @3x variants selected by pixel density

// 3. Base64 data URI
<Image
  source={{ uri: 'data:image/png;base64,iVBOR...' }}
  style={{ width: 100, height: 100 }}
/>
// Decoded in-memory — no caching, no network
// Avoid for large images (blocks JS thread during base64 decode)

// 4. Native asset (Android drawable / iOS asset catalog)
<Image
  source={{ uri: 'asset:/icon.png' }}  // Android drawable
  style={{ width: 24, height: 24 }}
/>

2. Image Loading Pipeline

Image Loading Pipeline (Network Image): 1. Source Resolution ├─ Parse URI, headers, cache policy └─ Determine loader (network, local, data URI) 2. Cache Check ├─ iOS: NSURLCache (HTTP cache) ├─ Android: Fresco disk cache → encoded memory cache → bitmap cache └─ Hit? → Skip to step 5 3. Download ├─ iOS: NSURLSession data task ├─ Android: OkHttp request via Fresco ├─ Progress callbacks (onProgress) └─ HTTP caching headers respected (Cache-Control, ETag) 4. Decode ├─ iOS: ImageIO (CGImageSource) — supports JPEG, PNG, GIF, HEIF, WebP ├─ Android: BitmapFactory — supports JPEG, PNG, GIF, WebP ├─ Decode happens on background thread ├─ Downsampling if image larger than display size └─ Memory allocation for decoded bitmap 5. Display ├─ Bitmap assigned to native image view ├─ Resize mode applied (cover, contain, stretch, etc.) ├─ onLoad callback fired └─ View composited into screen

2.1 Loading Callbacks

<Image
  source={{ uri: imageUrl }}
  style={{ width: 300, height: 200 }}
  onLoadStart={() => console.log('Loading started')}
  onProgress={({ nativeEvent: { loaded, total } }) => {
    console.log(`Progress: ${Math.round(loaded/total*100)}%`);
  }}
  onLoad={({ nativeEvent: { source: { width, height } } }) => {
    console.log(`Loaded: ${width}x${height}`);
  }}
  onError={({ nativeEvent: { error } }) => {
    console.log(`Error: ${error}`);
  }}
  onLoadEnd={() => console.log('Loading finished')}
/>

// Callback order: onLoadStart → onProgress* → onLoad/onError → onLoadEnd
// onProgress only fires for network images (not local/cached)

3. Image Caching

3.1 iOS Caching (NSURLCache)

// iOS uses NSURLCache — standard HTTP cache
// Respects HTTP caching headers:
// - Cache-Control: max-age=3600
// - ETag / If-None-Match
// - Last-Modified / If-Modified-Since

// Default cache size: 50MB memory, 200MB disk

// Cache control via source prop:
<Image
  source={{
    uri: 'https://example.com/photo.jpg',
    cache: 'force-cache',  // iOS only
  }}
/>

// cache options (iOS):
// 'default'     — use HTTP cache headers
// 'reload'      — skip cache, always download
// 'force-cache' — use cache even if expired
// 'only-if-cached' — only use cache, fail if not cached

3.2 Android Caching (Fresco)

Fresco Cache Architecture (Android): Request → Bitmap Memory Cache (L1) │ miss ↓ Encoded Memory Cache (L2) │ miss ↓ Disk Cache (L3) │ miss ↓ Network Fetch │ ↓ Store in all cache levels L1: Decoded bitmaps — fastest, limited by memory L2: Encoded bytes (JPEG/PNG) — needs decode, less memory L3: Disk files — persistent across app launches 
// Fresco cache configuration (in MainApplication.java)
// Default: 24MB bitmap cache, 2MB encoded cache, 40MB disk cache

// Clearing caches programmatically:
import { Image } from 'react-native';

// Query cache for a specific URL:
Image.queryCache(['https://example.com/photo.jpg'])
  .then(result => {
    // result: { 'https://example.com/photo.jpg': 'disk' | 'memory' }
    console.log(result);
  });
Caching Pitfall

React Native's built-in Image component has no cross-platform cache control. iOS uses NSURLCache (HTTP-based), Android uses Fresco (custom). The cache prop only works on iOS. For consistent cross-platform caching, use react-native-fast-image which provides a unified caching API on both platforms.

4. Progressive Loading

// Placeholder pattern: low-res → high-res
const ProgressiveImage = ({ thumbnailUri, fullUri, style }) => {
  const [loaded, setLoaded] = useState(false);

  return (
    <View style={style}>
      <Image
        source={{ uri: thumbnailUri }}
        style={[StyleSheet.absoluteFill, { opacity: loaded ? 0 : 1 }]}
        blurRadius={10}
      />
      <Image
        source={{ uri: fullUri }}
        style={StyleSheet.absoluteFill}
        onLoad={() => setLoaded(true)}
      />
    </View>
  );
};

// defaultSource — shows while loading (iOS only)
<Image
  source={{ uri: 'https://example.com/large.jpg' }}
  defaultSource={require('./placeholder.png')}
  style={{ width: 300, height: 200 }}
/>
// defaultSource only works on iOS
// Android: use the two-Image overlay pattern above

// Progressive JPEG support:
// iOS: Supported natively — renders progressively as data arrives
// Android: Fresco supports progressive JPEG rendering
// Enable in Fresco config: .setProgressiveRenderingEnabled(true)

5. Image Resize Modes

// resizeMode controls how image fits its container

// 'cover' (default) — scales to fill, may crop
<Image source={src} resizeMode="cover" style={{ width: 200, height: 200 }} />
// Maintains aspect ratio, fills container, crops excess
// iOS: UIViewContentModeScaleAspectFill
// Android: ScaleType.CENTER_CROP

// 'contain' — scales to fit, may letterbox
<Image source={src} resizeMode="contain" style={{ width: 200, height: 200 }} />
// Maintains aspect ratio, fits within container
// iOS: UIViewContentModeScaleAspectFit
// Android: ScaleType.FIT_CENTER

// 'stretch' — fills exactly, may distort
<Image source={src} resizeMode="stretch" style={{ width: 200, height: 200 }} />
// Ignores aspect ratio, fills container exactly
// iOS: UIViewContentModeScaleToFill
// Android: ScaleType.FIT_XY

// 'center' — no scaling, centered
<Image source={src} resizeMode="center" style={{ width: 200, height: 200 }} />
// Original size, centered in container

// 'repeat' — tiles the image (iOS only)
<Image source={src} resizeMode="repeat" style={{ width: 400, height: 400 }} />

6. SVG Support

// React Native does NOT support SVG natively
// Use react-native-svg for SVG rendering

import Svg, { Circle, Rect, Path } from 'react-native-svg';

<Svg width="100" height="100" viewBox="0 0 100 100">
  <Circle cx="50" cy="50" r="40" fill="blue" />
  <Rect x="10" y="10" width="30" height="30" fill="red" />
</Svg>

// SVG from file (using react-native-svg-transformer):
import Logo from './logo.svg';
<Logo width={120} height={40} />

// react-native-svg renders to:
// iOS: CAShapeLayer (Core Graphics paths)
// Android: Canvas draw operations
// NOT rasterized — truly vector, scales without quality loss

// Performance consideration:
// Complex SVGs with many paths are expensive to render
// For icons: prefer icon fonts or pre-rasterized PNGs
// For illustrations: SVG is fine (rendered once, cached)

7. Image Prefetching

// Prefetch images before they're displayed
import { Image } from 'react-native';

// Single image prefetch
await Image.prefetch('https://example.com/photo.jpg');

// Batch prefetch for a list
const urls = items.map(item => item.imageUrl);
await Promise.all(urls.map(url => Image.prefetch(url)));

// Prefetch with callback (deprecated in newer versions)
Image.prefetch(url)
  .then(() => console.log('Prefetched'))
  .catch(err => console.log('Prefetch failed', err));

// Get image dimensions before display
Image.getSize(
  'https://example.com/photo.jpg',
  (width, height) => {
    console.log(`Image: ${width}x${height}`);
  },
  (error) => {
    console.log('Failed to get size', error);
  }
);

// getSize with headers (for authenticated images)
Image.getSizeWithHeaders(
  'https://example.com/private.jpg',
  { Authorization: 'Bearer token' },
  (width, height) => { /* ... */ },
  (error) => { /* ... */ }
);

8. Memory Management

Memory Budget

A single 4000×3000 photo decoded to RGBA bitmap consumes 48MB of memory (4000 × 3000 × 4 bytes). iOS typically kills apps at 1-2GB memory usage. A photo gallery showing 10 full-resolution images would consume 480MB — nearly half the budget. Always resize images to display size before decoding.

// Memory calculation: width × height × 4 bytes (RGBA)
// 4000×3000 → 48,000,000 bytes → 48 MB
// 1000×750  → 3,000,000 bytes  → 3 MB
// 200×150   → 120,000 bytes    → 0.12 MB

// Strategy 1: Server-side resizing
const getOptimizedUrl = (url, width, height) =>
  `${url}?w=${width}&h=${height}&fit=crop&q=80`;

<Image
  source={{ uri: getOptimizedUrl(photo.url, 400, 300) }}
  style={{ width: 200, height: 150 }}
/>
// Request 2x display size for retina (200×150 display = 400×300 pixels)

// Strategy 2: Use thumbnails in lists
const renderItem = ({ item }) => (
  <Image
    source={{ uri: item.thumbnailUrl }}  // 200×200 thumbnail
    style={{ width: 100, height: 100 }}
  />
);

// Strategy 3: Fresco memory configuration (Android)
// In MainApplication.java:
// ImagePipelineConfig config = ImagePipelineConfig.newBuilder(context)
//   .setBitmapMemoryCacheParamsSupplier(new CustomCacheParams())
//   .setDownsampleEnabled(true)  // Auto-downsample large images
//   .build();

9. FastImage Library

react-native-fast-image wraps SDWebImage (iOS) and Glide (Android) for superior caching and performance compared to the built-in Image component.

import FastImage from 'react-native-fast-image';

<FastImage
  source={{
    uri: 'https://example.com/photo.jpg',
    headers: { Authorization: 'Bearer token' },
    priority: FastImage.priority.high,
    cache: FastImage.cacheControl.immutable,
  }}
  resizeMode={FastImage.resizeMode.cover}
  style={{ width: 200, height: 200 }}
/>

// Cache control options:
// FastImage.cacheControl.immutable — cache forever (URL-based versioning)
// FastImage.cacheControl.web — respect HTTP cache headers
// FastImage.cacheControl.cacheOnly — only show if cached

// Priority options:
// FastImage.priority.low
// FastImage.priority.normal
// FastImage.priority.high

// Preload multiple images:
FastImage.preload([
  { uri: 'https://example.com/1.jpg' },
  { uri: 'https://example.com/2.jpg' },
  { uri: 'https://example.com/3.jpg' },
]);

// Clear cache:
FastImage.clearMemoryCache();
FastImage.clearDiskCache();

10. Architect Considerations

Staff-Level Mental Model

Image handling is the second most common source of performance issues in React Native (after lists). The key architectural decisions are: (1) Server-side image optimization — always serve images at the size they'll be displayed, not original resolution. (2) Caching strategy — use URL-based versioning with immutable cache for maximum performance. (3) Memory budgeting — calculate the memory cost of your image-heavy screens and ensure you stay within platform limits. (4) Library choice — FastImage (or Expo Image) should be the default for any production app. The built-in Image component is adequate only for simple use cases.

10.1 Image Architecture Decision Tree

// Decision tree for image handling:

// Q: Is it an icon/small graphic?
//   → Use bundled assets (require('./icon.png'))
//   → Or icon font (react-native-vector-icons)
//   → Or SVG (react-native-svg) for scalable icons

// Q: Is it a user photo / content image?
//   → Use FastImage with CDN-resized URLs
//   → Implement progressive loading (thumbnail → full)
//   → Set appropriate cache policy

// Q: Is it in a list?
//   → Use thumbnails (not full resolution)
//   → Prefetch next page of images
//   → Consider placeholder shimmer while loading

// Q: Is it a hero/banner image?
//   → Prefetch on previous screen
//   → Use progressive JPEG
//   → Consider blur-up placeholder
