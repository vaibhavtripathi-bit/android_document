React Navigation Internals

1. Architecture Overview

React Navigation Architecture: NavigationContainer (root state manager) │ ├─ Manages global navigation state ├─ Handles deep linking (URL → state) ├─ Provides NavigationContext to tree ├─ Handles state persistence └─ Dispatches actions to navigators │ ├─ Stack.Navigator (manages stack state) │ ├─ Stack.Screen "Home" → HomeScreen │ ├─ Stack.Screen "Details" → DetailsScreen │ └─ Router: StackRouter (action → state reducer) │ ├─ Tab.Navigator (manages tab state) │ ├─ Tab.Screen "Feed" → FeedStack │ ├─ Tab.Screen "Profile" → ProfileStack │ └─ Router: TabRouter │ └─ Drawer.Navigator (manages drawer state) ├─ Drawer.Screen "Main" → MainTabs └─ Router: DrawerRouter

2. Navigation State Tree

// Navigation state is a recursive tree structure
// Each navigator owns a slice of the state tree

type NavigationState = {
  type: string;           // 'stack' | 'tab' | 'drawer'
  key: string;            // Unique navigator key
  index: number;          // Currently focused route index
  routeNames: string[];   // Available route names
  routes: Route[];        // Route objects
  stale: boolean;         // Whether state needs rehydration
  history?: any[];        // Tab/drawer history for back behavior
};

type Route = {
  key: string;            // Unique route key
  name: string;           // Route name (matches Screen name)
  params?: object;       // Route parameters
  state?: NavigationState; // Nested navigator state
};

// Example: Tab navigator with nested stacks
const state = {
  type: 'tab',
  index: 0,
  routes: [
    {
      name: 'HomeTab',
      state: {
        type: 'stack',
        index: 1,
        routes: [
          { name: 'Home' },
          { name: 'Details', params: { id: 42 } },
        ],
      },
    },
    {
      name: 'ProfileTab',
      // No nested state yet — lazy initialization
    },
  ],
};
Stale State

When navigation state is restored from persistence or deep link, it's marked as stale: true. This means the state may reference routes that no longer exist or may be missing new routes. React Navigation "rehydrates" stale state by merging it with the current navigator configuration, adding missing routes and removing invalid ones. This is why state persistence is resilient to app updates that change the navigation structure.

4. State Persistence

// State persistence saves/restores navigation state across app restarts

const PERSISTENCE_KEY = 'NAVIGATION_STATE_V1';

const App = () => {
  const [isReady, setIsReady] = useState(__DEV__ ? false : true);
  const [initialState, setInitialState] = useState();

  useEffect(() => {
    const restoreState = async () => {
      try {
        const savedState = await AsyncStorage.getItem(PERSISTENCE_KEY);
        if (savedState) {
          const state = JSON.parse(savedState);
          setInitialState(state);
        }
      } catch (e) {
        // Ignore restore errors — start fresh
      } finally {
        setIsReady(true);
      }
    };

    if (!isReady) restoreState();
  }, [isReady]);

  if (!isReady) return null;

  return (
    <NavigationContainer
      initialState={initialState}
      onStateChange={(state) =>
        AsyncStorage.setItem(PERSISTENCE_KEY, JSON.stringify(state))
      }
    >
      <RootNavigator />
    </NavigationContainer>
  );
};

// Best practice: only persist in development
// Production apps should start from a known state
// Exception: restore last tab for tab-based apps

5. Custom Navigators

import {
  createNavigatorFactory,
  useNavigationBuilder,
  TabRouter,
} from '@react-navigation/native';

// Custom navigator using the builder API
function CustomTabNavigator({ children, screenOptions, ...rest }) {
  const { state, navigation, descriptors, NavigationContent } =
    useNavigationBuilder(TabRouter, {
      children,
      screenOptions,
      ...rest,
    });

  return (
    <NavigationContent>
      <View style={{ flex: 1 }}>
        {state.routes.map((route, i) => (
          <View
            key={route.key}
            style={{
              display: i === state.index ? 'flex' : 'none',
              flex: 1,
            }}
          >
            {descriptors[route.key].render()}
          </View>
        ))}

        {/* Custom tab bar */}
        <View style={styles.tabBar}>
          {state.routes.map((route, i) => (
            <Pressable
              key={route.key}
              onPress={() => navigation.navigate(route.name)}
              style={[styles.tab, i === state.index && styles.activeTab]}
            >
              <Text>{descriptors[route.key].options.title || route.name}</Text>
            </Pressable>
          ))}
        </View>
      </View>
    </NavigationContent>
  );
}

export const createCustomTabNavigator = createNavigatorFactory(CustomTabNavigator);

7. Route Params

// Passing params:
navigation.navigate('Details', { id: 42, title: 'My Post' });

// Reading params:
const DetailsScreen = ({ route }) => {
  const { id, title } = route.params;
  return <Text>{title} (ID: {id})</Text>;
};

// Updating params on current screen:
navigation.setParams({ title: 'Updated Title' });

// Initial params (default values):
<Stack.Screen
  name="Details"
  component={DetailsScreen}
  initialParams={{ id: 0, title: 'Default' }}
/>

// Params should be serializable (for deep linking and persistence)
// DO: strings, numbers, booleans, plain objects, arrays
// DON'T: functions, class instances, Dates, Sets, Maps

// Anti-pattern: passing full objects as params
// Bad: navigation.navigate('Details', { post: fullPostObject })
// Good: navigation.navigate('Details', { postId: post.id })
// Fetch the full object in the Details screen using the ID
Params Serialization

Route params must be serializable for deep linking and state persistence to work. Passing non-serializable values (functions, class instances) will break state persistence and make deep linking impossible for that route. Always pass IDs and fetch data in the destination screen.

8. Navigation Events

// Navigation events allow screens to react to navigation changes

const ScreenWithEvents = ({ navigation }) => {
  useEffect(() => {
    // 'focus' — screen became the focused screen
    const unsubFocus = navigation.addListener('focus', () => {
      console.log('Screen focused');
    });

    // 'blur' — screen lost focus
    const unsubBlur = navigation.addListener('blur', () => {
      console.log('Screen blurred');
    });

    // 'beforeRemove' — screen about to be removed (can prevent!)
    const unsubRemove = navigation.addListener('beforeRemove', (e) => {
      if (hasUnsavedChanges) {
        e.preventDefault();  // Prevent navigation!
        Alert.alert(
          'Discard changes?',
          'You have unsaved changes.',
          [
            { text: 'Stay', style: 'cancel' },
            {
              text: 'Discard',
              style: 'destructive',
              onPress: () => navigation.dispatch(e.data.action),
            },
          ]
        );
      }
    });

    return () => {
      unsubFocus();
      unsubBlur();
      unsubRemove();
    };
  }, [navigation]);
};

9. TypeScript Integration

// Define param list types for type-safe navigation
type RootStackParamList = {
  Home: undefined;
  Details: { id: number; title: string };
  Profile: { userId: string };
  Settings: undefined;
};

// Create typed navigator
const Stack = createNativeStackNavigator<RootStackParamList>();

// Type-safe screen props
import type { NativeStackScreenProps } from '@react-navigation/native-stack';

type DetailsScreenProps = NativeStackScreenProps<RootStackParamList, 'Details'>;

const DetailsScreen = ({ route, navigation }: DetailsScreenProps) => {
  const { id, title } = route.params; // Typed: { id: number, title: string }

  // Type-safe navigation
  navigation.navigate('Profile', { userId: 'abc' }); // ✓
  navigation.navigate('Profile', { id: 1 });       // ✗ Type error!
  navigation.navigate('Unknown');                  // ✗ Type error!
};

// useNavigation with types
import { useNavigation } from '@react-navigation/native';
import type { NativeStackNavigationProp } from '@react-navigation/native-stack';

type NavigationProp = NativeStackNavigationProp<RootStackParamList>;

const SomeComponent = () => {
  const navigation = useNavigation<NavigationProp>();
  navigation.navigate('Details', { id: 42, title: 'Hello' });
};

10. Architect Considerations

Staff-Level Mental Model

React Navigation's architecture is a state machine where navigation actions are dispatched to routers that produce new state. The key architectural decisions are: (1) Navigator composition — design the navigator tree to match your app's information architecture, not its visual layout. (2) Type safety — invest in ParamList types early; they prevent runtime crashes and serve as documentation. (3) State serialization — keep route params serializable for deep linking and persistence. (4) Screen options — use screenOptions at the navigator level for consistency, per-screen options for overrides. (5) Custom navigators — use useNavigationBuilder for non-standard navigation patterns rather than fighting built-in navigators.
