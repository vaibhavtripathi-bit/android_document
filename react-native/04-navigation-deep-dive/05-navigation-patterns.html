1. Tab-Based Navigation

Tab-based navigation is the most common pattern in mobile apps. Each tab maintains its own navigation stack, and switching tabs preserves each stack's state.

Tab-Based Navigation Architecture: Tab.Navigator ├─ Tab "Home" │ └─ Stack.Navigator │ ├─ HomeScreen │ ├─ DetailsScreen │ └─ CommentsScreen │ ├─ Tab "Search" │ └─ Stack.Navigator │ ├─ SearchScreen │ └─ ResultsScreen │ └─ Tab "Profile" └─ Stack.Navigator ├─ ProfileScreen └─ EditProfileScreen Each tab has independent navigation history. Switching tabs does NOT reset the stack. 
const HomeStack = () => (
  <Stack.Navigator>
    <Stack.Screen name="Home" component={HomeScreen} />
    <Stack.Screen name="Details" component={DetailsScreen} />
  </Stack.Navigator>
);

const MainTabs = () => (
  <Tab.Navigator
    screenOptions={{
      headerShown: false,  // Hide tab navigator header (stacks have their own)
      tabBarHideOnKeyboard: true,
      lazy: true,  // Don't render tab content until first visit
    }}
  >
    <Tab.Screen name="HomeTab" component={HomeStack} />
    <Tab.Screen name="SearchTab" component={SearchStack} />
    <Tab.Screen name="ProfileTab" component={ProfileStack} />
  </Tab.Navigator>
);

// Reset tab stack when tab is pressed again:
<Tab.Screen
  name="HomeTab"
  component={HomeStack}
  listeners={({ navigation }) => ({
    tabPress: (e) => {
      const state = navigation.getState();
      const homeTab = state.routes.find(r => r.name === 'HomeTab');
      if (homeTab?.state?.index > 0) {
        navigation.dispatch(StackActions.popToTop());
      }
    },
  })}
/>

3. Nested Navigators

// Common nesting pattern: Root Stack → Auth/Main split

const RootNavigator = () => {
  const { isAuthenticated } = useAuth();

  return (
    <Stack.Navigator screenOptions={{ headerShown: false }}>
      {isAuthenticated ? (
        <Stack.Screen name="Main" component={MainDrawer} />
      ) : (
        <Stack.Screen name="Auth" component={AuthStack} />
      )}
      {/* Global modals accessible from anywhere */}
      <Stack.Group screenOptions={{ presentation: 'modal' }}>
        <Stack.Screen name="ImageViewer" component={ImageViewerScreen} />
        <Stack.Screen name="ShareSheet" component={ShareSheetScreen} />
      </Stack.Group>
    </Stack.Navigator>
  );
};

// Full nesting hierarchy:
// NavigationContainer
//   └─ RootStack
//        ├─ Auth (conditional)
//        │    ├─ Login
//        │    ├─ Register
//        │    └─ ForgotPassword
//        ├─ Main (conditional)
//        │    └─ Drawer
//        │         ├─ Tabs
//        │         │    ├─ HomeStack
//        │         │    ├─ SearchStack
//        │         │    └─ ProfileStack
//        │         └─ Settings
//        └─ Modals (group)
//             ├─ ImageViewer
//             └─ ShareSheet
Nesting Pitfall

Each navigator creates its own navigation context. A screen inside a nested stack navigator doesn't have direct access to the parent tab navigator's methods. Use navigation.getParent() to access parent navigators, or use the navigation ref for cross-navigator actions. Over-nesting (4+ levels) makes navigation logic complex and debugging difficult.

4. Authentication Flow

// Conditional navigation based on auth state
// This is the recommended pattern from React Navigation docs

const AuthContext = createContext(null);

const App = () => {
  const [state, dispatch] = useReducer(authReducer, {
    isLoading: true,
    isSignedIn: false,
    userToken: null,
  });

  useEffect(() => {
    const bootstrapAsync = async () => {
      const token = await SecureStore.getItemAsync('userToken');
      dispatch({ type: 'RESTORE_TOKEN', token });
    };
    bootstrapAsync();
  }, []);

  const authContext = useMemo(() => ({
    signIn: async (credentials) => {
      const token = await api.login(credentials);
      await SecureStore.setItemAsync('userToken', token);
      dispatch({ type: 'SIGN_IN', token });
    },
    signOut: async () => {
      await SecureStore.deleteItemAsync('userToken');
      dispatch({ type: 'SIGN_OUT' });
    },
  }), []);

  if (state.isLoading) return <SplashScreen />;

  return (
    <AuthContext.Provider value={authContext}>
      <NavigationContainer>
        <Stack.Navigator>
          {state.isSignedIn ? (
            <Stack.Screen name="Main" component={MainScreen} />
          ) : (
            <Stack.Screen
              name="Auth"
              component={AuthScreen}
              options={{ animationTypeForReplace: 'pop' }}
            />
          )}
        </Stack.Navigator>
      </NavigationContainer>
    </AuthContext.Provider>
  );
};
Why Conditional Rendering?

Using conditional rendering ({isSignedIn ? MainScreen : AuthScreen}) instead of imperative navigation (navigation.navigate('Login')) is the recommended pattern because: (1) It's declarative — auth state drives navigation state. (2) It prevents the user from navigating back to the auth screen after login. (3) It handles edge cases (token expiry, force logout) automatically. (4) The transition animation (animationTypeForReplace: 'pop') makes login feel natural.

6. Shared Element Transitions

// Shared element transitions animate an element from one screen to another
// Common in: photo galleries, card-to-detail, avatar-to-profile

// Using react-native-shared-element:
import { SharedElement } from 'react-navigation-shared-element';

// Source screen (list item):
const ListItem = ({ item, onPress }) => (
  <Pressable onPress={onPress}>
    <SharedElement id={`photo.${item.id}`}>
      <Image source={{ uri: item.photo }} style={styles.thumbnail} />
    </SharedElement>
    <SharedElement id={`title.${item.id}`}>
      <Text style={styles.title}>{item.title}</Text>
    </SharedElement>
  </Pressable>
);

// Destination screen (detail):
const DetailScreen = ({ route }) => {
  const { item } = route.params;
  return (
    <View>
      <SharedElement id={`photo.${item.id}`}>
        <Image source={{ uri: item.photo }} style={styles.hero} />
      </SharedElement>
      <SharedElement id={`title.${item.id}`}>
        <Text style={styles.heroTitle}>{item.title}</Text>
      </SharedElement>
    </View>
  );
};

// Navigator configuration:
DetailScreen.sharedElements = (route) => {
  const { item } = route.params;
  return [
    { id: `photo.${item.id}`, animation: 'move' },
    { id: `title.${item.id}`, animation: 'fade' },
  ];
};

  return (
    <View style={{ flex: 1, backgroundColor: 'rgba(0,0,0,0.5)' }}>
      <Pressable
        style={StyleSheet.absoluteFill}

        snapPoints={snapPoints}
        enablePanDownToClose
        onClose={() => navigation.goBack()}
      >
        <SheetContent />
      </BottomSheet>
    </View>
  );
};

8. Custom Transition Animations

// Custom transitions with JS stack (@react-navigation/stack)
import { createStackNavigator, CardStyleInterpolators } from '@react-navigation/stack';

const Stack = createStackNavigator();

// Built-in interpolators:
<Stack.Navigator
  screenOptions={{
    cardStyleInterpolator: CardStyleInterpolators.forHorizontalIOS,
    // forHorizontalIOS — slide from right
    // forVerticalIOS — slide from bottom
    // forModalPresentationIOS — iOS modal
    // forFadeFromBottomAndroid — Android fade
    // forRevealFromBottomAndroid — Android reveal
  }}
/>

// Custom interpolator:
const forFade = ({ current }) => ({
  cardStyle: {
    opacity: current.progress,
  },
});

const forSlideFromBottom = ({ current, layouts }) => ({
  cardStyle: {
    transform: [
      {
        translateY: current.progress.interpolate({
          inputRange: [0, 1],
          outputRange: [layouts.screen.height, 0],
        }),
      },
    ],
  },
});

<Stack.Screen
  name="FadeScreen"
  component={FadeScreen}
  options={{ cardStyleInterpolator: forFade }}
/>

9. Preventing Back Navigation

// Method 1: beforeRemove event (recommended)
useEffect(() => {
  const unsubscribe = navigation.addListener('beforeRemove', (e) => {
    if (!hasUnsavedChanges) return;

    e.preventDefault();

    Alert.alert(
      'Discard changes?',
      'You have unsaved changes. Are you sure you want to leave?',
      [
        { text: "Don't leave", style: 'cancel' },
        {
          text: 'Discard',
          style: 'destructive',
          onPress: () => navigation.dispatch(e.data.action),
        },
      ]
    );
  });

  return unsubscribe;
}, [navigation, hasUnsavedChanges]);

// Method 2: Disable gesture (native stack)
<Stack.Screen
  name="Checkout"
  component={CheckoutScreen}
  options={{
    gestureEnabled: false,  // Disable swipe-back
    headerBackVisible: false, // Hide back button
  }}
/>

// Method 3: Android hardware back button
import { BackHandler } from 'react-native';

useFocusEffect(
  useCallback(() => {
    const onBackPress = () => {
      if (isProcessing) {
        return true; // Prevent back
      }
      return false; // Allow default behavior
    };

    BackHandler.addEventListener('hardwareBackPress', onBackPress);
    return () =>
      BackHandler.removeEventListener('hardwareBackPress', onBackPress);
  }, [isProcessing])
);

10. Architect Considerations

Staff-Level Mental Model

Navigation patterns should mirror your app's information architecture, not its visual design. Key decisions: (1) Auth flow — always use conditional rendering, never imperative navigation for auth state changes. (2) Modal vs push — modals for creation/editing flows that can be dismissed, push for drill-down into content. (3) Nesting depth — keep it to 3 levels max (e.g., Drawer → Tabs → Stack). (4) Global modals — put them at the root stack level so they're accessible from anywhere. (5) Screen tracking — implement analytics tracking at the NavigationContainer level, not per-screen.

10.1 Screen Tracking for Analytics

// Centralized screen tracking
const routeNameRef = useRef();

<NavigationContainer
  ref={navigationRef}
  onReady={() => {
    routeNameRef.current = navigationRef.getCurrentRoute().name;
  }}
  onStateChange={() => {
    const previousRouteName = routeNameRef.current;
    const currentRoute = navigationRef.getCurrentRoute();
    const currentRouteName = currentRoute.name;

    if (previousRouteName !== currentRouteName) {
      analytics.trackScreen(currentRouteName, currentRoute.params);
    }

    routeNameRef.current = currentRouteName;
  }}
>
