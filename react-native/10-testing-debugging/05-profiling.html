Performance Profiling

Performance profiling in React Native requires tools that span the JS thread, UI thread, and native rendering pipeline. Staff engineers must correlate React render timing with native frame rates, identify memory leaks across the bridge, and establish automated performance regression detection.

6. Memory Profiling

// Common memory leak sources in React Native:
// 1. Event listeners not removed in cleanup
// 2. Timers (setInterval) not cleared
// 3. Closures capturing large objects
// 4. Native module references not released
// 5. Image cache growing unbounded

// Hermes heap snapshot
if (global.HermesInternal) {
  global.HermesInternal.createHeapSnapshot('/tmp/heap.heapsnapshot');
  // Open in Chrome DevTools вҶ’ Memory tab
}

// Monitor memory in development
const logMemory = () => {
  if (global.performance?.memory) {
    const { usedJSHeapSize, totalJSHeapSize } = global.performance.memory;
    console.log(`JS Heap: ${(usedJSHeapSize / 1024 / 1024).toFixed(1)}MB / ${(totalJSHeapSize / 1024 / 1024).toFixed(1)}MB`);
  }
};

// Detect leaks: take snapshots before and after navigation
// Compare retained objects вҖ” leaked objects appear in diff
Warning

React Native has two memory spaces: JS heap (managed by Hermes/JSC GC) and native heap (managed by ARC/GC). A JS heap snapshot won't show native memory leaks (e.g., unreleased images, native module allocations). Use Xcode Instruments (Allocations) for iOS native memory and Android Studio Profiler for Android native memory. Cross-reference both to get the full picture.

7. Network Profiling

// Flipper Network plugin вҖ” best for RN network debugging
// Shows: URL, method, status, request/response bodies,
//        headers, timing breakdown, size

// Programmatic network timing
const timedFetch = async (url: string, options?: RequestInit) => {
  const start = performance.now();
  try {
    const response = await fetch(url, options);
    const duration = performance.now() - start;
    analytics.trackNetworkTiming({
      url,
      duration,
      status: response.status,
      size: response.headers.get('content-length'),
    });
    return response;
  } catch (error) {
    const duration = performance.now() - start;
    analytics.trackNetworkError({ url, duration, error });
    throw error;
  }
};

9. Frame Rate Monitoring

// Built-in FPS monitor
// Dev Menu вҶ’ Show Perf Monitor
// Shows JS FPS and UI FPS in real-time

// Programmatic frame rate tracking
import { FrameRateLogger } from 'react-native';

FrameRateLogger.setGlobalOptions({
  debug: true,
  reportStackTraces: true,
});

// Custom frame drop detection
let lastFrameTime = performance.now();
const checkFrameRate = () => {
  const now = performance.now();
  const frameDuration = now - lastFrameTime;
  if (frameDuration > 32) { // >2 frames dropped (16ms Г— 2)
    console.warn(`Frame drop: ${frameDuration.toFixed(1)}ms`);
  }
  lastFrameTime = now;
  requestAnimationFrame(checkFrameRate);
};
requestAnimationFrame(checkFrameRate);

10. Architect Considerations

Architect Insight

Build a performance monitoring culture: (1) Establish performance budgets (TTI < 2s, FPS > 55, JS heap < 150MB). (2) Add automated performance tests in CI using Detox + custom metrics. (3) Track startup time, screen transition time, and scroll FPS as KPIs. (4) Profile on low-end devices (not just flagships) вҖ” use Android Go or iPhone SE as baseline. (5) Create a performance dashboard that tracks metrics across releases. (6) Use Hermes sampling profiler in production (low overhead) to catch real-world bottlenecks that don't reproduce in development.
