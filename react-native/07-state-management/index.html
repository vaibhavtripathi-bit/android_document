Series 7: State Management

From local component state to global stores, server caches, and persistent storage — mastering the full state management spectrum in React Native at Staff-level depth.

State management in React Native introduces unique challenges absent in web development: navigation state that must survive stack resets, persistent state across app kills, background/foreground transitions that invalidate caches, memory pressure from the OS, and the need to synchronize state between JavaScript and native layers. This series covers every dimension of state management a Google L6 candidate must command — from Redux and Zustand internals to MMKV persistence, TanStack Query caching, and cross-surface state synchronization in brownfield apps.

Documents in This Series

01 — State in React Native

State management considerations unique to mobile: navigation state, persistent state across restarts, background/foreground transitions, platform-specific state, memory pressure, and storage comparison (AsyncStorage vs MMKV vs SQLite).

02 — Redux in React Native

Redux Toolkit setup, Redux Persist for state persistence, middleware for navigation integration, Flipper debugging, selector optimization, normalized state, offline-first patterns, and migration strategies.

03 — Zustand & MobX

Zustand: lightweight stores with persistence middleware and MMKV integration. MobX: observables, computed values, reactions, and MobX-State-Tree for structured state with snapshots. Performance comparison across libraries.

04 — Server State in RN

TanStack Query caching and offline support. Apollo Client cache normalization and optimistic updates. Relay compiler-driven data fetching. Network state integration and mobile pagination patterns.

05 — Persistent Storage

AsyncStorage, MMKV (JSI-based synchronous), SQLite (react-native-quick-sqlite), Realm object database. Encrypted storage, Keychain/Keystore, migration strategies, and large dataset handling.

06 — State Synchronization

JS-to-native state sync, shared state in brownfield apps, multi-surface state, background updates, push notification state, deep link restoration, state machines, and undo/redo patterns.

07 — Interview Q&A

50+ L6-level questions covering state management patterns, Redux/Zustand/MobX internals, server state, persistent storage, and state synchronization in React Native.
