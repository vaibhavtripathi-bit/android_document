Series 12: Build & Deployment

From Metro bundler internals to OTA update strategies — the build and deployment pipeline every Staff-level mobile engineer must master for Google L6 interviews.

This series covers the complete build and deployment lifecycle of React Native applications at the depth expected for Staff-level interviews. We examine the Metro bundler's configuration and optimization, native build systems (Xcode and Gradle), the Expo ecosystem and its managed infrastructure, over-the-air update architectures, CI/CD pipeline design, code signing and provisioning, release management strategies, and monitoring. Each document focuses on the architectural decisions and trade-offs that distinguish a Staff engineer from a Senior one.

Documents in This Series

01 — Metro Configuration

metro.config.js deep dive, resolver configuration, transformer options, serializer customization, asset handling, source map generation, custom module resolution, monorepo support, platform-specific resolution, symlink handling, watch mode.

02 — Native Build Systems

Xcode build phases for RN, Gradle configuration for RN Android, CocoaPods integration, autolinking mechanism, build flavors/schemes, environment-specific builds, build variables, native dependency management, build optimization.

03 — Expo Ecosystem

Expo Go, Development builds, EAS Build, EAS Submit, EAS Update, config plugins, prebuild system, managed vs bare workflow, Expo Router, Expo Modules, migration between workflows.

04 — OTA Updates

CodePush architecture, EAS Update, update strategies (mandatory, optional, silent), bundle compatibility, rollback mechanisms, A/B testing, delta updates, app store policies, staged rollouts.

05 — CI/CD Pipelines

GitHub Actions / Bitrise / CircleCI for RN, build matrix strategies, caching (node_modules, CocoaPods, Gradle), parallel builds, artifact management, test automation integration, environment secrets.

06 — Code Signing & Provisioning

iOS provisioning profiles, Android keystore management, Fastlane match, automatic signing, CI signing, distribution certificates, App Store Connect API, Google Play service accounts.

07 — Release Management

Semantic versioning for mobile, staged rollouts, feature flags, A/B testing, beta distribution (TestFlight, Firebase App Distribution), release trains, hotfix workflows, rollback strategies.

08 — Interview Q&A

55+ L6-level interview questions covering Metro bundler, native builds, Expo, OTA updates, CI/CD, code signing, release management, and production monitoring.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6
Build Pipeline Architecture Staff engineers own the entire build pipeline — from Metro bundling through native compilation to store submission. Understanding how these stages compose and where bottlenecks arise is critical for scaling team velocity.
Metro Bundler Mastery Metro is the heart of the RN development experience. Deep configuration knowledge — custom resolvers, transformers, serializers — enables monorepo support, build optimization, and platform-specific code strategies that L5 engineers struggle with.
Native Build System Integration React Native sits atop Xcode and Gradle. Understanding build phases, autolinking, CocoaPods, and build flavors is essential for debugging build failures that block entire teams and for optimizing CI build times.
Expo vs Bare Trade-offs The Expo ecosystem has evolved dramatically. Staff engineers must evaluate managed vs bare workflows, EAS services, and config plugins — understanding when Expo accelerates delivery and when it constrains architecture.
OTA Update Strategy Over-the-air updates (CodePush, EAS Update) enable rapid iteration but introduce versioning complexity, compatibility risks, and app store policy concerns. Designing a safe, observable OTA strategy is a core Staff responsibility.
Release Engineering Staged rollouts, feature flags, beta distribution, and rollback mechanisms form the safety net for production releases. Staff engineers design these systems to balance speed with reliability across iOS and Android simultaneously.

Learning Objectives

  • Configure Metro bundler for complex scenarios including monorepos, custom resolvers, and platform-specific module resolution
  • Debug and optimize native build systems — Xcode build phases, Gradle tasks, CocoaPods, and autolinking
  • Evaluate the Expo ecosystem (managed workflow, EAS services, config plugins) and articulate when to adopt or eject
  • Design OTA update strategies that balance deployment velocity with stability, compatibility, and app store compliance
  • Architect CI/CD pipelines that handle the unique challenges of React Native (dual-platform builds, native dependencies, code signing)
  • Implement code signing and provisioning automation that works reliably across local development and CI environments
  • Design release management processes including staged rollouts, feature flags, and rollback mechanisms for mobile apps

Prerequisites

  • Production React Native experience with both iOS and Android builds
  • Basic familiarity with Xcode and Android Studio build processes
  • Understanding of JavaScript bundling concepts (modules, tree-shaking, source maps)
  • Experience with at least one CI/CD platform (GitHub Actions, Bitrise, CircleCI)
  • Familiarity with app store submission processes for both platforms
