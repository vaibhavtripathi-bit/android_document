Accessibility in React Native

Accessibility is not optional — it's a legal requirement (ADA, Section 508, EAA) and a quality signal. React Native maps accessibility props to native platform APIs (UIAccessibility on iOS, AccessibilityNodeInfo on Android). Staff engineers must understand the bridging layer, screen reader interaction models, and how to build WCAG-compliant mobile experiences.

2. Accessibility Roles

// Available accessibilityRole values
type AccessibilityRole =
  | 'none'           // no role
  | 'button'         // tappable element
  | 'link'           // navigates to URL
  | 'search'         // search field
  | 'image'          // image content
  | 'imagebutton'    // tappable image
  | 'header'         // section header
  | 'summary'        // summary of content
  | 'alert'          // important notification
  | 'checkbox'       // toggleable checkbox
  | 'combobox'       // dropdown/combo
  | 'menu'           // menu container
  | 'menubar'        // menu bar
  | 'menuitem'       // item in menu
  | 'progressbar'    // progress indicator
  | 'radio'          // radio button
  | 'radiogroup'     // group of radio buttons
  | 'scrollbar'      // scroll indicator
  | 'spinbutton'     // numeric stepper
  | 'switch'         // on/off toggle
  | 'tab'            // tab button
  | 'tablist'        // tab container

  | 'timer'          // countdown/timer

3. Accessibility State & Value

// accessibilityState — current interactive state
<View accessibilityState={{
  disabled: false,    // element is disabled
  selected: true,     // element is selected
  checked: true,      // checkbox/switch is checked (true | false | 'mixed')
  busy: false,        // element is loading
  expanded: true,     // expandable element is expanded
}} />

// accessibilityValue — current value for adjustable elements
<Slider
  accessibilityRole="adjustable"
  accessibilityLabel="Volume"
  accessibilityValue={{
    min: 0,
    max: 100,
    now: 75,
    text: "75 percent",  // overrides min/max/now announcement
  }}
/>

// Practical example: Accordion
const Accordion = ({ title, children, isOpen }) => (
  <View>
    <TouchableOpacity
      accessibilityRole="button"
      accessibilityState={{ expanded: isOpen }}
      accessibilityLabel={title}
      accessibilityHint={isOpen
        ? "Double tap to collapse"
        : "Double tap to expand"}
      onPress={toggle}
    >
      <Text>{title}</Text>
      <Icon name={isOpen ? 'chevron-up' : 'chevron-down'} />
    </TouchableOpacity>
    {isOpen && children}
  </View>
);

4. VoiceOver & TalkBack

4.1 Screen Reader Detection

import { AccessibilityInfo } from 'react-native';

// Check if screen reader is active
const useScreenReader = () => {
  const [isActive, setIsActive] = useState(false);

  useEffect(() => {
    AccessibilityInfo.isScreenReaderEnabled().then(setIsActive);

    const sub = AccessibilityInfo.addEventListener(
      'screenReaderChanged',
      setIsActive
    );
    return () => sub.remove();
  }, []);

  return isActive;
};

// Adapt UI for screen reader users
const ImageCarousel = () => {
  const isScreenReader = useScreenReader();

  if (isScreenReader) {
    // Show simple list instead of swipeable carousel
    return <AccessibleImageList images={images} />;
  }
  return <SwipeableCarousel images={images} />;
};

5. Focus Management

import { AccessibilityInfo, findNodeHandle } from 'react-native';

// Set accessibility focus programmatically
const FormScreen = () => {
  const errorRef = useRef(null);

  const onSubmitError = () => {
    // Move focus to error message
    const node = findNodeHandle(errorRef.current);
    if (node) {
      AccessibilityInfo.setAccessibilityFocus(node);
    }
  };

  return (
    <View>
      <Text
        ref={errorRef}
        accessibilityRole="alert"
        accessibilityLiveRegion="assertive"
      >
        {errorMessage}
      </Text>
    </View>
  );
};

// Modal focus trapping
const AccessibleModal = ({ visible, children }) => (
  <Modal
    visible={visible}
    accessibilityViewIsModal={true}  // iOS: traps focus in modal
    onShow={() => {
      // Focus first element in modal
      const node = findNodeHandle(firstElementRef.current);
      if (node) AccessibilityInfo.setAccessibilityFocus(node);
    }}
  >
    {children}
  </Modal>
);
importantForAccessibility (Android)

Android provides importantForAccessibility prop with values: 'auto', 'yes', 'no', 'no-hide-descendants'. The last value hides the element and all children from TalkBack — useful for decorative elements or off-screen content. iOS equivalent is accessibilityElementsHidden={true}.

6. Accessibility Actions

// Custom accessibility actions for swipeable list items
<View
  accessible={true}
  accessibilityLabel={`Message from ${sender}`}
  accessibilityActions={[
    { name: 'activate', label: 'Open message' },
    { name: 'delete', label: 'Delete message' },
    { name: 'archive', label: 'Archive message' },
    { name: 'magicTap', label: 'Quick reply' },  // iOS two-finger double-tap
    { name: 'escape', label: 'Close' },            // iOS two-finger Z gesture
    { name: 'increment' },  // for adjustable role
    { name: 'decrement' },  // for adjustable role
  ]}
  onAccessibilityAction={(event) => {
    switch (event.nativeEvent.actionName) {
      case 'delete':
        handleDelete();
        break;
      case 'archive':
        handleArchive();
        break;
      case 'magicTap':
        handleQuickReply();
        break;
    }
  }}
/>
Architect Insight

Swipe-to-delete and drag-to-reorder are inaccessible by default. Always provide accessibility actions as alternatives. The magicTap action (iOS two-finger double-tap) is ideal for the primary action on a screen (e.g., play/pause in a media player). The escape action (iOS two-finger Z-gesture) should dismiss modals and popovers.

7. Live Regions

// accessibilityLiveRegion — announce dynamic content changes
<Text accessibilityLiveRegion="polite">
  {itemCount} items in cart
</Text>

// 'polite' — announces when screen reader is idle
// 'assertive' — interrupts current announcement
// 'none' — no announcement (default)

// Practical: Toast/snackbar notification
const Toast = ({ message, type }) => (
  <View
    accessibilityRole="alert"
    accessibilityLiveRegion="assertive"
  >
    <Text>{message}</Text>
  </View>
);

// Practical: Loading state
const LoadingIndicator = ({ isLoading }) => (
  <View
    accessibilityRole="progressbar"
    accessibilityState={{ busy: isLoading }}
    accessibilityLiveRegion="polite"
    accessibilityLabel={isLoading ? "Loading content" : "Content loaded"}
  >
    {isLoading && <ActivityIndicator />}
  </View>
);

8. Semantic Grouping

// Group related elements for screen reader
<View
  accessible={true}
  accessibilityLabel="John Doe, Software Engineer, 5 mutual connections"
>
  <Image source={avatar} accessibilityElementsHidden />
  <Text>John Doe</Text>
  <Text>Software Engineer</Text>
  <Text>5 mutual connections</Text>
</View>

// Without grouping: screen reader reads each Text separately
// With grouping: reads combined label in one focus stop

// accessibilityElementsHidden (iOS) — hide from VoiceOver
<View accessibilityElementsHidden={true}>
  <DecorativeImage />
</View>

// importantForAccessibility (Android)
<View importantForAccessibility="no-hide-descendants">
  <DecorativeImage />
</View>

9. Accessibility Testing

// @testing-library/react-native — accessibility queries
import { render, screen } from '@testing-library/react-native';

// Query by accessibility label
const button = screen.getByLabelText('Submit form');

// Query by role
const heading = screen.getByRole('header', { name: 'Settings' });

// Query by accessibility state
const toggle = screen.getByRole('switch', { checked: true });

// Verify accessibility properties

expect(heading).toHaveAccessibilityValue({ text: '75%' });

Architect Insight

Build accessibility into your component library from day one. Every interactive component should: (1) Have a meaningful accessibilityLabel. (2) Set the correct accessibilityRole. (3) Reflect state via accessibilityState. (4) Meet 44×44pt minimum touch target size. (5) Maintain 4.5:1 contrast ratio. Create an accessibility linting rule that fails CI if components lack required props. Use react-native-ama in development builds to catch issues early.
