Internationalization (i18n) in React Native

Internationalization goes far beyond string translation — it encompasses RTL layout mirroring, date/number formatting, pluralization rules, script-specific typography, and dynamic language switching without app restart. Staff engineers must architect i18n systems that scale to dozens of locales without performance degradation.

1. react-native-localize

react-native-localize provides access to the device's locale settings — language, region, calendar, temperature unit, and timezone. It bridges native NSLocale (iOS) and Locale (Android) APIs.

import * as RNLocalize from 'react-native-localize';

// Get ordered list of user's preferred locales
const locales = RNLocalize.getLocales();
// [{ languageTag: 'en-US', languageCode: 'en', countryCode: 'US',
//    isRTL: false, scriptCode: 'Latn' }]

// Get user's preferred currencies
const currencies = RNLocalize.getCurrencies();
// ['USD', 'EUR']

// Calendar and temperature preferences
const calendar = RNLocalize.getCalendar();        // 'gregorian' | 'japanese' | ...
const tempUnit = RNLocalize.getTemperatureUnit(); // 'celsius' | 'fahrenheit'
const uses24h = RNLocalize.uses24HourClock();     // boolean
const timezone = RNLocalize.getTimeZone();       // 'America/New_York'

// Find best matching language from supported set
const bestLang = RNLocalize.findBestLanguageTag(
  ['en', 'fr', 'de', 'ja']
);
// { languageTag: 'en', isRTL: false }

// Listen for locale changes (user changes device language)
RNLocalize.addEventListener('change', () => {
  updateAppLocale();
});

2. i18next / react-i18next

import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as RNLocalize from 'react-native-localize';

const resources = {
  en: { translation: require('./locales/en.json') },
  fr: { translation: require('./locales/fr.json') },
  ja: { translation: require('./locales/ja.json') },
  ar: { translation: require('./locales/ar.json') },
};

const bestLang = RNLocalize.findBestLanguageTag(
  Object.keys(resources)
);

i18n
  .use(initReactI18next)
  .init({
    resources,
    lng: bestLang?.languageTag || 'en',
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
    react: { useSuspense: false },
  });

// Usage in components
import { useTranslation } from 'react-i18next';

const Greeting = () => {
  const { t } = useTranslation();
  return <Text>{t('greeting', { name: 'John' })}</Text>;
};

// locales/en.json
// { "greeting": "Hello, {{name}}!" }
// locales/fr.json
// { "greeting": "Bonjour, {{name}} !" }

2.1 Namespace Organization

// Organize translations by feature/screen
const resources = {
  en: {
    common: require('./locales/en/common.json'),
    auth: require('./locales/en/auth.json'),
    settings: require('./locales/en/settings.json'),
    errors: require('./locales/en/errors.json'),
  },
};

// Use specific namespace
const { t } = useTranslation('auth');
t('login.title');  // from auth namespace

// Multiple namespaces
const { t } = useTranslation(['auth', 'common']);
t('auth:login.title');
t('common:buttons.submit');

3. RTL Support

import { I18nManager } from 'react-native';

// Check current layout direction
I18nManager.isRTL;         // boolean
I18nManager.doLeftAndRightSwapInRTL;

// Force RTL (requires app restart)
I18nManager.forceRTL(true);
I18nManager.allowRTL(true);

// After changing RTL, restart the app
import RNRestart from 'react-native-restart';
RNRestart.restart();
RTL Layout Mirroring ════════════════════ LTR Layout RTL Layout ┌──────────────────────┐ ┌──────────────────────┐ │ [←] Title [...] │ │ [...] Title [→] │ ├──────────────────────┤ ├──────────────────────┤ │ 🖼 Name │ │ Name 🖼 │ │ Description │ │ Description │ │ Detail [>] │ │ [<] Detail │ ├──────────────────────┤ ├──────────────────────┤ │ 🖼 Name │ │ Name 🖼 │ │ Description │ │ Description │ └──────────────────────┘ └──────────────────────┘ Flexbox automatically mirrors: - flexDirection: 'row' → items flow right-to-left - marginStart/paddingStart → maps to correct side - textAlign: 'left' → becomes 'right'

3.1 RTL-Aware Styles

const styles = StyleSheet.create({
  row: {
    flexDirection: 'row',  // automatically mirrors in RTL
    paddingStart: 16,       // logical property (left in LTR, right in RTL)
    paddingEnd: 8,          // logical property
    marginStart: 12,
  },
  icon: {
    // Use transform to flip icons that have directional meaning
    transform: [{ scaleX: I18nManager.isRTL ? -1 : 1 }],
  },
  absolutePositioned: {
    position: 'absolute',
    start: 16,   // logical: left in LTR, right in RTL
    end: 16,     // logical: right in LTR, left in RTL
  },
});

// writingDirection for text alignment
<Text style={{ writingDirection: 'auto' }}>
  {mixedDirectionText}
</Text>
Warning

Not everything should mirror in RTL: (1) Media playback controls (play/pause/seek). (2) Checkmarks and progress bars. (3) Phone number fields. (4) Clocks and timelines. Use transform: [{ scaleX: -1 }] selectively for directional icons (arrows, chevrons) but not for universal symbols.

4. Date & Number Formatting

// Intl API (requires Hermes or polyfill on older JSC)
const formatDate = (date: Date, locale: string) => {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);
};

formatDate(new Date(), 'en-US');  // "March 28, 2026"
formatDate(new Date(), 'ja-JP');  // "2026年3月28日"
formatDate(new Date(), 'ar-SA');  // "٢٨ مارس ٢٠٢٦"

// Number formatting
const formatCurrency = (amount: number, locale: string, currency: string) => {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(amount);
};

formatCurrency(1234.56, 'en-US', 'USD');  // "$1,234.56"
formatCurrency(1234.56, 'de-DE', 'EUR');  // "1.234,56 €"
formatCurrency(1234.56, 'ja-JP', 'JPY');  // "￥1,235"

// Relative time formatting
const rtf = new Intl.RelativeTimeFormat('en', { numeric: 'auto' });
rtf.format(-1, 'day');    // "yesterday"
rtf.format(2, 'hour');    // "in 2 hours"
Hermes Intl Support

Hermes has built-in Intl support since React Native 0.70+ (with hermes.intl flag). For older versions or JSC, use intl polyfill or @formatjs/intl-* packages. Hermes Intl uses ICU data bundled with the engine, adding ~2-6MB to the binary depending on locale data included.

5. Pluralization

// i18next pluralization (uses CLDR plural rules)
// en.json
{
  "item_one": "{{count}} item",
  "item_other": "{{count}} items"
}

// ar.json (Arabic has 6 plural forms!)
{
  "item_zero": "لا عناصر",
  "item_one": "عنصر واحد",
  "item_two": "عنصران",
  "item_few": "{{count}} عناصر",
  "item_many": "{{count}} عنصرًا",
  "item_other": "{{count}} عنصر"
}

// Usage
t('item', { count: 0 });   // "0 items" (en) / "لا عناصر" (ar)
t('item', { count: 1 });   // "1 item" (en) / "عنصر واحد" (ar)
t('item', { count: 5 });   // "5 items" (en) / "5 عناصر" (ar)

6. Dynamic Language Switching

const LanguageSwitcher = () => {
  const { i18n } = useTranslation();

  const changeLanguage = async (lang: string) => {
    await i18n.changeLanguage(lang);

    // Persist selection
    await AsyncStorage.setItem('userLanguage', lang);

    // Handle RTL change if needed
    const isRTL = ['ar', 'he', 'fa', 'ur'].includes(lang);
    if (I18nManager.isRTL !== isRTL) {
      I18nManager.forceRTL(isRTL);
      I18nManager.allowRTL(isRTL);
      RNRestart.restart();  // RTL change requires restart
    }
  };

  return (
    <View>
      {['en', 'fr', 'ar', 'ja'].map(lang => (
        <TouchableOpacity
          key={lang}
          onPress={() => changeLanguage(lang)}
          accessibilityRole="radio"
          accessibilityState={{ checked: i18n.language === lang }}
        >
          <Text>{lang}</Text>
        </TouchableOpacity>
      ))}
    </View>
  );
};

7. Font Handling for Different Scripts

// Different scripts require different fonts
const getFontFamily = (script: string) => {
  const fontMap: Record<string, string> = {
    Latn: 'Inter',           // Latin (English, French, German)
    Arab: 'NotoSansArabic',  // Arabic script
    Jpan: 'NotoSansJP',      // Japanese
    Kore: 'NotoSansKR',      // Korean
    Hans: 'NotoSansSC',      // Simplified Chinese
    Deva: 'NotoSansDevanagari', // Hindi
    Thai: 'NotoSansThai',    // Thai
  };
  return fontMap[script] || 'System';
};

// Dynamic font loading based on locale
const useLocalizedFont = () => {
  const locales = RNLocalize.getLocales();
  const script = locales[0]?.scriptCode || 'Latn';
  return getFontFamily(script);
};
Warning

CJK (Chinese, Japanese, Korean) fonts are significantly larger (5-20MB per weight) compared to Latin fonts (~100KB). Use variable fonts where possible, and consider loading CJK fonts on demand rather than bundling them. System fonts handle CJK well on both platforms — prefer system fonts for body text in CJK locales.

10. Architect Considerations

Architect Insight

Design your i18n architecture for scale: (1) Use namespaced translation files per feature module — this enables lazy loading and reduces initial bundle impact. (2) Implement a translation key linting step in CI that detects unused keys and missing translations. (3) Use ICU MessageFormat syntax for complex messages (gender, plural, select). (4) Plan for text expansion — German text is typically 30% longer than English; Arabic/Hebrew require RTL layout mirroring. (5) Never concatenate translated strings — word order varies across languages. (6) Store user language preference server-side for push notification localization.
