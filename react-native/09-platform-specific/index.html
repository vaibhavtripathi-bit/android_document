Series 9: Platform-Specific Development

Mastering platform divergence — from conditional code patterns and OS-specific APIs to responsive design, accessibility, internationalization, and cross-platform web support at Staff-engineer depth.

React Native's promise of "learn once, write anywhere" requires deep understanding of where platforms diverge and how to handle those divergences architecturally. This series covers the Platform module internals, iOS and Android specific considerations, responsive design systems, accessibility compliance, internationalization pipelines, and React Native for Web — all at the depth expected for Google L6 interviews.

Documents in This Series

01 — Platform Module

Platform.OS, Platform.select, Platform.Version. Platform-specific file extensions (.ios.js, .android.js, .native.js, .web.js). Conditional platform code patterns. PlatformColor. Platform-specific styling decisions.

02 — iOS-Specific Considerations

Safe area handling, status bar management, keyboard avoidance, iOS permissions model, haptic feedback, iOS-specific components, Dynamic Island, and Live Activities integration.

03 — Android-Specific Considerations

Back button handling, Android runtime permissions, Material Design integration, notification channels, app links, split-screen support, Android 12+ splash screen, and ProGuard/R8 configuration.

04 — Responsive Design

Dimensions API, useWindowDimensions, Flexbox layout, percentage-based sizing, tablet support, landscape/portrait handling, dynamic font scaling, and safe area management across devices.

05 — Accessibility

accessible prop, accessibilityLabel/Role/State/Hint, VoiceOver and TalkBack testing, focus management, accessibility actions, live regions, semantic grouping, and WCAG compliance.

06 — Internationalization (i18n)

react-native-localize, i18next integration, RTL support with I18nManager, date/number formatting, pluralization, dynamic language switching, and translation management workflows.

07 — React Native for Web

react-native-web internals, shared codebase strategies, Next.js + RN Web, Expo for web, Solito for universal navigation, Tamagui for universal UI, and platform-specific web code.

08 — Interview Q&A

55+ L6-level questions covering platform-specific development, responsive design, accessibility, internationalization, and web support with detailed answers.
