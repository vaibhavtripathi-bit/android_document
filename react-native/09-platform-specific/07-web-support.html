React Native for Web

react-native-web maps React Native components to DOM elements, enabling a single codebase across iOS, Android, and web. Staff engineers must understand the mapping layer, bundle optimization, universal navigation patterns, and the trade-offs of shared vs platform-specific code in production universal apps.

1. How react-native-web Works

react-native-web provides a web-compatible implementation of React Native's core components and APIs. It works by aliasing react-native imports to react-native-web at the bundler level (webpack/Metro).

react-native-web Architecture ═════════════════════════════ Your Code Bundler (webpack) Output ───────── ───────────────── ────── import { View } alias resolution: from 'react-native' → 'react-native' → import { View } 'react-native-web' → from 'react-native-web' │ ↓ <div> with inline styles + className RN Component → DOM Element ──────────── ─────────── View → div Text → div (role="text") / span Image → img TextInput → input / textarea ScrollView → div (overflow: auto) TouchableOpacity → div (with event handlers) FlatList → div (virtualized) Modal → dialog / portal 
// webpack.config.js — aliasing react-native to react-native-web
module.exports = {
  resolve: {
    alias: {
      'react-native$': 'react-native-web',
    },
    extensions: ['.web.tsx', '.web.ts', '.web.js',
                 '.tsx', '.ts', '.js'],
  },
};

1.1 StyleSheet Implementation

// React Native StyleSheet on web
// Converts RN styles to atomic CSS classes

// Input:
StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    padding: 16,
  },
});

// Output (atomic CSS):
// .r-flex-1 { flex: 1 }
// .r-bg-fff { background-color: #fff }
// .r-p-16 { padding: 16px }
// className="r-flex-1 r-bg-fff r-p-16"

// Benefits of atomic CSS:
// - CSS size grows logarithmically (not linearly) with components
// - Styles are deduplicated across components
// - No specificity conflicts

    type="date"

5. Responsive Web Design with RN

// useWindowDimensions works on web too
import { useWindowDimensions } from 'react-native';

const ResponsiveLayout = () => {
  const { width } = useWindowDimensions();

  return (
    <View style={{
      flexDirection: width > 768 ? 'row' : 'column',
      maxWidth: 1200,
      marginHorizontal: 'auto',  // center on web
      width: '100%',
    }}>
      <Sidebar style={{ width: width > 768 ? 280 : '100%' }} />
      <Content style={{ flex: 1 }} />
    </View>
  );
};

6. Next.js + React Native Web

// next.config.js
const { withExpo } = require('@expo/next-adapter');

module.exports = withExpo({
  transpilePackages: [
    'react-native',
    'react-native-web',
    'expo',
    // Add any RN libraries that need transpilation
    'react-native-reanimated',
    'react-native-gesture-handler',
  ],
  webpack: (config) => {
    config.resolve.alias = {
      ...config.resolve.alias,
      'react-native$': 'react-native-web',
    };
    return config;
  },
});

// pages/index.tsx — uses RN components with SSR
import { View, Text, StyleSheet } from 'react-native';

export default function Home() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Universal App</Text>
    </View>
  );
}
Warning

SSR with react-native-web requires careful handling: (1) useWindowDimensions returns 0 during SSR — use fallback values. (2) Platform.OS is 'web' on both server and client. (3) Animations (Reanimated) don't work during SSR — use CSS animations as fallback. (4) Some RN libraries import native modules that crash on web — use .web.ts stubs.

8. Solito for Universal Navigation

// Solito bridges React Navigation (native) and Next.js Router (web)
import { createParam } from 'solito';
import { useLink } from 'solito/link';
import { useRouter } from 'solito/router';

// Universal link component
const ProfileLink = ({ userId }) => {
  const linkProps = useLink({
    href: `/profile/${userId}`,
  });
  // On native: navigates via React Navigation
  // On web: renders <a href="/profile/123">
  return <Text {...linkProps}>View Profile</Text>;
};

// Universal router
const SubmitButton = () => {
  const router = useRouter();
  return (
    <Button onPress={() => router.push('/success')} title="Submit" />
  );
};

// Universal params
const { useParam } = createParam<{ id: string }>();

const ProfileScreen = () => {
  const [id] = useParam('id');

};

9. Tamagui for Universal UI

// Tamagui: optimizing compiler for universal React Native + Web UI
import { Button, XStack, YStack, Text } from 'tamagui';

const Card = () => (
  <YStack
    padding="$4"
    borderRadius="$4"
    backgroundColor="$background"
    elevation="$2"
    $sm={{ padding: "$2" }}     // responsive breakpoint
    $gtMd={{ maxWidth: 600 }}   // greater than medium
  >
    <XStack gap="$2" alignItems="center">
      <Avatar />
      <Text fontSize="$5">Title</Text>
    </XStack>
    <Button theme="blue" size="$4">Action</Button>
  </YStack>
);

// Tamagui compiler optimization:
// - Extracts styles to CSS at build time (web)
// - Flattens component tree (removes wrapper Views)
// - Converts dynamic styles to static where possible
// - Result: faster rendering on both native and web
Tamagui vs NativeWind vs StyleSheet

Tamagui: Full UI kit with optimizing compiler, design tokens, responsive props. Best for universal apps. NativeWind: Tailwind CSS for React Native, compiles to StyleSheet. Good for teams familiar with Tailwind. StyleSheet: Built-in, zero dependencies, but no responsive props or design tokens. For Staff-level decisions: Tamagui for universal apps, NativeWind for mobile-only with Tailwind teams, StyleSheet for maximum control.

Architect Insight

Universal app architecture decisions: (1) Share business logic, state management, and API layers (70-80% code sharing is realistic). (2) Use platform-specific UI for interactions that differ fundamentally (navigation, gestures, notifications). (3) Adopt Expo Router or Solito early — retrofitting universal navigation is expensive. (4) Test on web from day one, not as an afterthought. (5) Use .native.ts / .web.ts for platform-specific service implementations. (6) Consider Tamagui for a unified design system that optimizes for both platforms. The key insight: optimize for developer velocity on the shared layer, and accept platform-specific code where UX demands it.
