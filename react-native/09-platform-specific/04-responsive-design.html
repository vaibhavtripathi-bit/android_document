Responsive Design in React Native

Unlike CSS media queries, React Native responsive design relies on Flexbox, the Dimensions API, and dynamic measurement hooks. Staff engineers must architect layout systems that gracefully handle phones, tablets, foldables, landscape orientation, split-screen, and dynamic type scaling.

1. Dimensions API

The Dimensions API provides screen and window measurements. The distinction between "screen" and "window" matters on Android where the window excludes the status bar and navigation bar.

import { Dimensions } from 'react-native';

const { width, height } = Dimensions.get('window');
const { width: screenW, height: screenH } = Dimensions.get('screen');

// 'window' — usable area (excludes status/nav bars on Android)
// 'screen' — full physical screen dimensions

// Listen for dimension changes (orientation, split-screen)
const subscription = Dimensions.addEventListener(
  'change',
  ({ window, screen }) => {
    console.log('Window:', window.width, window.height);
    console.log('Screen:', screen.width, screen.height);
  }
);
Warning

Dimensions.get() returns a snapshot at call time. Values become stale after orientation changes or split-screen resizes. Always use useWindowDimensions() hook or Dimensions.addEventListener for reactive layouts. Caching dimensions at module scope is a common source of layout bugs.

2. useWindowDimensions

import { useWindowDimensions } from 'react-native';

const ResponsiveGrid = () => {
  const { width, height, scale, fontScale } = useWindowDimensions();

  const numColumns = width > 768 ? 3 : width > 480 ? 2 : 1;
  const itemWidth = (width - (numColumns + 1) * 16) / numColumns;

  return (
    <FlatList
      data={items}
      numColumns={numColumns}
      key={numColumns}  // force re-render on column change
      renderItem={({ item }) => (
        <View style={{ width: itemWidth, margin: 8 }}>
          <Card item={item} />
        </View>
      )}
    />
  );
};

// Custom breakpoint hook
type Breakpoint = 'sm' | 'md' | 'lg' | 'xl';

const useBreakpoint = (): Breakpoint => {
  const { width } = useWindowDimensions();
  if (width < 480) return 'sm';
  if (width < 768) return 'md';
  if (width < 1024) return 'lg';
  return 'xl';
};
Hook Internals

useWindowDimensions subscribes to Dimensions change events internally and triggers re-renders. It returns { width, height, scale, fontScale }. The scale is the device pixel ratio (e.g., 3 for iPhone 15 Pro), and fontScale reflects the user's accessibility text size preference.

3. Flexbox Layout System

React Native uses Yoga (a C++ Flexbox implementation) for layout. It follows the CSS Flexbox spec with some key differences.

// Key Flexbox properties in React Native
const styles = StyleSheet.create({
  container: {
    flex: 1,
    flexDirection: 'column',    // default (CSS default is 'row')
    justifyContent: 'center',   // main axis alignment
    alignItems: 'stretch',      // cross axis alignment
    flexWrap: 'wrap',           // allow wrapping
    gap: 16,                    // RN 0.71+ (Yoga gap support)
    rowGap: 8,
    columnGap: 16,
  },
  item: {
    flexGrow: 1,
    flexShrink: 0,
    flexBasis: 'auto',
    alignSelf: 'flex-start',
  },
});

3.1 Common Responsive Patterns

// Responsive row-to-column layout
const ResponsiveRow = ({ children }) => {
  const { width } = useWindowDimensions();
  return (
    <View style={{
      flexDirection: width > 600 ? 'row' : 'column',
      gap: 16,
    }}>
      {children}
    </View>
  );
};

// Equal-width columns with flex
const EqualColumns = () => (
  <View style={{ flexDirection: 'row' }}>
    <View style={{ flex: 1 }}><Text>Col 1</Text></View>
    <View style={{ flex: 1 }}><Text>Col 2</Text></View>
    <View style={{ flex: 1 }}><Text>Col 3</Text></View>
  </View>
);

4. Percentage-Based Sizing

const styles = StyleSheet.create({
  halfWidth: {
    width: '50%',       // 50% of parent width
  },
  fullHeight: {
    height: '100%',     // 100% of parent height
  },
  margins: {
    marginHorizontal: '5%',  // percentage margins
  },
  padding: {
    padding: '3%',          // percentage padding
  },
  positioned: {
    position: 'absolute',
    top: '10%',
    left: '10%',
    width: '80%',
    height: '80%',
  },
});

// Limitation: percentage-based sizing requires the parent to have
// a defined dimension. A parent with flex:1 provides this.
// A parent with no height/flex makes percentage height = 0.

5. Aspect Ratio

const styles = StyleSheet.create({
  square: {
    width: '100%',
    aspectRatio: 1,       // height = width
  },
  widescreen: {
    width: '100%',
    aspectRatio: 16 / 9,  // 16:9 video container
  },
  portrait: {
    height: 300,
    aspectRatio: 3 / 4,   // width derived from height
  },
  thumbnail: {
    width: 120,
    aspectRatio: 1,
    borderRadius: 8,
  },
});

// Responsive image with aspect ratio
const ResponsiveImage = ({ uri, ratio = 16/9 }) => (
  <Image
    source={{ uri }}
    style={{ width: '100%', aspectRatio: ratio }}
    resizeMode="cover"
  />
);

6. Tablet Support

import { Platform, useWindowDimensions } from 'react-native';

const useDeviceType = () => {
  const { width } = useWindowDimensions();

  if (Platform.isPad || width >= 768) return 'tablet';
  if (width >= 600) return 'large-phone';
  return 'phone';
};

// Master-detail layout for tablets
const MasterDetailLayout = () => {
  const deviceType = useDeviceType();

  if (deviceType === 'tablet') {
    return (
      <View style={{ flexDirection: 'row', flex: 1 }}>
        <View style={{ width: 320, borderRightWidth: 1 }}>
          <MasterList />
        </View>
        <View style={{ flex: 1 }}>
          <DetailView />
        </View>
      </View>
    );
  }

  return <StackNavigator />;
};

// Responsive spacing scale
const useSpacing = () => {
  const device = useDeviceType();
  return {
    xs: device === 'tablet' ? 6 : 4,
    sm: device === 'tablet' ? 12 : 8,
    md: device === 'tablet' ? 24 : 16,
    lg: device === 'tablet' ? 48 : 24,
    xl: device === 'tablet' ? 64 : 32,
  };
};

7. Landscape/Portrait Handling

const useOrientation = () => {
  const { width, height } = useWindowDimensions();
  return width > height ? 'landscape' : 'portrait';
};

// Lock orientation (using react-native-orientation-locker)
import Orientation from 'react-native-orientation-locker';

// Lock to portrait
Orientation.lockToPortrait();

// Lock to landscape for video player
const VideoPlayer = () => {
  useEffect(() => {
    Orientation.lockToLandscape();
    return () => Orientation.lockToPortrait();
  }, []);

  return <Video style={{ flex: 1 }} />;
};

// Responsive layout based on orientation
const AdaptiveForm = () => {
  const orientation = useOrientation();
  const isLandscape = orientation === 'landscape';

  return (
    <View style={{ flexDirection: isLandscape ? 'row' : 'column' }}>
      <View style={{ flex: isLandscape ? 1 : undefined }}>
        <FormFields />
      </View>
      <View style={{ flex: isLandscape ? 1 : undefined }}>
        <Preview />
      </View>
    </View>
  );
};

8. Dynamic Font Scaling

import { PixelRatio, Text } from 'react-native';

// fontScale reflects user's accessibility text size preference
const fontScale = PixelRatio.getFontScale();
// 1.0 = default, >1.0 = larger text, <1.0 = smaller text

// Disable font scaling on specific text (use sparingly)
<Text allowFontScaling={false}>Fixed size text</Text>

// Cap maximum font scale
<Text maxFontSizeMultiplier={1.5}>Capped scaling</Text>

// Responsive font size utility
const scaledFontSize = (size: number, maxScale = 1.3) => {
  const fontScale = PixelRatio.getFontScale();
  const clampedScale = Math.min(fontScale, maxScale);
  return Math.round(size * clampedScale);
};

// Normalized pixel size (consistent across screen densities)
const normalize = (size: number) => {
  const { width } = Dimensions.get('window');
  const baseWidth = 375; // iPhone 6/7/8 width
  const scale = width / baseWidth;
  return Math.round(PixelRatio.roundToNearestPixel(size * scale));
};
Architect Insight

Never disable font scaling globally — it breaks accessibility. Instead, cap scaling with maxFontSizeMultiplier on containers where layout would break. Test your app at 200% font scale (the maximum on iOS). Common issues: text overflow, truncated labels, and broken layouts. Design your layouts to accommodate 2x text size from the start.

10. Architect Considerations

Architect Insight

Build a responsive design system as a foundational library: (1) Define breakpoints based on your target device matrix. (2) Create responsive hooks (useBreakpoint, useDeviceType, useOrientation). (3) Build responsive primitives (Row, Column, Grid) that adapt automatically. (4) Use a spacing/typography scale that adjusts per breakpoint. (5) Test on the smallest supported device at maximum font scale — this is your hardest constraint. Foldable devices add another dimension — use react-native-device-info to detect hinge position.
