Series 2: New Architecture

JSI, Fabric, TurboModules, Codegen, and Bridgeless Mode — the complete rewrite of React Native's internals that eliminates the bridge bottleneck and enables concurrent rendering.

This series covers React Native's new architecture at the depth expected for Google L6 / Staff-level interviews. We start with JSI (the C++ foundation), trace through Fabric (the new renderer) and TurboModules (the new module system), examine the Codegen pipeline, explore bridgeless mode, and cover concurrent rendering. Each document focuses on the architectural decisions and trade-offs that distinguish a Staff engineer.

Documents in This Series

01 — JSI Foundation

JavaScript Interface: C++ API for direct JS-native communication. HostObject, HostFunction, jsi::Runtime abstraction, synchronous calls, memory management, and JSI vs Bridge performance.

02 — Fabric Renderer

New rendering system: C++ ShadowNode tree, concurrent rendering support, synchronous layout, priority-based rendering, view flattening, and the mounting layer.

03 — TurboModules

JSI-based native module system: codegen from specs, lazy initialization, synchronous methods, type-safe interfaces, and migration from NativeModules.

04 — Codegen

Generating native interfaces from JS specs: Flow/TypeScript support, schema generation, type mapping, CLI integration, and build system integration.

05 — Bridgeless Mode

Running without the legacy bridge: RuntimeExecutor, CallInvoker, event loop integration, backward compatibility, and migration strategy.

06 — Concurrent Rendering

React 18 concurrent features in React Native: useTransition, Suspense, priority-based updates, interruptible rendering, and Fabric integration.

07 — Migration Guide

Migrating from old to new architecture: enabling Fabric/TurboModules, library compatibility, interop layers, performance benchmarking, and rollback strategies.

08 — Interview Q&A

55+ L6-level interview questions covering JSI, Fabric, TurboModules, Codegen, bridgeless mode, concurrent rendering, and migration strategies.
