Series 6: Performance Optimization

Performance is the single most scrutinized dimension of React Native at Staff level. This series dissects every layer — from the threading model and JS engine internals through rendering pipelines, memory management, startup time, animation systems, networking, and bundle size — with the depth expected at Google L6.

01

Performance Model

Threading architecture, 60fps target, frame drops, monitoring tools (Flipper, Systrace, Instruments), performance budgets, and bottleneck identification.

 02

JS Thread Optimization

Hermes engine, InteractionManager, useMemo/useCallback in RN, computation offloading, JSON parsing, and re-render reduction strategies.

 03

Rendering Optimization

FlatList tuning, React.memo, view flattening, FlashList, key stability, list item memoization, and minimizing native view updates.

 04

Memory Optimization

JS heap (Hermes vs JSC), native memory, leak patterns, profiling tools, image memory, large list memory, and low-memory handling.

 05

Startup Optimization

Cold vs warm start, Hermes bytecode, RAM bundles, inline requires, lazy initialization, splash screens, bundle size reduction, and tree shaking.

 06

Animation Performance

JS-driven vs native-driven, useNativeDriver, Reanimated UI thread, layout animations, gesture-driven animations, and frame rate monitoring.

 07

Network Optimization

Fetch API, image loading, request batching, caching (MMKV, AsyncStorage), offline support, GraphQL optimization, and request prioritization.

 08

Bundle Size

Metro analysis, Hermes bytecode size, tree shaking, code splitting, asset optimization, Proguard/R8, and OTA update size.

 09

Interview Q&A

60+ Staff-level questions on every performance dimension: threading, rendering, memory, startup, animation, network, and bundle size.
