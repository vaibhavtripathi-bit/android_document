Bundle Size Optimization

Bundle size directly impacts startup time (more code = more to load and execute), OTA update size (larger diffs = slower updates), and app store download size. This doc covers Metro bundle analysis, Hermes bytecode, tree shaking limitations, code splitting, and platform-specific binary optimizations.

Bundle Size Targets

For a well-optimized React Native app: JS bundle <2MB (Hermes bytecode), total app binary <30MB (iOS), <20MB (Android APK). Every 1MB of JS bundle adds ~50-100ms to startup time on mid-range devices. For OTA updates (CodePush/EAS Update), the diff size matters more than total size — aim for <500KB per update.

3. Tree Shaking Limitations

// Metro's tree shaking is LIMITED compared to webpack/Rollup

// Problem 1: CommonJS modules can't be tree-shaken
const _ = require('lodash'); // entire lodash included!
_.debounce(fn, 300);

// Fix: Use path imports
const debounce = require('lodash/debounce'); // only debounce

// Problem 2: Barrel exports include everything
// utils/index.ts exports 50 functions
import { formatDate } from './utils'; // all 50 included!

// Fix: Import directly from the file
import { formatDate } from './utils/date'; // only date utils

// Problem 3: Side effects prevent elimination
// A module that runs code at import time can't be removed
import './analytics-init'; // side effect — always included

// Problem 4: Dynamic requires defeat static analysis
const mod = require(condition ? './a' : './b'); // both included

// metro.config.js — enable experimental tree shaking
module.exports = {
  transformer: {
    getTransformOptions: async () => ({
      transform: {
        experimentalImportSupport: true,
        inlineRequires: true,
        nonInlinedRequires: [
          'React', 'react', 'react-native',
        ],
      },
    }),
  },
};

5. Code Splitting

// Screen-level lazy loading with React.lazy
const AdminDashboard = React.lazy(() =>
  import('./screens/AdminDashboard')
);
const AnalyticsScreen = React.lazy(() =>
  import('./screens/Analytics')
);

// Wrap lazy screens with Suspense
function LazyScreen({ component: Component, ...props }) {
  return (
    <Suspense fallback={<ScreenLoader />}>
      <Component {...props} />
    </Suspense>
  );
}

// Feature-level code splitting
async function loadChartLibrary() {
  const { VictoryChart, VictoryLine } = await import(
    'victory-native'
  );
  return { VictoryChart, VictoryLine };
}

// Conditional feature loading based on user role
function App() {
  const { user } = useAuth();

  return (
    <Navigator>
      <Screen name="Home" component={HomeScreen} />
      {user.isAdmin && (
        <Screen name="Admin">
          {() => (
            <Suspense fallback={<Loader />}>
              <AdminDashboard />
            </Suspense>
          )}
        </Screen>
      )}
    </Navigator>
  );
}

7. Font Subsetting

// Full font file: ~200-500KB per weight
// Subsetted font: ~20-50KB (only used characters)

// Subset fonts with pyftsubset (fonttools)
// pip install fonttools
// pyftsubset MyFont.ttf \
//   --output-file=MyFont-subset.ttf \
//   --unicodes="U+0000-00FF,U+2000-206F" \
//   --layout-features="kern,liga"

// Or use Google Fonts with text parameter
// https://fonts.googleapis.com/css2?family=Inter&text=ABCabc123

// Font loading strategy
import * as Font from 'expo-font';

async function loadFonts() {
  await Font.loadAsync({
    'Inter-Regular': require('./assets/fonts/Inter-Regular-subset.ttf'),
    'Inter-Bold': require('./assets/fonts/Inter-Bold-subset.ttf'),
  });
}

8. Android: Proguard/R8

// android/app/build.gradle
android {
  buildTypes {
    release {
      minifyEnabled true       // enable R8/Proguard
      shrinkResources true    // remove unused resources
      proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
                    'proguard-rules.pro'
    }
  }

  // Generate AAB instead of APK (30-50% smaller downloads)
  bundle {
    language { enableSplit true }
    density { enableSplit true }
    abi { enableSplit true }
  }
}

// proguard-rules.pro — React Native specific rules
// -keep class com.facebook.react.** { *; }
// -keep class com.facebook.hermes.** { *; }
// -dontwarn com.facebook.react.**

// ABI splits — separate APKs per architecture
// arm64-v8a: 64-bit ARM (most modern devices)
// armeabi-v7a: 32-bit ARM (older devices)
// x86_64: Intel (emulators)
splits {
  abi {
    enable true
    reset()
    include "armeabi-v7a", "arm64-v8a", "x86_64"
    universalApk false
  }
}

Staff-Level Mental Model

Bundle size optimization is a supply chain problem. Every npm install adds to your bundle. At Staff level, implement: (1) a bundle size budget in CI that fails builds exceeding the limit, (2) automated dependency auditing that flags large or unused packages, (3) per-screen bundle analysis to identify which screens contribute most to size, (4) a code splitting strategy that loads features on demand, and (5) OTA update size monitoring to ensure updates download quickly on cellular. The goal is not just a small initial bundle, but a system that prevents bundle bloat over time as the team grows.
