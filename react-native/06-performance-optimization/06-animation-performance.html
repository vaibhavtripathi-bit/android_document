Animation Performance

Animations are where React Native's multi-threaded architecture either shines or breaks down. The difference between 60fps butter and visible jank comes down to one question: does the animation cross the bridge every frame? This doc covers JS-driven vs native-driven animations, Reanimated's worklet system, and gesture-driven animation performance.

2. useNativeDriver

// useNativeDriver: true — animation runs entirely on native side
const fadeAnim = useRef(new Animated.Value(0)).current;

Animated.timing(fadeAnim, {
  toValue: 1,
  duration: 300,
  useNativeDriver: true,  // runs on UI thread
}).start();

// Apply to supported properties
<Animated.View style={{
  opacity: fadeAnim,                    // ✓ supported
  transform: [{                         // ✓ supported
    translateY: fadeAnim.interpolate({
      inputRange: [0, 1],
      outputRange: [50, 0],
    }),
  }],
}}>
  <Content />
</Animated.View>

useNativeDriver Limitation

useNativeDriver only supports non-layout properties (opacity, transform). Any property that triggers Yoga layout recalculation (width, height, padding, margin, flex) cannot use the native driver because layout must run on the shadow thread, not the UI thread. For layout animations, use Reanimated's LayoutAnimation API or LayoutAnimationConfig.

3. Reanimated: UI Thread Animations

// Reanimated 3 — worklet-based animations on UI thread
import Animated, {
  useSharedValue, useAnimatedStyle, withSpring,
  withTiming, withRepeat, withSequence,
  interpolate, Extrapolation, runOnJS,
} from 'react-native-reanimated';

function AnimatedCard() {
  const scale = useSharedValue(1);
  const rotation = useSharedValue(0);

  const animatedStyle = useAnimatedStyle(() => ({
    transform: [
      { scale: scale.value },
      { rotateZ: `${rotation.value}deg` },
    ],
    // Can animate ANY property — not limited like useNativeDriver
    backgroundColor: interpolateColor(
      scale.value,
      [1, 1.2],
      ['#ffffff', '#e3f2fd']
    ),
    borderRadius: interpolate(
      scale.value,
      [1, 1.2],
      [8, 16],
      Extrapolation.CLAMP
    ),
  }));

  const handlePress = () => {
    scale.value = withSequence(
      withSpring(1.2, { damping: 10, stiffness: 100 }),
      withSpring(1, { damping: 15 })
    );
    rotation.value = withSequence(
      withTiming(5, { duration: 100 }),
      withTiming(-5, { duration: 100 }),
      withTiming(0, { duration: 100 })
    );
  };

  return (
    <Animated.View style={[styles.card, animatedStyle]}>
      <TouchableOpacity onPress={handlePress}>
        <Text>Tap me</Text>
      </TouchableOpacity>
    </Animated.View>
  );
}
Reanimated Worklet Architecture

Reanimated compiles JavaScript functions marked with 'worklet' directive into a form that runs on the UI thread via a secondary JS runtime (not Hermes — a lightweight runtime on the UI thread). useSharedValue creates a value accessible from both JS and UI threads without bridge serialization. useAnimatedStyle creates a worklet that runs on every frame on the UI thread, reading shared values and producing style updates. This architecture provides the flexibility of JS with the performance of native animations.

4. Layout Animations

// Reanimated Layout Animations
import Animated, {
  FadeIn, FadeOut, SlideInRight, SlideOutLeft,
  Layout, LinearTransition,
} from 'react-native-reanimated';

function AnimatedList({ items }) {
  return (
    <View>
      {items.map((item) => (
        <Animated.View
          key={item.id}
          entering={FadeIn.duration(300).springify()}
          exiting={FadeOut.duration(200)}
          layout={LinearTransition.springify()}
        >
          <ListItem item={item} />
        </Animated.View>
      ))}
    </View>
  );
}

5. Gesture-Driven Animations

// react-native-gesture-handler + Reanimated
import { Gesture, GestureDetector } from
  'react-native-gesture-handler';
import Animated, {
  useSharedValue, useAnimatedStyle, withSpring,
  runOnJS,
} from 'react-native-reanimated';

function SwipeableCard({ onDismiss }) {
  const translateX = useSharedValue(0);
  const opacity = useSharedValue(1);

  const panGesture = Gesture.Pan()
    .onUpdate((event) => {
      // Runs on UI thread — no bridge crossing
      translateX.value = event.translationX;
      opacity.value = 1 - Math.abs(event.translationX) / 300;
    })
    .onEnd((event) => {
      if (Math.abs(event.translationX) > 150) {
        // Dismiss — animate off screen
        translateX.value = withSpring(
          event.translationX > 0 ? 400 : -400,
          { damping: 20 },
          () => runOnJS(onDismiss)()
        );
      } else {
        // Snap back
        translateX.value = withSpring(0);
        opacity.value = withSpring(1);
      }
    });

  const animatedStyle = useAnimatedStyle(() => ({
    transform: [{ translateX: translateX.value }],
    opacity: opacity.value,
  }));

  return (
    <GestureDetector gesture={panGesture}>
      <Animated.View style={[styles.card, animatedStyle]}>
        <CardContent />
      </Animated.View>
    </GestureDetector>
  );
}

6. Avoiding Bridge During Animations

// ANTI-PATTERN: Bridge crossing during animation
const scrollY = useRef(new Animated.Value(0)).current;

// This listener fires on JS thread for every scroll event
scrollY.addListener(({ value }) => {
  // JS thread work on every frame during scroll!
  setHeaderOpacity(Math.max(0, 1 - value / 200));
});

// CORRECT: Keep everything on native side
const scrollY = useSharedValue(0);

const headerStyle = useAnimatedStyle(() => ({
  opacity: interpolate(
    scrollY.value,
    [0, 200],
    [1, 0],
    Extrapolation.CLAMP
  ),
  transform: [{
    translateY: interpolate(
      scrollY.value,
      [0, 200],
      [0, -50],
      Extrapolation.CLAMP
    ),
  }],
}));

const scrollHandler = useAnimatedScrollHandler({
  onScroll: (event) => {
    // Runs on UI thread — zero bridge traffic
    scrollY.value = event.contentOffset.y;
  },
});

7. Frame Rate Monitoring

// Development-time FPS monitor
import { PerformanceObserver } from 'react-native';

function useFPSMonitor() {
  const [fps, setFps] = useState(60);

  useEffect(() => {
    if (!__DEV__) return;

    let frames = 0;
    let lastTime = performance.now();

    const measure = () => {
      frames++;
      const now = performance.now();
      if (now - lastTime >= 1000) {
        setFps(Math.round(frames * 1000 / (now - lastTime)));
        frames = 0;
        lastTime = now;
      }
      requestAnimationFrame(measure);
    };

    const id = requestAnimationFrame(measure);
    return () => cancelAnimationFrame(id);
  }, []);

  return fps;
}

// FPS overlay component
function FPSOverlay() {
  const fps = useFPSMonitor();
  const color = fps >= 55 ? 'green' : fps >= 40 ? 'orange' : 'red';

  return (
    <View style={styles.fpsOverlay}>
      <Text style={{ color }}>{fps} FPS</Text>
    </View>
  );
}

Staff-Level Mental Model

Animation performance in React Native reduces to one principle: keep animation logic off the JS thread. The JS thread is shared with React reconciliation, event handlers, and business logic — any of these can cause frame drops during animation. The evolution: (1) Animated with useNativeDriver — limited to opacity/transform but reliable, (2) Reanimated worklets — any property, any logic, runs on UI thread, (3) gesture handlers — touch processing on UI thread. At Staff level, the architecture decision is: use useNativeDriver for simple animations, Reanimated for complex/gesture-driven animations, and never use JS-driven animations for user-facing interactions.
