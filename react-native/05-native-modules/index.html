Series 5: Native Modules

Bridging JavaScript and native platform code is the defining capability of React Native. This series covers the full spectrum — from legacy NativeModules through TurboModules, custom UI components, platform API access, brownfield integration, cross-platform C++ modules, and the modern Expo Modules API.

01

Native Module Basics

Creating native modules on iOS (Objective-C/Swift) and Android (Java/Kotlin). RCT_EXPORT_MODULE, ReactPackage, callbacks, promises, events, and threading.

 02

TurboModule Creation

New Architecture TurboModules: spec files, codegen pipeline, platform implementation, type mapping, synchronous methods, lazy initialization, and migration.

 03

Native UI Components

Custom native views: ViewManager/RCTViewManager, prop mapping, event handling, direct commands, Fabric components, ShadowNode, and accessibility.

 04

Platform APIs

Camera, Geolocation, Biometrics, Push Notifications, File System. Permission flows, platform abstraction patterns, and graceful degradation.

 05

Native Integration (Brownfield)

Embedding React Native in existing apps. RCTRootView, ReactActivity, multiple surfaces, shared modules, navigation, and bundle loading strategies.

 06

C++ TurboModules

Cross-platform C++ native code: shared implementation, JSI HostObject, memory management, error handling, CMake integration, and third-party libraries.

 07

Expo Modules API

Modern native module development: Swift/Kotlin DSL, module/view definition, config plugins, prebuild system, EAS Build, and managed vs bare workflow.

 08

Interview Q&A

55+ Staff-level questions covering native modules, TurboModules, UI components, platform APIs, brownfield, C++, and Expo modules.
