Series 1: Architecture & Bridge

From the JSON-serialized bridge to the three-thread model — the internal mechanics every Staff-level mobile engineer must master for Google L6 interviews.

This series deconstructs React Native's original (pre-Fabric) architecture at the depth expected for a Google L6 / Staff-level interview. We trace how JavaScript communicates with native platforms through the Bridge, examine the threading model, explore the shadow tree and Yoga layout engine, and dissect the Metro bundler that packages it all. Each document focuses on internals that distinguish a Staff engineer from a Senior one.

Documents in This Series

01 — Bridge Architecture

The Bridge: JSON serialization over async message passing. BatchedBridge, MessageQueue, module registration, NativeModules proxy, and why the bridge is a bottleneck at scale.

02 — JavaScript Runtime

JavaScriptCore vs Hermes vs V8. Engine startup, bytecode precompilation, memory profiles, JSI as the foundation for the new architecture, and environment differences from browser JS.

03 — Native Module System

NativeModules: JS-to-native calls through the bridge. RCT_EXPORT_MODULE, ReactPackage, callback/Promise patterns, threading model, batch calls, and TurboModules as replacement.

04 — Shadow Tree & Yoga

Shadow tree concept, Yoga layout engine (C++ Flexbox), layout calculation process, Yoga node tree, style resolution, layout passes, and Yoga vs CSS Flexbox differences.

05 — Rendering Pipeline

Full pipeline: React reconciliation → shadow tree diff → native view mutations. UIManager, batch native calls, view flattening, setNativeProps, and platform-specific rendering.

06 — Threading Model

JS thread, main/UI thread, shadow thread. Thread communication, InteractionManager, requestAnimationFrame, timers, native animations on UI thread, and thread safety.

07 — Metro Bundler

Metro architecture: resolver, transformer, serializer. Delta bundling, RAM bundles, inline requires, tree shaking, source maps, and Hermes bytecode compilation integration.

08 — Interview Q&A

55+ L6-level interview questions covering bridge architecture, JS runtime, native modules, shadow tree, rendering pipeline, threading model, and Metro bundler internals.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6
Bridge Serialization Cost Understanding JSON serialization overhead explains performance cliffs in scroll handlers, animations, and high-frequency events
JavaScript Engine Selection Hermes vs JSC vs V8 trade-offs affect startup time, memory footprint, and debugging capabilities across platforms
Threading Architecture The three-thread model dictates what can run synchronously, why animations jank, and how to architect responsive UIs
Yoga Layout Engine Cross-platform layout consistency depends on understanding Yoga's Flexbox subset and its performance characteristics
Metro Bundler Internals Bundle size, startup time, and developer experience all flow from Metro's architecture — RAM bundles and delta bundling are key optimizations
Native Module Design Efficient JS-native communication patterns are critical for features like camera, Bluetooth, and file system access at scale

Learning Objectives

  • Explain the complete bridge communication pipeline including JSON serialization, batching, and async message passing
  • Compare JavaScript engines (JSC, Hermes, V8) with specific metrics for startup time, memory, and bytecode support
  • Design native modules with correct threading, callback patterns, and migration path to TurboModules
  • Trace the full rendering pipeline from React reconciliation through shadow tree to native view mutations
  • Describe the three-thread model and explain when each thread is involved in common operations
  • Explain Metro bundler's architecture and how RAM bundles, delta bundling, and Hermes bytecode improve performance
  • Answer 55+ Staff-level interview questions on React Native architecture with confidence

Prerequisites

  • Solid JavaScript/TypeScript fundamentals and React knowledge
  • Experience building production React Native applications
  • Basic understanding of iOS (UIKit) and Android (View system) native frameworks
  • Familiarity with mobile performance profiling concepts
