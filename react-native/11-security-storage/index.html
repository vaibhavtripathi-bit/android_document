Series 11: Security & Storage

From Keychain/Keystore internals to App Store privacy manifests — the security architecture every Staff-level mobile engineer must command for Google L6 interviews.

This series covers the full security surface of React Native applications at the depth expected for Staff-level interviews. We examine secure storage primitives, network hardening, code protection, authentication flows, data protection regulations, dependency supply-chain security, and app store compliance. Each document focuses on the architectural decisions and trade-offs that distinguish a Staff engineer from a Senior one.

55+ L6-level interview questions covering secure storage, network security, code protection, authentication, data protection, dependency security, and app store requirements.

Key Themes for Staff-Level Interviews

Theme Why It Matters at L6
Defense in Depth Staff engineers design layered security — no single control is sufficient. Understanding how storage, network, and runtime protections compose is critical.
Platform Security Primitives Keychain/Keystore, Secure Enclave/StrongBox, and hardware-backed attestation are the foundation — knowing their guarantees and limitations separates L6 from L5.
Regulatory Compliance GDPR, CCPA, and app store privacy requirements directly impact architecture. Staff engineers must design systems that are compliant by default.
Supply Chain Security React Native's dependency graph (npm + CocoaPods + Gradle) creates a large attack surface. Understanding and mitigating supply chain risks is an L6 responsibility.
Authentication Architecture Token lifecycle, biometric integration, and OAuth flows must be designed for security, UX, and reliability — a core Staff-level design challenge.
Code Protection Trade-offs JavaScript bundles are inherently inspectable. Understanding what obfuscation can and cannot achieve, and when to move logic server-side, is key.

Learning Objectives

  • Design a secure storage strategy using platform-native Keychain/Keystore with appropriate access levels and biometric protection
  • Implement certificate pinning and network hardening that survives certificate rotation and doesn't break CI/CD
  • Evaluate code protection techniques and articulate their limitations for JavaScript-based mobile apps
  • Architect authentication flows (OAuth 2.0/PKCE, biometrics, MFA) with secure token lifecycle management
  • Design data protection systems that comply with GDPR/CCPA while maintaining usability
  • Establish dependency security processes that mitigate supply chain attacks across npm and native dependencies
  • Navigate app store security requirements including privacy manifests, data safety sections, and dynamic code policies

Prerequisites

  • Production React Native experience with both iOS and Android
  • Basic understanding of cryptographic primitives (symmetric/asymmetric encryption, hashing, signing)
  • Familiarity with OAuth 2.0 and JWT concepts
  • Experience with app store submission processes
